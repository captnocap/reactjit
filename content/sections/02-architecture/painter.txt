=== OVERVIEW ===
The painter (lua/painter.lua) is the final stage of the rendering pipeline. After layout.lua
computes absolute positions and sizes for every node, the painter walks the tree and issues
Love2D graphics calls to render each node to the screen.

The painter handles:

Backgrounds and borders. Every View node can have a backgroundColor, borderRadius,
borderWidth, and borderColor. Rounded rectangles are drawn when borderRadius > 0. Box shadows
are rendered before the background with a blur simulation.

Text rendering. Text nodes use Love2D's font rasterizer. The painter resolves font size
(default 14px), font family (with font caching), text alignment, line height, letter spacing,
text truncation with ellipsis, and number-of-lines clamping. For letter spacing, each
character is drawn individually (known performance cost).

Images. Image nodes are drawn with scaling to fit their computed bounds, respecting opacity
and borderRadius clipping. Images are loaded and ref-counted by the tree module.

Opacity. Opacity propagates multiplicatively down the tree. If a parent has opacity 0.5 and a
child has opacity 0.5, the child renders at 0.25. This is applied to the Love2D drawing color
alpha channel before each draw call.

Overflow clipping. Two strategies depending on borderRadius:
  - borderRadius > 0: stencil-based clipping with nesting support (stencilDepth tracks depth)
  - borderRadius = 0: scissor-based rectangular clipping using intersectScissor

Z-index sorting. Children are sorted by zIndex before painting. This is paint-order only --
it does not affect layout computation. The ZIndex module provides stable sorting so nodes with
the same zIndex maintain their source order.

Transforms. The painter supports translate, rotate, and scale transforms with configurable
transform origin. Transforms are applied via love.graphics.translate/rotate/scale calls
within a push/pop pair.

Gradients. Linear gradient backgrounds are supported with horizontal, vertical, and diagonal
directions using mesh-based rendering.

=== API / SYNTAX ===
Visual style properties consumed by the painter:

  Backgrounds:
    backgroundColor: string          -- hex, named, rgb(), hsl(), or table
    opacity: number                  -- 0 to 1, multiplied down the tree

  Borders:
    borderRadius: number             -- corner radius in pixels
    borderWidth: number              -- border stroke width
    borderColor: string              -- border stroke color
    borderTopWidth, borderBottomWidth, borderLeftWidth, borderRightWidth: number
    borderTopColor, borderBottomColor, borderLeftColor, borderRightColor: string

  Shadows:
    shadow: { offsetX, offsetY, blur, spread?, color }

  Gradients:
    backgroundGradient: { direction, colors: [color1, color2, ...] }

  Text:
    color: string                    -- text color
    fontSize: number                 -- default: 14
    fontFamily: string               -- path to .ttf/.otf file
    fontWeight: 'normal' | 'bold'
    textAlign: 'left' | 'center' | 'right'  -- default: 'left'
    lineHeight: number               -- override line height in pixels
    letterSpacing: number            -- extra pixels between characters
    numberOfLines: number            -- max visible lines (truncates remainder)
    textOverflow: 'ellipsis'         -- adds '...' when text is truncated

  Overflow:
    overflow: 'visible' | 'hidden' | 'scroll'  -- default: 'visible'

  Stacking:
    zIndex: number                   -- paint order (higher = on top). Layout is unaffected.

  Transforms:
    transform: {
      translateX?: number,
      translateY?: number,
      rotate?: number,               -- radians
      scaleX?: number,
      scaleY?: number,
      originX?: number,              -- 0-1 (fraction of element width)
      originY?: number,              -- 0-1 (fraction of element height)
    }

Specialized node types painted by dedicated Lua modules (lazy-loaded):
  TextInput, TextEditor, CodeBlock, VideoPlayer, Slider, Fader, Knob, Switch,
  Checkbox, Radio, Select, PianoKeyboard, StepSequencer, Scene3D, Emulator, Chart2D

=== EXAMPLES ===
Example 1: Box with background, border, shadow, and opacity
---
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

function StyledCard() {
  const c = useThemeColors();
  return (
    <Box style={{ width: '100%', height: '100%', padding: 24, backgroundColor: c.bg }}>
      <Box style={{
        padding: 20,
        backgroundColor: c.surface,
        borderRadius: 12,
        borderWidth: 1,
        borderColor: c.border,
        opacity: 0.9,
        shadow: { offsetX: 0, offsetY: 4, blur: 12, color: 'rgba(0,0,0,0.3)' },
      }}>
        <Text style={{ fontSize: 18, color: c.text }}>{`Card with shadow`}</Text>
      </Box>
    </Box>
  );
}

Example 2: Z-index for layered content
---
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

function Layered() {
  const c = useThemeColors();
  return (
    <Box style={{ width: '100%', height: '100%', backgroundColor: c.bg }}>
      {/* These are siblings. zIndex controls paint order, not layout. */}
      <Box style={{ position: 'absolute', top: 20, left: 20, width: 200, height: 200,
        backgroundColor: c.surface, zIndex: 1, borderRadius: 8 }}>
        <Text style={{ fontSize: 14, color: c.text, padding: 8 }}>{`Behind`}</Text>
      </Box>
      <Box style={{ position: 'absolute', top: 40, left: 40, width: 200, height: 200,
        backgroundColor: c.primary, zIndex: 2, borderRadius: 8 }}>
        <Text style={{ fontSize: 14, color: c.text, padding: 8 }}>{`In front`}</Text>
      </Box>
    </Box>
  );
}

=== PLATFORM NOTES ===
Love2D:
All rendering uses Love2D's OpenGL 2.1 backend. Text rasterization, image decoding, and
stencil operations are native Love2D calls. The painter runs within love.draw() and issues
draw calls directly to the Love2D graphics context. The frame budget is 100,000 paintNode
calls per pass -- exceeding this triggers an error (likely infinite loop).

=== CRITICAL RULES ===
- Never use raw love.graphics.setScissor with content-space coordinates inside a node renderer. setScissor operates in screen coordinates and ignores the current transform stack (scroll offsets). Use love.graphics.transformPoint to convert content coords to screen coords, then use love.graphics.intersectScissor to respect parent clip regions.
- zIndex is paint-order only. It does not affect layout computation. Two overlapping elements at different z-indices still occupy the same layout space.
- Opacity multiplies down the tree. A parent at 0.5 and a child at 0.5 renders the child at 0.25. There is no "isolation" -- opacity always inherits.
- Letter spacing triggers per-character rendering which is expensive. Only use it when necessary.
