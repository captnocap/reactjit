=== OVERVIEW ===
ReactJIT is a Love2D rendering framework that uses React as its layout declaration layer.
React does not render to a DOM. There is no browser, no CSS, no HTML. React declares a tree
of primitives (Box, Text, Image) with style props, and Lua turns that tree into OpenGL draw
calls via Love2D.

The rendering pipeline has five stages:

1. React reconciler -- diffs your component tree and emits mutation commands (CREATE, UPDATE,
   REMOVE, APPEND). Event handlers stay in JavaScript; only serializable props cross the bridge.

2. QuickJS bridge -- the FFI transport layer. Commands are JSON-serialized and passed from the
   embedded QuickJS runtime to Lua via C trampolines. Events flow the other direction: Lua
   queues input events, JS polls them each frame.

3. Instance tree (tree.lua) -- Lua's retained element tree. Mutation commands are applied here
   to build and maintain the in-memory node hierarchy. This is the "DOM equivalent" for ReactJIT.

4. Layout engine (layout.lua) -- a pixel-perfect flexbox implementation in Lua. Computes
   absolute {x, y, w, h} for every node. Supports flex direction, grow/shrink/basis, alignment,
   padding, margin, gap, percentage units, viewport units, calc(), and min/max constraints.

5. Painter (painter.lua) -- walks the laid-out tree and issues Love2D graphics calls. Handles
   backgrounds, borders, text rendering, images, video, opacity, overflow clipping, z-index
   sorting, transforms, shadows, and gradients.

Love2D owns the run loop. Each frame: Lua calls into JS to poll events and tick timers, JS
flushes any pending mutation commands back to Lua, Lua applies commands to the tree, runs
layout, and paints. React never drives the frame -- it only responds when Lua asks.
