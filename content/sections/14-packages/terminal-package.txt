=== OVERVIEW ===
@reactjit/terminal provides components and hooks for embedding Claude Code sessions in a ReactJIT application. ClaudeCanvas is the visual component -- a Lua-rendered PTY canvas with damage-driven rendering, semantic classification, and scroll. useClaude manages permission/question/status state. useSessionChrome polls the classified token stream for status bar text, prompt text, and cursor position.

ClaudeCanvas renders the Lua-side "ClaudeCanvas" capability node. It is a hittable surface that accepts keyboard input, displays the terminal grid, and classifies each row semantically (user input, assistant output, status bar, tool use, etc.).

The proxy input pattern: an Input component displays the prompt text from useSessionChrome, forwards keystrokes to the ClaudeCanvas via keystrokeTarget/submitTarget, and never maintains its own text buffer. The classified token stream is the single source of truth.

=== API / SYNTAX ===
```tsx
import { ClaudeCanvas, useClaude, useSessionChrome } from '@reactjit/terminal';
```

---- ClaudeCanvas ----

| Prop         | Type    | Default   | Description                        |
|--------------|---------|-----------|------------------------------------|
| sessionId    | string  | 'default' | Session identifier                 |
| debugVisible | boolean | false     | Show debug overlay                 |
| style        | Style   | undefined | Container style                    |

---- useClaude() ----

Returns ClaudeState:

| Field            | Type                    | Description                        |
|------------------|-------------------------|------------------------------------|
| perm             | PermissionInfo \| null  | Pending permission request         |
| question         | QuestionInfo \| null    | Pending question prompt            |
| status           | string                  | Current status ('idle', etc)       |
| autoAccept       | boolean                 | Auto-accept mode                   |
| toggleAutoAccept | () => Promise<void>     | Toggle auto-accept                 |
| onPerm           | (event) => void         | Wire to onPermissionRequest        |
| onPermResolved   | () => void              | Wire to onPermissionResolved       |
| onQuestion       | (event) => void         | Wire to onQuestionPrompt           |
| onStatusChange   | (event) => void         | Wire to onStatusChange             |
| respond          | (choice: number) => void| Respond to permission prompt       |
| respondQuestion  | (optionIndex) => void   | Respond to question prompt         |

PermissionInfo: { action, target, question }
QuestionInfo: { question, options[] }

---- useSessionChrome(sessionId?) ----

Returns SessionChromeState:

| Field          | Type   | Description                          |
|----------------|--------|--------------------------------------|
| statusLeft     | string | Left status bar text                 |
| statusRight    | string | Right status bar text                |
| placeholder    | string | Input placeholder text               |
| promptText     | string | Current prompt text (from classifier)|
| cursorPosition | number | Cursor offset (-1 if not on input)   |

Example 2: Claude with permission handling
---
```tsx
import { Box, Text, Pressable } from '@reactjit/core';
import { ClaudeCanvas, useClaude } from '@reactjit/terminal';
import { useThemeColors } from '@reactjit/theme';

export function ClaudeWithPermissions() {
  const colors = useThemeColors();
  const claude = useClaude();

  return (
    <Box style={{ width: '100%', height: '100%' }}>
      <ClaudeCanvas sessionId="default" style={{ flexGrow: 1 }} />
      {claude.perm && (
        <Box style={{ paddingLeft: 12, paddingTop: 8, paddingBottom: 8, backgroundColor: colors.bgElevated, gap: 4 }}>
          <Text style={{ color: colors.text, fontSize: 12 }}>
            {`${claude.perm.action}: ${claude.perm.target}`}
          </Text>
          <Box style={{ flexDirection: 'row', gap: 8 }}>
            <Pressable onPress={() => claude.respond(1)}>
              <Text style={{ color: colors.primary }}>Allow</Text>
            </Pressable>
            <Pressable onPress={() => claude.respond(0)}>
              <Text style={{ color: colors.muted }}>Deny</Text>
            </Pressable>
          </Box>
        </Box>
      )}
    </Box>
  );
}
```

Example 3: Proxy input pattern
---
```tsx
import { Box, Input, Text } from '@reactjit/core';
import { ClaudeCanvas, useSessionChrome } from '@reactjit/terminal';
import { useThemeColors } from '@reactjit/theme';

export function ClaudeWithInput() {
  const colors = useThemeColors();
  const chrome = useSessionChrome('default');

  return (
    <Box style={{ width: '100%', height: '100%' }}>
      <ClaudeCanvas sessionId="default" style={{ flexGrow: 1 }} />
      <Box style={{ paddingLeft: 12, paddingRight: 12, paddingTop: 8, paddingBottom: 8, backgroundColor: colors.bgElevated }}>
        <Input
          value={chrome.promptText}
          placeholder={chrome.placeholder}
          keystrokeTarget="ClaudeCanvas"
          submitTarget="ClaudeCanvas"
          style={{ color: colors.text, fontSize: 14 }}
        />
      </Box>
      <Box style={{ flexDirection: 'row', justifyContent: 'space-between', paddingLeft: 12, paddingRight: 12, paddingTop: 4, paddingBottom: 4 }}>
        <Text style={{ color: colors.muted, fontSize: 10 }}>{chrome.statusLeft}</Text>
        <Text style={{ color: colors.muted, fontSize: 10 }}>{chrome.statusRight}</Text>
      </Box>
    </Box>
  );
}
```

Platforms: love2d

=== CRITICAL RULES ===
- The Input in the proxy pattern must NEVER maintain its own text buffer. It displays chrome.promptText and forwards keystrokes via keystrokeTarget/submitTarget. The classified token stream is the single source of truth.
- useSessionChrome polls claude:classified at 100ms intervals. This is the only poller needed for prompt/status state.
- useClaude event handlers (onPerm, onPermResolved, onQuestion, onStatusChange) must be wired to the Native element's event props when using the low-level API.
- ClaudeCanvas is a Lua capability node, not a Box. It handles its own rendering, hit testing, and keyboard input.
- respond(1) = allow, respond(0) = deny for permission prompts. respondQuestion takes the option index.
- autoAccept state survives HMR -- it syncs from Lua on mount.
