=== OVERVIEW ===
@reactjit/math provides mathematical primitives for game development, simulations, and data visualization. All vector/matrix/quaternion operations are pure TypeScript with no bridge dependency. Heavy computational workloads (noise fields, FFT, bezier sampling) delegate to Lua via bridge RPCs.

Vectors use plain tuple arrays ([x, y] for Vec2, [x, y, z] for Vec3, [x, y, z, w] for Vec4). Matrix operations use 16-element flat arrays in column-major order. This avoids object allocation overhead and makes serialization across the bridge trivial.

The library is organized into pure math (no React) and React hooks:

**Pure math** (import and use anywhere):
- Vec2, Vec3, Vec4 -- static method namespaces (add, sub, mul, dot, cross, normalize, lerp, distance, rotate, etc.)
- Mat4 -- 4x4 matrix (identity, translate, rotateX/Y/Z, scale, perspective, lookAt, invert, multiply)
- Quat -- quaternion (fromEuler, fromAxisAngle, slerp, multiply, toMat4, normalize)
- BBox2, BBox3 -- axis-aligned bounding boxes (fromPoints, contains, intersects, expand)
- Geometry -- distancePointToSegment, distancePointToRect, circleContainsPoint, circleIntersectsRect, lineIntersection
- Interpolation -- lerp, inverseLerp, smoothstep, smootherstep, remap, clamp, wrap, damp, step, pingPong, moveTowards, smoothDamp

**React hooks** (bridge-backed for heavy compute):
- useVec2, useVec3, useVec4 -- reactive vector state
- useMat4, useTransform -- reactive matrix state with transform helpers
- useLerp, useSmoothstep, useDistance, useBBox, useIntersection -- memoized math
- useNoise, useNoiseField -- Perlin/simplex noise via Lua
- useFFT -- Fast Fourier Transform via Lua
- useBezier -- bezier curve sampling via Lua
- useMathPool -- batch multiple math operations in a single RPC

=== API / SYNTAX ===
```tsx
import {
  // Pure math (no React)
  Vec2, Vec3, Vec4, Mat4, Quat,
  BBox2, BBox3, distancePointToSegment, circleContainsPoint, lineIntersection,
  lerp, inverseLerp, smoothstep, smootherstep, remap, clamp, wrap, damp,
  moveTowards, moveTowardsAngle, smoothDamp,
  // React hooks
  useVec2, useVec3, useVec4, useMat4, useTransform,
  useLerp, useSmoothstep, useDistance, useBBox, useIntersection,
  useNoise, useNoiseField, useFFT, useBezier, useMathPool,
} from '@reactjit/math';
```

Vec2 type: `[number, number]`
Vec3 type: `[number, number, number]`
Vec4 type: `[number, number, number, number]`
Mat4 type: `number[]` (16 elements, column-major)

|------------|-------------------------------|--------------------------------|

| cross      | (a, b) => number              | 2D cross product (scalar)      |

=== EXAMPLES ===
Example 1: Vector math
---
```tsx
import { Vec2, Vec3 } from '@reactjit/math';

const a: Vec2 = [10, 20];
const b: Vec2 = [30, 40];

const sum = Vec2.add(a, b);           // [40, 60]
const dist = Vec2.distance(a, b);     // 28.28...
const mid = Vec2.lerp(a, b, 0.5);     // [20, 30]
const dir = Vec2.normalize(Vec2.sub(b, a)); // unit vector from a to b
const rotated = Vec2.rotate(a, Math.PI / 4); // rotate 45 degrees
```

Example 2: Transform matrix from position/rotation/scale

```tsx

export function TransformDemo() {

    </Box>
  );
}
```

Example 6: Batch math operations
---
```tsx
import { useMathPool } from '@reactjit/math';

export function BatchDemo() {
  const pool = useMathPool();

  // Enqueue multiple operations
  const id1 = pool.enqueue('noise2d', { x: 10, y: 20, seed: 42 });
  const id2 = pool.enqueue('noise2d', { x: 11, y: 20, seed: 42 });
  const id3 = pool.enqueue('bezier', { points: [[0,0],[50,100],[100,0]], segments: 20 });

  // All execute in a single RPC when flushed
  pool.flush();

  // Results available after flush resolves
  const noise1 = pool.result(id1);
  const noise2 = pool.result(id2);
  const curve = pool.result(id3);
}
```

=== PLATFORM NOTES ===
Love2D:
  useNoise and useNoiseField use Love2D's love.math.noise (Simplex noise) via bridge RPC. useFFT delegates to a Lua FFT implementation. useBezier uses Love2D's love.math.newBezierCurve for sampling. useMathPool batches all enqueued operations into a single bridge RPC for efficiency. Pure math functions (Vec2, Mat4, lerp, etc.) run entirely in TypeScript with no Lua dependency.

=== CRITICAL RULES ===
- Vec2/Vec3/Vec4 are tuple types, not objects. Use Vec2.add(a, b), not a.add(b). All methods are static.
- Mat4 is a 16-element flat array in column-major order. This matches GLSL conventions but differs from row-major math notation.
- useNoise and useNoiseField are async (Lua RPC). They return null until the first result arrives. Always handle the null case.
- useMathPool flush is async. Results are not available until the RPC resolves and triggers a re-render.
- Pure math functions have zero allocations beyond the returned tuple/array. They are safe to call in render loops.
