=== OVERVIEW ===
@reactjit/finance provides financial data analysis and visualization. The package has four layers:

1. **Pure indicator functions** -- SMA, EMA, WMA, RSI, MACD, Stochastic, Bollinger Bands, VWAP, OBV, ATR, ROC, pivot points, and candlestick pattern detection. All accept number[] or OHLCV[] and return aligned arrays. No React dependency.
2. **Portfolio utilities** -- portfolioSnapshot computes total value, P&L, and allocation weights. sharpeRatio, maxDrawdown, and equityToReturns provide risk metrics.
3. **React hooks** -- useTechnicalAnalysis (computes all standard indicators via Lua RPC), usePortfolio (holdings CRUD + metrics), useSyntheticCandles (demo data generator), usePriceFeed (CoinGecko polling + Binance WebSocket), useOHLCVHistory (historical candles), useWalletConnectV2 (relay connection), useSecurePortfolio (encrypted storage).
4. **Display components** -- TickerSymbol, TickerTape (scrolling Lua-owned widget), HoldingRow, PortfolioCard, OrderBookPanel, RSIGauge, MACDPanel, IndicatorLegend.

All heavy computation (indicator calculation for useTechnicalAnalysis, portfolio snapshots) runs via Lua RPCs. The pure TS indicator functions are available for client-side use when Lua is not needed.

Formatting utilities (formatCurrency, formatCompact, formatPercent, formatVolume, formatPrice, formatBps, spreadBps) handle display formatting for financial values.

OHLCV interface:
| Field  | Type   | Description                  |
|--------|--------|------------------------------|
| time   | number | Unix timestamp (seconds)     |
| open   | number | Opening price                |
| high   | number | Highest price                |
| low    | number | Lowest price                 |
| close  | number | Closing price                |
| volume | number | Volume traded                |

Holding interface:
| Field        | Type   | Description                 |
|--------------|--------|-----------------------------|
| symbol       | string | Ticker symbol               |
| quantity     | number | Units held                  |
| avgCost      | number | Average cost per unit       |
| currentPrice | number | Current market price        |

Indicator function signatures:
```tsx
sma(data: number[], period: number): number[]
ema(data: number[], period: number): number[]
rsi(data: number[], period?: number): number[]    // default: 14
macd(data: number[], fast?: number, slow?: number, signal?: number): MACDPoint[]
bollingerBands(data: number[], period?: number, multiplier?: number): BollingerBand[]
vwap(candles: OHLCV[]): IndicatorPoint[]
atr(candles: OHLCV[], period?: number): IndicatorPoint[]
detectPatterns(candles: OHLCV[]): PatternSignal[]
```

usePriceFeed options:

=== EXAMPLES ===
Example 1: Computing indicators from candle data
---
```tsx
import { sma, rsi, bollingerBands } from '@reactjit/finance';

const closes = candles.map(c => c.close);

const sma20 = sma(closes, 20);    // NaN for first 19 entries, then rolling average
const rsi14 = rsi(closes);         // RSI with default period 14
const bands = bollingerBands(closes, 20, 2); // upper, middle, lower bands
```

Example 2: Full technical analysis via Lua
---
```tsx
import { useTechnicalAnalysis, useSyntheticCandles } from '@reactjit/finance';
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function TADemo() {
  const c = useThemeColors();
  const { candles } = useSyntheticCandles({ count: 60, startPrice: 100 });
  const ta = useTechnicalAnalysis(candles);

  const lastRSI = ta.rsi14[ta.rsi14.length - 1];
  const lastMACD = ta.macd[ta.macd.length - 1];

  return (
    <Box style={{ padding: 16, gap: 8 }}>
      <Text style={{ color: c.text, fontSize: 14 }}>
        {`RSI(14): ${isNaN(lastRSI) ? '--' : lastRSI.toFixed(1)}`}
      </Text>
      <Text style={{ color: c.text, fontSize: 14 }}>
        {`MACD: ${isNaN(lastMACD.macd) ? '--' : lastMACD.macd.toFixed(4)}`}
      </Text>
      <Text style={{ color: c.text, fontSize: 14 }}>
        {`Patterns: ${ta.patterns.map(p => p.type).join(', ') || 'none'}`}
      </Text>
    </Box>
  );
}
```

Example 3: Live price feed with ticker tape
---
```tsx
import { usePriceFeed, TickerTape } from '@reactjit/finance';
import { Box } from '@reactjit/core';

export function PriceTicker() {
  const feed = usePriceFeed({
    symbols: ['BTC', 'ETH', 'SOL'],
    pollInterval: 30000,
  });

  const items = feed.all.map(q => ({
    symbol: q.symbol,
    price: q.price,
    change: q.change24h,
  }));

  return (
    <Box style={{ width: '100%' }}>
      <TickerTape items={items} />
    </Box>
  );
}
```

Example 5: Pattern detection
---
```tsx
import { detectPatterns } from '@reactjit/finance';
import type { OHLCV, PatternSignal } from '@reactjit/finance';

const patterns: PatternSignal[] = detectPatterns(candles);

// Each pattern has: type, index (candle position), confidence (0-1)
// Types: doji, hammer, shooting_star, bullish_engulfing, bearish_engulfing,
//        double_top, double_bottom, higher_high, lower_low
for (const p of patterns) {
  console.log(`${p.type} at index ${p.index} (${(p.confidence * 100).toFixed(0)}%)`);
}
```

=== PLATFORM NOTES ===
Love2D:
  useTechnicalAnalysis sends the full OHLCV array to Lua for computation. For large datasets (1000+ candles), this is more efficient than computing in JS because Lua has direct LuaJIT-optimized math. The CoinGecko OHLC endpoint does not include volume data -- volume is set to 0 in useOHLCVHistory results. TickerTape is a Lua-owned host widget that handles horizontal scrolling internally for low-latency rendering.

=== CRITICAL RULES ===
- Indicator functions return NaN for warmup periods. The first `period - 1` values of sma/ema/rsi are NaN. Always check for NaN before displaying values.
- usePriceFeed defaults to CoinGecko REST polling only. Set wsEnabled: true to add real-time Binance WebSocket updates.
- useSyntheticCandles generates random walk data seeded deterministically. Use it for demos and testing, not real analysis.
- CoinGecko has rate limits. The free tier allows ~10 calls/minute. Set pollInterval to 30000+ or provide an apiKey for higher limits.
- TickerTape and TickerSymbol render via Lua host elements. They are not React component trees -- children are not supported.
