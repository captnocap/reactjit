=== OVERVIEW ===
This page covers the most common errors and visual bugs in ReactJIT, why they happen,
and how to fix them. Each entry includes the symptom, the cause, and a before/after code
fix.

=== EXAMPLES ===
Example 1: Mixed text children stack vertically
---
Symptom: Text like "Hello Alice!" renders as three lines instead of one.

Cause: Mixed children in Text (string literals + JSX expressions) create separate __TEXT__
nodes that stack vertically in the column layout.

Before (broken):
```tsx
// BAD: creates 3 separate text nodes that stack vertically
<Text style={{ color: c.text, fontSize: 14 }}>
  Hello {name}!
</Text>
```

After (fixed):
```tsx
// GOOD: single template literal = single text node
<Text style={{ color: c.text, fontSize: 14 }}>
  {`Hello ${name}!`}
</Text>
```

The linter catches this as `no-mixed-text-children`.
---
Platforms: love2d

Example 2: Empty Box renders at unexpected size
---
Symptom: A Box with no children renders at 1/4 of its parent's size instead of zero.

Cause: Empty surface nodes (Box, Image, Video) use the proportional fallback -- they get
1/4 of the parent's available space. This cascades: an unsized Box in an 800px parent
is 200px; a nested unsized Box inside it is 50px.

Fix: Either give it explicit dimensions or add children (which triggers content auto-sizing).

```tsx
// Proportional fallback: 1/4 of parent
<Box />

// Explicit dimensions: exactly 0
<Box style={{ width: 0, height: 0 }} />

// Content auto-sizing: sizes to children
<Box>
  <Text style={{ fontSize: 14 }}>Content</Text>
</Box>

// flexGrow: fills remaining space
<Box style={{ flexGrow: 1 }} />
```
---
Platforms: love2d

Example 3: ScrollView shows nothing
---
Symptom: A ScrollView with children renders as zero height or is invisible.

Cause: ScrollView is not a surface -- it does not get the proportional fallback. It needs
explicit height (or flexGrow) to define its scroll viewport.

Before (broken):
```tsx
// BAD: ScrollView has no height -> viewport is zero
<ScrollView>
  <Text style={{ fontSize: 14 }}>Content</Text>
</ScrollView>
```

After (fixed):
```tsx
// GOOD: flexGrow gives it a viewport
<ScrollView style={{ flexGrow: 1 }}>
  <Text style={{ fontSize: 14 }}>Content</Text>
</ScrollView>

// Also GOOD: explicit height
<ScrollView style={{ height: 400 }}>
  <Text style={{ fontSize: 14 }}>Content</Text>
</ScrollView>
```
---
Platforms: love2d

Example 4: Root container does not fill the window
---
Symptom: The root component renders at 1/4 of the window size.

Cause: The root container needs explicit width and height. The proportional fallback does
not apply at the root because the root IS the viewport reference.

Before (broken):
```tsx
// BAD: root Box uses proportional fallback -> 1/4 viewport
export function App() {
  return (
    <Box>
      <Text style={{ fontSize: 16 }}>Hello</Text>
    </Box>
  );
}
```

After (fixed):
```tsx
// GOOD: root fills the viewport
export function App() {
  return (
    <Box style={{ width: '100%', height: '100%' }}>
      <Text style={{ fontSize: 16 }}>Hello</Text>
    </Box>
  );
}
```
---
Platforms: love2d

Example 5: Stale theme colors after switching themes
---
Symptom: Some colors do not update when the theme changes. The app partially shows the
old theme.

Cause: Module-level const objects capture theme values at eval time. When applyTheme()
mutates the color map, module-level references go stale.

Before (broken):
```tsx
// BAD: captures theme values at module load time
const STYLES = {
  header: { backgroundColor: C.accent },  // stale after theme switch
};
```

After (fixed):
```tsx
// GOOD: read theme colors inside the component
function Header() {
  const c = useThemeColors();
  return (
    <Box style={{ backgroundColor: c.primary }}>
      <Text style={{ color: c.text, fontSize: 16 }}>Header</Text>
    </Box>
  );
}
```
---
Platforms: love2d

Example 6: Linter rejects paddingHorizontal/paddingVertical
---
Symptom: Build fails with a linter error about paddingHorizontal or paddingVertical.

Cause: ReactJIT does not support paddingHorizontal or paddingVertical shorthand properties.
Use the individual side props instead.

Before (broken):
```tsx
// BAD: linter rejects these
<Box style={{ paddingHorizontal: 16, paddingVertical: 8 }}>
```

After (fixed):
```tsx
// GOOD: use individual side props
<Box style={{ paddingLeft: 16, paddingRight: 16, paddingTop: 8, paddingBottom: 8 }}>
```
---
Platforms: love2d

Example 7: Event handler silently does nothing
---
Symptom: An onClick or custom event handler is attached to a Box but never fires.

Cause: The Box component uses an explicit whitelist for event handlers. If a handler is
not in the destructure list in primitives.tsx, it silently disappears -- extractHandlers
in hostConfig never sees it, hasHandlers is false, and hit testing skips the node.

Fix: Check that the handler prop is listed in all four places:
1. BoxProps in types.ts (type definition)
2. Box() destructure in primitives.tsx (extract from props)
3. Box() createElement call in primitives.tsx (pass to host element)
4. bridge.subscribe() in eventDispatcher.ts (Lua-to-JS routing)
---
Platforms: love2d

Example 8: CRLF line endings cause parse errors
---
Symptom: Files created on Windows or by some tools have \r\n line endings, causing
unexpected parse errors or build failures.

Cause: ReactJIT is a Linux-native project. CRLF line endings can confuse parsers.

Fix: Run dos2unix on the affected file. Do not rewrite the file manually.

```bash
dos2unix path/to/file.tsx
```
---
Platforms: love2d

=== CRITICAL RULES ===
- The linter (rjit lint) catches many of these errors before they become runtime bugs. Always run it after modifying components.
- When debugging layout, use the F12 inspector to see computed dimensions. The three sizing tiers (explicit > content auto-sizing > proportional fallback) resolve in order.
- If a visual bug only appears after a theme switch, it is almost certainly the stale theme cache pattern. Search for module-level objects that reference theme colors.
