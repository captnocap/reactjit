=== OVERVIEW ===
Answers to the most common questions about ReactJIT's architecture, design decisions,
and how to accomplish tasks that would be straightforward in a browser environment.

=== EXAMPLES ===
Example 1: Where is the DOM?
---
There is no DOM. ReactJIT does not use react-dom. There is no browser, no HTML, no CSS,
no document object, no window object.

React renders to a custom reconciler that emits mutation commands (CREATE, UPDATE, REMOVE,
APPEND). These commands cross the QuickJS bridge to Lua, which maintains an in-memory
element tree. Lua runs flexbox layout on this tree and paints it using Love2D's OpenGL
backend.

The components are Box (not div), Text (not span), Image (not img), and Pressable (not
button). They accept style props that map directly to the flex layout engine.

```tsx
// There is no DOM equivalent. This is the entire rendering target:
<Box style={{ flexDirection: 'row', padding: 16, gap: 8 }}>
  <Text style={{ fontSize: 14 }}>Hello</Text>
  <Image src="icon.png" style={{ width: 24, height: 24 }} />
</Box>
```

Example 2: Why no CSS?
---
CSS does not exist in ReactJIT. Styles are JavaScript objects passed via the style prop.
The layout engine implements a subset of CSS flexbox, but it is a clean-room Lua
implementation -- not a browser engine.

Supported style properties map to flexbox concepts: flexDirection, justifyContent,
alignItems, flexGrow, flexShrink, flexBasis, width, height, padding, margin, gap,
position (relative/absolute), overflow, borderRadius, backgroundColor, opacity, and more.

What is NOT supported:
- CSS selectors, class names, or stylesheets
- CSS Grid
- CSS transitions or @keyframes (use useAnimation/useSpring instead)
- Media queries (use useBreakpoint or useWindowDimensions)
- Pseudo-elements (::before, ::after)
- Cascading -- styles only apply to the element they are set on

```tsx
// Styles are always inline objects, not class names
<Box style={{
  flexDirection: 'row',
  justifyContent: 'space-between',
  alignItems: 'center',
  padding: 16,
  gap: 8,
  backgroundColor: c.surface,
  borderRadius: 8,
}}>
  <Text style={{ fontSize: 16, fontWeight: 'bold', color: c.text }}>Title</Text>
</Box>
```

Example 3: Why flexGrow instead of hardcoded heights?
---
Hardcoded pixel heights break at different window sizes. A panel with height: 260 in a
600px window leaves 340px of dead space. Resize to 800px and there is 540px of dead space.
Resize to 500px and it overflows.

flexGrow: 1 tells the element to absorb whatever space remains after siblings are measured.
It works at any resolution without manual math.

```tsx
// BAD: hardcoded heights break at different window sizes
<Box style={{ width: '100%', height: '100%' }}>
  <Box style={{ height: 48 }}>{/* header */}</Box>
  <Box style={{ height: 500 }}>{/* content -- what if window is 400px tall? */}</Box>
  <Box style={{ height: 32 }}>{/* footer */}</Box>
</Box>

// GOOD: flexGrow absorbs remaining space at any resolution
<Box style={{ width: '100%', height: '100%' }}>
  <Box style={{ height: 48 }}>{/* header: fixed */}</Box>
  <Box style={{ flexGrow: 1 }}>{/* content: fills remaining space */}</Box>
  <Box style={{ height: 32 }}>{/* footer: fixed */}</Box>
</Box>
```

Example 4: How do I add a new feature to the framework?
---
Use the capability system. It is the standard pattern for adding new Lua-side features
with React integration and AI discoverability.

Step 1: Register the capability in Lua (create a file in lua/capabilities/):
```lua
local Capabilities = require("lua.capabilities")

Capabilities.register("MyFeature", {
  visual = false,
  schema = {
    value = { type = "number", default = 0, desc = "The value" },
  },
  events = { "onChange" },
  create  = function(nodeId, props) return {} end,
  update  = function(nodeId, props, prev, state) end,
  tick    = function(nodeId, state, dt, pushEvent, props) end,
  destroy = function(nodeId, state) end,
})
```

Step 2: Create a one-liner React wrapper:
```tsx
import { Native } from '@reactjit/core';

export function MyFeature(props: {
  value?: number;
  onChange?: (e: { value: number }) => void;
}) {
  return <Native type="MyFeature" {...props} />;
}
```

Step 3: Use it:
```tsx
<MyFeature value={42} onChange={(e) => console.log(e.value)} />
```

An AI agent discovers it automatically via useCapabilities() -- no documentation needed
beyond the schema.

Example 5: How do I use browser APIs (fetch, localStorage, WebSocket)?
---
Browser APIs do not exist in ReactJIT. The runtime is LuaJIT + QuickJS, not a browser.
For equivalent functionality:

- HTTP requests: use useFetch() from @reactjit/core or bridge.rpc('http:get', { url })
- Persistent storage: use useLocalStore() from @reactjit/core (backed by SQLite)
- WebSocket: use useWebSocket() from @reactjit/core (backed by Lua luasocket)
- Clipboard: use useClipboard() from @reactjit/core
- Timers: use the Timer capability component or useLuaInterval()

```tsx
import { useLocalStore, useFetch } from '@reactjit/core';

function DataComponent() {
  // Persistent storage (survives app restart)
  const [data, setData] = useLocalStore('myData', { count: 0 });

  // HTTP fetch (Lua-backed, not browser fetch)
  const { data: apiData, loading } = useFetch('https://api.example.com/data');

  return null;
}
```

Example 6: Why does my component render but not respond to clicks?
---
Check these in order:

1. Is the component wrapped in Pressable? Plain Box does not have press behavior by
   default. Use Pressable for tap handling, or attach onClick directly to Box.

2. Does Box have the handler in its whitelist? Box uses an explicit prop whitelist. If
   your handler prop is not destructured in primitives.tsx, it silently disappears.

3. Is there a higher-zIndex element covering it? The inspector (F12) shows which node
   the hit test finds. If another element is on top, it intercepts the event.

4. Is the element inside a scroll container? Scroll containers adjust hit test coordinates.
   If the ScrollView has no explicit height, it may have zero viewport and not be
   intercepting correctly.

5. Is display: 'none' or visibility: 'hidden' set? Hidden elements are excluded from
   hit testing entirely.

=== CRITICAL RULES ===
- There is no DOM and there never will be. Do not attempt to use document, window, or any Web API.
- react-dom is not a dependency. Components do not render HTML elements.
- All styles are inline objects. There are no CSS files, class names, or style sheets.
- The capability system is the correct way to add new features. If a user needs to call bridge.rpc() directly, the feature is not finished -- wrap it in a capability.
