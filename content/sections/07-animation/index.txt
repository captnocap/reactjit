=== OVERVIEW ===
ReactJIT has two animation layers that work together:

JS-driven animations run inside the QuickJS runtime. They are driven by Lua's love.update(dt) frame loop via tickAnimations() — there are no independent JS timers or requestAnimationFrame calls. Every frame, Lua calls the bridge, which advances all active animations in lockstep with the renderer. JS animations cause React re-renders each frame during motion, so they are best for values that feed into layout or conditional logic.

Lua-driven animations run entirely inside the Lua painter. Style properties like transition and animation (keyframes) are declared in JSX and interpolated by Lua without crossing the bridge. They produce zero JS re-renders during motion and are ideal for hover effects, infinite loops, and visual polish.

JS-driven hooks:
- AnimatedValue — mutable container that notifies listeners when its value changes
- useAnimation — creates an AnimatedValue and subscribes a component to its changes
- useSpring — springs to a target value using physics simulation (stiffness, damping, mass)
- useTransition — transitions to a target value using timing-based easing
- parallel, sequence, stagger, loop — compose multiple animations together

Lua-driven style properties:
- style.transition — per-property or all-property interpolation (duration, easing, delay)
- style.animation — keyframe-based loops with direction, fill mode, and play state
- strokeDasharray / strokeDashoffset — SVG-style dash patterns and animated reveals

Animation presets wrap common patterns into one-liners:
- usePulse — oscillating value for breathing effects and attention indicators
- useCountUp — animated number counter for statistics and scores
- useTypewriter — character-by-character text reveal
- useShake — triggered horizontal shake for error feedback
- useEntrance — one-shot fade + slide for mount animations
- useBounce — spring-based bounce to a target value
- useRepeat — looping 0-to-1 progress value for custom animations

All animation APIs are exported from @reactjit/core.

Example 2: Lua-driven transition (zero JS re-renders)
---
```tsx
import { Box, Pressable, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function AnimatedButton() {
  const c = useThemeColors();
  const [hovered, setHovered] = useState(false);
  const [pressed, setPressed] = useState(false);

  return (
    <Pressable
      onPress={() => {}}
      onHoverIn={() => setHovered(true)}
      onHoverOut={() => setHovered(false)}
      onPressIn={() => setPressed(true)}
      onPressOut={() => setPressed(false)}
    >
      <Box style={{
        padding: 12,
        borderRadius: 8,
        backgroundColor: hovered ? c.primary : c.surface,
        transform: {
          scaleX: pressed ? 0.95 : 1,
          scaleY: pressed ? 0.95 : 1,
        },
        shadowBlur: hovered ? 12 : 0,
        transition: { all: { duration: 200, easing: 'easeOut' } },
      }}>
        <Text style={{ color: c.text, fontSize: 14 }}>{'Click me'}</Text>
      </Box>
    </Pressable>
  );
}
```

Platforms: love2d

Example 3: Lua-driven keyframe animation (infinite spin)
---
```tsx
<Box style={{
  width: 40,
  height: 40,
  borderRadius: 20,
  borderWidth: 3,
  borderColor: c.primary,
  borderTopColor: 'transparent',
  animation: {
    keyframes: {
      0: { transform: { rotate: 0 } },
      100: { transform: { rotate: 360 } },
    },
    duration: 1000,
    iterations: -1,
    easing: 'linear',
  },
}} />
```
