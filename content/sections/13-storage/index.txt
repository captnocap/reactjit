=== OVERVIEW ===
The `@reactjit/storage` package provides a complete CRUD storage layer for ReactJIT applications. It combines Zod-inspired schema validation, pluggable storage adapters, a query engine, and a migration system into a cohesive API that works identically across all targets.

The core idea: define a schema once, pick an adapter for your target, and get type-safe create/read/update/delete operations with automatic validation. The same application code works against in-memory storage during development, Love2D's filesystem in production, and SQLite for server-side tools.

Key pieces:

- **z schema builder** -- lightweight runtime validation with full TypeScript inference. No Zod dependency.
- **useCRUD hook** -- React hook that returns typed CRUD methods plus reactive query hooks (useQuery, useListQuery).
- **createCRUD** -- same CRUD interface without React, for scripts and CLI tools.
- **Adapters** -- MemoryAdapter, Love2DFileAdapter, TerminalSQLiteAdapter, LocalStorageAdapter, IndexedDBAdapter.
- **Query engine** -- filtering ($eq, $gt, $contains...), sorting, and pagination applied uniformly across all adapters.
- **Migrations** -- versioned record transforms that run on read, with automatic write-back.
