=== OVERVIEW ===
The storage system provides persistent data storage across all ReactJIT targets with a unified API. It combines schema validation, CRUD operations, reactive queries, and migrations. Multiple adapters handle platform differences: Love2DFileAdapter uses love.filesystem, LocalStorageAdapter and IndexedDBAdapter work in browsers, TerminalSQLiteAdapter uses Node.js SQLite, and MemoryAdapter provides in-memory storage for testing.

=== API / SYNTAX ===

Core Components:
- `StorageProvider` — React context provider for the default adapter
- `useCRUD(collection, schema, options?)` — React hook for CRUD + reactive queries
- `createCRUD(collection, schema, adapter, options?)` — Non-React CRUD for scripts/backends
- `useStorage()` — Direct access to the underlying storage adapter
- `z` — Schema builder for validation (exported from @reactjit/storage)

Adapters:
- `MemoryAdapter()` — In-memory storage (lost on close)
- `Love2DFileAdapter({ rpc, format? })` — Love2D filesystem (JSON/Markdown/Text)
- `LocalStorageAdapter({ prefix? })` — Browser localStorage (< 5MB)
- `IndexedDBAdapter({ dbName?, version? })` — Browser IndexedDB (larger datasets)
- `TerminalSQLiteAdapter({ dbPath? })` — Node.js SQLite (default: ./data.db)

Schema Types (z.*):
- `z.string()`, `z.number()`, `z.boolean()`, `z.date()`
- `z.object({ ... })`, `z.array(schema)`
- `.optional()`, `.nullable()`, `.default(value)`

CRUD Methods (from useCRUD):
- `create(data)` → Promise<string> (returns new ID)
- `get(id)` → Promise<T | null>
- `update(id, partial)` → Promise<void>
- `delete(id)` → Promise<void>
- `list(query?)` → Promise<T[]>

Reactive Hooks (from useCRUD):
- `useQuery(id)` → { data, loading, error, refetch }
- `useListQuery(query?)` → { data, loading, error, refetch }

Query Object:
{
  where?: { field: value, field2: { $gt: 10 } },
  orderBy?: 'fieldName',
  order?: 'asc' | 'desc',
  limit?: number,
  offset?: number
}

Query Operators:
$eq, $ne, $gt, $gte, $lt, $lte, $in, $contains

=== EXAMPLES ===

Example 1: Basic CRUD with MemoryAdapter
---
import { StorageProvider, useCRUD, z, MemoryAdapter } from '@reactjit/storage';

const NoteSchema = z.object({
  id: z.string(),
  title: z.string(),
  body: z.string(),
  createdAt: z.date(),
});

function App() {
  const adapter = new MemoryAdapter();
  return (
    <StorageProvider adapter={adapter}>
      <NotesList />
    </StorageProvider>
  );
}

function NotesList() {
  const notes = useCRUD('notes', NoteSchema);
  const { data, loading, refetch } = notes.useListQuery();

  const handleCreate = async () => {
    await notes.create({
      id: 'n1',
      title: 'My Note',
      body: 'Content here',
      createdAt: new Date(),
    });
    refetch();
  };

  if (loading) return <Text fontSize={14}>Loading...</Text>;

  return (
    <Box>
      {data.map(note => (
        <Text key={note.id} fontSize={14}>{note.title}</Text>
      ))}
    </Box>
  );
}

Example 2: Love2D File Storage
---
import { StorageProvider, useCRUD, z, Love2DFileAdapter } from '@reactjit/storage';
import { useLoveRPC } from '@reactjit/core';

const SaveGameSchema = z.object({
  id: z.string(),
  level: z.number(),
  score: z.number(),
  playerName: z.string(),
});

function App() {
  const rpc = useLoveRPC();
  const adapter = new Love2DFileAdapter({ rpc, format: 'json' });

  return (
    <StorageProvider adapter={adapter}>
      <SaveGameManager />
    </StorageProvider>
  );
}

function SaveGameManager() {
  const saves = useCRUD('saves', SaveGameSchema);
  const { data } = saves.useListQuery({ orderBy: 'level', order: 'desc' });

  const handleSave = async () => {
    await saves.create({
      id: `save-${Date.now()}`,
      level: 5,
      score: 1000,
      playerName: 'Player1',
    });
  };

  return (
    <Box>
      {data.map(save => (
        <Text key={save.id} fontSize={14}>
          {save.playerName} - Level {save.level} - {save.score}
        </Text>
      ))}
    </Box>
  );
}

Example 3: Web with IndexedDB
---
import { StorageProvider, useCRUD, z, IndexedDBAdapter } from '@reactjit/storage';

const TaskSchema = z.object({
  id: z.string(),
  text: z.string(),
  completed: z.boolean().default(false),
  priority: z.number(),
});

function App() {
  const adapter = new IndexedDBAdapter({ dbName: 'taskdb', version: 1 });

  return (
    <StorageProvider adapter={adapter}>
      <TaskList />
    </StorageProvider>
  );
}

function TaskList() {
  const tasks = useCRUD('tasks', TaskSchema);
  const { data, refetch } = tasks.useListQuery({
    where: { completed: false },
    orderBy: 'priority',
    order: 'desc',
  });

  const handleToggle = async (id: string, completed: boolean) => {
    await tasks.update(id, { completed: !completed });
    refetch();
  };

  return (
    <Box>
      {data.map(task => (
        <Box key={task.id} flexDirection="row">
          <Text fontSize={14}>{task.text}</Text>
          <Pressable onPress={() => handleToggle(task.id, task.completed)}>
            <Text fontSize={12}>{task.completed ? '✓' : '○'}</Text>
          </Pressable>
        </Box>
      ))}
    </Box>
  );
}

Example 4: Query Filtering
---
const products = useCRUD('products', ProductSchema);

// Get expensive items
const expensive = await products.list({
  where: { price: { $gt: 100 } }
});

// Get electronics in stock
const electronics = await products.list({
  where: {
    category: 'electronics',
    stock: { $gt: 0 }
  },
  orderBy: 'price',
  order: 'asc',
  limit: 10
});

// Search by partial match
const results = await products.list({
  where: { name: { $contains: 'phone' } }
});

Example 5: Schema Validation
---
const UserSchema = z.object({
  id: z.string(),
  username: z.string(),
  email: z.string(),
  age: z.number().optional(),
  bio: z.string().nullable(),
  role: z.string().default('user'),
  tags: z.array(z.string()),
});

const users = useCRUD('users', UserSchema);

// Valid
await users.create({
  id: 'u1',
  username: 'alice',
  email: 'alice@example.com',
  tags: ['admin', 'moderator'],
});

// Invalid — throws ValidationError
await users.create({
  id: 'u2',
  username: 'bob',
  // missing required 'email'
  tags: ['user'],
});

Example 6: Non-React CRUD (Scripts/Backends)
---
import { createCRUD, z, TerminalSQLiteAdapter } from '@reactjit/storage';

const LogSchema = z.object({
  id: z.string(),
  level: z.string(),
  message: z.string(),
  timestamp: z.date(),
});

const adapter = new TerminalSQLiteAdapter({ dbPath: './logs.db' });
const logs = createCRUD('logs', LogSchema, adapter);

// Use in scripts without React
async function logError(message: string) {
  await logs.create({
    id: `log-${Date.now()}`,
    level: 'error',
    message,
    timestamp: new Date(),
  });
}

async function getRecentErrors() {
  return await logs.list({
    where: { level: 'error' },
    orderBy: 'timestamp',
    order: 'desc',
    limit: 100,
  });
}

Example 7: Migrations
---
const UserSchema = z.object({
  id: z.string(),
  name: z.string(),
  email: z.string(),
  settings: z.object({
    theme: z.string(),
    notifications: z.boolean(),
  }),
});

const users = useCRUD('users', UserSchema, {
  migrations: {
    1: (data) => ({
      ...data,
      email: data.email || `${data.username}@example.com`,
    }),
    2: (data) => ({
      ...data,
      settings: data.settings || { theme: 'dark', notifications: true },
    }),
  },
  autoMigrate: true,
});

// Old records are migrated automatically on read
const user = await users.get('u1');

=== PLATFORM NOTES ===
Love2D: Uses Love2DFileAdapter backed by love.filesystem. Data stored in save directory (OS-dependent: ~/.local/share/love/<identity>/ on Linux, ~/Library/Application Support/LOVE/<identity>/ on macOS, %APPDATA%\LOVE\<identity>\ on Windows). Supports JSON, Markdown, and Text formats. Requires RPC connection via useLoveRPC().

Web: Use LocalStorageAdapter for simple key-value storage (5-10MB limit) or IndexedDBAdapter for larger datasets with structured queries. IndexedDB is async, transactional, and supports indexes. LocalStorage is synchronous but has lower limits.

Terminal: Use TerminalSQLiteAdapter with Node.js v22+ built-in node:sqlite module. Synchronous API wrapped in async interface. Creates one table per collection. Data persists in SQLite file (default: ./data.db).

MemoryAdapter: Works on all platforms. Data lost when app closes. Useful for testing, prototyping, and ephemeral state.

=== CRITICAL RULES ===
- Every collection must have a schema — schema.parse() is called on all reads/writes
- Schema must include an `id: z.string()` field — CRUD methods expect it
- Use StorageProvider at app root OR pass { adapter } to useCRUD options
- Query operators ($gt, $gte, etc.) work in-memory for all adapters except TerminalSQLiteAdapter (which uses SQL WHERE)
- Reactive hooks (useQuery, useListQuery) do NOT auto-refetch on mutations — call refetch() manually
- Love2DFileAdapter requires an RPC caller from useLoveRPC() — it calls Lua-side storage:get/set/list/delete handlers
- IndexedDBAdapter is async — all operations return Promises
- LocalStorageAdapter stores everything as JSON strings — do not store large objects
- TerminalSQLiteAdapter requires Node.js v22+ for node:sqlite — does not work in browsers
- Migrations run on every read — keep migration functions fast and idempotent
- autoMigrate: true runs migrations automatically; autoMigrate: false requires manual migration trigger
- createCRUD is for scripts/backends without React — do not use it in components
- Do not mutate data returned from get() or list() — it may be cached
