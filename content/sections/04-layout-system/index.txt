=== OVERVIEW ===
ReactJIT uses a Flexbox layout model implemented entirely in Lua (layout.lua). Every Box
is a flex container by default, with flexDirection defaulting to 'column'. There are no
block, inline, or grid display modes -- everything is flex. The layout engine computes
absolute pixel positions and sizes from the style tree that React produces.

The engine is pixel-perfect. Flex distribution math, cursor positions, and post-layout
sizes are exact to the pixel. If you encounter a layout overflow or sizing problem, look
at the component first -- does it hardcode a width that overflows its parent? The layout
engine itself is not the source of bugs.

Because Love2D renders to a fixed canvas (not a scrolling web page), the layout system
demands explicit sizing at the root level. Beyond that, containers auto-size from their
children, and empty surface nodes use a proportional fallback (1/4 of parent). The static
linter (rjit lint) catches common layout mistakes at build time.

The layout engine supports:
- Absolute units, percentages, vw/vh viewport units, and calc() expressions
- flexDirection row/column, flexWrap, flexGrow, flexShrink, flexBasis
- justifyContent, alignItems, alignSelf
- Padding (all sides + per-side), margin, gap
- display: flex | none, position: relative | absolute
- minWidth/maxWidth/minHeight/maxHeight clamping
- Intrinsic text measurement with font metrics, lineHeight, letterSpacing
- aspectRatio
- Fit-content sizing (width: 'fit-content' or 'fit')

=== API / SYNTAX ===
The layout system is configured through the `style` prop on Box, Text, and Image primitives.
All style properties are defined in the Style interface (packages/core/src/types.ts).

Layout-related style categories:
  - Flexbox: flexDirection, justifyContent, alignItems, alignSelf, flexGrow, flexShrink, flexBasis, flexWrap, gap, display
  - Sizing: width, height, minWidth, minHeight, maxWidth, maxHeight, aspectRatio
  - Spacing: padding, paddingLeft/Right/Top/Bottom, margin, marginLeft/Right/Top/Bottom, gap
  - Positioning: position (relative | absolute), top, left, right, bottom
  - Visual: backgroundColor, borderRadius, borderWidth, borderColor, opacity, overflow, zIndex, shadows, gradients
  - Text: color, fontSize, fontFamily, fontWeight, textAlign, lineHeight, letterSpacing
  - Transforms: transform object with translateX/Y, rotate, scaleX/Y, skewX/Y, originX/Y

Supported units:
  - number: pixels (e.g. width: 200)
  - string '%': percentage of parent (e.g. width: '50%')
  - string 'vw': percentage of viewport width (e.g. width: '50vw')
  - string 'vh': percentage of viewport height (e.g. height: '50vh')
  - string 'calc()': arithmetic with % and px (e.g. width: 'calc(100% - 40px)')
  - string 'fit-content' or 'fit': shrink-wrap to content size

=== EXAMPLES ===
Example 1: Full-viewport root container
---
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

function App() {
  const c = useThemeColors();
  return (
    <Box style={{ width: '100%', height: '100%', backgroundColor: c.bg }}>
      <Text style={{ fontSize: 24, color: c.text }}>Hello, ReactJIT</Text>
    </Box>
  );
}
---
Platforms: love2d

Example 2: Row layout with space-between
---
const c = useThemeColors();
<Box style={{
  flexDirection: 'row',
  width: '100%',
  justifyContent: 'space-between',
  alignItems: 'center',
  padding: 16,
}}>
  <Text style={{ fontSize: 18, color: c.text }}>Left</Text>
  <Text style={{ fontSize: 18, color: c.text }}>Right</Text>
</Box>
---
Platforms: love2d

=== PLATFORM NOTES ===
Love2D:
Fixed canvas rendering. Root must fill the viewport with width: '100%', height: '100%'.
The Lua layout engine (layout.lua) computes all positions. Text measurement uses Love2D's
font rasterizer. The canvas resolution is set in conf.lua (typically 1280x720 or 1920x1080).
Percentage values resolve against the parent container's computed size.

=== CRITICAL RULES ===
- Root containers MUST have width: '100%', height: '100%' -- the proportional fallback does not apply at the root because the root IS the viewport
- Use flexGrow: 1 for space-filling elements -- never hardcode pixel heights to fill a known window size
- ScrollView needs explicit height or flexGrow -- scroll containers are excluded from auto-sizing
- Do not mix text and expressions in Text -- use template literals instead
