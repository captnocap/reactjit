=== OVERVIEW ===
useDebug registers arbitrary data under a key on `globalThis.__debug`. The inspector console and watch expressions can read this data live. Automatically unregisters on unmount.

This is the primary way to expose component state for debugging. No serialization overhead -- data is written directly to a global object.

=== API / SYNTAX ===
```tsx
import { useDebug, getDebugData, registerDebug } from '@reactjit/core';
```

Hook:
```tsx
function useDebug(key: string, data: any): void
```

Utilities:
```tsx
function getDebugData(): Record<string, any>
function registerDebug(key: string, data: any): () => void
```

Parameters:
| Param | Type   | Default  | Description                          |
|-------|--------|----------|--------------------------------------|
| key   | string | required | Unique identifier (usually component name) |
| data  | any    | required | Data to expose for debugging         |

=== EXAMPLES ===
Example 1: Expose component state
---
```tsx
import { useState } from 'react';
import { Box, Text } from '@reactjit/core';
import { useDebug } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function PlayerHUD({ health, mana, position }: {
  health: number;
  mana: number;
  position: { x: number; y: number };
}) {
  const colors = useThemeColors();
  useDebug('PlayerHUD', { health, mana, position });

  return (
    <Box style={{ padding: 8 }}>
      <Text style={{ color: colors.text }}>{`HP: ${health} | MP: ${mana}`}</Text>
    </Box>
  );
}
// Console: __debug['PlayerHUD'].health
// Watch:   :watch __debug['PlayerHUD'].health
```
---
Platforms: love2d

Example 2: Register debug data outside React
---
```tsx
import { registerDebug } from '@reactjit/core';

// In a game loop or service
const unregister = registerDebug('GameLoop', { fps: 60, entities: 42 });

// Later, clean up
unregister();
```
---
Platforms: love2d

=== CRITICAL RULES ===
- Keys must be unique across your app. Using the same key from two components overwrites one's data.
- Data is updated every render. Keep it cheap -- avoid allocating large objects just for debugging.
- useDebug auto-unregisters on unmount. registerDebug returns an unregister function you must call manually.
