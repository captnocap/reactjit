=== OVERVIEW ===
useLoveSend returns a stable send function for fire-and-forget commands to Lua. Commands are batched and flushed on the next bridge cycle. There is no return value or acknowledgment.

Use this when you need to notify Lua of something but do not need a response. For request/response patterns, use useLoveRPC.

=== API / SYNTAX ===
```tsx
import { useLoveSend } from '@reactjit/core';
```

Signature:
```tsx
function useLoveSend(): (type: string, payload?: any) => void
```

Returns:
| Type                                  | Description                          |
|---------------------------------------|--------------------------------------|
| (type: string, payload?: any) => void | Send function for fire-and-forget    |

=== EXAMPLES ===
Example 1: Send a command on press
---
```tsx
import { useLoveSend } from '@reactjit/core';

function FireButton() {
  const send = useLoveSend();

  return (
    <Pressable onPress={() => send('weapon:fire', { angle: 45 })}>
      <Text>Fire</Text>
    </Pressable>
  );
}
```
---
Platforms: love2d

Example 2: Analytics events
---
```tsx
import { useLoveSend } from '@reactjit/core';

function useAnalytics() {
  const send = useLoveSend();

  return {
    track: (event: string, data?: any) => send('analytics:track', { event, ...data }),
    pageView: (page: string) => send('analytics:pageview', { page }),
  };
}
```
---
Platforms: love2d

=== CRITICAL RULES ===
- The send function is memoized. It is safe to use in dependency arrays and will not cause re-renders or re-subscriptions.
- Commands are batched and flushed each bridge cycle. They are not instant -- there is a one-frame delay before Lua sees them.
- Do not use send for request/response patterns. There is no mechanism to receive a result. Use useLoveRPC for that.
