=== OVERVIEW ===
useWindowDimensions returns the current viewport width and height, updating reactively on window resize. The value is cached across remounts so tab switches pick up the last known size instantly without waiting for a new resize event from Lua.

This file also covers three declarative window control hooks: useWindowSize, useWindowPosition, and useWindowAlwaysOnTop. These hooks set window properties as side effects -- mount the component to apply the setting, unmount to optionally revert.

=== API / SYNTAX ===
```tsx
import { useWindowDimensions, useWindowSize, useWindowPosition, useWindowAlwaysOnTop } from '@reactjit/core';
```

useWindowDimensions:
```tsx
function useWindowDimensions(): { width: number; height: number }
```

useWindowSize:
```tsx
function useWindowSize(width: number, height: number, options?: WindowControlOptions): void
```

useWindowPosition:
```tsx
function useWindowPosition(x: number, y: number, options?: WindowControlOptions): void
```

useWindowAlwaysOnTop:
```tsx
function useWindowAlwaysOnTop(onTop: boolean, options?: { windowId?: number; revert?: boolean }): void
```

WindowControlOptions:
| Option    | Type    | Default | Description                                        |
|-----------|---------|---------|----------------------------------------------------|
| windowId  | number  | 0       | Target window (0 = main, 1+ = child windows)      |
| animate   | boolean | false   | Smoothly animate the transition                    |
| duration  | number  | 300     | Animation duration in ms (only when animate=true)  |
| revert    | boolean | false   | Revert to previous value on unmount                |

=== EXAMPLES ===
Example 1: Responsive layout
---
```tsx
import { useWindowDimensions } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

function ResponsiveLayout() {
  const colors = useThemeColors();
  const { width, height } = useWindowDimensions();
  const isWide = width > 800;

  return (
    <Box style={{ flexDirection: isWide ? 'row' : 'column', flexGrow: 1 }}>
      <Box style={{ width: isWide ? 200 : '100%', backgroundColor: colors.bgElevated }}>
        <Text style={{ color: colors.text }}>Sidebar</Text>
      </Box>
      <Box style={{ flexGrow: 1 }}>
        <Text style={{ color: colors.text }}>{`${width}x${height}`}</Text>
      </Box>
    </Box>
  );
}
```
---
Platforms: love2d

Example 2: Set window size on mount
---
```tsx
import { useWindowSize } from '@reactjit/core';

function CompactMode() {
  useWindowSize(400, 300, { animate: true, revert: true });

  return <Text>Compact view -- window reverts on unmount</Text>;
}
```
---
Platforms: love2d

Example 3: Pin window on top
---
```tsx
import { useWindowAlwaysOnTop } from '@reactjit/core';

function PinnedOverlay({ pinned }: { pinned: boolean }) {
  useWindowAlwaysOnTop(pinned, { revert: true });

  return <Text>{pinned ? 'Pinned' : 'Unpinned'}</Text>;
}
```
---
Platforms: love2d

Example 4: Position a child window
---
```tsx
import { useWindowPosition, useWindowSize } from '@reactjit/core';

function ToolWindow() {
  useWindowSize(300, 200, { windowId: 1 });
  useWindowPosition(100, 100, { windowId: 1, animate: true });

  return <Text>Tool window at 100, 100</Text>;
}
```

=== PLATFORM NOTES ===
Love2D:
  The viewport event is pushed from Lua whenever the Love2D window is resized. The initial value comes from the first viewport event after bridge initialization. The module-level cache ensures that if a component unmounts and remounts (e.g., tab switch), it immediately gets the last known dimensions instead of starting at 0x0.

  Window control hooks (useWindowSize, useWindowPosition, useWindowAlwaysOnTop) send RPC calls to Lua which forwards them to love.window APIs. Animation support depends on the Lua-side window manager interpolation.

=== CRITICAL RULES ===
- useWindowDimensions returns { width: 0, height: 0 } before the first viewport event. Design your layout to handle this gracefully (use flexGrow instead of pixel math based on window size).
- The revert option on window control hooks captures the previous value at mount time and restores it on unmount. If the window was resized by the user between mount and unmount, the revert goes back to the captured value, not the current one.
- Window control hooks are declarative side effects. Changing the width/height/position props re-applies the setting. Do not call them conditionally -- mount/unmount the component instead.
