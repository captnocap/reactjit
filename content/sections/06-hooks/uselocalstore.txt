=== OVERVIEW ===
useLocalStore is a drop-in replacement for useState that persists values to a SQLite database on the Lua side. Values survive app restarts. Writes are debounced (300ms) to avoid excessive disk I/O during rapid state changes.

On mount, the hook loads the stored value asynchronously. Until the stored value arrives, the component renders with the provided default. If no stored value exists, the default persists.

=== API / SYNTAX ===
```tsx
import { useLocalStore } from '@reactjit/core';
```

Signature:
```tsx
function useLocalStore<T>(
  key: string,
  defaultValue: T,
  options?: { namespace?: string }
): [T, (value: T | ((prev: T) => T)) => void]
```

Parameters:
| Parameter    | Type   | Default | Description                                |
|--------------|--------|---------|--------------------------------------------|
| key          | string | required | Storage key within the namespace           |
| defaultValue | T      | required | Default value if nothing is stored         |
| options.namespace | string | 'app' | Storage namespace (separate databases)  |

Returns:
| Index | Type                                      | Description            |
|-------|-------------------------------------------|------------------------|
| [0]   | T                                         | Current value          |
| [1]   | (value: T or (prev: T) => T) => void      | Setter (supports updater functions) |

=== EXAMPLES ===
Example 1: Persistent counter
---
```tsx
import { useLocalStore } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

function Counter() {
  const colors = useThemeColors();
  const [count, setCount] = useLocalStore('counter', 0);

  return (
    <Pressable onPress={() => setCount(c => c + 1)}>
      <Text style={{ color: colors.text, fontSize: 24 }}>{`Count: ${count}`}</Text>
    </Pressable>
  );
}
```
---
Platforms: love2d

Example 2: Theme preference with namespace
---
```tsx
import { useLocalStore } from '@reactjit/core';

function useThemePreference() {
  const [theme, setTheme] = useLocalStore('selected', 'catppuccin', {
    namespace: 'theme',
  });

  return { theme, setTheme };
}
```

Platforms: love2d

Example 3: Storing structured data
---
```tsx
import { useLocalStore } from '@reactjit/core';

function BookmarkManager() {
  const [bookmarks, setBookmarks] = useLocalStore<string[]>('bookmarks', []);

  const addBookmark = (url: string) => {
    setBookmarks(prev => [...prev, url]);
  };

  const removeBookmark = (url: string) => {
    setBookmarks(prev => prev.filter(b => b !== url));
  };

  return (
    <Box>
      {bookmarks.map((b, i) => (
        <Text key={i}>{b}</Text>
      ))}
    </Box>
  );
}
```

=== CRITICAL RULES ===
- Writes are debounced at 300ms. If the component unmounts before the debounce fires, the last value may not be persisted. For critical saves, consider calling the RPC directly via useBridge.
- The setter supports updater functions like useState: `setCount(prev => prev + 1)`. This is safe for rapid updates.
- The namespace separates storage databases. Different namespaces cannot see each other's keys. Use this to isolate feature-specific data.
- The stored value loads asynchronously on mount. The component will briefly render with defaultValue before the stored value arrives. Design your UI to handle this flash gracefully.
- Values are serialized as JSON through the bridge. Functions, symbols, and circular references cannot be stored.
