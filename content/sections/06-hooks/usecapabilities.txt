=== OVERVIEW ===
useCapabilities queries the Lua runtime for all registered native capabilities and their schemas. This is the AI discovery hook -- an agent calls it once to learn what components are available, what props they accept, and what events they emit, then generates JSX accordingly.

Capabilities are registered on the Lua side via `Capabilities.register()`. Each capability has a schema describing its props (name, type, default) and a list of events it emits.

=== API / SYNTAX ===
```tsx
import { useCapabilities } from '@reactjit/core';
```

Signature:
```tsx
function useCapabilities(): {
  capabilities: Record<string, CapabilitySchema> | null;
  loading: boolean;
}
```

Types:
```tsx
interface CapabilitySchema {
  schema: Record<string, { type: string; default?: any; description?: string }>;
  events: string[];
}
```

Returns:
| Field        | Type                                  | Description                        |
|--------------|---------------------------------------|------------------------------------|
| capabilities | Record<string, CapabilitySchema> \| null | Map of capability name to schema |
| loading      | boolean                               | True while fetching from Lua       |

=== EXAMPLES ===
Example 1: List available capabilities
---
```tsx
import { Box, Text } from '@reactjit/core';
import { useCapabilities } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function CapabilityBrowser() {
  const colors = useThemeColors();
  const { capabilities, loading } = useCapabilities();

  if (loading) return <Text style={{ color: colors.muted }}>Loading...</Text>;
  if (!capabilities) return <Text style={{ color: colors.muted }}>No capabilities</Text>;

  return (
    <Box style={{ gap: 8, padding: 16 }}>
      {Object.entries(capabilities).map(([name, cap]) => (
        <Box key={name} style={{ gap: 2 }}>
          <Text style={{ color: colors.text, fontSize: 16 }}>{name}</Text>
          <Text style={{ color: colors.muted, fontSize: 12 }}>
            {`Props: ${Object.keys(cap.schema).join(', ')}`}
          </Text>
          <Text style={{ color: colors.muted, fontSize: 12 }}>
            {`Events: ${cap.events.join(', ') || 'none'}`}
          </Text>
        </Box>
      ))}
    </Box>
  );
}
```

Example 2: AI agent using capability discovery
---
```tsx
import { useCapabilities } from '@reactjit/core';

export function useAgentContext() {
  const { capabilities } = useCapabilities();

  // Pass capabilities to an AI agent so it knows what components exist
  // capabilities = {
  //   Audio: { schema: { src: { type: "string" }, playing: { type: "bool" } }, events: ["onProgress", "onEnded"] },
  //   Timer: { schema: { interval: { type: "number" } }, events: ["onTick"] },
  // }
  return { capabilities };
}
```

=== CRITICAL RULES ===
- The hook fetches once on mount. It does not poll. If capabilities change at runtime (unlikely), remount the component.
- Uses useBridgeOptional internally. Returns null capabilities and loading=false when no bridge is present.
- The schema is the documentation. Capability names map to component types in JSX (e.g. Audio -> `<Audio ...props />`).
