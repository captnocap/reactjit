=== OVERVIEW ===
@reactjit/storage provides schema-validated CRUD operations with reactive
React hooks and a plain-JS interface for non-React code. The same API works
on every target: Love2D stores data in love.filesystem, terminals use SQLite
via Node.js built-in node:sqlite, web targets use localStorage or IndexedDB,
and any target can use the in-memory adapter for tests or ephemeral state.
Schema validation uses the built-in z builder — a lightweight Zod-inspired
system that gives you runtime validation and TypeScript type inference with no
external dependencies.

Import: import { StorageProvider, useCRUD, useStorage, createCRUD, z,
         MemoryAdapter, Love2DFileAdapter, TerminalSQLiteAdapter,
         LocalStorageAdapter, IndexedDBAdapter } from '@reactjit/storage';

=== API / SYNTAX ===

---- z — Schema builder ----------------------------------------------------

The z object is the entry point for defining schemas. All methods return a
Schema<T> instance that carries both a TypeScript type and a runtime parse
function.

  z.string()                       StringSchema
  z.number()                       NumberSchema
  z.boolean()                      BooleanSchema
  z.date()                         DateSchema
  z.array(itemSchema)              ArraySchema<T>
  z.object(shape)                  ObjectSchema<S>
  z.enum(values)                   EnumSchema<T>
  z.literal(value)                 LiteralSchema<T>

Common modifiers (chainable on any schema):
  .optional()                      Makes field optional (T | undefined)
  .nullable()                      Allows null (T | null)
  .default(value)                  Provides a default (implies optional)

String-specific:
  .min(n)                          Minimum character count
  .max(n)                          Maximum character count
  .email()                         Email format check
  .url()                           http/https URL format check
  .matches(regexp, message?)       Custom regex

Number-specific:
  .min(n)                          Minimum value (inclusive)
  .max(n)                          Maximum value (inclusive)
  .int()                           Integer only

Array-specific:
  .min(n)                          Minimum item count

Object-specific:
  .strict()                        Strip unknown fields
  .merge(other)                    Merge two object schemas
  .pick(...keys)                   New schema with only named keys
  .omit(...keys)                   New schema without named keys
  .partial()                       Make all fields optional

Schema methods:
  schema.parse(value): T           Parse and validate. Throws ValidationError.
  schema.safeParse(value):         Returns { success, data } | { success, error }
    SafeParseResult<T>

Type inference:
  type MyType = z.infer<typeof MySchema>

---- StorageProvider -------------------------------------------------------

Context provider. Wrap your app or a subtree with one StorageProvider. All
useCRUD and useStorage calls in that subtree will use the provided adapter by
default.

  <StorageProvider adapter={StorageAdapter}>
    {children}
  </StorageProvider>

  Props:
    adapter   StorageAdapter   Required. See adapter section below.
    children  ReactNode        Required.

---- useStorage() ----------------------------------------------------------

Returns the raw StorageAdapter from context. Use this when you need direct
adapter access beyond what useCRUD provides.

  useStorage(): StorageAdapter

  Throws if called outside a <StorageProvider>.

  StorageAdapter interface:
    get(collection, id): Promise<any | null>
    set(collection, id, data): Promise<void>
    delete(collection, id): Promise<boolean>
    list(collection, query?): Promise<any[]>
    format?: StorageFormat             // 'json' | 'markdown' | 'text'

---- useCRUD<T>(collection, schema, options?) --------------------------------

Full CRUD interface for a single collection. All operations validate data
against the provided schema. Returns an object with async methods and two
reactive query hooks for subscribing to data in components.

  useCRUD<T extends Record<string, any>>(
    collection: string,
    schema:     Schema<T>,
    options?:   CRUDOptions,
  ): CRUDHandle<T>

  Arguments:
    collection  string       Collection name (acts as a namespace / table name).
    schema      Schema<T>    z.object(...) schema for type T.
    options     CRUDOptions  Optional overrides.

  CRUDOptions:
    adapter?:      StorageAdapter                  Override context adapter.
    format?:       'json' | 'markdown' | 'text'   Storage format.
    migrations?:   Record<number, MigrationFn>     Version migration map.
    autoMigrate?:  boolean                         Apply migrations on read.

  CRUDHandle<T>:
    create(data: T): Promise<string>
      Validates data, generates an id if not present, writes record.
      Returns the id string.

    get(id: string): Promise<T | null>
      Reads and validates one record. Returns null if not found.

    update(id: string, partial: Partial<T>): Promise<void>
      Merges partial into existing record, re-validates, writes back.
      Throws if record does not exist.

    delete(id: string): Promise<void>
      Removes a record. Resolves even if the record was not found.

    list(query?: Query): Promise<T[]>
      Returns all records matching the optional query, schema-validated.

    useQuery(id: string): QueryResult<T>
      Reactive hook. Fetches one record and subscribes to refetch.
      QueryResult<T>: { data: T | null; loading: boolean;
                        error: Error | null; refetch: () => void }

    useListQuery(query?: Query): ListResult<T>
      Reactive hook. Fetches all matching records.
      ListResult<T>: { data: T[]; loading: boolean;
                       error: Error | null; refetch: () => void }

  Note: useQuery and useListQuery are hooks returned from useCRUD, not
  standalone hooks. They follow React hook rules and must be called
  unconditionally inside a component.

---- createCRUD<T>(collection, schema, adapter, options?) -------------------

Non-React version of useCRUD. Identical CRUD operations (create, get, update,
delete, list) but without useQuery / useListQuery. Use in scripts, CLI tools,
background workers, or any context without a React tree.

  createCRUD<T extends Record<string, any>>(
    collection: string,
    schema:     Schema<T>,
    adapter:    StorageAdapter,
    options?:   CreateCRUDOptions,
  ): CRUDMethods<T>

  CreateCRUDOptions:
    migrations?:   Record<number, MigrationFn>
    autoMigrate?:  boolean

  CRUDMethods<T>:
    create(data: T): Promise<string>
    get(id: string): Promise<T | null>
    update(id: string, partial: Partial<T>): Promise<void>
    delete(id: string): Promise<void>
    list(query?: Query): Promise<T[]>

---- Query -----------------------------------------------------------------

All list() calls and useListQuery() accept an optional Query object.

  interface Query {
    where?:   WhereClause        Field-level filter conditions.
    orderBy?: string             Field name to sort by.
    order?:   'asc' | 'desc'    Sort direction. Default: 'asc'.
    limit?:   number             Maximum records to return.
    offset?:  number             Number of records to skip.
  }

  WhereClause:
    Record<string, value | Partial<Record<ComparisonOp, value>>>

  ComparisonOp values:
    '$eq'        Equal (same as plain value)
    '$ne'        Not equal
    '$gt'        Greater than
    '$gte'       Greater than or equal
    '$lt'        Less than
    '$lte'       Less than or equal
    '$in'        Value is in array
    '$contains'  String contains substring

---- Adapters ---------------------------------------------------------------

MemoryAdapter
  In-memory. No persistence. Works on every target. Useful for tests and
  development.

    new MemoryAdapter()
    .clear()          Wipes all data across all collections.

Love2DFileAdapter
  Love2D target only. Reads/writes files via love.filesystem RPC.
  Data lives in the Love2D save directory: save/<collection>/<id>.<ext>

    new Love2DFileAdapter({ rpc, format? })

    Options:
      rpc     (method, args?) => Promise<T>   Required. Pass the result of
                                              useLoveRPC or bridge.rpc.
      format  'json' | 'markdown' | 'text'   Default: 'json'.

TerminalSQLiteAdapter
  Terminal target. Uses Node.js v22+ built-in node:sqlite for a SQLite file.
  Each collection is a separate table. Synchronous reads/writes wrapped in
  async interface.

    new TerminalSQLiteAdapter({ dbPath? })

    Options:
      dbPath  string   Path to SQLite file. Default: './data.db'.

    Extra methods:
      rawQuery(sql, params?): Promise<any[]>   Parameterized raw SQL.
      transaction(fn): Promise<void>           Wraps fn in BEGIN/COMMIT/ROLLBACK.
      close(): void                            Release the database file handle.

LocalStorageAdapter
  Web target. Uses window.localStorage. Suitable for small data (< 5 MB total).

    new LocalStorageAdapter({ prefix? })

    Options:
      prefix  string   Key prefix to namespace this adapter. Default: 'ilr:'.

    Extra method:
      clearCollection(collection): Promise<void>

IndexedDBAdapter
  Web target. Uses window.indexedDB. Better for larger datasets.

    new IndexedDBAdapter({ dbName?, collections? })

    Options:
      dbName?      string     IndexedDB database name. Default: 'reactjit-storage'.
      collections? string[]   Known collection names for initial schema creation.

    Extra method:
      clearCollection(collection): Promise<void>

---- Migrations -------------------------------------------------------------

Migrations track schema evolution via a _version integer on each record. When
autoMigrate is true, records are migrated on every read and written back with
the new version.

  migrations: Record<number, MigrationFn>
  MigrationFn = (data: any) => any

  Migrations run sequentially. A record at version 0 with migrations defined
  for versions 1, 2, 3 will have all three functions applied in order.

  Helper utilities (importable from @reactjit/storage):
    getLatestVersion(migrations): number
    migrateRecord(data, migrations, autoMigrate): any
    stampVersion(data, migrations): any
    createMigratingAdapter(adapter, migrations, autoMigrate): StorageAdapter

=== EXAMPLES ===

---- Define a schema and type -----------------------------------------------

import { z } from '@reactjit/storage';

const NoteSchema = z.object({
  id:        z.string(),
  title:     z.string().min(1).max(100),
  body:      z.string(),
  pinned:    z.boolean().default(false),
  createdAt: z.number(),
});

type Note = z.infer<typeof NoteSchema>;

---- Love2D app: wire up StorageProvider ------------------------------------

import { StorageProvider, Love2DFileAdapter } from '@reactjit/storage';
import { useLoveRPC } from '@reactjit/core';

function AppWithStorage({ children }) {
  const rpc = useLoveRPC;   // pass the hook factory, not a result
  const adapter = new Love2DFileAdapter({ rpc });
  return (
    <StorageProvider adapter={adapter}>
      {children}
    </StorageProvider>
  );
}

---- useCRUD basic usage ----------------------------------------------------

import { useCRUD, z } from '@reactjit/storage';

function NotesScreen() {
  const Notes = useCRUD('notes', NoteSchema);
  const { data, loading, refetch } = Notes.useListQuery();

  async function addNote() {
    await Notes.create({
      id: '',                   // omit or leave empty — id is generated
      title: 'New note',
      body: '',
      createdAt: Date.now(),
    });
    refetch();
  }

  if (loading) return <Text fontSize={14}>Loading...</Text>;

  return (
    <Box width="100%" height="100%">
      {data.map(note => (
        <Text key={note.id} fontSize={16}>{note.title}</Text>
      ))}
      <Pressable onPress={addNote}>
        <Text fontSize={14}>Add note</Text>
      </Pressable>
    </Box>
  );
}

---- useCRUD reactive single-record query -----------------------------------

function NoteDetail({ id }: { id: string }) {
  const Notes = useCRUD('notes', NoteSchema);
  const { data: note, loading, error } = Notes.useQuery(id);

  if (loading) return <Text fontSize={14}>Loading...</Text>;
  if (error)   return <Text fontSize={14}>Error: {error.message}</Text>;
  if (!note)   return <Text fontSize={14}>Not found</Text>;

  return <Text fontSize={16}>{note.title}</Text>;
}

---- Query with filtering and sorting ---------------------------------------

const { data } = Notes.useListQuery({
  where:   { pinned: true },
  orderBy: 'createdAt',
  order:   'desc',
  limit:   20,
});

// Operator syntax
const recent = await Notes.list({
  where: { createdAt: { $gt: Date.now() - 86400000 } },
});

---- Inline adapter override (no StorageProvider needed) --------------------

const localAdapter = new MemoryAdapter();

function Standalone() {
  const Items = useCRUD('items', ItemSchema, { adapter: localAdapter });
  // ...
}

---- createCRUD outside React (script / CLI) --------------------------------

import { createCRUD, TerminalSQLiteAdapter, z } from '@reactjit/storage';

const db = new TerminalSQLiteAdapter({ dbPath: './app.db' });

const Users = createCRUD('users', z.object({
  id:    z.string(),
  email: z.string().email(),
  name:  z.string(),
}), db);

await Users.create({ id: 'u1', email: 'alice@example.com', name: 'Alice' });
const all = await Users.list();
console.log(all);
db.close();

---- Migrations: rename a field between versions ----------------------------

const migrations = {
  // Version 1: add missing 'email' field to old records
  1: (data) => ({ ...data, email: data.email ?? '' }),
  // Version 2: rename 'username' to 'name'
  2: (data) => {
    const { username, ...rest } = data;
    return { ...rest, name: username ?? rest.name };
  },
};

const Users = useCRUD('users', UserSchema, {
  migrations,
  autoMigrate: true,   // migrate records transparently on read
});

import { StorageProvider, LocalStorageAdapter } from '@reactjit/storage';

const adapter = new LocalStorageAdapter({ prefix: 'myapp:' });

function App() {

const adapter = new IndexedDBAdapter({

---- Schema: optional fields and defaults ------------------------------------

const TaskSchema = z.object({
  id:          z.string(),
  title:       z.string().min(1),
  done:        z.boolean().default(false),
  priority:    z.enum(['low', 'medium', 'high']).default('medium'),
  tags:        z.array(z.string()).default([]),
  dueDate:     z.date().optional(),
  description: z.string().nullable().default(null),
});

type Task = z.infer<typeof TaskSchema>;

---- safeParse for validation without throwing --------------------------------

const result = NoteSchema.safeParse(untrustedInput);
if (!result.success) {
  console.error(result.error.issues);
} else {
  await Notes.create(result.data);
}
