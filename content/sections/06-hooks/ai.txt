=== OVERVIEW ===
The @reactjit/ai package brings LLM integration into ReactJIT apps as composable hooks. It supports OpenAI-compatible APIs and Anthropic natively, handles token streaming over SSE, and runs full agentic tool execution loops without leaving React. Provider configuration is shared via context so API keys and model selection are declared once at the app root and inherited by every hook beneath.

The package is organized into four capability layers:

  - Chat and completion — useChat, useCompletion
  - Provider utilities — useModels, useAIConfig, AIProvider
  - Key storage — useAPIKeys
  - Tool integrations — useBrowser, createBrowserTools, useMCPServer

All hooks handle streaming, loading state, and errors uniformly. Tool calls in useChat run concurrently and loop back to the model automatically — the agent loop is built in.

=== API / SYNTAX ===

--- AIProvider ---

Wrap your app (or a subtree) to set default AI configuration for all hooks below it. Any hook option passed directly to useChat / useCompletion overrides the context value.

  import { AIProvider } from '@reactjit/ai';

  <AIProvider config={AIConfig}>
    {children}
  </AIProvider>

AIConfig fields:
  provider      AIProviderType   'openai' | 'anthropic' | 'custom'
  model         string           Model ID (e.g. 'gpt-4o', 'claude-opus-4-6')
  apiKey        string?          API key for the provider
  baseURL       string?          Override the API base URL (useful for local models)
  temperature   number?          Sampling temperature (0–2)
  maxTokens     number?          Max tokens in the response
  systemPrompt  string?          System prompt injected into every conversation
  proxy         string?          SOCKS5 or HTTP proxy URL

--- useAIConfig ---

Read the current AI configuration from context. Returns null when no AIProvider ancestor is present; hooks handle this gracefully.

  import { useAIConfig } from '@reactjit/ai';

  const config: AIConfig | null = useAIConfig();

Parameters: none
Returns: AIConfig | null

--- useChat ---

Conversational AI with streaming, message history, and a built-in agentic tool loop. Supports both OpenAI and Anthropic. Streams each token into the live message list as it arrives. When the model returns tool calls, executes them concurrently and feeds results back for the next round — automatically, up to maxToolRounds.

  import { useChat } from '@reactjit/ai';

  const result: ChatResult = useChat(options?: ChatOptions);

ChatOptions fields (all optional — values override the AIProvider context):
  provider        AIProviderType          Provider to use
  model           string                  Model ID
  apiKey          string                  API key
  baseURL         string                  Override base URL
  temperature     number                  Sampling temperature
  maxTokens       number                  Max response tokens
  systemPrompt    string                  System-level instruction
  proxy           string                  SOCKS5/HTTP proxy
  initialMessages Message[]              Pre-load conversation history
  tools           ToolDefinition[]        Tool definitions the model may call
  maxToolRounds   number                  Max agentic loop rounds (default: 10)
  onChunk         (chunk: string) => void Called for each streamed token
  onToolCall      (call: ToolCall) => void Called when the model invokes a tool
  onError         (error: Error) => void  Called on any error

Returns: ChatResult
  messages    Message[]                                  Full conversation history including tool calls and results
  send        (content: string) => Promise<void>         Send a user message and stream the response
  isLoading   boolean                                    True while any round (LLM or tools) is in flight
  isStreaming  boolean                                   True while tokens are arriving from the model
  stop        () => void                                 Abort the current stream and tool loop
  error       Error | null                               Last error, or null
  setMessages (msgs: Message[] | ((prev: Message[]) => Message[])) => void   Direct message history manipulation

Message fields:
  role         MessageRole   'system' | 'user' | 'assistant' | 'tool'
  content      string | ContentBlock[]
  toolCalls    ToolCall[]?   Tool invocations on assistant messages
  toolCallId   string?       On tool result messages, the matching call ID
  name         string?       Tool name on result messages

ToolCall fields:
  id         string   Unique call identifier
  name       string   Tool name
  arguments  string   JSON-encoded argument string

ToolDefinition fields:
  name         string                          Tool name the model uses
  description  string                          Natural language description
  parameters   Record<string, any>             JSON Schema for the arguments
  execute      (args: any) => Promise<any>     Function called when the model invokes the tool

--- useCompletion ---

Single-shot text completion with streaming. Does not maintain conversation history — each call to complete() is independent. Ideal for summaries, transformations, and one-off generation tasks.

  import { useCompletion } from '@reactjit/ai';

  const result: CompletionResult = useCompletion(options?: CompletionOptions);

CompletionOptions fields (all optional — override AIProvider context):
  provider      AIProviderType
  model         string
  apiKey        string
  baseURL       string
  temperature   number
  maxTokens     number
  systemPrompt  string
  proxy         string
  onChunk       (chunk: string) => void
  onError       (error: Error) => void

Returns: CompletionResult
  completion   string                               Full accumulated text so far (updates during streaming)
  complete     (prompt: string) => Promise<string>  Run a completion, resolves with the full text
  isLoading    boolean
  isStreaming  boolean
  stop         () => void
  error        Error | null

--- useModels ---

Fetch the list of available models from a provider. For OpenAI-compatible endpoints, calls GET /v1/models. For Anthropic, returns a static curated list (Anthropic has no models endpoint). Fetches automatically on mount and whenever provider/baseURL/apiKey changes.

  import { useModels } from '@reactjit/ai';

  const result: ModelsResult = useModels(options?: Partial<AIConfig>);

Parameters:
  options  Partial<AIConfig>  Same fields as AIConfig — override context values to query a specific provider

Returns: ModelsResult
  models   ModelInfo[]   Sorted list of available models
  loading  boolean
  error    Error | null
  refetch  () => void    Manually trigger a fresh fetch

ModelInfo fields:
  id        string           Model identifier string (e.g. 'gpt-4o')
  name      string           Display name (equals id for OpenAI-compat)
  provider  AIProviderType

--- useAPIKeys ---

Manage stored API keys. Persists keys across sessions via @reactjit/storage when a StorageProvider ancestor is present; falls back to in-memory storage without one. Keys are stored in a collection named 'ai_keys'.

  import { useAPIKeys } from '@reactjit/ai';

  const result: APIKeysResult = useAPIKeys();

Parameters: none

Returns: APIKeysResult
  keys       APIKeyRecord[]                                            All stored key records
  setKey     (record: Omit<APIKeyRecord, 'id'> & { id?: string }) => Promise<string>   Upsert a key record, returns the assigned ID
  deleteKey  (id: string) => Promise<void>                            Delete a key by ID
  getKey     (provider: AIProviderType) => APIKeyRecord | undefined   Lookup the first key for a provider
  loading    boolean                                                   True while loading keys from storage on mount

APIKeyRecord fields:
  id        string           Auto-generated or caller-supplied identifier
  provider  AIProviderType   Provider this key belongs to
  apiKey    string           The raw API key string
  label     string?          Human-readable label (e.g. 'Work account')
  baseURL   string?          Custom base URL associated with this key
  models    string[]?        Specific models this key has access to

--- useBrowser ---

Control a stealth browse session (anti-fingerprint Firefox) from React components. The browser runs as a separate process on the host machine, listening on TCP port 7331. Commands route through Lua's browse.lua worker thread via the QuickJS bridge. Returns loading and error state so you can reflect browser activity in the UI.

  import { useBrowser } from '@reactjit/ai';

  const browser: BrowseResult = useBrowser(options?: BrowseOptions);

BrowseOptions fields:
  host  string   Browse session host (default: '127.0.0.1')
  port  number   Browse session TCP port (default: 7331)

Returns: BrowseResult
  navigate        (url: string) => Promise<PageContent>                            Go to a URL
  click           (selector: string) => Promise<PageContent>                       Click an element by CSS selector
  typeText        (selector: string, text: string) => Promise<PageContent>         Type into an element (does NOT submit)
  extractContent  () => Promise<PageContent>                                       Re-extract current page content
  screenshot      (path?: string) => Promise<string>                               Take a screenshot (returns base64 PNG or saves to path)
  back            () => Promise<PageContent>                                       Go back in browser history
  forward         () => Promise<PageContent>                                       Go forward in browser history
  refresh         () => Promise<PageContent>                                       Reload the current page
  executeJs       (script: string) => Promise<any>                                 Run JavaScript in the page context
  listTabs        () => Promise<{ index: number; title: string; url: string }[]>   List all open tabs
  useTab          (index: number) => Promise<PageContent>                          Switch to a tab by index
  openTab         (url: string) => Promise<void>                                   Open a new background tab
  ping            () => Promise<boolean>                                           Check if the browse session is reachable
  raw             (cmd: Record<string, any>) => Promise<any>                       Send an arbitrary command
  loading         boolean                                                           True while a command is in flight
  error           Error | null

PageContent fields:
  url    string
  title  string
  text   string
  links  { text: string; href: string }[]
  forms  { id: string; action: string; inputs: any[] }[]

--- createBrowserTools ---

Build a ToolDefinition[] array that gives an AI agent full control over the browser. Pass the result directly into useChat's tools option. The agent can then navigate, click, type, take screenshots, and manage tabs autonomously using natural language instructions.

  import { createBrowserTools } from '@reactjit/ai';

  const tools: ToolDefinition[] = createBrowserTools(options?: BrowseOptions);

Parameters:
  options  BrowseOptions  Same host/port options as useBrowser

Returns: ToolDefinition[]

Tools exposed to the model:
  browser_navigate     Go to a URL
  browser_click        Click an element by CSS selector
  browser_type         Type text into an input field
  browser_extract      Re-extract current page content (for AJAX updates)
  browser_back         Go back in history
  browser_tabs         List open tabs
  browser_use_tab      Switch to a tab by index
  browser_open_tab     Open a new background tab
  browser_execute_js   Run JavaScript in page context
  browser_screenshot   Take a screenshot

Page content returned to the model is automatically truncated to 8000 characters and 50 links to avoid flooding the context window.

--- useMCPServer ---

Connect to an MCP (Model Context Protocol) server and expose its tools as ToolDefinitions ready to pass into useChat. Handles the full lifecycle: connect, initialize handshake, tool discovery, permission filtering, and optional confirmation gates. Reconnects automatically when transport parameters change.

  import { useMCPServer } from '@reactjit/ai';

  const result: MCPServerResult = useMCPServer(config: MCPServerConfig);

MCPServerConfig fields:
  name         string               Human-readable name for the server (used in error messages)
  transport    MCPTransportType     'stdio' | 'sse' | 'streamable-http'
  command      string?              (stdio) Command to spawn (e.g. 'npx')
  args         string[]?            (stdio) Arguments to the command
  env          Record<string,string>? (stdio) Extra environment variables
  url          string?              (sse / streamable-http) Server endpoint URL
  headers      Record<string,string>? (sse / streamable-http) Extra request headers
  timeout      number?              Request timeout in ms (default: 30000)
  permissions  MCPPermissionsConfig?  Allowlist of tools to expose (required — no permissions = no tools)
  onConfirm    (toolName: string, args: any) => Promise<boolean>?  Gate for tools with confirm: true

MCPPermissionsConfig shape:
  tools   Record<string, MCPToolPermission>   Tool name → permission entry
  lastDiscovered   string?                    ISO timestamp of last tool discovery
  tokenBudget      { totalIfAllEnabled: number; note: string }?

MCPToolPermission fields:
  enabled        boolean   Whether to expose this tool to the model
  confirm        boolean   Whether to call onConfirm before executing
  description    string?   Override the tool description shown to the model
  inputSchema    Record<string, any>?   Override the tool's input schema
  tokenEstimate  number?   Pre-computed token cost of this tool's definition

Returns: MCPServerResult
  status          'connecting' | 'ready' | 'error'   Current connection state
  tools           ToolDefinition[]                    Enabled tools, ready for useChat
  availableTools  string[]                            All tool names discovered from the server (regardless of permissions)
  error           Error | null
  disconnect      () => void                          Manually close the connection

--- MCPClient ---

Low-level MCP client class. Used by useMCPServer internally and available for imperative use cases such as the CLI linter, build-time tool discovery, or custom React effects.

  import { MCPClient } from '@reactjit/ai';

  const client = new MCPClient(config: MCPServerConfig);

Methods:
  connect()                                       Connect and complete the initialize handshake
  listTools(): Promise<MCPTool[]>                 Discover all tools from the server
  callTool(name, args): Promise<MCPToolCallResult>  Call a named tool
  close()                                         Close the connection
  getServerInfo()                                 Return the server name and version from the handshake

Properties:
  status  MCPClientStatus   'disconnected' | 'connecting' | 'ready' | 'error'
  error   Error | null

MCPTool fields:
  name         string
  description  string
  inputSchema  Record<string, any>   JSON Schema for arguments

--- estimateToolTokens / estimateToolBudget ---

Estimate the context window cost of tool definitions before passing them to a model. Useful for deciding which MCP tools to enable when working with tight context budgets.

  import { estimateToolTokens, estimateToolBudget } from '@reactjit/ai';

  const tokens: number = estimateToolTokens(tool: MCPTool);
  const budget: { total: number; note: string } = estimateToolBudget(tools: MCPTool[]);

estimateToolTokens:
  Parameters: tool  MCPTool
  Returns: number — estimated token count for that tool's definition

estimateToolBudget:
  Parameters: tools  MCPTool[]
  Returns: { total: number; note: string } — total tokens and a human-readable percentage of a 128K context window

=== EXAMPLES ===

-- Basic chat UI --

A minimal streaming chat interface backed by Anthropic Claude:

  import { AIProvider, useChat } from '@reactjit/ai';
  import { Box, Text, TextInput, Pressable, ScrollView } from '@reactjit/core';

  // Wrap the app once with provider config
  function App() {
    return (
      <AIProvider config={{ provider: 'anthropic', model: 'claude-opus-4-6', apiKey: 'sk-ant-...' }}>
        <ChatScreen />
      </AIProvider>
    );
  }

  function ChatScreen() {
    const { messages, send, isLoading } = useChat({
      systemPrompt: 'You are a helpful assistant embedded in a Love2D application.',
    });
    const [input, setInput] = useState('');

    const submit = async () => {
      if (!input.trim() || isLoading) return;
      const text = input;
      setInput('');
      await send(text);
    };

    return (
      <Box width="100%" height="100%" flexDirection="column" padding={12} gap={8}>
        <ScrollView width="100%" flexGrow={1}>
          {messages.map((msg, i) => (
            <Box key={i} backgroundColor={msg.role === 'user' ? '#1e3a5f' : '#2a2a2a'} padding={8} marginBottom={4}>
              <Text fontSize={11} color="#888">{msg.role}</Text>
              <Text fontSize={13}>{typeof msg.content === 'string' ? msg.content : ''}</Text>
            </Box>
          ))}
        </ScrollView>
        <Box flexDirection="row" gap={8} width="100%">
          <TextInput
            value={input}
            onChangeText={setInput}
            placeholder="Type a message..."
            fontSize={13}
            flexGrow={1}
          />
          <Pressable
            onPress={submit}
            backgroundColor={isLoading ? '#555' : '#4CAF50'}
            padding={8}
          >
            <Text fontSize={13}>{isLoading ? 'Wait...' : 'Send'}</Text>
          </Pressable>
        </Box>
      </Box>
    );
  }

-- Agent with tools --

Wire an agent that can fetch weather data using a custom tool:

  import { useChat } from '@reactjit/ai';
  import type { ToolDefinition } from '@reactjit/ai';
  import { Box, Text } from '@reactjit/core';

  const weatherTool: ToolDefinition = {
    name: 'get_weather',
    description: 'Get the current temperature and conditions for a city.',
    parameters: {
      type: 'object',
      properties: {
        city: { type: 'string', description: 'City name' },
      },
      required: ['city'],
    },
    execute: async ({ city }: { city: string }) => {
      // Replace with a real API call
      return { city, temperature: 22, conditions: 'Partly cloudy' };
    },
  };

  function WeatherAgent() {
    const { messages, send, isLoading, isStreaming } = useChat({
      provider: 'openai',
      model: 'gpt-4o',
      apiKey: 'sk-...',
      tools: [weatherTool],
      maxToolRounds: 5,
      onToolCall: (call) => {
        console.log('Tool called:', call.name, call.arguments);
      },
    });

    return (
      <Box width="100%" flexDirection="column" gap={8} padding={12}>
        <Box onPress={() => send("What is the weather like in Tokyo and London?")}>
          <Text fontSize={14}>Ask about weather</Text>
        </Box>
        {isStreaming && <Text fontSize={12} color="#888">Thinking...</Text>}
        {messages.filter(m => m.role === 'assistant' && typeof m.content === 'string').map((m, i) => (
          <Text key={i} fontSize={13}>{m.content as string}</Text>
        ))}
      </Box>
    );
  }

-- API key settings screen --

Let users enter and persist their own API keys:

  import { AIProvider, useAPIKeys, useModels } from '@reactjit/ai';
  import { StorageProvider } from '@reactjit/storage';
  import { Box, Text, TextInput, Pressable } from '@reactjit/core';
  import { useState } from 'react';

  function SettingsScreen() {
    return (
      // StorageProvider is required for useAPIKeys to persist keys to disk
      <StorageProvider adapter="love">
        <APIKeyForm />
      </StorageProvider>
    );
  }

  function APIKeyForm() {
    const { keys, setKey, getKey, loading } = useAPIKeys();
    const [draft, setDraft] = useState('');

    const openaiKey = getKey('openai');

    const save = async () => {
      await setKey({ provider: 'openai', apiKey: draft, label: 'OpenAI' });
      setDraft('');
    };

    if (loading) return <Text fontSize={13}>Loading...</Text>;

    return (
      <Box flexDirection="column" gap={12} padding={16}>
        <Text fontSize={14}>OpenAI API Key</Text>
        {openaiKey && (
          <Text fontSize={12} color="#888">
            Saved: {openaiKey.apiKey.slice(0, 8)}...
          </Text>
        )}
        <TextInput
          value={draft}
          onChangeText={setDraft}
          placeholder="sk-..."
          fontSize={13}
          secureTextEntry
        />
        <Pressable onPress={save} backgroundColor="#4CAF50" padding={8}>
          <Text fontSize={13}>Save Key</Text>
        </Pressable>
      </Box>
    );
  }

-- AI agent with browser and MCP tools --

Combine browser control and an MCP filesystem server into a single agent:

  import { useChat, useBrowser, createBrowserTools, useMCPServer } from '@reactjit/ai';
  import { Box, Text, Pressable } from '@reactjit/core';
  import mcpPermissions from '../mcp.tools.json';

  function ResearchAgent() {
    // Connect to a local MCP filesystem server
    const filesystem = useMCPServer({
      name: 'filesystem',
      transport: 'stdio',
      command: 'npx',
      args: ['-y', '@modelcontextprotocol/server-filesystem', '/tmp/research'],
      permissions: mcpPermissions.filesystem,
      onConfirm: async (toolName, args) => {
        // Show a confirmation dialog before destructive operations
        if (toolName.includes('write') || toolName.includes('delete')) {
          return window.confirm(`Allow ${toolName}?`);
        }
        return true;
      },
    });

    // Combine browser tools with filesystem tools
    const allTools = [
      ...createBrowserTools(),
      ...filesystem.tools,
    ];

    const { messages, send, isLoading } = useChat({
      provider: 'anthropic',
      model: 'claude-opus-4-6',
      tools: allTools,
      maxToolRounds: 20,
      systemPrompt: 'You are a research agent. Browse the web to gather information and save findings to /tmp/research.',
    });

    const status = filesystem.status;

    return (
      <Box width="100%" flexDirection="column" gap={8} padding={12}>
        <Text fontSize={12} color={status === 'ready' ? '#4CAF50' : '#888'}>
          Filesystem MCP: {status}
        </Text>
        <Pressable
          onPress={() => send('Research the latest ReactJIT releases and save a summary to /tmp/research/notes.txt')}
          backgroundColor={isLoading ? '#555' : '#1565C0'}
          padding={10}
        >
          <Text fontSize={13}>Start Research</Text>
        </Pressable>
        {messages.filter(m => m.role === 'assistant').slice(-1).map((m, i) => (
          <Text key={i} fontSize={12} color="#ccc">
            {typeof m.content === 'string' ? m.content.slice(0, 300) : ''}
          </Text>
        ))}
      </Box>
    );
  }
