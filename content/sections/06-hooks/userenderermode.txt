=== OVERVIEW ===
Returns the current renderer mode as either `'web'` or `'native'`. The mode is set by `<RendererProvider>` at the root of the application. Primitives like `Box`, `Text`, and `Image` use this internally to switch between DOM elements (web) and reconciler instances (native). Application code can use it for conditional rendering based on the rendering target.

=== API / SYNTAX ===
Import:
```typescript
import { useRendererMode, RendererProvider } from '@reactjit/core';
```

Hook signature:
```typescript
useRendererMode(): RendererMode
```

Type:
```typescript
type RendererMode = 'web' | 'native';
```

Return value: `'web'` or `'native'`

Provider signature:
```typescript
<RendererProvider mode={'web' | 'native'}>
  {children}
</RendererProvider>
```

Example 1: Conditional feature based on renderer
---
function InteractiveDemo() {
  const mode = useRendererMode();

  return (
    <Box>
      <Text fontSize={16}>Renderer: {mode}</Text>
      {mode === 'native' && (
        <Text fontSize={14}>Inspector available (F12)</Text>
      )}
      {mode === 'web' && (
        <Text fontSize={14}>Use browser DevTools</Text>
      )}
    </Box>
  );
}

Example 2: Platform-specific styling

function Button({ label, onPress }) {
  const mode = useRendererMode();

  const buttonStyle = {
    padding: mode === 'web' ? 12 : 16,
    backgroundColor: mode === 'web' ? '#007bff' : '#0066cc',
    borderRadius: mode === 'web' ? 4 : 0,
  };

  return (
    <Pressable onPress={onPress}>
      <Box style={buttonStyle}>
        <Text fontSize={14} color="#ffffff">{label}</Text>
      </Box>
    </Pressable>
  );
}

Example 3: Conditional hook usage
---
function AnimatedComponent() {
  const mode = useRendererMode();

  // useSpring works on all targets, but might have different configs
  const springConfig = mode === 'web'
    ? { tension: 170, friction: 26 }
    : { tension: 120, friction: 20 };

  const spring = useSpring({
    from: { opacity: 0 },
    to: { opacity: 1 },
    config: springConfig,
  });

  return <Box style={{ opacity: spring.opacity }}>...</Box>;
}

Example 4: Debug overlay for native targets
---
function DebugOverlay({ visible, stats }) {
  const mode = useRendererMode();

  // Only show debug overlay on native targets
  if (mode !== 'native' || !visible) return null;

  return (
    <Box style={{
      position: 'absolute',
      top: 0,
      left: 0,
      padding: 8,
      backgroundColor: 'rgba(0, 0, 0, 0.8)',
    }}>
      <Text fontSize={12} color="#00ff00">FPS: {stats.fps}</Text>
      <Text fontSize={12} color="#00ff00">Memory: {stats.memory}MB</Text>
    </Box>
  );
}

Example 5: Provider setup at app root
---
import { createRoot } from '@reactjit/native';
import { RendererProvider } from '@reactjit/core';

function App() {
  return (
    <Box style={{ width: '100%', height: '100%' }}>
      <Text fontSize={24}>Hello ReactJIT</Text>
    </Box>
  );
}

// Native target setup
const root = createRoot();
root.render(
  <RendererProvider mode="native">
    <App />
  </RendererProvider>
);

Example 7: Feature detection pattern
---
function FeatureGate({ feature, children }) {
  const mode = useRendererMode();

  const features = {
    web: ['dom-events', 'css-animations'],
    native: ['inspector', 'lua-bridge', 'visual-debugger'],
  };

  const available = features[mode].includes(feature);

  return available ? <>{children}</> : null;
}

function Demo() {
  return (
    <Box>
      <FeatureGate feature="inspector">
        <Text fontSize={14}>Press F12 to open inspector</Text>
      </FeatureGate>
      <FeatureGate feature="dom-events">
        <Text fontSize={14}>Right-click for context menu</Text>
      </FeatureGate>
    </Box>
  );
}

Example 8: Logging wrapper
---
function Logger({ message }) {
  const mode = useRendererMode();

  useEffect(() => {
    if (mode === 'native') {
      // Logs to Love2D console or Lua print
      console.log(`[NATIVE] ${message}`);
    } else {
      // Logs to browser DevTools
      console.log(`[WEB] ${message}`);
    }
  }, [message, mode]);

  return null;
}

=== CRITICAL RULES ===
- Must be used within a `<RendererProvider>`. Throws error if context is missing.
- The mode is static for the lifetime of the app. Switching modes requires remounting the entire tree.
- Primitives (`Box`, `Text`, `Image`) automatically detect mode. Application code rarely needs to switch on mode unless implementing platform-specific features.
- `'native'` mode = Love2D, terminal, nvim, cc, hs, awesome targets. `'web'` mode = browser DOM.
- Default context value is `'web'` if no provider exists (for compatibility), but production apps should always use explicit provider.
- Do NOT use this for target-specific builds. Use esbuild `--define` or environment variables for compile-time target detection.
- Use this for runtime conditional rendering, not bundling logic.
- Mode does NOT indicate specific target (Love2D vs terminal vs nvim). Use environment variables or bridge queries for target-specific logic.
