=== OVERVIEW ===
useBridge returns the IBridge instance from React context. This is the low-level escape hatch -- most code should use the higher-level hooks (useLoveRPC, useLoveEvent, useLoveSend) instead. Use useBridge when you need direct access to the bridge for custom communication patterns.

The bridge must be provided by a BridgeProvider higher in the tree. Calling useBridge outside a provider throws an error.

=== API / SYNTAX ===
```tsx
import { useBridge } from '@reactjit/core';
```

Signature:
```tsx
function useBridge(): IBridge
```

IBridge interface:
| Method                                          | Description                                |
|-------------------------------------------------|--------------------------------------------|
| send(type: string, payload?: any)               | Queue a command for Lua (batched)           |
| flush()                                         | Write all queued commands to Lua            |
| subscribe(type: string, fn: Listener): Unsub    | Listen for events from Lua                 |
| rpc<T>(method: string, args?, timeout?): Promise | Call a Lua function, await response        |
| setState(key: string, value: any)               | Set a shared state key                      |
| isReady(): boolean                              | Whether transport is initialized            |
| onReady(callback: () => void)                   | Register callback for bridge ready          |
| destroy()                                       | Tear down transport resources               |

=== EXAMPLES ===
Example 1: Direct RPC call
---
```tsx
import { useBridge } from '@reactjit/core';

function SaveButton() {
  const bridge = useBridge();

  const handleSave = async () => {
    await bridge.rpc('file:save', { path: 'data.json', content: '{}' });
  };

  return (
    <Pressable onPress={handleSave}>
      <Text>Save</Text>
    </Pressable>
  );
}
```
---
Platforms: love2d

Example 2: Custom event subscription
---
```tsx
import { useBridge } from '@reactjit/core';

function useCustomPoller(key: string) {
  const bridge = useBridge();
  const [data, setData] = useState(null);

  useEffect(() => {
    const unsub = bridge.subscribe(`data:${key}`, (payload) => {
      setData(payload);
    });
    return unsub;
  }, [bridge, key]);

  return data;
}
```
---
Platforms: love2d

=== CRITICAL RULES ===
- useBridge throws if called outside a BridgeProvider. The provider is set up automatically by the renderer -- you only need to worry about this if building custom root components.
- Prefer the higher-level hooks (useLoveRPC, useLoveEvent, useLoveSend) over raw bridge access. They handle cleanup and ref stability for you.
- The bridge instance is stable across re-renders. It is safe to use in dependency arrays.
