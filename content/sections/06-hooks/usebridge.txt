=== OVERVIEW ===
Returns the `IBridge` transport instance that handles communication between JavaScript (React) and the native runtime (Lua for Love2D, Node.js IPC for grid targets). Allows direct access to low-level bridge methods: pushing events, calling Lua functions, subscribing to bridge events, and querying runtime state. Required for custom integrations, native API bindings, and performance-critical operations that bypass the reconciler. Only works on native targets — throws error if used in web mode.

=== API / SYNTAX ===
Import:
```typescript
import { useBridge, BridgeProvider } from '@reactjit/core';
import type { IBridge } from '@reactjit/core';
```

Hook signature:
```typescript
useBridge(): IBridge
```

Return value: `IBridge` instance

Provider signature:
```typescript
<BridgeProvider bridge={IBridge}>
  {children}
</BridgeProvider>
```

IBridge interface (simplified):
```typescript
interface IBridge {
  // Push mutation to native runtime
  push(mutation: Mutation): void;

  // Call Lua function and get result (Love2D only)
  call(fn: string, ...args: any[]): any;

  // Subscribe to bridge events
  on(event: string, handler: Function): void;
  off(event: string, handler: Function): void;

  // Query runtime capabilities
  query(key: string): any;

  // Batch mutations (performance)
  batch(fn: () => void): void;
}
```

Common bridge events:
- `pointerDown`, `pointerUp`, `pointerMove`
- `keyDown`, `keyUp`, `textInput`
- `resize`, `focus`, `blur`
- `tick` (frame event, Love2D only)
- Custom events from Lua via `love.event.push('reactEvent', ...)`

Example 1: Direct Lua function call (Love2D)
---
function GameAPI() {
  const bridge = useBridge();

  const loadLevel = (levelId: number) => {
    // Calls Lua: love.handlers.loadLevel(levelId)
    const result = bridge.call('love.handlers.loadLevel', levelId);
    console.log('Level loaded:', result);
  };

  return (
    <Pressable onPress={() => loadLevel(1)}>
      <Text fontSize={16}>Load Level 1</Text>
    </Pressable>
  );
}

Example 2: Subscribe to custom bridge events
---
function CustomEventListener() {
  const bridge = useBridge();
  const [message, setMessage] = useState('');

  useEffect(() => {
    const handler = (data: any) => {
      setMessage(data.text);
    };

    bridge.on('customEvent', handler);
    return () => bridge.off('customEvent', handler);
  }, [bridge]);

  return <Text fontSize={14}>{message}</Text>;
}

Example 3: Query runtime capabilities
---
function RuntimeInfo() {
  const bridge = useBridge();
  const [info, setInfo] = useState(null);

  useEffect(() => {
    const capabilities = {
      version: bridge.query('version'),
      platform: bridge.query('platform'),
      renderer: bridge.query('renderer'),
    };
    setInfo(capabilities);
  }, [bridge]);

  if (!info) return null;

  return (
    <Box>
      <Text fontSize={14}>Version: {info.version}</Text>
      <Text fontSize={14}>Platform: {info.platform}</Text>
      <Text fontSize={14}>Renderer: {info.renderer}</Text>
    </Box>
  );
}

Example 4: Batched mutations for performance
---
function BulkUpdate({ items }) {
  const bridge = useBridge();

  const updateAll = () => {
    // Batches all mutations in the callback into a single bridge roundtrip
    bridge.batch(() => {
      items.forEach(item => {
        // Each update normally triggers a bridge push
        // Batch mode accumulates them
        updateItem(item);
      });
    });
  };

  return (
    <Pressable onPress={updateAll}>
      <Text fontSize={16}>Update {items.length} items</Text>
    </Pressable>
  );
}

function DirectMutation() {
  const bridge = useBridge();

  const setProperty = () => {
    // Bypass reconciler: push mutation directly
    bridge.push({
      type: 'setProperty',
      nodeId: 123,
      property: 'backgroundColor',
      value: '#ff0000',

  // WARNING: This bypasses React's reconciler state.
  // Only use for performance-critical hot paths or native integrations.

Example 6: Frame tick subscription (Love2D)
---
function GameLoop() {
  const bridge = useBridge();
  const [frame, setFrame] = useState(0);
  const [deltaTime, setDeltaTime] = useState(0);

  useEffect(() => {
    const tick = (dt: number) => {
      setDeltaTime(dt);
      setFrame(f => f + 1);
    };

    bridge.on('tick', tick);
    return () => bridge.off('tick', tick);
  }, [bridge]);

  const fps = deltaTime > 0 ? Math.round(1 / deltaTime) : 0;

  return (
    <Box>
      <Text fontSize={14}>Frame: {frame}</Text>
      <Text fontSize={14}>FPS: {fps}</Text>
    </Box>
  );
}

function FileLoader() {
  const bridge = useBridge();
  const [content, setContent] = useState('');

  const loadFile = async (path: string) => {

      </Pressable>
      {content && (
        <Text fontSize={12}>{content}</Text>
      )}
    </Box>
  );
}

=== CRITICAL RULES ===
- Must be used within a `<BridgeProvider>`. Throws error if context is missing.
- Only works on native targets (Love2D, terminal, nvim, cc, hs, awesome). Web mode has no bridge.
- Direct mutations bypass React reconciler state. Use sparingly and only for performance-critical hot paths.
- Event handlers registered via `bridge.on()` MUST be cleaned up via `bridge.off()` in useEffect cleanup. Memory leaks otherwise.
- `bridge.call()` is synchronous on Love2D (QuickJS FFI), asynchronous on grid targets (IPC). Check target before use.
- Batched mutations improve performance for bulk updates (10+ mutations per frame). Non-batched is fine for typical component updates.
- Bridge events are lowercase (`pointerDown`, not `PointerDown`). Event names are target-specific — check bridge implementation.
- `bridge.query()` is target-specific. Love2D supports `version`, `platform`, `renderer`. Grid targets vary.
- Mutation format is internal and unstable. Prefer reconciler methods (setState, props) unless implementing a native binding.
- Bridge is NOT reactive. Changes pushed directly don't trigger React re-renders. Use state if you need reactivity.
