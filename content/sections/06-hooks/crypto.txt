=== OVERVIEW ===
The @reactjit/crypto package exposes pure-JavaScript cryptographic primitives that work across every ReactJIT target, including QuickJS inside Love2D. All implementations come from the @noble suite — a set of audited, dependency-free libraries that are widely used in production (millions of weekly downloads, no native code). The package is organized into four functional groups: hashing (sha256, sha512, BLAKE2b/2s/3, HMAC), symmetric encryption (AES-256-GCM, ChaCha20-Poly1305 with scrypt KDF), asymmetric signatures/key exchange (Ed25519, X25519), and token generation. Encoding utilities (hex, base64) are also re-exported. None of these are React hooks — they are synchronous or Promise-returning functions you call directly from hooks, effects, or event handlers.

=== API / SYNTAX ===

--- sha256 ---

Compute a SHA-256 digest.

  import { sha256 } from '@reactjit/crypto';

  const result: HashResult = sha256(input: string | Uint8Array);

Parameters:
  input  string | Uint8Array   Data to hash (strings are UTF-8 encoded)

Returns: HashResult
  hex    string       Lowercase hex digest (64 chars)
  base64 string       Base64-encoded digest
  bytes  Uint8Array   Raw 32-byte digest

--- sha512 ---

Compute a SHA-512 digest.

  import { sha512 } from '@reactjit/crypto';

  const result: HashResult = sha512(input: string | Uint8Array);

Parameters:
  input  string | Uint8Array

Returns: HashResult (128-char hex, 64 raw bytes)

--- hash_blake2b ---

BLAKE2b with configurable output length. Default: 32 bytes.

  import { hash_blake2b } from '@reactjit/crypto';

  const result: HashResult = hash_blake2b(input: string | Uint8Array, bytes?: number);

Parameters:
  input  string | Uint8Array
  bytes  number   Output length in bytes. Default: 32. Range: 1–64.

Returns: HashResult

--- hash_blake2s ---

BLAKE2s (always 32 bytes). Faster than BLAKE2b on 32-bit platforms.

  import { hash_blake2s } from '@reactjit/crypto';

  const result: HashResult = hash_blake2s(input: string | Uint8Array);

Parameters:
  input  string | Uint8Array

Returns: HashResult

--- hash_blake3 ---

BLAKE3 with configurable output length. Default: 32 bytes. Fastest of the three BLAKE variants.

  import { hash_blake3 } from '@reactjit/crypto';

  const result: HashResult = hash_blake3(input: string | Uint8Array, bytes?: number);

Parameters:
  input  string | Uint8Array
  bytes  number   Output length in bytes. Default: 32.

Returns: HashResult

--- hmacSHA256 ---

HMAC-SHA256. Use for message authentication, API request signing, and webhook signature generation.

  import { hmacSHA256 } from '@reactjit/crypto';

  const result: HashResult = hmacSHA256(key: string | Uint8Array, message: string | Uint8Array);

Parameters:
  key      string | Uint8Array   Shared secret
  message  string | Uint8Array   Message to authenticate

Returns: HashResult (64-char hex)

--- hmacSHA512 ---

HMAC-SHA512. Same semantics as hmacSHA256 with a 128-char hex output.

  import { hmacSHA512 } from '@reactjit/crypto';

  const result: HashResult = hmacSHA512(key: string | Uint8Array, message: string | Uint8Array);

Returns: HashResult

--- timingSafeEqual ---

Compare two byte sequences in constant time. Safe for comparing secrets — the comparison time does not depend on where the first difference appears.

  import { timingSafeEqual } from '@reactjit/crypto';

  const equal: boolean = timingSafeEqual(a: string | Uint8Array, b: string | Uint8Array);

Parameters:
  a  string | Uint8Array
  b  string | Uint8Array

Returns: boolean — false immediately (with length leak) if lengths differ; otherwise constant-time value comparison.

--- encrypt ---

Password-based authenticated encryption using scrypt KDF + AES-256-GCM (default) or ChaCha20-Poly1305. Returns a JSON-serializable EncryptedData envelope that bundles the algorithm, ciphertext, nonce, salt, and KDF parameters together so the receiver does not need to remember them separately.

  import { encrypt } from '@reactjit/crypto';

  const envelope: EncryptedData = encrypt(plaintext: string, password: string, options?: EncryptOptions);

Parameters:
  plaintext  string          Data to encrypt
  password   string          User-provided password (derived to a 256-bit key via scrypt)
  options    EncryptOptions  Optional configuration

EncryptOptions fields:
  algorithm  'aes-256-gcm' | 'chacha20-poly1305' | 'xchacha20-poly1305'
             Default: 'aes-256-gcm'. Choose chacha20-poly1305 on platforms without hardware AES acceleration.

Returns: EncryptedData
  algorithm   'aes-256-gcm' | 'chacha20-poly1305' | 'xchacha20-poly1305'
  ciphertext  string   Base64-encoded ciphertext (includes GCM/Poly1305 authentication tag)
  nonce       string   Base64-encoded nonce/IV
  salt        string   Base64-encoded 32-byte salt
  kdf         'scrypt'
  kdfParams   { N, r, p }   scrypt cost parameters (N=131072, r=8, p=1 by default)

--- decrypt ---

Decrypt an EncryptedData envelope produced by encrypt. Reads KDF parameters from the envelope, re-derives the key, and authenticates + decrypts.

  import { decrypt } from '@reactjit/crypto';

  const plaintext: string = decrypt(data: EncryptedData, password: string);

Parameters:
  data      EncryptedData   Envelope from encrypt (or stored JSON)
  password  string          The same password used to encrypt

Returns: string — decrypted plaintext
Throws on wrong password (authentication tag mismatch) or malformed input.

--- encryptRaw ---

Encrypt raw bytes with a caller-managed 32-byte key. No KDF step. Use this when you already have a key from a Diffie-Hellman exchange or another key derivation flow.

  import { encryptRaw } from '@reactjit/crypto';

  const { ciphertext, nonce } = encryptRaw(
    plaintext: Uint8Array,
    key: Uint8Array,
    algo?: 'aes-256-gcm' | 'chacha20-poly1305' | 'xchacha20-poly1305',
  );

Parameters:
  plaintext  Uint8Array   Data to encrypt
  key        Uint8Array   32-byte key
  algo       string       Encryption algorithm. Default: 'aes-256-gcm'

Returns: { ciphertext: Uint8Array, nonce: Uint8Array }

--- decryptRaw ---

Decrypt raw bytes with a caller-managed key and nonce.

  import { decryptRaw } from '@reactjit/crypto';

  const plaintext: Uint8Array = decryptRaw(
    ciphertext: Uint8Array,
    key: Uint8Array,
    nonce: Uint8Array,
    algo?: 'aes-256-gcm' | 'chacha20-poly1305' | 'xchacha20-poly1305',
  );

Parameters:
  ciphertext  Uint8Array
  key         Uint8Array   32-byte key
  nonce       Uint8Array   Nonce used during encryption
  algo        string       Must match the algorithm used during encryption. Default: 'aes-256-gcm'

Returns: Uint8Array — decrypted bytes
Throws on authentication failure.

--- generateSigningKeys ---

Generate an Ed25519 key pair for digital signatures.

Parameters: none

Returns: KeyPair

--- sign ---

Sign a message string with an Ed25519 private key.

  import { sign } from '@reactjit/crypto';

  const signed: SignedMessage = sign(privateKeyHex: string, message: string);

Parameters:
  privateKeyHex  string   Hex-encoded private key from generateSigningKeys
  message        string   Message to sign

Returns: SignedMessage
  message    string   The original message
  signature  string   Hex-encoded Ed25519 signature (128 chars)
  publicKey  string   Hex-encoded public key derived from the private key
  algorithm  'ed25519'

--- diffieHellman ---

Compute a shared secret from your X25519 private key and the other party's public key. Both sides derive the same secret without transmitting it.

  import { diffieHellman } from '@reactjit/crypto';

  const sharedSecret: string = diffieHellman(privateKeyHex: string, publicKeyHex: string);

Parameters:
  privateKeyHex  string   Your hex-encoded X25519 private key
  publicKeyHex   string   The other party's hex-encoded X25519 public key

Returns: string — hex-encoded 32-byte shared secret

=== EXAMPLES ===

-- File integrity checker --

Hash files on disk and verify them later to detect tampering:

  import { sha256, toHex } from '@reactjit/crypto';
  import { useState } from 'react';
  import { Box, Text } from '@reactjit/core';

  // Precomputed SHA-256 of known-good assets
  const EXPECTED: Record<string, string> = {
    'config.json': 'b94d27b9934d3e08a52e52d7da7dabfac484efe04294e576b3736efb0cd78896',
    'bundle.js':   'e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855',
  };

  function IntegrityChecker({ content, filename }: { content: string; filename: string }) {
    const actual = sha256(content).hex;
    const expected = EXPECTED[filename];
    const ok = expected ? actual === expected : null;

    return (
      <Box padding={10} backgroundColor={ok === false ? '#400' : '#1a1a1a'}>
        <Text fontSize={13}>{filename}</Text>
        <Text fontSize={11}>{actual.slice(0, 16)}...</Text>
        {ok === false && <Text fontSize={11}>MISMATCH</Text>}
        {ok === true && <Text fontSize={11}>OK</Text>}
        {ok === null && <Text fontSize={11}>Unknown</Text>}
      </Box>
    );
  }

-- Password-encrypted settings store --

Encrypt user settings with a password before writing to disk, decrypt on load:

  import { encrypt, decrypt } from '@reactjit/crypto';
  import { useState } from 'react';
  import { Box, Text } from '@reactjit/core';

  // Encryption is synchronous but scrypt is CPU-heavy — run in an effect or worker
  function SettingsPanel({ password }: { password: string }) {
    const [saved, setSaved] = useState(false);

    function handleSave(settings: Record<string, unknown>) {
      // Serialize settings to JSON, encrypt with user's password
      const plaintext = JSON.stringify(settings);
      const envelope = encrypt(plaintext, password, { algorithm: 'chacha20-poly1305' });

      // envelope is a plain object — safe to JSON.stringify and write to disk
      const stored = JSON.stringify(envelope);

      // In a real app: write `stored` to a file via Love2D FFI or useLoveRPC
      console.log('Encrypted:', stored.slice(0, 60) + '...');
      setSaved(true);
    }

    function handleLoad(storedJson: string) {
      const envelope = JSON.parse(storedJson);
      const plaintext = decrypt(envelope, password);
      const settings = JSON.parse(plaintext);
      console.log('Loaded settings:', settings);
    }

    return (
      <Box padding={16} flexDirection="column" gap={8}>
        <Box
          backgroundColor="#2196F3"
          padding={10}
          onPress={() => handleSave({ theme: 'dark', fontSize: 14 })}
        >
          <Text fontSize={14}>Save Settings</Text>
        </Box>
        {saved && <Text fontSize={12}>Saved (encrypted)</Text>}
      </Box>
    );
  }

-- End-to-end encrypted chat channel --

Use X25519 DH to establish a shared secret, then encrypt each message with that secret:

  import {
    generateDHKeys,
    diffieHellman,
    encryptRaw,
    decryptRaw,
    randomBytes,
    toHex,
    fromHex,
  } from '@reactjit/crypto';

  // Each peer generates a DH key pair and publishes their public key
  const alice = generateDHKeys();
  const bob   = generateDHKeys();

  // Exchange public keys over any channel (can be plaintext)
  // Both sides derive the same shared secret
  const aliceShared = diffieHellman(alice.privateKey, bob.publicKey);
  const bobShared   = diffieHellman(bob.privateKey, alice.publicKey);
  // aliceShared === bobShared

  // Encode shared secret as a key for encryptRaw
  const key = fromHex(aliceShared); // 32 bytes

  // Alice encrypts a message
  const msgBytes = new TextEncoder().encode('Hello Bob');
  const { ciphertext, nonce } = encryptRaw(msgBytes, key, 'chacha20-poly1305');

  // Bob decrypts using the same shared key
  const decrypted = decryptRaw(ciphertext, fromHex(bobShared), nonce, 'chacha20-poly1305');
  const message = new TextDecoder().decode(decrypted);
  // message === 'Hello Bob'

  // Wire this up to useWebSocket or useServer for a live encrypted channel
