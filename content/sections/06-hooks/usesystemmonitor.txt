=== OVERVIEW ===
useSystemMonitor returns comprehensive system monitoring data at htop/nvtop level. It polls via useLuaInterval and provides per-core CPU usage, detailed memory breakdown (including buffers, cached, swap), process list, GPU stats (NVIDIA/AMD), network interface I/O rates, disk I/O rates, and task counts.

Every section carries a `.toSysLog(path)` method for structured JSON logging.

=== API / SYNTAX ===
```tsx
import { useSystemMonitor, formatRate, formatTotalBytes } from '@reactjit/core';
```

Signature:
```tsx
function useSystemMonitor(
  interval?: number,
  opts?: { processLimit?: number }
): SystemMonitor
```

Types:
```tsx
interface SystemMonitor {
  cpu: CpuInfo;
  memory: DetailedMemory;
  processes: ProcessInfo[];
  tasks: TaskCounts;
  gpu: GpuInfo[] | null;
  network: NetworkInterface[];
  disk: DiskDevice[];
  loading: boolean;
  toSysLog: (path: string) => void;
}

interface CpuInfo {
  cores: CoreInfo[];
  total: number;                    // Overall CPU usage %
  loadAvg: [number, number, number]; // 1, 5, 15 min load averages
}

interface CoreInfo {
  id: number;
  usage: number;   // Total usage %
  user: number;    // User-space %
  system: number;  // Kernel %
  iowait: number;  // I/O wait %
}

interface DetailedMemory {
  total: number; used: number; free: number;
  available: number; buffers: number; cached: number;
  swap: { total: number; used: number };
  unit: 'GiB';
}

interface ProcessInfo {
  user: string; pid: number; cpu: number; mem: number;
  vsz: number; rss: number; tty: string; stat: string; command: string;
}

interface GpuInfo {
  name: string; vendor: 'nvidia' | 'amd';
  utilization: number; memUsed: number; memTotal: number;
  temperature: number; power: number;
}

interface NetworkInterface {
  name: string; rxBytes: number; txBytes: number;
  rxRate: number; txRate: number;
}

interface DiskDevice {
  name: string; readRate: number; writeRate: number;
  reads: number; writes: number;
}
```

Parameters:
| Param            | Type   | Default | Description                        |
|------------------|--------|---------|------------------------------------|
| interval         | number | 1000    | Ms between refreshes               |
| opts.processLimit| number | 20      | Max processes to return            |

Helpers:
| Function         | Signature                    | Description              |
|------------------|------------------------------|--------------------------|
| formatRate       | (bytesPerSec: number) => string | "1.2 MB/s"           |
| formatTotalBytes | (bytes: number) => string    | "4.2 GB"                 |

=== EXAMPLES ===
Example 1: CPU and memory dashboard
---
```tsx
import { Box, Text } from '@reactjit/core';
import { useSystemMonitor } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function MonitorPanel() {
  const colors = useThemeColors();
  const sys = useSystemMonitor(1000);

  return (
    <Box style={{ gap: 8, padding: 16 }}>
      <Text style={{ color: colors.text }}>
        {`CPU: ${sys.cpu.total.toFixed(1)}% | Load: ${sys.cpu.loadAvg.join(', ')}`}
      </Text>
      <Text style={{ color: colors.text }}>
        {`RAM: ${sys.memory.used.toFixed(1)} / ${sys.memory.total.toFixed(1)} GiB`}
      </Text>
      {sys.gpu?.[0] && (
        <Text style={{ color: colors.text }}>
          {`GPU: ${sys.gpu[0].name} ${sys.gpu[0].utilization}% ${sys.gpu[0].temperature}C`}
        </Text>
      )}
      <Text style={{ color: colors.muted }}>
        {`Tasks: ${sys.tasks.running} running, ${sys.tasks.sleeping} sleeping`}
      </Text>
    </Box>
  );
}
```

=== CRITICAL RULES ===
- Requires the Lua bridge. Returns empty/loading state without it.
- GPU data is only available when nvidia-smi (NVIDIA) or rocm-smi (AMD) is installed. Returns null otherwise.
- processLimit caps the number of processes returned to avoid serializing thousands of entries across the bridge each poll.
- Pick a unique interval value to avoid timer collisions. See useLuaInterval docs.
