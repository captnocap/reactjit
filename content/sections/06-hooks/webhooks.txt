=== OVERVIEW ===
The @reactjit/webhooks package covers both sides of the webhook pattern: receiving HTTP POST payloads from external services (GitHub, Stripe, CI systems, etc.) and sending signed JSON payloads to external endpoints. The receiver hook (useWebhook) starts an HTTP server via the Love2D bridge and verifies HMAC-SHA256 signatures using the same algorithm GitHub, Slack, and most modern platforms use. The sender hook (useWebhookSender) wraps fetch with automatic signing, exponential-backoff retries, and React state for loading/error tracking. An imperative sendWebhook function is also exported for use outside React.

=== API / SYNTAX ===

--- useWebhook ---

Start an HTTP listener and receive incoming webhook events as React state. Each arriving request is parsed, optionally signature-verified, and prepended to the events array. Requires the Love2D HTTP server bridge (Lua-side httpserver module).

  import { useWebhook } from '@reactjit/webhooks';

  const result: WebhookReceiverResult = useWebhook(
    port: number | null,
    path: string,
    options?: WebhookReceiverOptions,
  );

Parameters:
  port     number | null   Port to listen on. Pass null to disable the server.
  path     string          URL path to handle (e.g. '/webhook/github')
  options  WebhookReceiverOptions   Optional configuration (see below)

WebhookReceiverOptions fields:
  secret           string     Shared secret for HMAC-SHA256 signature verification. If omitted, event.verified is always null.
  signatureHeader  string     Header name carrying the signature. Default: 'x-hub-signature-256' (GitHub format).
  maxEvents        number     Maximum events to keep in the queue (newest first). Default: 100.
  responseStatus   number     HTTP status code sent back to the caller. Default: 200.
  responseBody     string     Response body sent back. Default: '{"ok":true}'.
  methods          string[]   Filter by HTTP method. Default: accepts all methods.
  host             string     Host/interface to bind to. Default: Lua server default.

Returns: WebhookReceiverResult
  events         WebhookEvent[]      All received events, newest first (up to maxEvents)
  latest         WebhookEvent | null The most recently received event, or null if empty
  ready          boolean             True once the HTTP server is bound and listening
  totalReceived  number              Running count of all events received (including evicted ones)
  clear          () => void          Empty the event queue without stopping the server

WebhookEvent fields:
  id        string                        Auto-generated unique event ID (e.g. 'wh_1700000000_abc123')
  timestamp string                        ISO timestamp of when the event was received
  method    string                        HTTP method (usually 'POST')
  path      string                        Request path
  headers   Record<string, string>        All request headers
  body      T                             Parsed body — JSON object if Content-Type is application/json, raw string otherwise
  rawBody   string                        Original unparsed request body
  query     Record<string, string>        Query parameters from the URL
  verified  boolean | null                HMAC verification result. true = valid, false = invalid, null = no secret configured

--- useWebhookSender ---

React hook for sending webhook payloads with loading and error state tracking. Wraps sendWebhook with default options that apply to every call made through the returned send function.

  import { useWebhookSender } from '@reactjit/webhooks';

  const result: WebhookSenderResult = useWebhookSender(options?: WebhookSenderOptions);

Parameters:
  options  WebhookSenderOptions   Optional defaults for all sends

WebhookSenderOptions fields:
  headers  Record<string, string>   Default headers merged into every request
  secret   string                   Default shared secret for HMAC signing
  retries  number                   Default retry count. Default: 3

Returns: WebhookSenderResult
  send        (url: string, payload: any, options?: WebhookSendOptions) => Promise<WebhookSendResult>
              Send a payload. Per-call options override the hook-level defaults.
  sending     boolean                  True while a send is in flight
  lastResult  WebhookSendResult | null Result of the last completed send
  error       Error | null             Error from the last failed send, or null

WebhookSendResult fields:
  ok        boolean   True if the server responded with a 2xx status
  status    number    HTTP response status code (0 on network failure)
  body      string    Raw response body
  attempts  number    Total number of attempts made (1 + retries used)

--- sendWebhook (imperative) ---

Send a webhook payload to a URL without React state. Use this in event handlers, Lua RPC callbacks, or any context where a hook is not appropriate.

  import { sendWebhook } from '@reactjit/webhooks';

  const result: WebhookSendResult = await sendWebhook(
    url: string,
    payload: any,
    options?: WebhookSendOptions,
  );

Parameters:
  url      string               Destination URL
  payload  any                  Payload to send. Objects are serialized as JSON automatically.
  options  WebhookSendOptions   Optional per-call configuration

WebhookSendOptions fields:
  method           string                   HTTP method. Default: 'POST'
  headers          Record<string, string>   Additional headers merged with Content-Type: application/json
  secret           string                   Shared secret — generates an HMAC-SHA256 signature header
  signatureHeader  string                   Header name for the signature. Default: 'x-hub-signature-256'
  retries          number                   Retry count on 5xx or network failure. Default: 0 (no retries)
  retryDelay       number                   Base delay in ms between retries. Doubles each retry (exponential backoff). Default: 1000

Returns: Promise<WebhookSendResult>
Does not throw — errors are captured in the result object.

--- hmacSHA256 (utility) ---

Compute an HMAC-SHA256 hex digest. Exported for use in custom signing schemes or signature verification outside of webhooks. Backed by @noble/hashes (audited).

  import { hmacSHA256 } from '@reactjit/webhooks';

  const hex: string = hmacSHA256(key: string, message: string);

Parameters:
  key      string   Shared secret
  message  string   Message to sign

Returns: string — lowercase hex digest (64 characters)

--- timingSafeEqual (utility) ---

Compare two hex strings in constant time to prevent timing side-channel attacks. Use this when comparing received signatures against computed ones.

  import { timingSafeEqual } from '@reactjit/webhooks';

  const equal: boolean = timingSafeEqual(a: string, b: string);

Parameters:
  a  string   First hex string
  b  string   Second hex string

Returns: boolean — false immediately if lengths differ (length leakage is acceptable; value is not)

=== EXAMPLES ===

-- GitHub webhook receiver --

Listen for GitHub push events, verify the signature, and display the latest commit message:

  import { useWebhook } from '@reactjit/webhooks';
  import { Box, Text } from '@reactjit/core';

  const SECRET = 'my-github-webhook-secret';

  function GitHubListener() {
    const { events, latest, ready } = useWebhook(9090, '/webhook/github', {
      secret: SECRET,
      methods: ['POST'],
      maxEvents: 20,
    });

    return (
      <Box width="100%" height="100%" flexDirection="column" gap={4} padding={12}>
        <Box flexDirection="row" gap={8} alignItems="center">
          <Box
            width={8}
            height={8}
            backgroundColor={ready ? '#4CAF50' : '#888'}
          />
          <Text fontSize={14}>{ready ? 'Listening on :9090' : 'Starting...'}</Text>
        </Box>

        {latest && (
          <Box padding={10} backgroundColor="#1c1c1c">
            <Text fontSize={12}>
              {latest.verified === false ? '[UNVERIFIED] ' : ''}
              {latest.body?.head_commit?.message ?? 'No commit message'}
            </Text>
            <Text fontSize={10}>{latest.timestamp}</Text>
          </Box>
        )}

        <Text fontSize={11}>Total received: {events.length}</Text>
      </Box>
    );
  }

-- Webhook sender with retry and signing --

Send a deployment notification to an external service, showing progress in the UI:

  import { useWebhookSender } from '@reactjit/webhooks';
  import { Box, Text } from '@reactjit/core';

  const ENDPOINT = 'https://hooks.example.com/deploy';
  const SECRET = 'shared-deploy-secret';

  function DeployButton({ version }: { version: string }) {
    const { send, sending, lastResult, error } = useWebhookSender({
      secret: SECRET,
      retries: 3,
    });

    async function handleDeploy() {
      await send(ENDPOINT, {
        event: 'deploy',
        version,
        timestamp: new Date().toISOString(),
      });
    }

    return (
      <Box flexDirection="column" gap={8} padding={16}>
        <Box
          backgroundColor={sending ? '#555' : '#2196F3'}
          padding={12}
          onPress={sending ? undefined : handleDeploy}
        >
          <Text fontSize={14}>{sending ? 'Deploying...' : 'Deploy'}</Text>
        </Box>

        {lastResult && (
          <Text fontSize={12}>
            {lastResult.ok
              ? `Sent (${lastResult.attempts} attempt(s))`
              : `Failed: HTTP ${lastResult.status}`}
          </Text>
        )}

        {error && (
          <Text fontSize={12}>{error.message}</Text>
        )}
      </Box>
    );
  }

-- Event log dashboard --

Receive webhooks from multiple sources on different paths and display a unified event log:

  import { useWebhook } from '@reactjit/webhooks';
  import { Box, Text } from '@reactjit/core';

  function WebhookDashboard() {
    const github = useWebhook(9090, '/hooks/github', { secret: 'gh-secret' });
    const stripe = useWebhook(9090, '/hooks/stripe', { secret: 'stripe-secret' });

    // Merge events from both sources and sort by timestamp
    const allEvents = [
      ...github.events.map((e) => ({ ...e, source: 'GitHub' })),
      ...stripe.events.map((e) => ({ ...e, source: 'Stripe' })),
    ].sort((a, b) => b.timestamp.localeCompare(a.timestamp));

    return (
      <Box width="100%" height="100%" flexDirection="column">
        <Box padding={10} backgroundColor="#222">
          <Text fontSize={15}>Webhook Log</Text>
        </Box>

        {allEvents.map((event) => (
          <Box
            key={event.id}
            flexDirection="row"
            gap={8}
            padding={8}
            backgroundColor={event.verified === false ? '#300' : '#111'}
          >
            <Box backgroundColor="#333" padding={4}>
              <Text fontSize={10}>{event.source}</Text>
            </Box>
            <Text fontSize={12}>
              {event.method} {event.path}
            </Text>
            {event.verified === false && (
              <Text fontSize={10}>BAD SIG</Text>
            )}
          </Box>
        ))}
      </Box>
    );
  }
