=== OVERVIEW ===
useHotState is a drop-in replacement for useState that survives HMR (hot module replacement). During `reactjit dev`, when you save a file, the JS context is destroyed and recreated -- but Lua keeps running. useHotState stores values in a Lua memory table that persists across reloads.

On HMR, the Lua reload path injects all stored atoms into `globalThis.__hotstateCache` before the new bundle evaluates. useHotState reads from this cache synchronously on first render -- zero flash, zero async delay. The value appears instantly as if the reload never happened.

Unlike useLocalStore (SQLite, survives restarts), useHotState is purely in-memory. Close the app and the state is gone. Use it for ephemeral UI state you do not want to lose on every code change: sidebar open/closed, scroll position, selected tab, editor cursor position.

=== API / SYNTAX ===
```tsx
import { useHotState } from '@reactjit/core';
```

Signature:
```tsx
function useHotState<T>(
  key: string,
  defaultValue: T
): [T, (value: T | ((prev: T) => T)) => void]
```

Parameters:
| Parameter    | Type   | Description                                      |
|--------------|--------|--------------------------------------------------|
| key          | string | Unique key identifying this atom across reloads  |
| defaultValue | T      | Initial value on fresh start (no cache)          |

Returns:
| Index | Type                                      | Description                   |
|-------|-------------------------------------------|-------------------------------|
| [0]   | T                                         | Current value                 |
| [1]   | (value: T or (prev: T) => T) => void      | Setter (supports updaters)    |

=== EXAMPLES ===
Example 1: Sidebar state that survives HMR
---
```tsx
import { useHotState } from '@reactjit/core';

function Sidebar() {
  const [open, setOpen] = useHotState('sidebar', true);

  if (!open) return null;

  return (
    <Box style={{ width: 250, backgroundColor: '#1e1e2e' }}>
      <Text>Sidebar content</Text>
      <Pressable onPress={() => setOpen(false)}>
        <Text>Close</Text>
      </Pressable>
    </Box>
  );
}
```
---
Platforms: love2d

Example 2: Selected tab persists across code changes
---
```tsx
import { useHotState } from '@reactjit/core';

function SettingsPanel() {
  const [tab, setTab] = useHotState('settings.tab', 'general');

  return (
    <Box>
      <Box style={{ flexDirection: 'row', gap: 8 }}>
        <Pressable onPress={() => setTab('general')}><Text>General</Text></Pressable>
        <Pressable onPress={() => setTab('display')}><Text>Display</Text></Pressable>
        <Pressable onPress={() => setTab('audio')}><Text>Audio</Text></Pressable>
      </Box>
      <Text>{`Active tab: ${tab}`}</Text>
    </Box>
  );
}
```
---
Platforms: love2d

=== CRITICAL RULES ===
- useHotState survives HMR but NOT app restarts. For persistent state, use useLocalStore.
- The key must be globally unique across the entire app. Use dotted namespaces like `'settings.tab'` or `'sidebar.width'` to avoid collisions.
- Writes go to Lua immediately (no debounce, unlike useLocalStore). This is a memory table write, not disk I/O.
- The injection cache is read synchronously on first render. There is no loading state or async delay -- the value appears instantly after HMR.
- The setter supports updater functions: `setValue(prev => prev + 1)`.
