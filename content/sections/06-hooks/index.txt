=== OVERVIEW ===
Hooks are the React-side API for talking to Lua. Every hook in this section consumes the bridge via React context -- they work identically regardless of the transport layer (QuickJS FFI or WASM Module.FS).

React declares layout and diffs the tree. Hooks are how you reach beyond that into the runtime: subscribe to events, call Lua functions, share state, read the clipboard, bind hotkeys, persist data.

All hooks import from `@reactjit/core`:

```tsx
import { useLove, useLoveEvent, useLoveRPC, useLoveState, useLoveReady, useLoveSend } from '@reactjit/core';
import { useWindowDimensions, useHotkey, useClipboard } from '@reactjit/core';
import { useLocalStore, useHotState, useBreakpoint, useBridge } from '@reactjit/core';
```

Bridge hooks (React <-> Lua):
- useBridge -- direct access to the bridge instance
- useLove -- subscribe to an event and get a send function
- useLoveEvent -- subscribe to a Lua event (fire-and-forget listener)
- useLoveRPC -- call a Lua function and await the result
- useLoveState -- bidirectional shared state between React and Lua
- useLoveReady -- true when the bridge is initialized
- useLoveSend -- fire-and-forget message to Lua
- useLoveOverlays -- position overlay windows in WASM mode

Window and input hooks:
- useWindowDimensions -- reactive viewport width/height
- useWindowSize -- declaratively set window size
- useWindowPosition -- declaratively set window position
- useWindowAlwaysOnTop -- pin/unpin window on top
- useHotkey -- bind keyboard shortcuts
- useClipboard -- read/write system clipboard

State hooks:
- useLocalStore -- persistent state backed by SQLite (survives restarts)
- useHotState -- in-memory state that survives HMR (but not restarts)
- useBreakpoint -- responsive breakpoint based on viewport width

Networking:
- useFetch -- HTTP requests via Lua fetch polyfill
- useWebSocket -- persistent WebSocket connection
- usePeerServer -- host a WebSocket server for P2P
- useScrape -- declarative web scraping with CSS selectors

Timers and automation:
- useLuaInterval -- Lua-side interval timer
- useIFTTT -- declarative trigger-action wiring (If This Then That)

System monitoring:
- useSystemInfo -- OS, CPU, memory, uptime
- useSystemMonitor -- htop-level per-core CPU, GPU, processes, network, disk
- usePorts -- open port monitoring with process kill

Search:
- useSearch -- synchronous in-memory search
- useFuzzySearch -- scored fuzzy matching
- useAsyncSearch -- async search with loading state
- useSearchHistory -- persistent search history (SQLite)
- useCommandSearch -- command palette filtering

Events:
- useEventBus -- create an in-memory event bus
- useEvent -- subscribe to bus channels
- useEmit -- emit events on a channel

Terminal:
- usePTY -- PTY terminal sessions with output accumulation
- useSemanticTerminal -- classified PTY with playback controls

Multi-agent:
- useFleet -- manage concurrent Claude Code sessions

Utilities:
- useId, useUUID -- ID generation via Lua
- useTimeAgo, useFormatDate -- date/time formatting via Lua
- useScale -- viewport-proportional scaling
- useDebug -- expose data for inspector/console
- useOverlay -- game overlay mode control
- useGifRecorder, useRecorder -- capture as GIF/MP4

Hardware:
- usePin, usePWM, useSerial, useI2C -- GPIO hardware hooks

Discovery:
- useCapabilities -- list all registered Lua capabilities and their schemas
