=== OVERVIEW ===
A family of search hooks for filtering, ranking, and managing search queries. All hooks are stateless and pure React -- they deal with data, not input. Input lifecycle (debouncing, keystrokes) lives in Lua-owned components like SearchBar.

The family includes:
- useSearch -- synchronous substring filter
- useFuzzySearch -- scored fuzzy matching with ranking
- useAsyncSearch -- async fetcher with loading/error state
- useSearchHistory -- persistent search history backed by SQLite
- useSearchHighlight -- split text into matched/unmatched segments
- useCommandSearch -- fuzzy-rank CommandDef entries for command palettes
- useSearchSchema -- inspect which fields are being searched

When no `key` is specified, all hooks auto-detect searchable fields by inspecting the first item's string and number properties.

=== API / SYNTAX ===
```tsx
import {
  useSearch, useFuzzySearch, useAsyncSearch,
  useSearchHistory, useSearchHighlight, useCommandSearch,
  useSearchSchema, detectSearchableFields,
} from '@reactjit/core';
```

useSearch:
```tsx
function useSearch<T>(items: T[], query: string, options?: UseSearchOptions<T>): T[]
```

useFuzzySearch:
```tsx
function useFuzzySearch<T>(items: T[], query: string, options?: UseFuzzySearchOptions<T>): {
  results: FuzzySearchResult<T>[];
  items: T[];
}
```

useAsyncSearch:
```tsx
function useAsyncSearch<T>(
  fetcher: (query: string) => Promise<T[]>,
  query: string,
  options?: UseAsyncSearchOptions
): { results: T[]; loading: boolean; error: Error | null }
```

useSearchHistory:
```tsx
function useSearchHistory(options?: UseSearchHistoryOptions): {
  history: string[];
  push: (query: string) => void;
  remove: (query: string) => void;
  clear: () => void;
}
```

useSearchHighlight:
```tsx
function useSearchHighlight(text: string, query: string): HighlightPart[]
```

useCommandSearch:
```tsx
function useCommandSearch(commands: CommandDef[], query: string, options?: { limit?: number }): CommandDef[]
```

Options:
| Option         | Type                        | Default   | Hook       | Description                     |
|----------------|-----------------------------|-----------|------------|---------------------------------|
| key            | keyof T \| (keyof T)[]      | auto      | search/fuzzy | Field(s) to search            |
| matcher        | (item, query) => boolean    | undefined | search     | Custom match function           |
| limit          | number                      | unlimited | all        | Max results                     |
| minLength      | number                      | 0 or 1   | search/fuzzy/async | Min query length          |
| showAllOnEmpty | boolean                     | true      | search/fuzzy | Return all items when empty   |
| sortByScore    | boolean                     | true      | fuzzy      | Sort by match quality           |
| minScore       | number                      | 1         | fuzzy      | Minimum score threshold         |
| debounce       | number                      | 0         | async      | Debounce ms                     |
| maxEntries     | number                      | 20        | history    | Max persisted entries           |
| storeKey       | string                      | 'searchHistory' | history | SQLite store key          |

=== EXAMPLES ===
Example 1: Basic substring search
---
```tsx
import { useState } from 'react';
import { Box, Text } from '@reactjit/core';
import { useSearch } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

const ITEMS = [
  { id: 1, name: 'Alice', role: 'Admin' },
  { id: 2, name: 'Bob', role: 'User' },
  { id: 3, name: 'Carol', role: 'Admin' },
];

export function UserSearch({ query }: { query: string }) {
  const colors = useThemeColors();
  const results = useSearch(ITEMS, query, { key: 'name', limit: 10 });

  return (
    <Box style={{ gap: 4 }}>
      {results.map(user => (
        <Text key={user.id} style={{ color: colors.text }}>{user.name}</Text>
      ))}
    </Box>
  );
}
```

Example 2: Fuzzy search with scores
---
```tsx
import { Box, Text } from '@reactjit/core';
import { useFuzzySearch } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

const COMMANDS = [
  { id: 1, label: 'Open File', group: 'File' },
  { id: 2, label: 'Save As', group: 'File' },
  { id: 3, label: 'Find Replace', group: 'Edit' },
];

export function FuzzyResults({ query }: { query: string }) {
  const colors = useThemeColors();
  const { results } = useFuzzySearch(COMMANDS, query, { key: 'label' });

  return (
    <Box style={{ gap: 4 }}>
      {results.map(({ item, score }) => (
        <Text key={item.id} style={{ color: colors.text }}>
          {`${item.label} (score: ${score})`}
        </Text>
      ))}
    </Box>
  );
}
```

Example 3: Highlight matching text
---
```tsx
import { Box, Text } from '@reactjit/core';
import { useSearchHighlight } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function HighlightedResult({ text, query }: { text: string; query: string }) {
  const colors = useThemeColors();
  const parts = useSearchHighlight(text, query);

  return (
    <Box style={{ flexDirection: 'row' }}>
      {parts.map((part, i) => (
        <Text
          key={i}
          style={{
            color: part.match ? colors.primary : colors.text,
            fontWeight: part.match ? 'bold' : undefined,
          }}
        >
          {part.text}
        </Text>
      ))}
    </Box>
  );
}
```

Example 4: Persistent search history
---
```tsx
import { Box, Text, Pressable } from '@reactjit/core';
import { useSearchHistory } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function SearchWithHistory({ onSelect }: { onSelect: (q: string) => void }) {
  const colors = useThemeColors();
  const { history, push, clear } = useSearchHistory({ maxEntries: 10 });

  return (
    <Box style={{ gap: 4 }}>
      <Text style={{ color: colors.muted }}>Recent Searches</Text>
      {history.map(q => (
        <Pressable key={q} onPress={() => onSelect(q)}>
          <Text style={{ color: colors.text }}>{q}</Text>
        </Pressable>
      ))}
      <Pressable onPress={clear}>
        <Text style={{ color: colors.primary }}>Clear History</Text>
      </Pressable>
    </Box>
  );
}
```

=== CRITICAL RULES ===
- When no `key` is provided, the hooks auto-detect searchable fields by inspecting the first item. Fields named id, key, uuid, type, url, src, path, color, icon are excluded from auto-detection.
- useSearchHistory uses useLocalStore internally, which persists to SQLite. It survives app restarts.
- useAsyncSearch cancels stale requests automatically. The fetcher function is captured by ref.
- useSearchHighlight returns a single non-match part when query is empty -- it does not split on nothing.
