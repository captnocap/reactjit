=== OVERVIEW ===
useLoveRPC returns a stable callback that calls a named Lua RPC method and returns a Promise with the result. This is the primary hook for request/response communication with Lua -- querying data, triggering actions that return results, or calling any registered RPC handler.

The returned function is memoized by method name. It is safe to store in a ref or call from effects without causing re-subscriptions.

=== API / SYNTAX ===
```tsx
import { useLoveRPC } from '@reactjit/core';
```

Signature:
```tsx
function useLoveRPC<T = any>(method: string): (args?: any, timeoutMs?: number) => Promise<T>
```

Parameters:
| Parameter | Type   | Description                     |
|-----------|--------|---------------------------------|
| method    | string | Lua RPC method name to call     |

Returns:
| Type                                        | Description                              |
|---------------------------------------------|------------------------------------------|
| (args?: any, timeoutMs?: number) => Promise  | Async function that calls the RPC method |

=== EXAMPLES ===
Example 1: Query nearby entities
---
```tsx
import { useLoveRPC } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

function NearbyList() {
  const colors = useThemeColors();
  const getNearby = useLoveRPC<Entity[]>('getNearby');
  const [entities, setEntities] = useState<Entity[]>([]);

  useEffect(() => {
    getNearby({ x: 100, y: 200, range: 500 }).then(setEntities);
  }, []);

  return (
    <Box>
      {entities.map((e, i) => (
        <Text key={i} style={{ color: colors.text }}>{e.name}</Text>
      ))}
    </Box>
  );
}
```

Platforms: love2d

Example 2: Polling with useLuaInterval
---
```tsx
import { useLoveRPC, useLuaInterval } from '@reactjit/core';

function SystemMonitor() {
  const getStats = useLoveRPC<{ fps: number; memory: number }>('system:stats');
  const [stats, setStats] = useState({ fps: 0, memory: 0 });

  useLuaInterval(1000, () => {
    getStats().then(setStats);
  });

  return <Text>{`FPS: ${stats.fps} | Memory: ${stats.memory}MB`}</Text>;
}
```

Platforms: love2d

Example 3: RPC with timeout
---
```tsx
import { useLoveRPC } from '@reactjit/core';

function SlowQuery() {
  const query = useLoveRPC<Results>('db:query');

  const handleSearch = async (term: string) => {
    try {
      const results = await query({ term }, 5000); // 5 second timeout
      setResults(results);
    } catch (e) {
      console.log('Query timed out');
    }
  };

  // ...
}
```

=== CRITICAL RULES ===
- The returned function is memoized on `method`. Changing the method string creates a new function.
- Store the RPC function in a ref if calling it from a setInterval or other non-React timer. The function identity is stable per method name, but the ref pattern avoids stale closure issues.
- RPC calls cross the bridge once per call. For high-frequency polling, use useLuaInterval to tick in Lua and push results via events instead.
- The optional timeoutMs parameter defaults to the bridge's global timeout. If the Lua side does not respond within the timeout, the Promise rejects.
