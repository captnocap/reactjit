=== OVERVIEW ===
useFleet manages N concurrent Claude Code sessions. Each agent gets independent status, permission, and question state. RPCs are session-scoped -- every call includes the agent's ID so Lua routes it to the correct PTY.

Auto-accept is global (one toggle for all agents), matching the Lua-side behavior. The hook syncs the auto-accept state from Lua on mount.

Each agent in the returned array provides both state (status, perm, question) and actions (send, stop, respond). It also provides event handler props to wire onto `<Native type="ClaudeCode">` elements.

=== API / SYNTAX ===
```tsx
import { useFleet } from '@reactjit/core';
```

Signature:
```tsx
function useFleet(options: FleetOptions): FleetResult
```

Types:
```tsx
interface FleetOptions {
  workingDir: string;
  agents: FleetAgentConfig[];
}

interface FleetAgentConfig {
  id: string;
  model?: 'sonnet' | 'opus' | 'haiku';
  label?: string;
  workingDir?: string;
}

interface FleetResult {
  agents: FleetAgentState[];
  autoAccept: boolean;
  toggleAutoAccept: () => void;
  focused: string | null;
  setFocused: (id: string | null) => void;
}

interface FleetAgentState {
  id: string;
  model: string;
  label: string;
  status: string;
  perm: FleetPermission | null;
  question: FleetQuestion | null;
  send: (message: string) => void;
  stop: () => void;
  respond: (choice: number) => void;
  respondQuestion: (optionIndex: number) => void;
  // Event handlers for <Native type="ClaudeCode" />
  onStatusChange: (e: any) => void;
  onPermissionRequest: (e: any) => void;
  onPermissionResolved: () => void;
  onQuestionPrompt: (e: any) => void;
}

interface FleetPermission {
  action: string;
  target: string;
  question: string;
}

interface FleetQuestion {
  question: string;
  options: string[];
}
```

=== EXAMPLES ===
Example 1: Two-agent fleet
---
```tsx
import { Box, Text, Pressable } from '@reactjit/core';
import { useFleet } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function FleetPanel() {
  const colors = useThemeColors();
  const fleet = useFleet({
    workingDir: '/home/user/project',
    agents: [
      { id: 'architect', model: 'opus', label: 'Architect' },
      { id: 'worker', model: 'sonnet', label: 'Worker' },
    ],
  });

  return (
    <Box style={{ gap: 12, padding: 16 }}>
      {fleet.agents.map(agent => (
        <Box key={agent.id} style={{ gap: 4 }}>
          <Text style={{ color: colors.text }}>
            {`${agent.label} (${agent.model}): ${agent.status}`}
          </Text>
          {agent.perm && (
            <Box style={{ flexDirection: 'row', gap: 8 }}>
              <Text style={{ color: colors.primary }}>{agent.perm.question}</Text>
              <Pressable onPress={() => agent.respond(1)}>
                <Text style={{ color: colors.text }}>Approve</Text>
              </Pressable>
              <Pressable onPress={() => agent.respond(3)}>
                <Text style={{ color: colors.muted }}>Deny</Text>
              </Pressable>
            </Box>
          )}
          <Pressable onPress={() => agent.send('Fix the build errors')}>
            <Text style={{ color: colors.primary }}>Send Task</Text>
          </Pressable>
        </Box>
      ))}
      <Pressable onPress={fleet.toggleAutoAccept}>
        <Text style={{ color: colors.muted }}>
          {`Auto-accept: ${fleet.autoAccept ? 'ON' : 'OFF'}`}
        </Text>
      </Pressable>
    </Box>
  );
}
```

=== CRITICAL RULES ===
- Agent IDs must be unique within a fleet. They scope all RPC calls to the correct session.
- Permission responses: 1=approve, 2=allow-all, 3=deny. Question responses are 1-indexed option numbers.
- Auto-accept is a global toggle -- it applies to ALL agents, not per-agent.
- The event handler props (onStatusChange, onPermissionRequest, etc.) are designed to be spread onto Native capability nodes. They update the per-agent state map internally.
