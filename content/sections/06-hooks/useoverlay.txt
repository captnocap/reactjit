=== OVERVIEW ===
useOverlay manages a transparent game overlay mode where the Love2D window floats above other applications. The overlay can be interactive (captures mouse/keyboard), passthrough (clicks go through to the app behind), or hidden.

The hook polls the Lua overlay state every 500ms and provides control methods for mode, opacity, and toggling.

Overlay mode requires X11 compositing support. The `hasX11` field indicates whether the system supports it.

=== API / SYNTAX ===
```tsx
import { useOverlay } from '@reactjit/core';
```

Signature:
```tsx
function useOverlay(): OverlayState
```

Types:
```tsx
type OverlayMode = 'interactive' | 'passthrough' | 'hidden';

interface OverlayState {
  enabled: boolean;
  mode: OverlayMode;
  opacity: number;
  hotkey: string;
  hasX11: boolean;
  setMode: (mode: OverlayMode) => void;
  setOpacity: (opacity: number) => void;
  toggle: () => void;
}
```

Returns:
| Field      | Type                      | Description                              |
|------------|---------------------------|------------------------------------------|
| enabled    | boolean                   | Whether overlay mode is active           |
| mode       | OverlayMode               | Current mode                             |
| opacity    | number                    | Window opacity (0-1)                     |
| hotkey     | string                    | Configured toggle hotkey (e.g. 'f6')     |
| hasX11     | boolean                   | Whether X11 compositing is available     |
| setMode    | (mode) => void            | Switch between interactive/passthrough/hidden |
| setOpacity | (opacity) => void         | Set window opacity                       |
| toggle     | () => void                | Toggle overlay on/off                    |

=== EXAMPLES ===
Example 1: Overlay HUD
---
```tsx
import { Box, Text, Pressable } from '@reactjit/core';
import { useOverlay } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function OverlayControls() {
  const colors = useThemeColors();
  const overlay = useOverlay();

  if (!overlay.hasX11) {
    return <Text style={{ color: colors.muted }}>Overlay requires X11</Text>;
  }

  return (
    <Box style={{ gap: 8, padding: 16 }}>
      <Text style={{ color: colors.text }}>
        {`Overlay: ${overlay.enabled ? overlay.mode : 'off'}`}
      </Text>
      <Pressable onPress={overlay.toggle}>
        <Text style={{ color: colors.primary }}>
          {overlay.enabled ? 'Disable Overlay' : 'Enable Overlay'}
        </Text>
      </Pressable>
      {overlay.enabled && (
        <Box style={{ flexDirection: 'row', gap: 8 }}>
          <Pressable onPress={() => overlay.setMode('interactive')}>
            <Text style={{ color: colors.text }}>Interactive</Text>
          </Pressable>
          <Pressable onPress={() => overlay.setMode('passthrough')}>
            <Text style={{ color: colors.text }}>Passthrough</Text>
          </Pressable>
        </Box>
      )}
    </Box>
  );
}
```

=== CRITICAL RULES ===
- Overlay mode only works on Linux with X11 compositing (picom, compton, etc.). Wayland is not supported.
- The hook polls every 500ms via setInterval, not useLuaInterval.
- In passthrough mode, all mouse and keyboard events pass through the window to the application behind it.
- opacity of 0 makes the window fully transparent; 1 is fully opaque.
