=== OVERVIEW ===
The `@reactjit/apis` package provides React hooks for consuming external APIs. Each service gets its own set of hooks that return reactive `{ data, loading, error, refetch }` results. Polling is driven by Lua-side timers, not JS setInterval.

All hooks follow the same pattern: pass credentials (API key or OAuth token) as the first argument, pass null to disable fetching. Data arrives as typed objects. Errors surface in the `error` field.

Import everything from `@reactjit/apis`:

```tsx
import { useSpotifyNowPlaying, useGitHubUser, useWeatherCurrent } from '@reactjit/apis';
import { useAPI, useAPIMutation, bearer, qs } from '@reactjit/apis';
import { useServiceKey, useSettingsRegistry } from '@reactjit/apis';
```

Base utilities (for building custom integrations):
- useAPI -- reactive data fetching with optional polling and rate limiting
- useAPIMutation -- imperative POST/PUT/PATCH/DELETE calls
- bearer(token) -- returns `{ Authorization: 'Bearer ...' }` header object
- qs(params) -- builds a query string from an object, filtering null/undefined values

Key management:
- useServiceKey(serviceId, fieldKey) -- reads an API key from persistent Lua storage
- useServiceKeys(serviceId) -- reads all keys for a service
- useSettingsRegistry(services?) -- sends service definitions to the Lua settings overlay (call once at app startup)

Service registry:
- builtinServices -- array of all built-in ServiceDefinition objects
- Individual definitions: spotify, tmdb, github, weather, homeassistant, lastfm, plex, jellyfin, trakt, notion, todoist, ynab, google, hue, nasa, coingecko, steam, telegram

Rate limiting:
- All hooks accept an optional `rateLimit: { maxCalls, perSeconds }` config
- Requests exceeding the budget queue automatically and drain when slots open
- Rate limits are shared per domain -- all hooks hitting the same host share one budget

Service categories:
- Media: Spotify, TMDB, Plex, Jellyfin, Trakt, Last.fm
- Dev: GitHub, Steam, NASA
- AI: OpenAI, Anthropic
- Smart Home: Home Assistant, Philips Hue
- Productivity: Notion, Todoist, Google (Calendar, Sheets)
- Finance: YNAB, CoinGecko
- Social: Telegram, OpenWeatherMap

Pre-built display components (one-liner UI for common data):
- SpotifyNowPlaying, SpotifyRecentTracks, LastFMNowPlaying, LastFMRecentTracks, LastFMTopArtists
- TMDBTrending, CoinMarkets, GitHubProfile, GitHubRepos, APOD
- TrackRow, ArtistRow, RepoCard, GitHubUserCard, APODCard
- NowPlayingCard, MediaPosterCard, CoinTickerRow, StatCard, ActivityRow, HueLightBadge

=== EXAMPLES ===
Example 1: Using useAPI for a custom endpoint
---
```tsx
import { Box, Text } from '@reactjit/core';
import { useAPI, bearer } from '@reactjit/apis';
import { useThemeColors } from '@reactjit/theme';

interface Post { id: number; title: string; }

export function Posts() {
  const colors = useThemeColors();
  const { data, loading, error } = useAPI<Post[]>(
    'https://api.example.com/posts',
    { headers: bearer('my-token'), interval: 30000 },
  );

  if (loading) return <Text style={{ color: colors.muted }}>Loading...</Text>;
  if (error) return <Text style={{ color: colors.primary }}>{`Error: ${error.message}`}</Text>;

  return (
    <Box style={{ gap: 4 }}>
      {data?.map(post => (
        <Text key={post.id} style={{ color: colors.text }}>{post.title}</Text>
      ))}
    </Box>
  );
}
```

Example 2: Key management with useServiceKey
---
```tsx
import { Text } from '@reactjit/core';
import { useServiceKey, useSpotifyNowPlaying } from '@reactjit/apis';
import { useThemeColors } from '@reactjit/theme';

export function NowPlaying() {
  const colors = useThemeColors();
  const { key } = useServiceKey('spotify', 'token');
  const { data } = useSpotifyNowPlaying(key);

  return (
    <Text style={{ color: colors.text }}>
      {data?.item?.name ?? 'Not playing'}
    </Text>
  );
}
```

Platforms: love2d

Example 3: Registering services at app startup
---
```tsx
import { useSettingsRegistry, builtinServices } from '@reactjit/apis';

const myService = {
  id: 'my-api',
  name: 'My API',
  category: 'custom' as const,
  auth: {
    type: 'bearer' as const,
    fields: [{ key: 'token', label: 'API Token' }],
  },
};

export function App() {
  useSettingsRegistry([...builtinServices, myService]);
  return <MyApp />;
}
```

Platforms: love2d

Example 4: Mutation with useAPIMutation
---
```tsx
import { Pressable, Text } from '@reactjit/core';
import { useAPIMutation, bearer } from '@reactjit/apis';
import { useThemeColors } from '@reactjit/theme';

export function CreatePost() {
  const colors = useThemeColors();
  const { execute, loading } = useAPIMutation(bearer('my-token'));

  const handleCreate = () => {
    execute('https://api.example.com/posts', {
      body: { title: 'New Post', content: 'Hello world' },
    });
  };

  return (
    <Pressable onPress={handleCreate}>
      <Text style={{ color: colors.primary }}>
        {loading ? 'Creating...' : 'Create Post'}
      </Text>
    </Pressable>
  );
}
```
