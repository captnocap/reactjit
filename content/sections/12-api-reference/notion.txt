=== OVERVIEW ===
The Notion hooks in @reactjit/apis connect to the Notion REST API v1. You need an internal integration token — create one at https://www.notion.so/my-integrations and share it with the databases or pages you want to read.

Read hooks (useNotionPage, useNotionBlocks) use reactive polling via useAPI. The database query hooks (useNotionDatabases, useNotionDatabase) use the POST-based search and query endpoints internally, so they are implemented with useAPIMutation but behave like read hooks — they auto-fetch on mount and expose a refetch() method. Write operations go through useNotionMutation, which returns imperative functions.

The Notion API version pinned by these hooks is 2022-06-28.

=== API / SYNTAX ===

--- useNotionDatabases ---

Search for all databases the integration has been granted access to.

  import { useNotionDatabases } from '@reactjit/apis';

  const result: APIResult<NotionPaginated<NotionDatabase>> = useNotionDatabases(
    token: string | null,
  );

Parameters:
  token  string | null  Internal integration token

Returns: APIResult<NotionPaginated<NotionDatabase>>

NotionDatabase fields:
  id           string              UUID of the database
  title        NotionRichText[]    Database title as rich text
  description  NotionRichText[]
  url          string              notion.so URL
  properties   Record<string, { id: string; type: string; name: string }>
               Map of property key to schema definition

NotionPaginated<T> fields:
  results      T[]                 Page of results
  has_more     boolean             Whether more pages exist
  next_cursor  string | null       Cursor for next page, or null

--- useNotionDatabase ---

Query rows from a specific database with optional filter and sort. Uses Notion's database query endpoint (POST).

  import { useNotionDatabase } from '@reactjit/apis';

  const result: APIResult<NotionPaginated<NotionPage>> = useNotionDatabase(
    token: string | null,
    databaseId: string | null,
    opts?: { filter?: any; sorts?: any[] },
  );

Parameters:
  token       string | null   Integration token
  databaseId  string | null   UUID of the database to query
  opts        object | undefined
    filter    any             Notion filter object (see Notion API docs)
    sorts     any[]           Array of Notion sort objects

Returns: APIResult<NotionPaginated<NotionPage>>

NotionPage fields:
  id                string
  url               string
  created_time      string              ISO 8601
  last_edited_time  string              ISO 8601
  properties        Record<string, any>  Property values (shape varies by property type)
  icon              { type: string; emoji?: string } | null
  cover             { type: string; external?: { url: string } } | null

--- useNotionPage ---

Fetch a single Notion page by its ID.

  import { useNotionPage } from '@reactjit/apis';

  const result: APIResult<NotionPage> = useNotionPage(
    token: string | null,
    pageId: string | null,
  );

Parameters:
  token   string | null  Integration token
  pageId  string | null  UUID of the page

Returns: APIResult<NotionPage>

--- useNotionBlocks ---

Fetch the child blocks of a page or block. Returns the top-level block tree only — call recursively with a block's ID to fetch nested children.

  import { useNotionBlocks } from '@reactjit/apis';

  const result: APIResult<NotionPaginated<NotionBlock>> = useNotionBlocks(
    token: string | null,
    blockId: string | null,
  );

Parameters:
  token    string | null  Integration token
  blockId  string | null  UUID of the page or parent block

Returns: APIResult<NotionPaginated<NotionBlock>>

NotionBlock fields:
  id            string   UUID of the block
  type          string   'paragraph', 'heading_1', 'bulleted_list_item', 'to_do', 'image', etc.
  has_children  boolean  Whether this block has nested children
  [key: string] any      Type-specific content at the key matching the type field

--- useNotionMutation ---

Get imperative write operations: create pages and update existing page properties.

  import { useNotionMutation } from '@reactjit/apis';

  const mutation = useNotionMutation(token: string | null);

Parameters:
  token  string | null  Integration token

Returns: object
  createPage   (parentId: string, properties: Record<string, any>, children?: any[]) => Promise<any>
               Create a new page in a database. parentId is the database UUID.
               properties follows the Notion property value format.
               children is an optional array of block objects.
  updatePage   (pageId: string, properties: Record<string, any>) => Promise<any>
               Update properties of an existing page. Only provided keys are changed.
  loading      boolean        True while any mutation is in flight
  error        Error | null   Last mutation error, or null

--- Shared Types ---

NotionRichText fields:
  type        'text'
  text        { content: string; link?: { url: string } | null }
  plain_text  string  Convenience field — the text content without formatting

=== EXAMPLES ===

-- Render a task database --

Query a database of tasks filtered to incomplete items, sorted by priority:

  import { useNotionDatabase } from '@reactjit/apis';
  import { Box, Text } from '@reactjit/core';

  const TOKEN = process.env.NOTION_TOKEN;
  const DB_ID = 'your-database-uuid-here';

  function TaskList() {
    const { data, loading, refetch } = useNotionDatabase(TOKEN, DB_ID, {
      filter: {
        property: 'Done',
        checkbox: { equals: false },
      },
      sorts: [{ property: 'Priority', direction: 'descending' }],
    });

    if (loading) return <Text fontSize={13}>Loading tasks...</Text>;

    const pages = data?.results ?? [];

    return (
      <Box gap={6} width="100%">
        {pages.map((page) => {
          // Property values use Notion's typed structure
          const titleProp = page.properties['Name'];
          const title = titleProp?.title?.[0]?.plain_text ?? 'Untitled';
          return (
            <Box key={page.id} padding={10} backgroundColor="#1e1e2e">
              <Text fontSize={14}>{title}</Text>
              <Text fontSize={11}>{page.last_edited_time.slice(0, 10)}</Text>
            </Box>
          );
        })}
      </Box>
    );
  }

-- Create a page in a database --

Add a new row to a Notion database when the user submits a form:

  import { useNotionMutation } from '@reactjit/apis';
  import { Box, Text } from '@reactjit/core';
  import { useState } from 'react';

  const TOKEN = process.env.NOTION_TOKEN;
  const DB_ID = 'your-database-uuid-here';

  function QuickCapture() {
    const [input, setInput] = useState('');
    const { createPage, loading, error } = useNotionMutation(TOKEN);

    async function submit() {
      if (!input.trim()) return;
      await createPage(DB_ID, {
        Name: { title: [{ text: { content: input } }] },
      });
      setInput('');
    }

    return (
      <Box gap={8}>
        <Text fontSize={13}>Quick capture</Text>
        {/* TextInput wired to setInput + onSubmit */}
        <Box onPress={submit} padding={10} backgroundColor="#4CAF50">
          <Text fontSize={13}>{loading ? 'Saving...' : 'Add to Notion'}</Text>
        </Box>
        {error && <Text fontSize={11}>{error.message}</Text>}
      </Box>
    );
  }
