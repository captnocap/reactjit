=== OVERVIEW ===
Hooks for the Notion API. Auth uses an internal integration token (Bearer). Create one at https://www.notion.so/my-integrations.

The integration must be shared with the pages/databases you want to access (Connections setting in Notion).

useNotionDatabases and useNotionDatabase use POST requests internally (Notion's search and query endpoints require POST), but still return reactive `{ data, loading, error, refetch }` results. useNotionPage and useNotionBlocks use standard GET requests.

useNotionMutation provides imperative page creation and updates.

All hooks use Notion API version 2022-06-28.

=== API / SYNTAX ===
```tsx
import {
  useNotionDatabases,
  useNotionDatabase,
  useNotionPage,
  useNotionBlocks,
  useNotionMutation,
} from '@reactjit/apis';
```

---- useNotionDatabases

```tsx
function useNotionDatabases(
  token: string | null
): APIResult<NotionPaginated<NotionDatabase>>
```

List all databases the integration can access. Uses Notion's search endpoint with a database filter. Auto-fetches on mount.

---- useNotionDatabase

```tsx
function useNotionDatabase(
  token: string | null,
  databaseId: string | null,
  opts?: { filter?: any; sorts?: any[] }
): APIResult<NotionPaginated<NotionPage>>
```

Query a database for its pages. Supports Notion filter and sort objects. Auto-fetches on mount.

---- useNotionPage

```tsx
function useNotionPage(
  token: string | null,
  pageId: string | null
): APIResult<NotionPage>
```

Fetch a single page by ID. Returns the page's properties, icon, and cover.

---- useNotionBlocks

```tsx
function useNotionBlocks(
  token: string | null,
  blockId: string | null
): APIResult<NotionPaginated<NotionBlock>>
```

Fetch the child blocks of a page or block. Use the page ID as blockId to get the page's content blocks.

---- useNotionMutation

```tsx
function useNotionMutation(
  token: string | null
): {
  createPage: (parentId: string, properties: Record<string, any>, children?: any[]) => Promise;
  updatePage: (pageId: string, properties: Record<string, any>) => Promise;
  loading: boolean;
  error: Error | null;
}
```

Imperative mutations. `createPage` adds a page to a database. `updatePage` modifies page properties.

---- Types

NotionPage:
| Field            | Type                 | Description                   |
|------------------|----------------------|-------------------------------|
| id               | string               | Page ID (UUID)                |
| url              | string               | Notion page URL               |
| created_time     | string               | Creation timestamp            |
| last_edited_time | string               | Last edit timestamp           |
| properties       | Record<string, any>  | Page properties by name       |
| icon             | object?              | `{ type, emoji? }` or null    |
| cover            | object?              | `{ type, external? }` or null |

NotionDatabase:
| Field       | Type              | Description                         |
|-------------|-------------------|-------------------------------------|
| id          | string            | Database ID (UUID)                  |
| title       | NotionRichText[]  | Database title                      |
| description | NotionRichText[]  | Database description                |
| url         | string            | Notion database URL                 |
| properties  | Record            | Map of property name to `{ id, type, name }` |

NotionBlock:
| Field        | Type    | Description               |
|--------------|---------|---------------------------|
| id           | string  | Block ID                  |
| type         | string  | 'paragraph', 'heading_1', etc. |
| has_children | boolean | Whether block has children |

NotionRichText:
| Field      | Type   | Description             |
|------------|--------|-------------------------|
| type       | string | Always 'text'           |
| text       | object | `{ content, link? }`    |
| plain_text | string | Plain text content      |

NotionPaginated:
| Field       | Type    | Description               |
|-------------|---------|---------------------------|
| results     | T[]     | Array of results          |
| has_more    | boolean | Whether more pages exist  |
| next_cursor | string? | Cursor for next page      |

=== EXAMPLES ===
Example 1: List databases
---
```tsx
import { Box, Text } from '@reactjit/core';
import { useNotionDatabases } from '@reactjit/apis';
import { useThemeColors } from '@reactjit/theme';

export function DatabaseList({ token }: { token: string }) {
  const colors = useThemeColors();
  const { data, loading } = useNotionDatabases(token);

  if (loading) return <Text style={{ color: colors.muted }}>Loading...</Text>;

  return (
    <Box style={{ gap: 4 }}>
      {data?.results.map(db => (
        <Text key={db.id} style={{ color: colors.text, fontSize: 13 }}>
          {db.title[0]?.plain_text ?? 'Untitled'}
        </Text>
      ))}
    </Box>
  );
}
```

Platforms: love2d

Example 2: Query a database with filters
---
```tsx
import { Box, Text } from '@reactjit/core';
import { useNotionDatabase } from '@reactjit/apis';
import { useThemeColors } from '@reactjit/theme';

export function TaskList({ token, dbId }: { token: string; dbId: string }) {
  const colors = useThemeColors();
  const { data } = useNotionDatabase(token, dbId, {
    filter: {
      property: 'Status',
      select: { equals: 'In Progress' },
    },
    sorts: [{ property: 'Priority', direction: 'descending' }],
  });

  return (
    <Box style={{ gap: 4 }}>
      <Text style={{ color: colors.text, fontSize: 14, fontWeight: 'bold' }}>In Progress</Text>
      {data?.results.map(page => {
        const title = page.properties.Name?.title?.[0]?.plain_text ?? 'Untitled';
        return (
          <Text key={page.id} style={{ color: colors.text, fontSize: 12 }}>{title}</Text>
        );
      })}
    </Box>
  );
}
```

Platforms: love2d

Example 3: Create a page
---
```tsx
import { Box, Text, Pressable } from '@reactjit/core';
import { useNotionMutation } from '@reactjit/apis';
import { useThemeColors } from '@reactjit/theme';

export function QuickAdd({ token, dbId }: { token: string; dbId: string }) {
  const colors = useThemeColors();
  const { createPage, loading } = useNotionMutation(token);

  const handleAdd = () => {
    createPage(dbId, {
      Name: { title: [{ text: { content: 'New Task' } }] },
      Status: { select: { name: 'To Do' } },
    });
  };

  return (
    <Pressable onPress={handleAdd}>
      <Text style={{ color: colors.primary, fontSize: 14 }}>
        {loading ? 'Creating...' : 'Add Task'}
      </Text>
    </Pressable>
  );
}
```

Platforms: love2d

=== CRITICAL RULES ===
- The integration must be connected to each database/page via Notion's Connections setting. Unconnected pages return 404.
- Property access varies by type. Title properties use `page.properties.Name.title[0].plain_text`. Select uses `.select.name`. Check Notion's API docs for each property type.
- useNotionDatabases and useNotionDatabase auto-fetch on mount. They do not re-fetch when options change. Call `refetch()` manually after changing filters.
- Database IDs from Notion URLs must be formatted as UUIDs (with dashes). The 32-character hex string from the URL needs dashes inserted.
