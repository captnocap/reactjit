=== OVERVIEW ===
The Telegram hooks wrap the official Telegram Bot API. They cover three concerns: identifying the bot (`useTelegramBot`), polling for incoming messages and callback queries (`useTelegramUpdates`), and sending messages or photos imperatively (`useTelegramSend`).

Auth: Create a bot via @BotFather in Telegram. BotFather gives you a token string of the form `1234567890:ABCDEFGHIJabcdefghij`. Pass it as `botToken` to every hook. The token is embedded directly in the API URL path — treat it as a secret.

=== API / SYNTAX ===

--- useTelegramBot ---

Fetch the identity of the bot associated with the given token. Use this to confirm the token is valid and to get the bot's username and capability flags.

  import { useTelegramBot } from '@reactjit/apis/telegram';

  const result: APIResult<{ ok: boolean; result: TelegramBotInfo }> = useTelegramBot(
    botToken: string | null,
  );

Parameters:
  botToken  string | null   Telegram bot token from BotFather. Pass null to disable.

Returns: APIResult<{ ok: boolean; result: TelegramBotInfo }>
  data.ok      boolean
  data.result  TelegramBotInfo

TelegramBotInfo fields:
  id                            number
  is_bot                        boolean    Always true for bots
  first_name                    string     Bot display name
  username                      string     Bot username (without @)
  can_join_groups               boolean
  can_read_all_group_messages   boolean
  supports_inline_queries       boolean

--- useTelegramUpdates ---

Poll for incoming updates (messages, edited messages, callback queries) using Telegram's long-polling `getUpdates` method. Refetches on a configurable interval.

  import { useTelegramUpdates } from '@reactjit/apis/telegram';

  const result: APIResult<{ ok: boolean; result: TelegramUpdate[] }> = useTelegramUpdates(
    botToken: string | null,
    opts?:    { offset?: number; limit?: number; interval?: number },
  );

Parameters:
  botToken       string | null   Telegram bot token. Pass null to disable.
  opts.offset    number          Return only updates with update_id >= offset. Pass the last update_id + 1 to acknowledge processed updates.
  opts.limit     number          Maximum number of updates to return per poll. Defaults to 20. Max 100.
  opts.interval  number          Polling interval in milliseconds. Defaults to 5000 (5 seconds).

Returns: APIResult<{ ok: boolean; result: TelegramUpdate[] }>
  data.ok      boolean
  data.result  TelegramUpdate[]

TelegramUpdate fields:
  update_id        number                      Globally unique update ID. Use for offset tracking.
  message          TelegramMessage | undefined  A new incoming message
  edited_message   TelegramMessage | undefined  An edited message
  callback_query   object | undefined           A callback from an inline keyboard button

TelegramMessage fields:
  message_id           number
  from                 TelegramUser | undefined
  chat                 TelegramChat
  date                 number                   Unix timestamp
  text                 string | undefined
  photo                Array<{ file_id, width, height }> | undefined
  document             { file_id, file_name?, mime_type? } | undefined
  reply_to_message     TelegramMessage | undefined

TelegramChat fields:
  id          number
  type        'private' | 'group' | 'supergroup' | 'channel'
  title       string | undefined
  username    string | undefined
  first_name  string | undefined
  last_name   string | undefined

TelegramUser fields:
  id              number
  is_bot          boolean
  first_name      string
  last_name       string | undefined
  username        string | undefined
  language_code   string | undefined

--- useTelegramSend ---

Imperative mutation hook for sending messages and photos to any chat your bot has access to.

  import { useTelegramSend } from '@reactjit/apis/telegram';

  const sender = useTelegramSend(botToken: string | null);

Parameters:
  botToken  string | null   Telegram bot token. Pass null to disable (all methods reject with an error).

Returns: object
  sendMessage   (chatId, text, opts?) => Promise<any>
  sendPhoto     (chatId, photoUrl, opts?) => Promise<any>
  loading       boolean   True while any mutation request is in flight
  error         Error | null

sendMessage signature:
  sendMessage(
    chatId:            number | string,
    text:              string,
    opts?: {
      parseMode?:         'HTML' | 'Markdown' | 'MarkdownV2';
      replyToMessageId?:  number;
    },
  ): Promise<any>

  chatId              number | string   Telegram chat ID or @username
  text                string            Message text
  opts.parseMode      string            Enable HTML or Markdown formatting in text
  opts.replyToMessageId  number         Reply to a specific message by ID

sendPhoto signature:
  sendPhoto(
    chatId:   number | string,
    photoUrl: string,
    opts?: {
      caption?: string;
    },
  ): Promise<any>

  chatId      number | string   Target chat ID
  photoUrl    string            Public URL of the image to send
  opts.caption  string          Optional caption text shown below the photo

=== EXAMPLES ===

-- Message monitor panel --

Poll for new messages every 3 seconds and display the last 20 in a scrollable list. Track the offset to avoid re-showing old messages.

  import { useTelegramUpdates } from '@reactjit/apis/telegram';
  import { useState, useEffect } from 'react';
  import { Box, Text, ScrollView } from '@reactjit/core';

  const BOT_TOKEN = 'your-bot-token';

  function MessageMonitor() {
    const [offset, setOffset] = useState<number | undefined>(undefined);
    const [messages, setMessages] = useState<string[]>([]);

    const { data } = useTelegramUpdates(BOT_TOKEN, {
      offset,
      limit: 20,
      interval: 3000,
    });

    useEffect(() => {
      if (!data?.result?.length) return;

      // Acknowledge the updates by advancing offset past the last one
      const last = data.result[data.result.length - 1];
      setOffset(last.update_id + 1);

      // Accumulate text messages only
      const incoming = data.result
        .filter((u) => u.message?.text)
        .map((u) => {
          const sender = u.message!.from?.first_name ?? 'Unknown';
          return `${sender}: ${u.message!.text}`;
        });

      setMessages((prev) => [...prev, ...incoming].slice(-50));
    }, [data]);

    return (
      <Box width="100%" height="100%" padding={12} gap={8}>
        <Text fontSize={16}>Telegram Monitor</Text>
        <ScrollView width="100%" flexGrow={1}>
          <Box gap={4}>
            {messages.map((msg, i) => (
              <Text key={i} fontSize={12}>{msg}</Text>
            ))}
            {messages.length === 0 && (
              <Text fontSize={12}>Waiting for messages...</Text>
            )}
          </Box>
        </ScrollView>
      </Box>
    );
  }

-- Alert sender --

Render a button that sends a formatted notification to a hardcoded alert channel when pressed.

  import { useTelegramSend } from '@reactjit/apis/telegram';
  import { Box, Text } from '@reactjit/core';

  const BOT_TOKEN = 'your-bot-token';
  const ALERT_CHAT = -1001234567890; // channel or group ID (negative for groups)

  function AlertButton({ label, message }: { label: string; message: string }) {
    const { sendMessage, loading, error } = useTelegramSend(BOT_TOKEN);

    const handlePress = async () => {
      const timestamp = new Date().toISOString();
      await sendMessage(
        ALERT_CHAT,
        `<b>${label}</b>\n${message}\n<i>${timestamp}</i>`,
        { parseMode: 'HTML' },
      );
    };

    return (
      <Box gap={6}>
        <Box
          backgroundColor={loading ? '#555' : '#2196F3'}
          padding={12}
          onPress={handlePress}
        >
          <Text fontSize={14}>{loading ? 'Sending...' : `Send: ${label}`}</Text>
        </Box>
        {error && <Text fontSize={11}>Failed: {error.message}</Text>}
      </Box>
    );
  }
