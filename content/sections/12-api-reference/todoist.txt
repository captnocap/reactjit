=== OVERVIEW ===
The Todoist hooks wrap the Todoist REST API v2 for reading and writing task data. All read hooks accept a Bearer token as the first argument and return the standard APIResult shape ({ data, loading, error, refetch }). Pass null for the token to disable fetching.

Mutations are handled by useTodoistMutation, which returns named functions for creating, completing, reopening, updating, and deleting tasks.

Get your API token from Todoist Settings > Integrations > Developer.

=== API / SYNTAX ===
```tsx
import {
  useTodoistTasks,
  useTodoistProjects,
  useTodoistSections,
  useTodoistLabels,
  useTodoistComments,
  useTodoistMutation,
} from '@reactjit/apis';
```

---- useTodoistTasks

```tsx
function useTodoistTasks(
  token: string | null,
  opts?: { projectId?: string; label?: string; filter?: string }
): APIResult<TodoistTask[]>
```

| Param          | Type           | Default   | Description                                    |
|----------------|----------------|-----------|------------------------------------------------|
| token          | string \| null | required  | Todoist API token, or null to disable           |
| opts.projectId | string         | undefined | Filter tasks to a specific project              |
| opts.label     | string         | undefined | Filter tasks by label name                      |
| opts.filter    | string         | undefined | Todoist filter expression (e.g. "today & p1")   |

---- useTodoistProjects

```tsx
function useTodoistProjects(token: string | null): APIResult<TodoistProject[]>
```

---- useTodoistSections

```tsx
function useTodoistSections(
  token: string | null,
  projectId: string | null
): APIResult<TodoistSection[]>
```

Requires both token and projectId to be non-null.

---- useTodoistLabels

```tsx
function useTodoistLabels(token: string | null): APIResult<TodoistLabel[]>
```

---- useTodoistComments

```tsx
function useTodoistComments(
  token: string | null,
  opts: { taskId?: string; projectId?: string }
): APIResult<TodoistComment[]>
```

Requires at least one of taskId or projectId.

---- useTodoistMutation

```tsx
function useTodoistMutation(token: string | null): {
  createTask: (content: string, opts?: { projectId?: string; priority?: number; dueString?: string; labels?: string[] }) => Promise<any>;
  completeTask: (taskId: string) => Promise<any>;
  reopenTask: (taskId: string) => Promise<any>;
  deleteTask: (taskId: string) => Promise<any>;
  updateTask: (taskId: string, updates: Partial<Pick<TodoistTask, 'content' | 'description' | 'priority' | 'labels'>>) => Promise<any>;
  loading: boolean;
  error: Error | null;
}
```

---- Types

TodoistTask:
| Field        | Type                          | Description                      |
|--------------|-------------------------------|----------------------------------|
| id           | string                        | Task ID                          |
| content      | string                        | Task title                       |
| description  | string                        | Task description                 |
| is_completed | boolean                       | Whether the task is done         |
| priority     | 1 \| 2 \| 3 \| 4             | Priority (4 = highest, 1 = none) |
| due          | object \| null                | Due date info (date, datetime)   |
| labels       | string[]                      | Applied label names              |
| project_id   | string                        | Parent project ID                |
| section_id   | string                        | Parent section ID (optional)     |
| parent_id    | string \| null                | Parent task ID for subtasks      |
| order        | number                        | Sort position                    |
| created_at   | string                        | ISO creation timestamp           |
| url          | string                        | Web URL for the task             |

=== EXAMPLES ===
Example 1: List tasks for a project
---
```tsx
import { Box, Text } from '@reactjit/core';
import { useTodoistTasks } from '@reactjit/apis';
import { useThemeColors } from '@reactjit/theme';

export function TaskList({ token, projectId }: { token: string; projectId: string }) {
  const colors = useThemeColors();
  const { data: tasks, loading, error } = useTodoistTasks(token, { projectId });

  if (loading) return <Text style={{ color: colors.muted }}>Loading tasks...</Text>;
  if (error) return <Text style={{ color: colors.primary }}>{`Error: ${error.message}`}</Text>;

  return (
    <Box style={{ gap: 4 }}>
      {tasks?.map(task => (
        <Text key={task.id} style={{ color: task.is_completed ? colors.muted : colors.text }}>
          {`${task.is_completed ? '[x]' : '[ ]'} ${task.content}`}
        </Text>
      ))}
    </Box>
  );
}
```

Example 2: Create and complete tasks
---
```tsx
import { useState } from 'react';
import { Box, Text, Pressable, Input } from '@reactjit/core';
import { useTodoistTasks, useTodoistMutation } from '@reactjit/apis';
import { useThemeColors } from '@reactjit/theme';

export function TaskManager({ token }: { token: string }) {
  const colors = useThemeColors();
  const { data: tasks, refetch } = useTodoistTasks(token);
  const { createTask, completeTask, loading } = useTodoistMutation(token);
  const [newTask, setNewTask] = useState('');

  const handleAdd = async () => {
    if (!newTask.trim()) return;
    await createTask(newTask, { priority: 2, dueString: 'today' });
    setNewTask('');
    refetch();
  };

  const handleComplete = async (taskId: string) => {
    await completeTask(taskId);
    refetch();
  };

  return (
    <Box style={{ gap: 8 }}>
      <Input value={newTask} onChangeText={setNewTask} onSubmit={handleAdd} placeholder="New task..." />
      {tasks?.filter(t => !t.is_completed).map(task => (
        <Pressable key={task.id} onPress={() => handleComplete(task.id)}>
          <Text style={{ color: colors.text }}>{task.content}</Text>
        </Pressable>
      ))}
    </Box>
  );
}
```

Platforms: love2d

=== CRITICAL RULES ===
- Priority values are inverted from what you might expect: 4 is the highest priority (p1 in the UI), 1 is no priority.
- useTodoistComments requires at least one of taskId or projectId. If neither is provided, the hook is disabled.
- useTodoistSections requires both token and projectId to be non-null.
- After mutations, call refetch() on the relevant read hook to update the UI.
