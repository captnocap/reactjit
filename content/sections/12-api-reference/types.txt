=== OVERVIEW ===
This is the complete reference for core type definitions exported from @reactjit/core. These types define the prop interfaces for all primitive components, the event shape for input handling, and the foundational Color type used throughout the framework.

All types are defined in packages/core/src/types.ts.

=== API / SYNTAX ===
```tsx
import type {
  Style,
  Color,
  LoveEvent,
  LayoutEvent,
  BoxProps,
  TextProps,
  ImageProps,
  VideoProps,
  ScrollViewProps,
  InputProps,
  FlatListProps,
  ContextMenuProps,
  FocusGroupProps,
  EmulatorProps,
  RenderProps,
  TooltipProp,
  TooltipConfig,
  CapabilitySchema,
} from '@reactjit/core';
```

---- Color

```tsx
type Color = string | [number, number, number, number?];
```

Colors can be:
- CSS hex strings: "#ff0000", "#ff000080" (with alpha)
- CSS rgba strings: "rgba(255, 0, 0, 0.5)"
- Love2D number arrays: [1, 0, 0] or [1, 0, 0, 0.5] (channels 0-1)

---- LayoutEvent

```tsx
interface LayoutEvent {
  targetId?: number;
  x: number;
  y: number;
  width: number;
  height: number;
}
```

Passed to the onLayout handler when an element's position or size changes.

---- CapabilitySchema

```tsx
interface CapabilitySchema {
  schema: Record<string, {
    type: string;
    desc?: string;
    default?: any;
    min?: number;
    max?: number;
    values?: string[];
  }>;
  events: string[];
  visual: boolean;
}
```

Returned by the capabilities:list RPC. Used by AI agents to discover available capabilities and their props/events.

=== EXAMPLES ===
Example 1: Typed event handling
---
```tsx
import { Box, Text, Pressable } from '@reactjit/core';
import type { LoveEvent } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function EventDemo() {
  const colors = useThemeColors();

  const handleClick = (e: LoveEvent) => {
    console.log(`Clicked at (${e.x}, ${e.y}) with button ${e.button}`);
  };

  const handleKey = (e: LoveEvent) => {
    if (e.key === 'return' && e.ctrl) {
      console.log('Ctrl+Enter pressed');
    }
  };

  return (
    <Box onKeyDown={handleKey} style={{ gap: 8 }}>
      <Pressable onClick={handleClick}>
        <Text style={{ color: colors.primary }}>Click me</Text>
      </Pressable>
    </Box>
  );
}
```

Platforms: love2d

Example 2: Box shorthand props
---
```tsx
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function ShorthandDemo() {
  const colors = useThemeColors();

  return (
    <Box fill direction="col" gap={12} padding={16} bg={colors.bg}>
      <Box direction="row" gap={8} align="center">
        <Box w={40} h={40} radius={20} bg={colors.primary} />
        <Text style={{ color: colors.text }}>Avatar + name</Text>
      </Box>
      <Box grow bg={colors.surface} radius={8} padding={12}>
        <Text style={{ color: colors.text }}>Content area with flexGrow</Text>
      </Box>
    </Box>
  );
}
```

=== CRITICAL RULES ===
- Text children must use template literals: {`Score: ${score}`} not <Text>Score: {score}</Text>. Mixed children create separate text nodes that stack vertically.
- BoxProps shorthand props (direction, gap, bg, etc.) have lower priority than style={}. If both are set, style wins.
- InputProps (formerly TextInputProps) is the unified text input interface. TextInputProps and TextEditorProps are deprecated aliases.
- LoveEvent fields are optional because different event types populate different subsets. Always check for existence before use.
- All event handlers receive LoveEvent, not DOM events. There is no event.preventDefault(), event.target, or event.currentTarget in the DOM sense. Use stopPropagation() for bubbling control.
