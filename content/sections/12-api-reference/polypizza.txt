=== OVERVIEW ===
The Poly Pizza hooks fetch 3D model metadata from the Poly Pizza API. Authentication uses an API key sent as the x-api-key header by default (configurable to query parameter via opts.apiKeyIn).

Three hooks are provided: usePolyPizzaModel for fetching a single model by ID, usePolyPizzaSearch for searching models by keyword, and usePolyPizzaModelWithAttribution which wraps the model hook with normalized attribution metadata.

Two helper functions handle attribution: polyPizzaAttributionFields normalizes the response into a consistent structure, and polyPizzaAttributionLine builds a human-readable attribution string for credits.

All hooks accept endpoint overrides via options, allowing them to adapt to API version changes without code modifications.

=== API / SYNTAX ===
```tsx
import {
  usePolyPizzaModel,
  usePolyPizzaSearch,
  usePolyPizzaModelWithAttribution,
  polyPizzaAttributionFields,
  polyPizzaAttributionLine,
} from '@reactjit/apis';
```

---- usePolyPizzaModel

```tsx
function usePolyPizzaModel(
  apiKey: string | null,
  modelId: string | null,
  opts?: PolyPizzaModelOptions
): APIResult<PolyPizzaModel>
```

| Param          | Type   | Default          | Description                              |
|----------------|--------|------------------|------------------------------------------|
| apiKey         | string \| null | required | API key, or null to disable              |
| modelId        | string \| null | required | Model ID or slug                         |
| opts.baseUrl   | string | poly.pizza/api/v1.1 | API base URL                        |
| opts.modelPath | string | "/models/{id}"   | Endpoint template ({id} is replaced)     |
| opts.apiKeyIn  | string | "header"         | How to send the key ("header" or "query")|

---- usePolyPizzaSearch

```tsx
function usePolyPizzaSearch(
  apiKey: string | null,
  query: string | null,
  opts?: PolyPizzaSearchOptions
): APIResult<PolyPizzaSearchResponse>
```

| Param          | Type   | Default            | Description                          |
|----------------|--------|--------------------|--------------------------------------|
| apiKey         | string \| null | required   | API key                              |
| query          | string \| null | required   | Search query, or null to disable     |
| opts.searchPath| string | "/models/search"   | Search endpoint path                 |
| opts.queryKey  | string | "q"                | Query parameter name                 |
| opts.page      | number | undefined          | Page number                          |
| opts.perPage   | number | undefined          | Results per page                     |

The response may contain results in results, items, or data fields depending on API version. Check all three.

---- usePolyPizzaModelWithAttribution

```tsx
function usePolyPizzaModelWithAttribution(
  apiKey: string | null,
  modelId: string | null,
  opts?: PolyPizzaModelOptions
): PolyPizzaModelWithAttributionResult
```

Returns the same APIResult as usePolyPizzaModel plus:
| Field           | Type                       | Description                         |
|-----------------|----------------------------|-------------------------------------|
| attribution     | PolyPizzaAttribution \| null| Normalized attribution fields       |
| attributionLine | string \| null              | Human-readable attribution string   |

---- polyPizzaAttributionFields

```tsx
function polyPizzaAttributionFields(payload: unknown): PolyPizzaAttribution
```

Extracts and normalizes attribution info from a model response payload. Handles multiple response shapes by probing common field paths for author, license, title, and source URL.

Returns:
| Field           | Type          | Description                          |
|-----------------|---------------|--------------------------------------|
| id              | string \| null | Model ID                            |
| title           | string        | Model title (falls back to ID)       |
| author          | string        | Author name (falls back to "Unknown")|
| license         | string        | License name                         |
| licenseUrl      | string \| null | License URL                         |
| sourceUrl       | string \| null | Link to the original model          |
| attributionText | string \| null | Raw attribution text from API       |

---- polyPizzaAttributionLine

```tsx
function polyPizzaAttributionLine(payload: unknown): string
```

Builds a formatted attribution string: "Title" by Author. License: License Name (url) -- Source: url

=== EXAMPLES ===
Example 1: Fetch a model with attribution
---
```tsx
import { Box, Text } from '@reactjit/core';
import { usePolyPizzaModelWithAttribution } from '@reactjit/apis';
import { useThemeColors } from '@reactjit/theme';

export function ModelInfo({ apiKey, modelId }: { apiKey: string; modelId: string }) {
  const colors = useThemeColors();
  const { data, attribution, attributionLine, loading } =
    usePolyPizzaModelWithAttribution(apiKey, modelId);

  if (loading) return <Text style={{ color: colors.muted }}>Loading model...</Text>;
  if (!data) return <Text style={{ color: colors.primary }}>Model not found</Text>;

  return (
    <Box style={{ gap: 6 }}>
      <Text style={{ color: colors.text, fontWeight: 'bold', fontSize: 16 }}>
        {attribution?.title ?? 'Untitled'}
      </Text>
      <Text style={{ color: colors.muted }}>{`By ${attribution?.author ?? 'Unknown'}`}</Text>
      <Text style={{ color: colors.muted, fontSize: 12 }}>{`License: ${attribution?.license}`}</Text>
      {attributionLine && (
        <Text style={{ color: colors.muted, fontSize: 10 }}>{attributionLine}</Text>
      )}
    </Box>
  );
}
```

Example 2: Search for models
---
```tsx
import { useState } from 'react';
import { Box, Text, Input } from '@reactjit/core';
import { usePolyPizzaSearch, polyPizzaAttributionFields } from '@reactjit/apis';
import { useThemeColors } from '@reactjit/theme';

export function ModelSearch({ apiKey }: { apiKey: string }) {
  const colors = useThemeColors();
  const [query, setQuery] = useState<string | null>(null);
  const { data, loading } = usePolyPizzaSearch(apiKey, query);

  const models = data?.results ?? data?.items ?? [];

  return (
    <Box style={{ gap: 8 }}>
      <Input
        placeholder="Search 3D models..."
        onSubmit={(text) => setQuery(text || null)}
      />
      {loading && <Text style={{ color: colors.muted }}>Searching...</Text>}
      {models.map((model, i) => {
        const attr = polyPizzaAttributionFields(model);
        return (
          <Box key={attr.id ?? i} style={{ gap: 2 }}>
            <Text style={{ color: colors.text }}>{attr.title}</Text>
            <Text style={{ color: colors.muted, fontSize: 12 }}>{`by ${attr.author} | ${attr.license}`}</Text>
          </Box>
        );
      })}
    </Box>
  );
}
```

=== CRITICAL RULES ===
- The Poly Pizza API response shape can vary by version. The hooks handle this internally, but search results may be in results, items, or data fields. Check all three.
- polyPizzaAttributionFields is defensive -- it probes multiple field paths to extract author, title, and license. It never throws.
- Always include attribution when displaying models. Most Poly Pizza models are Creative Commons licensed and require credit.
- The API key is sent as x-api-key header by default. Set opts.apiKeyIn to "query" if the API expects it as a query parameter instead.
