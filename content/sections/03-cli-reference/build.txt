=== OVERVIEW ===
`rjit build` bundles your project for development or production. Without arguments, it
produces a Love2D dev build (love/bundle.js). With a platform target, it produces a
self-contained distribution binary.

Every build path runs the static linter first. Lint errors exit with code 1 before esbuild
runs. Post-build bundle checks (duplicate contexts, crypto miner detection) run on the
compiled output. The build also auto-runs `rjit update` to sync runtime files before
bundling (skipped for the storybook and with --no-update).

TSL files (.tsl) in src/ are automatically transpiled to Lua before linting and bundling.
TSL transpilation errors block the build. Video files (.mp4, .mkv, etc.) in the project
are auto-converted to .ogv (Theora) format for Love2D compatibility when ffmpeg is available.

=== API / SYNTAX ===
Usage:
  rjit build [target] [flags]

Dev builds (bundle only, no packaging):
  rjit build                     Love2D dev build (default)
  rjit build love                Love2D dev build (explicit)

Production builds (self-contained binaries):
  rjit build linux               Self-extracting Linux binary (x64)
  rjit build macos               macOS bundle (Intel x64)
  rjit build macmseries          macOS bundle (Apple Silicon arm64)
  rjit build windows             Windows archive (x64)
  rjit build web                 WASM bundle (love.js / Emscripten)
  rjit build dist:love           Self-extracting Love2D binary (host platform)

Flags:
  --no-lint              Skip the lint gate (use sparingly)
  --no-update            Skip auto-update of runtime files
  --debug                Include debug symbols in dist builds
  --target <platform>    Cross-compile target (linux-x64, macos-x64, macos-arm64, windows-x64)

Build pipeline (dev build):
  1. Auto-update runtime files (rjit update)
  2. Transpile .tsl files to .lua
  3. Run static linter (rjit lint)
  4. Bundle with esbuild (IIFE format, ES2020 target, JSX automatic)
  5. Post-build bundle checks (duplicate contexts, miner detection)
  6. Output: love/bundle.js

Build pipeline (dist:love):
  1-5. Same as dev build
  6. Copy lua/, lib/, assets, fonts into staging directory
  7. Create .love archive (zip of staging directory)
  8. Fuse Love2D binary with .love archive
  9. Bundle shared libraries (libquickjs.so, system deps)
  10. Generate self-extracting binary with embedded launcher script
  11. Output: dist/<project-name>

Entry point resolution:
  Love2D: src/main-love.tsx, then src/main.tsx
  Web: src/main-wasm.tsx, then src/main-web.tsx

esbuild flags (from targets.mjs):
  Love2D: --format=iife --global-name=ReactJIT --target=es2020 --jsx=automatic
          --external:react-dom --external:child_process
  Web:    --format=iife --global-name=ReactJIT --target=es2020 --jsx=automatic
          --external:child_process

=== EXAMPLES ===
Example 1: Development build (most common)
---
cd my-app
rjit build
# Output: love/bundle.js
# Run: love .
---
Platforms: love2d

Example 2: Production Linux binary
---
rjit build linux
# Output: dist/my-app (self-extracting binary)
# Run: ./dist/my-app
---
Platforms: love2d

Example 3: macOS Apple Silicon build
---
rjit build macmseries
# Output: dist/my-app.app (macOS bundle)
---
Platforms: love2d

Example 4: Skip lint for quick iteration
---
rjit build --no-lint
# Skips linter -- useful for rapid prototyping, not for final builds
---
Platforms: love2d

Example 5: WASM build for browser deployment
---
rjit build web
# Output: web/bundle.js (served via love.js / Emscripten canvas)
---
Platforms: love2d

=== CRITICAL RULES ===
- Lint errors block the build -- fix them or use --no-lint (not recommended for production)
- Never run raw esbuild commands -- the CLI sets format, global-name, externals, and target correctly
- Production builds (dist:love) bundle ALL shared libraries including glibc for zero-dependency deployment
- The build auto-runs rjit update before bundling -- use --no-update to skip if you manage runtime files manually
- Video files are auto-converted to .ogv if ffmpeg is installed -- this is required for Love2D video playback
