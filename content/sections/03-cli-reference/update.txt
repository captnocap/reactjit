=== OVERVIEW ===
The `reactjit update` command synchronizes the framework runtime files (lua/, lib/, and
reactjit/) from the CLI runtime directory into the current project. It replaces these
directories entirely, bringing the project up to date with the latest framework code.

This command never touches project-specific files (src/, main.lua, conf.lua, package.json).
It is the second step in the framework update pipeline: first `make cli-setup` copies source-of-truth
files into the CLI runtime, then `reactjit update` copies from the CLI runtime into each project.

=== API / SYNTAX ===
Usage:
  reactjit update

Prerequisite:
  Must be run from inside an ReactJIT project directory (detected by presence of main.lua
  or src/ directory).

What it syncs:
  cli/runtime/lua/         ->  <project>/lua/
  cli/runtime/lib/         ->  <project>/lib/
  cli/runtime/reactjit/  ->  <project>/reactjit/

What it does NOT touch:
  src/              Project source code
  main.lua          Love2D entry point
  conf.lua          Love2D configuration
  package.json      Project dependencies
  packaging/        Build customizations

The update is destructive for the synced directories -- existing lua/, lib/, and reactjit/
directories are deleted and replaced entirely.

=== EXAMPLES ===
Example 1: After editing framework source files
---
# Edit source-of-truth files:
#   lua/layout.lua
#   packages/shared/src/primitives.tsx

# Propagate to CLI runtime:
make cli-setup

# Sync into project:
cd storybook
reactjit update

# Rebuild:
reactjit build
---
Platforms: All

Example 2: Hydrate an existing project
---
cd my-project
reactjit update
---
Platforms: All

=== PLATFORM NOTES ===
The update command works for all targets. The synced files include Lua runtime (needed for
Love2D), the QuickJS shared library (needed for Love2D FFI bridge), and the TypeScript
framework packages (needed for all targets).

Note: `reactjit build` runs update automatically before building (unless --no-update is
passed). You only need to run update manually when using `reactjit dev` or `love .`
directly.

=== CRITICAL RULES ===
- Always run `make cli-setup` in the monorepo root BEFORE running `reactjit update`
- The CLI runtime must be populated (`cli/runtime/lua/` and `cli/runtime/reactjit/` must exist)
- Never edit lua/, lib/, or reactjit/ in a project directly -- those are disposable copies managed by update
- The update deletes and replaces the target directories entirely -- any local modifications are lost
- Must be run from inside an ReactJIT project directory
