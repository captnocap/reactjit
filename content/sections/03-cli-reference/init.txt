=== OVERVIEW ===
`rjit init <name>` creates a new ReactJIT project directory with all the files needed to
start building. It copies the Lua runtime, libquickjs.so, framework TypeScript packages,
a starter App.tsx, Love2D entry points (main.lua, conf.lua), and a package.json. It then
runs npm install automatically.

In interactive mode (default when run in a terminal), it presents a checkbox menu for
optional packages. Use arrow keys to navigate, space to toggle, 'a' to toggle all, and
enter to confirm. In non-interactive mode (piped input or CI), it uses defaults.

The init command can also convert existing code from other frameworks into a ReactJIT
project via the --convert flag or interactive conversion flow.

=== API / SYNTAX ===
Usage:
  rjit init <project-name> [flags]

Flags:
  --minimal              Include no optional packages (core + renderer only)
  --all                  Include all optional packages
  --convert <file>       Convert an existing file from another framework
  --theme                Include @reactjit/theme (default: on)
  --router               Include @reactjit/router (default: on)
  --storage              Include @reactjit/storage (default: on)
  --3d                   Include @reactjit/3d
  --audio                Include @reactjit/audio
  --media                Include @reactjit/media
  --game                 Include @reactjit/game
  --apis                 Include @reactjit/apis
  --server               Include @reactjit/server
  --rss                  Include @reactjit/rss
  --webhooks             Include @reactjit/webhooks
  --geo                  Include @reactjit/geo
  --crypto               Include @reactjit/crypto
  --ai                   Include @reactjit/ai
  --controls             Include @reactjit/controls

Default packages (included unless --minimal):
  @reactjit/theme        Design tokens, theme switcher, dark/light support
  @reactjit/router       Navigation and URL routing
  @reactjit/storage      CRUD, schemas, data persistence

Project structure created:
  <name>/
    src/
      App.tsx            Starter application component
      main.tsx           Love2D entry point (renders App into the reconciler)
    lua/                 Lua runtime (layout engine, painter, bridge, etc.)
    lib/                 Native libraries (libquickjs.so)
    reactjit/            Framework TypeScript source (core, renderer, selected packages)
    main.lua             Love2D bootstrap
    conf.lua             Love2D window configuration
    package.json         Dependencies (react, react-reconciler, esbuild, typescript)
    tsconfig.json        Generated with path aliases for selected packages

Supported migration frameworks (--convert):
  .js / .cjs / .mjs      Blessed (Node.js TUI)
  .py                     Tkinter or PyQt6/PySide6 (Python GUI)
  .swift                  SwiftUI (iOS/macOS)
  .dart                   Flutter
  .html / .htm / .tsx / .jsx   HTML / React (Web)

=== EXAMPLES ===
Example 1: Create a minimal project
---
rjit init my-app --minimal
cd my-app
rjit dev
---
Platforms: love2d

Example 2: Create a project with all packages
---
rjit init dashboard --all
---
Platforms: love2d

Example 3: Create with specific packages
---
rjit init synth-app --audio --controls --ai
---
Platforms: love2d

Example 4: Convert an existing Blessed TUI app
---
rjit init my-app --convert legacy-tui.js
# Converted code is in my-app/src/App.tsx
---
Platforms: love2d

=== CRITICAL RULES ===
- The project directory must not already exist -- init refuses to overwrite
- If lua/ or lib/ runtime files are missing from the CLI, run make cli-setup first (monorepo development only)
- The generated tsconfig.json includes path aliases only for selected packages -- add more later by editing paths manually or re-scaffolding
- npm install runs automatically unless stdin is not a TTY (piped/CI mode)
