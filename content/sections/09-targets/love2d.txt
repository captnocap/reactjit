=== OVERVIEW ===
Love2D is the default and primary target. It runs LuaJIT + Love2D + OpenGL 2.1 on desktop platforms (Linux, macOS, Windows). JavaScript executes inside an embedded QuickJS runtime linked as a shared library (`libquickjs.so`). The bridge between JS and Lua is synchronous -- mutation commands and events are exchanged via C FFI trampolines within the same process.

Every ReactJIT app has two Lua entry files:

- `conf.lua` -- Love2D configuration (window title, size, vsync, MSAA, enabled modules)
- `main.lua` -- application bootstrap (loads ReactJIT, wires Love2D callbacks, initializes the bridge)

The CLI generates both when you run `rjit init`. You rarely need to edit `main.lua` -- it follows a standard pattern. `conf.lua` is where you set window properties.

Love2D owns the run loop. Each frame:
1. `love.update(dt)` calls `ReactJIT.update(dt)` -- polls JS for mutation commands, runs layout
2. `love.draw()` calls `ReactJIT.draw()` -- walks the laid-out tree and issues OpenGL draw calls
3. Input callbacks (`love.keypressed`, `love.mousepressed`, etc.) forward to `ReactJIT.safeCall()` -- events are queued for JS to poll on the next frame

React never drives the frame. It only processes events and flushes commands when Lua asks during `love.update()`.

=== API / SYNTAX ===

---- conf.lua -------------------------------------------------------------

Standard Love2D configuration. Set window properties and enable/disable modules.

```lua
function love.conf(t)
  t.identity = "my-app"           -- save directory name
  t.window.title = "My App"       -- window title bar text
  t.window.width = 800            -- initial window width
  t.window.height = 600           -- initial window height
  t.window.vsync = 1              -- 1 = vsync on, 0 = off
  t.window.msaa = 4               -- multisample anti-aliasing (0, 2, 4, 8)
  t.window.resizable = true       -- allow window resizing
  t.modules.joystick = true       -- enable gamepad support
  t.modules.physics = false       -- disable Box2D (not used by ReactJIT)
end
```

---- main.lua -------------------------------------------------------------

Application entry point. Loads ReactJIT, sets up error handling, and wires Love2D callbacks.

```lua
local ReactJIT = require("lua.init")

love.errorhandler = require("lua.bsod")

function love.load()
  love.graphics.setBackgroundColor(0.04, 0.04, 0.06)
  ReactJIT.init({
    mode = "native",
    bundlePath = "love/bundle.js",
    libpath = "lib/libquickjs",
  })
end

function love.update(dt)
  ReactJIT.update(dt)
end

function love.draw()
  ReactJIT.draw()
end
```

ReactJIT.init options:
| Option     | Type   | Description                                      |
|------------|--------|--------------------------------------------------|
| mode       | string | `"native"` for desktop Love2D                   |
| bundlePath | string | Path to the esbuild output (JS bundle)           |
| libpath    | string | Path to libquickjs shared library (without extension) |

---- Input callbacks ------------------------------------------------------

All input is forwarded to ReactJIT via `safeCall()`, which wraps the call in pcall for error resilience:

```lua
function love.keypressed(key, scancode, isrepeat)
  ReactJIT.safeCall("keypressed", key, scancode, isrepeat)
end

function love.mousepressed(x, y, button)
  ReactJIT.safeCall("mousepressed", x, y, button)
end

function love.wheelmoved(x, y)
  ReactJIT.safeCall("wheelmoved", x, y)
end

function love.resize(w, h)
  ReactJIT.safeCall("resize", w, h)
end

function love.textinput(text)
  ReactJIT.safeCall("textinput", text)
end

function love.filedropped(file)
  ReactJIT.safeCall("filedropped", file)
end
```

---- Multi-window ---------------------------------------------------------

Each `<Window>` component spawns a child Love2D process connected over TCP via `lua/window_manager.lua`. Child windows run their own Love2D instance with `lua/child_window/main.lua` and receive layout subtrees from the parent process. This is IPC-based, not threading -- each window has its own OpenGL context.

---- Build commands -------------------------------------------------------

```bash
rjit dev                   # Watch + HMR, launches Love2D automatically
rjit build                 # Dev build (outputs love/bundle.js)
rjit build linux           # Self-extracting Linux binary (x64)
rjit build macos           # macOS bundle (Intel x64)
rjit build macmseries      # macOS bundle (Apple Silicon arm64)
rjit build windows         # Windows archive (x64)
rjit build dist:love       # Self-extracting Linux binary with bundled glibc
```

=== PLATFORM NOTES ===
Love2D:
  OpenGL 2.1 is the graphics backend. This means no compute shaders, no instanced rendering, and no geometry shaders. The painter uses immediate-mode draw calls (love.graphics.rectangle, love.graphics.print, etc.) with stencil-based overflow clipping.

  LuaJIT provides the Lua runtime -- standard Lua 5.1 with JIT compilation and FFI. The QuickJS shared library is loaded via LuaJIT's FFI module.

  MSAA (multisample anti-aliasing) is set in conf.lua. The default template uses MSAA 4. Set to 0 if you need maximum performance on low-end hardware.

  VSync is enabled by default (t.window.vsync = 1). This caps the frame rate to the display refresh rate. Set to 0 for uncapped rendering (useful for benchmarking, not for production).

=== CRITICAL RULES ===
- Never run `love .` directly. Use `rjit dev` which handles esbuild watch and Love2D launch together. Direct launching causes duplicate processes.
- The `mode` field in ReactJIT.init must be `"native"` for desktop Love2D. The WASM mode uses a different bridge transport.
- Edit Lua files in the monorepo root `lua/` directory, not in project-local copies. Run `make cli-setup` then `rjit update` to propagate changes.
- `conf.lua` is read by Love2D before any Lua code runs. It cannot require other modules or use Love2D APIs.
- `love.errorhandler` should be set to `require("lua.bsod")` for the custom error screen. Without it, Love2D's default blue error screen has no back button or reload capability.
