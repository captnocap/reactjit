=== OVERVIEW ===
ReactJIT runs on LuaJIT inside Love2D. The entire rendering pipeline -- layout, painting,
event dispatch, text measurement, animation -- executes in Lua. React runs inside an
embedded QuickJS runtime that communicates with Lua via C FFI trampolines.

The runtime stack:

  Love2D (C/C++) -- owns the run loop, OpenGL context, font rasterizer, event pump
    LuaJIT -- layout engine, painter, event system, capabilities, devtools
      QuickJS (via FFI) -- React reconciler, component tree, event handlers

LuaJIT provides near-native performance for the layout engine and painter. The QuickJS
runtime is embedded as a shared library (libquickjs.so) loaded via LuaJIT FFI. The bridge
uses three host functions:

  __hostFlush(json) -- JS sends mutation commands to Lua (CREATE, UPDATE, REMOVE, APPEND)
  __hostGetEvents() -- JS polls the event queue from Lua (click, keydown, etc.)
  __hostMeasureText(text, fontSize, fontWeight, fontFamily, maxWidth, letterSpacing) --
    JS requests text measurement from Love2D's font rasterizer

All Lua source files live at the monorepo root in lua/. This is the source of truth.
Files are copied to projects via the CLI sync pipeline:

  lua/ -> make cli-setup -> cli/runtime/lua/ -> reactjit update -> project/lua/

Never edit lua files inside a project directory or cli/runtime/ -- those are disposable
copies. Always edit the source at the monorepo root.

=== EXAMPLES ===
Example 1: Adding a new Lua module
---
```lua
-- lua/my_feature.lua
-- Source of truth: monorepo root lua/

local M = {}

function M.init(tree, bridge)
  -- Called once at startup with references to tree and bridge modules
end

function M.update(dt)
  -- Called every frame from love.update
end

function M.draw()
  -- Called every frame from love.draw (after painter)
end

return M
```

Then wire it into the main loop in lua/init.lua:

```lua
local MyFeature = require("lua.my_feature")
MyFeature.init(tree, bridge)

-- In love.update:
MyFeature.update(dt)

-- In love.draw:
MyFeature.draw()
```

Example 2: Registering a capability instead of a raw module
---
```lua
-- lua/capabilities/countdown.lua
-- Preferred pattern: register as a capability for React integration

local Capabilities = require("lua.capabilities")

Capabilities.register("Countdown", {
  visual = false,  -- no visual output (non-visual capability)

  schema = {
    seconds = { type = "number", default = 10, desc = "Starting seconds" },
    running = { type = "bool", default = true, desc = "Whether counting" },
  },

  events = { "onTick", "onComplete" },

  create = function(nodeId, props)
    return { remaining = props.seconds or 10, elapsed = 0 }
  end,

  update = function(nodeId, props, prev, state)
    if props.seconds ~= prev.seconds then
      state.remaining = props.seconds
      state.elapsed = 0
    end
  end,

  tick = function(nodeId, state, dt, pushEvent, props)
    if not (props.running ~= false) then return end
    state.elapsed = state.elapsed + dt
    if state.elapsed >= 1 then
      state.elapsed = state.elapsed - 1
      state.remaining = state.remaining - 1
      pushEvent({
        type = "capability",
        payload = {
          type = "capability",
          targetId = nodeId,
          handler = "onTick",
          remaining = state.remaining,
        },
      })
      if state.remaining <= 0 then
        pushEvent({
          type = "capability",
          payload = {
            type = "capability",
            targetId = nodeId,
            handler = "onComplete",
          },
        })
      end
    end
  end,

  destroy = function(nodeId, state)
    -- cleanup if needed
  end,
})
```

Example 3: Using the capability from React
---
```tsx
import { Native } from '@reactjit/core';
import { useState } from 'react';

// One-liner wrapper -- the user never sees the bridge
function Countdown(props: {
  seconds: number;
  running?: boolean;
  onTick?: (e: { remaining: number }) => void;
  onComplete?: () => void;
}) {
  return <Native type="Countdown" {...props} />;
}

// Usage
export function TimerApp() {
  const [remaining, setRemaining] = useState(10);

  return (
    <Countdown
      seconds={10}
      running
      onTick={(e) => setRemaining(e.remaining)}
      onComplete={() => console.log('Done!')}
    />
  );
}
```

=== CRITICAL RULES ===
- Always edit lua/ at the monorepo root. Never edit cli/runtime/lua/ or project/lua/ directly -- those are disposable copies overwritten by the sync pipeline.
- After editing any Lua file, run: make cli-setup, then cd project && reactjit update && reactjit build.
- The storybook reads Lua via symlink (storybook/love/lua -> ../../lua). Never replace this symlink with a real directory.
- Capability files in lua/capabilities/ are auto-loaded by capabilities.lua. No manual registration wiring is needed.
- LuaJIT has a 65KB bytecode limit per function. If a function gets too large, split it into multiple local functions.
- The QuickJS runtime is single-threaded. Heavy computation in JS blocks the frame. Move expensive work to Lua (which runs in the Love2D thread) or use Lua coroutines for frame-distributed work.
