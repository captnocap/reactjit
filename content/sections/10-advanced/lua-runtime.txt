=== OVERVIEW ===
The Lua runtime consists of a collection of modules that collectively implement the rendering
pipeline for the Love2D target. The entry point is init.lua, which detects the
operating mode, loads the appropriate bridge, and initializes all submodules. On every
frame, the runtime ticks the JavaScript engine, drains mutation commands, advances
animations, runs layout, and paints the tree.

All Lua files live at lua/ in the monorepo root (source of truth) and are copied into
projects via `make cli-setup` and `reactjit update`.

=== API / SYNTAX ===
Module Inventory:

  init.lua           - Entry point. Detects mode (native/web/canvas), loads all
                       modules, implements love.update/draw/input callbacks.
                       Manages HMR, interaction styles, deferred mount protocol.

  tree.lua           - Retained instance tree. Receives mutation commands from JS
                       (createNode, updateProps, appendChild, removeChild, etc.)
                       and maintains the live node tree. Tracks dirty state.

  layout.lua         - Flexbox layout engine. Computes {x, y, w, h} for every node.
                       Handles flex-grow/shrink, wrap, justify, align, padding,
                       margin, gap, text measurement, min/max clamping.

  painter.lua        - Love2D painter. Walks computed tree and issues love.graphics
                       calls. Handles backgrounds, borders, text, images, shadows,
                       gradients, transforms, opacity, clipping, scrollbars.

  events.lua         - Event handling. Hit testing, hover tracking, drag state,
                       event creation (click, release, keydown, wheel, touch,
                       gamepad), bubble path construction.

  measure.lua        - Text measurement. Font loading/caching by (size, family,
                       weight) tuple. measureText() for layout, getFont() for
                       painter. Letter spacing width calculation.

  bridge_quickjs.lua - QuickJS FFI bridge. Loads libquickjs.so via LuaJIT FFI.
                       eval(), callGlobal(), tick(), drainCommands(), pushEvent().

  bridge_fs.lua      - Module.FS bridge. File-based communication via Emscripten
                       shared filesystem. poll(), flush(), emit().

  animate.lua        - Lua-side transitions and keyframe animations. Processes
                       CSS-style transition declarations and animation keyframes.
                       Interpolates style values per frame.

  images.lua         - Image loading and caching. Manages love.graphics.newImage
                       instances. Handles async loading and placeholder display.

  focus.lua          - Focus manager for Lua-owned inputs (TextEditor). Tracks
                       which node has focus, handles focus/blur transitions.

  texteditor.lua     - Lua-owned multi-line text editor. Handles cursor, selection,
                       line numbers, scrolling, key input, mouse selection.
                       Renders directly via love.graphics calls.

  errors.lua         - Error overlay. Displays runtime errors as a full-screen
                       overlay with stack traces. Uses raw love.graphics calls
                       (independent of the tree/layout system).

  inspector.lua      - Visual inspector (F12). Shows node boundaries, computed
                       dimensions, style properties. Highlights hovered nodes.
                       Displays layout timing and paint timing.

  console.lua        - Interactive eval console (backtick key). Evaluates JS
                       expressions via the bridge and displays results.

  screenshot.lua     - Headless screenshot capture. Waits for the first stable
                       frame, captures the framebuffer, saves as PNG.

  zindex.lua         - Z-index sorting. Returns paint-ordered child lists based
                       on zIndex style property (stable sort, ascending).

  storage.lua        - Persistent storage RPC handlers. Provides get/set/remove
                       methods backed by love.filesystem.

  target_love2d.lua  - Love2D target configuration. Contains target-specific
                       constants and setup.

  http.lua           - Async HTTP client. Supports GET/POST/PUT/DELETE with headers,
                       proxy (HTTP/SOCKS5), and concurrent request pooling via
                       love.thread. Bridges to LuaSocket socket.http library.

  network.lua        - WebSocket network manager. Manages multiple concurrent
                       WebSocket connections with auto-reconnect, backoff,
                       and SOCKS5 proxy support (for Tor tunneling).

  websocket.lua      - Pure Lua WebSocket client (vendored from flaribbit/love2d-
                       lua-websocket). Implements RFC 6455 frame encoding/decoding.
                       Low-level; use network.lua for high-level connection management.

  json.lua           - JSON encoder/decoder. Used internally to serialize/deserialize
                       mutation commands between JS and Lua. Handles objects, arrays,
                       strings, numbers, booleans, null.

  socks5.lua         - SOCKS5 proxy client (RFC 1928 + RFC 1929). Establishes TCP
                       tunnels through SOCKS5 proxies with no-auth and username/
                       password auth. Used by http.lua and network.lua for Tor support.

  tor.lua            - Tor subprocess manager. Auto-starts Tor, creates hidden
                       services, polls for bootstrap completion, graceful shutdown.
                       Opt-in via config.tor in ReactLove.init().

  codeblock.lua      - Syntax-highlighted code block renderer. Direct love.graphics
                       rendering (like texteditor.lua). Highlights keywords, imports,
                       strings, numbers, comments.

  textselection.lua  - Text selection and clipboard copy for Text nodes. Manages
                       global selection state, drag-to-select, multi-line extraction,
                       Ctrl+C / Cmd+C copy support.

  log_colors.lua     - ANSI color tag library for Lua-side logging. Provides color-
                       coded print() wrappers (log.info, log.warn, log.error, etc.)
                       for readable terminal output from Lua modules.

  debug_log.lua      - Structured debug logging with source location. Wraps log_colors
                       and adds file:line prefixes. Used internally by runtime modules;
                       toggle verbosity without editing call sites.

  devtools.lua       - Developer tools aggregator. Coordinates inspector, console,
                       screenshot, and debug_log. Loaded automatically in dev mode;
                       stripped from production builds.

  settings.lua       - Persistent key-value store for framework settings (inspector
                       position, theme selection, window state). Backed by love.filesystem.

  theme_menu.lua     - In-app theme switcher overlay (Ctrl+T). Lists all registered
                       themes, applies them live, and persists the selection via settings.lua.

  system_panel.lua   - System info overlay. Shows frame time, memory usage, node count,
                       layout timing, and paint timing. Toggled via a dev shortcut.

  permit.lua         - Runtime permission guard. Enforces capability-level permissions
                       declared in the app manifest. Blocks RPC calls that exceed
                       declared scope and logs violations via audit.lua.

  audit.lua          - Security audit log. Records all RPC calls, capability invocations,
                       and permission decisions to a local audit trail. Used with permit.lua.

  manifest.lua       - App manifest loader. Reads the app's capability declarations,
                       metadata, and permission scope. Feeds permit.lua and the
                       capability registry.

  cart_reader.lua    - Cartridge reader for self-contained .rjit bundles. Extracts
                       embedded Lua runtime, JS bundle, and assets from a single-file
                       distribution. Enables the portable cart deployment format.

  capabilities.lua   - Capability registry. All native capabilities register here
                       via Capabilities.register(). Provides the capabilities:list
                       RPC used by useCapabilities().

  pty.lua            - PTY process manager. Spawns pseudo-terminal processes, pipes
                       output to the bridge, handles resize (SIGWINCH), and manages
                       session lifecycles for Terminal and SemanticTerminal.

  vterm.lua          - VT100/VT220 terminal emulator (pure Lua). Decodes raw PTY
                       byte streams into a Unicode text buffer. Used by SemanticTerminal
                       to strip ANSI codes before classification.

  semantic_graph.lua - Semantic graph builder. Aggregates classified rows into turns,
                       nodes, and state transitions. Emits onGraphUpdate and onStateChange
                       events to the React layer.

  session_recorder.lua - PTY output recorder. Captures raw PTY bytes with timestamps
                         into a .rec.lua file for later playback. See session-recording docs.

  session_player.lua - Playback engine for .rec.lua recordings. Drives the vterm +
                       classifier pipeline at recorded timing (with speed control and
                       seeking). Used by SemanticTerminal in playback mode.

=== EXAMPLES ===
Example 1: Module Loading Order (Native Mode)
---
When ReactLove.init() is called in native mode:

  1. resolveBasePath()                  -- Find the lua/ directory
  2. require("lua.bridge_quickjs")      -- Load QuickJS FFI bridge
  3. BridgeQJS.new(libpath)             -- Create bridge with libquickjs.so
  4. require("lua.measure")             -- Text measurement module
  5. require("lua.images")              -- Image cache
  6. require("lua.animate")             -- Animation engine
  7. require("lua.tree")                -- Retained tree
  8. tree.init({ images, animate })     -- Wire dependencies
  9. animate.init({ tree })             -- Wire back-reference
  10. require("lua.layout")             -- Layout engine
  11. layout.init({ measure })          -- Wire text measurement
  12. require("lua.painter")            -- Painter
  13. painter.init({ measure, images }) -- Wire dependencies
  14. require("lua.events")             -- Event system
  15. events.setTreeModule(tree)         -- Wire tree reference
  16. require("lua.texteditor")         -- Text editor
  17. texteditor.init({ measure })      -- Wire measurement
  18. bridge:eval(bundleJS)             -- Load the JS bundle into QuickJS

  Always-loaded (independent, no init):
  - require("lua.errors")              -- Error overlay
  - require("lua.inspector")           -- F12 inspector
  - require("lua.console")             -- Backtick console
  - require("lua.focus")               -- Focus manager
  - require("lua.log_colors")          -- Color logging
  - require("lua.debug_log")           -- Structured debug logging
  - require("lua.devtools")            -- Dev tool aggregator (dev builds only)
  - require("lua.settings")            -- Persistent settings store
  - require("lua.theme_menu")          -- Theme switcher overlay
  - require("lua.system_panel")        -- System info overlay
  - require("lua.permit")              -- Permission guard
  - require("lua.audit")               -- Audit log
  - require("lua.manifest")            -- App manifest loader
  - require("lua.cart_reader")         -- Cartridge format reader
