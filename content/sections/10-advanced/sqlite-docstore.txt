=== OVERVIEW ===
ReactJIT provides two complementary layers for data persistence: a low-level SQLite3 wrapper (lua/sqlite.lua) using LuaJIT FFI for direct SQL access, and a high-level schema-free document store (lua/docstore.lua) offering a MongoDB-like API. Both are accessible from React via the RPC bridge (useBridge). SQLite uses the system libsqlite3.so shared library, with automatic fallback to a stub module if unavailable. DocStore builds on SQLite, storing JSON documents in a single table with query operators for filtering.

=== API / SYNTAX ===

**Lua API — SQLite (lua/sqlite.lua)**

```lua
local sqlite = require("lua.sqlite")

-- Open database (file or in-memory)
local db = sqlite.open("saves.db")   -- file-backed
local db = sqlite.open()              -- in-memory

-- Check availability
if not sqlite.available then
  -- libsqlite3.so not found
end

-- Execute statements (DDL, DML)
db:exec("CREATE TABLE IF NOT EXISTS saves (id INTEGER PRIMARY KEY, data TEXT)")
db:exec("INSERT INTO saves (data) VALUES (?)", "hello world")

-- Query with results
local rows = db:query("SELECT * FROM saves")
for _, row in ipairs(rows) do
  print(row.id, row.data)
end

-- Scalar query (single value)
local count = db:scalar("SELECT COUNT(*) FROM saves")

-- Check existence (returns boolean)
local exists = db:exists("SELECT 1 FROM saves WHERE id = ?", 5)

-- Get insert ID and change count
local id = db:lastInsertRowId()
local changes = db:changes()

-- Close database
db:close()
```

**Lua API — DocStore (lua/docstore.lua)**

```lua
local docstore = require("lua.docstore")

-- Open document store
local db = docstore.open("game.db")   -- file-backed
local db = docstore.open()            -- in-memory

-- Save documents (auto-generates _id if missing)
db:save("players", { name = "Link", hp = 100 })
db:save("players", { name = "Zelda", hp = 80, role = "princess" })

-- Find all documents in a collection
local all = db:find("players")

-- Find with query
local strong = db:find("players", { hp = { gte = 50 } })
local link = db:findOne("players", { name = "Link" })

-- Update document
db:update("players", link._id, { hp = 120 })

-- Remove document
db:remove("players", link._id)

-- Drop collection
db:drop("players")

-- List collections
local collections = db:collections()

-- Close database
db:close()
```

**Query Operators (DocStore)**

```lua
-- Exact match
{ field = value }

-- Comparison operators
{ field = { gt = 10 } }          -- greater than
{ field = { gte = 10 } }         -- greater than or equal
{ field = { lt = 10 } }          -- less than
{ field = { lte = 10 } }         -- less than or equal
{ field = { ne = "x" } }         -- not equal

-- Pattern matching
{ field = { like = "%pattern%" } }  -- SQL LIKE

-- Array contains (JSON arrays)
{ field = { contains = "value" } }

-- Nested fields (dot notation)
{ ["stats.hp"] = { gte = 50 } }

-- Multiple fields (AND logic)
{ name = "Link", hp = { gt = 90 } }
```

**React/TypeScript API (via RPC bridge)**

Currently, SQLite and DocStore are Lua-only. To expose them to React:

1. Register RPC handlers in lua/init.lua:
```lua
rpcHandlers["db:save"] = function(args)
  return db:save(args.collection, args.doc)
end

rpcHandlers["db:find"] = function(args)
  return db:find(args.collection, args.query)
end
```

2. Call from React:
```tsx
import { useBridge } from '@reactjit/core';

function Component() {
  const bridge = useBridge();

  const savePlayer = async (data: any) => {
    await bridge.rpc('db:save', {
      collection: 'players',
      doc: data
    });
  };

  const loadPlayers = async () => {
    const players = await bridge.rpc('db:find', {
      collection: 'players',
      query: { hp: { gte: 50 } }
    });
    return players;
  };
}
```

=== EXAMPLES ===

Example 1: Game Save System (Lua)
---
local sqlite = require("lua.sqlite")
local db = sqlite.open("savegame.db")

-- Initialize schema
db:exec([[
  CREATE TABLE IF NOT EXISTS saves (
    slot INTEGER PRIMARY KEY,
    name TEXT,
    level INTEGER,
    playtime REAL,
    data TEXT,
    timestamp TEXT DEFAULT (datetime('now'))
  )
]])

-- Save game
local function saveGame(slot, playerData)
  db:exec(
    "INSERT OR REPLACE INTO saves (slot, name, level, playtime, data) VALUES (?, ?, ?, ?, ?)",
    slot,
    playerData.name,
    playerData.level,
    playerData.playtime,
    json.encode(playerData)
  )
end

-- Load game
local function loadGame(slot)
  local row = db:query("SELECT * FROM saves WHERE slot = ?", slot)[1]
  if not row then return nil end
  return json.decode(row.data)
end

-- List saves
local function listSaves()
  return db:query("SELECT slot, name, level, playtime, timestamp FROM saves ORDER BY timestamp DESC")
end

Example 2: Schema-Free Player Inventory (DocStore)
---
local docstore = require("lua.docstore")
local db = docstore.open("inventory.db")

-- Add items (no schema needed)
db:save("items", { name = "Master Sword", type = "weapon", damage = 50, durability = 100 })
db:save("items", { name = "Hylian Shield", type = "shield", defense = 40, durability = 80 })
db:save("items", { name = "Red Potion", type = "consumable", healing = 100, quantity = 3 })

-- Find all weapons
local weapons = db:find("items", { type = "weapon" })

-- Find consumables with quantity > 1
local stackables = db:find("items", {
  type = "consumable",
  quantity = { gt = 1 }
})

-- Find items with durability below 50%
local damaged = db:find("items", { durability = { lt = 50 } })

-- Update item durability
local sword = db:findOne("items", { name = "Master Sword" })
if sword then
  db:update("items", sword._id, { durability = sword.durability - 10 })
end

Example 3: Multi-Collection Game State (DocStore)
---
local docstore = require("lua.docstore")
local db = docstore.open("rpg.db")

-- Initialize game state across multiple collections
local function newGame(playerName)
  -- Player profile
  local player = db:save("players", {
    name = playerName,
    level = 1,
    xp = 0,
    gold = 100,
    stats = { hp = 100, mp = 50, str = 10, int = 8 }
  })

  -- Starting inventory
  db:save("inventory", { playerId = player._id, itemId = "wooden_sword", equipped = true })
  db:save("inventory", { playerId = player._id, itemId = "cloth_armor", equipped = true })

  -- Quest log
  db:save("quests", { playerId = player._id, questId = "tutorial", status = "active", progress = 0 })

  return player
end

-- Load full game state
local function loadGameState(playerId)
  return {
    player = db:findOne("players", { _id = playerId }),
    inventory = db:find("inventory", { playerId = playerId }),
    quests = db:find("quests", { playerId = playerId, status = "active" })
  }
end

-- Nested field queries
local function findStrongPlayers()
  return db:find("players", { ["stats.str"] = { gte = 15 } })
end

Example 4: React Integration via RPC
---
// lua/init.lua — register handlers
local docstore = require("lua.docstore")
local gameDb = docstore.open("game.db")

rpcHandlers["game:save"] = function(args)
  return gameDb:save(args.collection, args.doc)
end

rpcHandlers["game:find"] = function(args)
  return gameDb:find(args.collection, args.query)
end

// React component
import { useBridge } from '@reactjit/core';
import { useState, useEffect } from 'react';

function PlayerList() {
  const bridge = useBridge();
  const [players, setPlayers] = useState([]);

  useEffect(() => {
    bridge.rpc('game:find', {
      collection: 'players',
      query: { level: { gte: 10 } }
    }).then(setPlayers);
  }, []);

  const createPlayer = async (name: string) => {
    await bridge.rpc('game:save', {
      collection: 'players',
      doc: { name, level: 1, hp: 100 }
    });
    // Refresh list
  };

  return (
    <Box>
      {players.map(p => (
        <Text key={p._id} fontSize={14}>
          {p.name} — Level {p.level}
        </Text>
      ))}
    </Box>
  );
}

=== CRITICAL RULES ===
- **Check availability before use** — `sqlite.available` is false if libsqlite3.so is not found. Wrap DB logic in availability checks to prevent crashes.
- **SQLite uses OS filesystem** — paths are absolute or relative to the process CWD, NOT Love2D's save directory. Use `love.filesystem.getSource()` to resolve paths when needed.
- **DocStore auto-generates _id** — every saved document gets a UUID `_id` field if not provided. Use this for updates and deletes.
- **Query operators require tables** — direct value queries like `{ hp = 100 }` work, but operators like `{ hp = { gt = 100 } }` require wrapping the condition in a table.
- **Nested field queries use dot notation** — access nested JSON fields with `{ ["stats.hp"] = value }`. Sanitization strips all characters except alphanumeric, underscore, and dot.
- **Multiple query fields are AND'd** — `{ name = "Link", hp = { gt = 50 } }` means `name = "Link" AND hp > 50`. For OR logic, perform multiple queries.
- **Prepared statements are parameterized** — all Lua API methods use `?` placeholders for safety. Never concatenate user input into SQL strings.
- **Close databases when done** — call `db:close()` to release file handles, especially in long-running apps or when switching save files.
- **In-memory databases are ephemeral** — `sqlite.open()` without a path creates a database that exists only in RAM and is lost when the app closes.
- **DocStore stores JSON** — all documents are JSON-encoded. Lua tables are automatically serialized. Functions, userdata, and circular references are not supported.
- **System dependency** — requires `libsqlite3.so.0` (Debian/Ubuntu: `apt install libsqlite3-0`). The framework does NOT bundle SQLite by default — projects must include it in `lib/` for distribution.
