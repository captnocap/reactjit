=== OVERVIEW ===
ReactJIT includes a Chrome-style devtools panel that opens with F12. The panel docks to the
bottom of the window and provides multiple tabs for inspecting different aspects of the
running application.

Tabs:

- Elements -- the visual inspector. Hover over the canvas to highlight nodes with margin
  (orange), padding (green), and content (blue) overlays. Click to select a node and see
  its computed styles, dimensions, parent/child hierarchy, and handler list. Inline style
  editing is supported -- click a style value to edit it live.

- Console -- JavaScript eval console running inside the QuickJS context. Supports watch
  expressions (:watch expr) that update every frame. Debug data from useDebug is accessible
  via __debug['key'].

- Network -- displays bridge traffic (mutation commands and events) with timing.

- Performance -- frame timing graph showing layout and paint milliseconds per frame.

- Wireframe -- renders the element tree as a wireframe overlay with all borders visible.

- Logs -- captures debug log output from Lua modules.

- Source -- live source code viewer that shows the TypeScript source of the selected
  component (when source maps are available).

The panel supports two modes:

- Docked (default) -- the panel occupies the bottom portion of the window. The app viewport
  shrinks to fit above it. Toggle with F12.

- Pop-out -- the panel detaches into a separate Love2D window via Ctrl+Shift+D. The app
  gets the full viewport back. The devtools window shows the same tabs and tracks the same
  tree. Pop back with Ctrl+Shift+D or Escape.

Canvas overlays (hover highlight, selected outline, tooltip, performance bar) always render
on the main canvas regardless of which tab is active or whether the panel is popped out.

=== API / SYNTAX ===
```tsx
import { DevToolsEmbed } from '@reactjit/core';
```

DevToolsEmbed Props:
| Prop  | Type           | Default   | Description                           |
|-------|----------------|-----------|---------------------------------------|
| w     | number/string  | undefined | Width of the embed region             |
| h     | number/string  | undefined | Height of the embed region            |
| style | Style          | undefined | Style applied to the embed container  |

Keyboard shortcuts:
| Key           | Action                                    |
|---------------|-------------------------------------------|
| F12           | Toggle devtools open/closed               |
| `             | Open console tab (opens devtools if closed)|
| Tab           | Toggle tree panel sidebar (in Elements)   |
| Ctrl+Shift+D  | Toggle pop-out / dock back               |
| Escape        | Close devtools or clear selection          |

=== EXAMPLES ===
Example 1: Embedding devtools in the React tree
---
```tsx
import { Box, DevToolsEmbed } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function AppWithDevTools() {
  const c = useThemeColors();

  return (
    <Box style={{ width: '100%', height: '100%' }}>
      <Box style={{ flexGrow: 1, padding: 16, backgroundColor: c.bg }}>
        {/* App content */}
      </Box>
      <DevToolsEmbed style={{ width: '100%', height: 300 }} />
    </Box>
  );
}
```

Example 2: Using the console for live debugging
---
```
Press ` to open the console, then type:

> __debug                         // all registered debug entries
> __debug['PlayerHUD']            // specific component data
> __debug['PlayerHUD'].health     // specific field

Watch expressions (update every frame):
> :watch __debug['PlayerHUD'].health
> :watch __debug['GameState'].score

Remove a watch:
> :unwatch 0

Clear output:
> :clear
```

Platforms: love2d

=== PLATFORM NOTES ===
Love2D:
  The devtools panel is implemented entirely in Lua (lua/devtools/main.lua). It draws
  using raw Love2D graphics calls and does not participate in the React tree or layout
  pipeline. When closed, it costs one boolean check per frame.

  The DevToolsEmbed component is a visual capability (lua/capabilities/devtools_embed.lua)
  that renders the real devtools panel into a node's computed bounds. It is the same panel
  as F12, just embedded in the layout.

  Pop-out mode uses the window_manager to create a separate Love2D window. The devtools
  panel renders there while canvas overlays continue on the main window.

  The performance tab shows:
  - FPS counter with color coding (green > 55fps, yellow > 30fps, red below)
  - Layout time in milliseconds
  - Paint time in milliseconds
  - Total node count in the tree

=== CRITICAL RULES ===
- DevToolsEmbed needs explicit dimensions (w/h or style width/height). It is a visual capability and follows the surface sizing rules.
- The console evaluates expressions in the QuickJS context, not a browser context. DOM APIs do not exist.
- Inline style editing in the Elements tab applies changes live to the Lua tree. Changes are not persisted -- they reset on the next React re-render that touches that node.
- The devtools panel consumes keyboard and mouse events when it has focus. Click outside the panel to return input to the app.
