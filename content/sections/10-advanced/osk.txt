=== OVERVIEW ===
The On-Screen Keyboard (lua/osk.lua) is a Lua-owned singleton overlay that enables text input when using a gamepad or controller. When a TextInput gains focus via gamepad navigation, the OSK opens automatically and intercepts all gamepad input. D-pad navigates keys, A button selects, B button closes. Text is injected into the target TextInput using the existing event pipeline (textinput and keydown events). The OSK supports multiple keyboard layouts (QWERTY, AZERTY, QWERTZ, Spanish, numeric, symbols) and is context-aware based on the TextInput's keyboardType prop.

=== API / SYNTAX ===

**Lua API (lua/osk.lua)**

The OSK is automatically initialized by the framework. Most apps do not need to interact with it directly.

```lua
local OSK = require("lua.osk")

-- Initialize (called by framework in lua/init.lua)
OSK.init({ measure = measureFunction })

-- Check if OSK is open
local isOpen = OSK.isOpen()

-- Open OSK for a specific TextInput node
OSK.open(node, joystickId, pushEventCallback)
-- node: TextInput instance tree node
-- joystickId: integer ID of the gamepad
-- pushEventCallback: function to inject text events

-- Close OSK
OSK.close()

-- Handle gamepad input (called by framework)
OSK.handleGamepadPressed(button, joystickId)  -- "dpup", "dpdown", "dpleft", "dpright", "a", "b"
OSK.handleGamepadAxis(axis, value, joystickId)  -- leftx/lefty for analog stick navigation

-- Update (called in love.update)
OSK.update(dt)

-- Draw (called in love.draw if isOpen)
OSK.draw()
```

**Keyboard Layouts**

Available layouts (internal, selected based on context):
- QWERTY (default, English)
- AZERTY (French)
- QWERTZ (German)
- SPANISH
- NUMERIC (for keyboardType="numeric")
- SYMBOLS / SYMBOLS2 (accessible via #+= key)

**Key Actions**

Keys can be characters ("a", "1", "!") or actions:
- `shift` — toggle uppercase
- `capslock` — lock uppercase (not implemented in base OSK)
- `backspace` — delete previous character
- `submit` — submit/close (triggers "return" keydown event)
- `space` — insert space character
- `symbols` / `symbols2` — switch to symbol layouts
- `letters` — return to letter layout
- `numeric` — switch to numeric layout

**Context-Aware Behavior**

The OSK checks `node.props.keyboardType` when opening:
- `"numeric"` → opens NUMERIC layout
- `"default"` or undefined → opens QWERTY (or user's locale layout)

**React Integration**

The OSK is fully automatic. React components do not need to invoke it — it opens when a TextInput gains focus via gamepad.

```tsx
import { TextInput } from '@reactjit/core';

// OSK opens automatically when focused with gamepad
<TextInput keyboardType="numeric" />   // opens numeric keypad
<TextInput keyboardType="default" />   // opens QWERTY
```

Example 1: Automatic OSK Behavior
---
import React, { useState } from 'react';
import { Box, Text, TextInput } from '@reactjit/core';

function GamepadTextEntry() {
  const [name, setName] = useState('');
  const [score, setScore] = useState('');

  return (
    <Box style={{ gap: 16 }}>
      {/* OSK opens as QWERTY when focused with gamepad */}
      <Box>
        <Text fontSize={12} color="#888">Player Name</Text>
        <TextInput
          value={name}
          onChangeText={setName}
          placeholder="Enter name..."
          style={{ fontSize: 14 }}
        />
      </Box>

      {/* OSK opens as NUMERIC when focused with gamepad */}
      <Box>
        <Text fontSize={12} color="#888">Score</Text>
        <TextInput
          value={score}
          onChangeText={setScore}
          keyboardType="numeric"
          placeholder="0"
          style={{ fontSize: 14 }}
        />
      </Box>
    </Box>
  );
}

// Gamepad usage:
// 1. Navigate to TextInput with D-pad
// 2. Press A to focus → OSK opens
// 3. Navigate keys with D-pad, select with A
// 4. Press B to close OSK

Example 3: OSK Event Flow (Internal)
---
-- When gamepad A is pressed on a focused TextInput:
-- lua/init.lua calls OSK.open(node, joystickId, pushEvent)

-- OSK opens and takes over gamepad input.
-- When user selects a key:

-- For character keys ("a", "1", "!"):
OSK.injectText("a")  -- calls pushEvent("textinput", { text = "a" })

-- For backspace:
pushEvent("keydown", { key = "backspace", node = targetNode })

-- For submit (Enter):
pushEvent("keydown", { key = "return", node = targetNode })
OSK.close()

-- Target TextInput receives these events via normal event dispatch
-- and updates its state accordingly. No special handling needed.
---
Platforms: Love2D

Example 4: Multi-Layout Form
---
import React, { useState } from 'react';
import { Box, Text, TextInput } from '@reactjit/core';

function RegistrationForm() {
  const [username, setUsername] = useState('');
  const [email, setEmail] = useState('');
  const [age, setAge] = useState('');
  const [password, setPassword] = useState('');

  return (
    <Box style={{ gap: 12 }}>
      {/* QWERTY layout */}
      <Box>
        <Text fontSize={12}>Username</Text>
        <TextInput
          value={username}
          onChangeText={setUsername}
          keyboardType="default"
        />
      </Box>

      {/* QWERTY layout (email optimized would need custom keyboardType) */}
      <Box>
        <Text fontSize={12}>Email</Text>
        <TextInput
          value={email}
          onChangeText={setEmail}
          keyboardType="default"
        />
      </Box>

      {/* NUMERIC layout */}
      <Box>
        <Text fontSize={12}>Age</Text>
        <TextInput
          value={age}
          onChangeText={setAge}
          keyboardType="numeric"
        />
      </Box>

      {/* QWERTY layout */}
      <Box>
        <Text fontSize={12}>Password</Text>
        <TextInput
          value={password}
          onChangeText={setPassword}
          secureTextEntry
          keyboardType="default"
        />
      </Box>
    </Box>
  );
}

// OSK automatically switches between QWERTY and NUMERIC
// as user navigates between fields with gamepad.

Example 5: OSK Navigation Patterns
---
-- Gamepad Controls (automatic, no config needed):

-- D-pad / Left Stick:
--   Up    → move selection up
--   Down  → move selection down
--   Left  → move selection left
--   Right → move selection right

-- A button:
--   Select highlighted key (insert character or perform action)

-- B button:
--   Close OSK without submitting

-- Analog stick navigation:
--   Smooth cursor movement with acceleration
--   NAV_SPEED controls sensitivity

-- D-pad repeat:
--   Hold D-pad direction for continuous movement
--   NAV_REPEAT_DELAY: initial delay before repeat starts (0.4s)
--   NAV_REPEAT_RATE: repeat interval (0.1s)

-- Shift key behavior:
--   Press Shift → next character is uppercase
--   Shift is "sticky" — highlighted after press, deactivated after next char
---
Platforms: Love2D

=== CRITICAL RULES ===
- **Love2D only** — the OSK is Lua-owned and uses Love2D's drawing and gamepad APIs. Not available in terminal, web, or other targets.
- **Singleton pattern** — only one OSK can be open at a time. Opening for a new TextInput closes the previous OSK.
- **Gamepad-triggered only** — the OSK does NOT open for mouse/keyboard focus. It only opens when a TextInput is focused via gamepad navigation.
- **Intercepts all gamepad input** — when OSK is open, gamepad events do NOT propagate to the game/app. Press B to close OSK and resume normal input.
- **No touch support yet** — the OSK is designed for gamepad navigation. Touch/mouse support would require additional hit testing and event handling.
- **Context-aware layout selection** — `keyboardType` prop determines initial layout. Supported values: `"default"`, `"numeric"`. Other types (email, url, phone) fall back to default.
- **Layout switching during session** — users can switch between letters, symbols, and numeric layouts using action keys (ABC, #+=, 123) during a single OSK session.
- **Shift is sticky** — pressing Shift highlights the key and uppercases the NEXT character only. Not a toggle. For caps lock behavior, hold Shift and press multiple keys.
- **Submit closes OSK** — pressing Enter/submit closes the OSK and triggers a "return" keydown event on the TextInput. Apps can handle this to submit forms.
- **Text injection uses existing events** — OSK does NOT directly mutate TextInput state. It pushes "textinput" and "keydown" events, which are handled by the normal event dispatcher. This keeps the OSK decoupled from React.
- **Requires measure.lua** — OSK uses the text measurement API to calculate key sizes and layout. `OSK.init({ measure = ... })` is required.
- **No custom keyboard layouts from React** — keyboard layouts are Lua-side constants. To add a new layout (e.g., Cyrillic, Arabic), edit `lua/osk.lua` directly.
- **Backspace only deletes one character** — no support for word deletion or selection manipulation. Backspace triggers a single "backspace" keydown event.
- **No paste support** — OSK is character-by-character input only. For clipboard paste, use keyboard shortcuts (Ctrl+V) when OSK is closed.
- **Analog stick navigation is smooth** — left stick provides cursor-style navigation with acceleration. D-pad navigation is discrete (one key per press).
- **Visual feedback** — selected key is highlighted. Pressed keys briefly change color. Shift key shows "active" state when pressed.
