=== OVERVIEW ===
NavPanel renders a vertical sidebar with grouped navigation items. Each section has a title and a list of selectable items. The active item is highlighted with a distinct background and border.

NavPanel supports an optional header slot (any ReactNode) displayed above the sections, and a `contentAlign` prop to center section titles and item labels. It uses `overflow: 'scroll'` internally, so it handles long lists of items gracefully.

Colors are theme-aware via `useThemeColorsOptional()` with dark-theme fallbacks.

=== API / SYNTAX ===
```tsx
import { NavPanel } from '@reactjit/core';
```

Props:
| Prop         | Type                 | Default | Description                                  |
|--------------|----------------------|---------|----------------------------------------------|
| sections     | NavSection[]         | required| Array of navigation sections                 |
| activeId     | string               | undefined | ID of the currently active item            |
| onSelect     | (id: string) => void | undefined | Callback when an item is pressed           |
| header       | ReactNode            | undefined | Content rendered above all sections        |
| width        | number               | 180     | Panel width in pixels                        |
| contentAlign | 'start' \| 'center' | 'start' | Alignment of section titles and item labels  |
| style        | Style                | undefined | Outer container style overrides            |

NavSection:
| Field | Type       | Description              |
|-------|------------|--------------------------|
| title | string     | Section heading text     |
| items | NavItem[]  | Items within the section |

NavItem:
| Field | Type   | Description           |
|-------|--------|-----------------------|
| id    | string | Unique item identifier |
| label | string | Display text          |

=== EXAMPLES ===
Example 1: Basic sidebar
---
```tsx
const [active, setActive] = useState('dashboard');

<NavPanel
  sections={[
    { title: 'Main', items: [
      { id: 'dashboard', label: 'Dashboard' },
      { id: 'analytics', label: 'Analytics' },
    ]},
    { title: 'Settings', items: [
      { id: 'profile', label: 'Profile' },
      { id: 'preferences', label: 'Preferences' },
    ]},
  ]}
  activeId={active}
  onSelect={setActive}
/>
```
---
Platforms: love2d

Example 2: NavPanel with header
---

```tsx
const colors = useThemeColors();
const [active, setActive] = useState('home');

    { title: 'Navigation', items: [
      { id: 'home', label: 'Home' },
      { id: 'explore', label: 'Explore' },
      { id: 'library', label: 'Library' },
    ]},
  ]}
  activeId={active}
  onSelect={setActive}
/>
```

Example 3: Sidebar with content area
---
```tsx
const colors = useThemeColors();
const [active, setActive] = useState('overview');

<FlexRow style={{ width: '100%', height: '100%' }}>
  <NavPanel
    sections={[
      { title: 'Docs', items: [
        { id: 'overview', label: 'Overview' },
        { id: 'api', label: 'API Reference' },
        { id: 'examples', label: 'Examples' },
      ]},
    ]}
    activeId={active}
    onSelect={setActive}
  />
  <Box style={{ flexGrow: 1, padding: 16 }}>
    <Text style={{ color: colors.text, fontSize: 16 }}>{`Selected: ${active}`}</Text>
  </Box>
</FlexRow>
```

Platforms: love2d

=== CRITICAL RULES ===
- NavPanel has a fixed default `width` of 180px. Set `width` explicitly or use `style` to override if your sidebar needs a different size.
- The panel uses `overflow: 'scroll'` internally. In a full-height layout, give the NavPanel or its parent a defined height (e.g., `flexGrow: 1`) so the scroll region has a viewport.
- Section titles are rendered uppercased automatically.
