=== OVERVIEW ===
Radio and RadioGroup provide exclusive single-selection from a set of options. RadioGroup wraps multiple Radio children and manages which option is selected. Each Radio renders as a `Radio` host element -- Lua handles the circle rendering, fill animation, and click detection.

RadioGroup uses React context to share selection state. Radio reads from this context to determine whether it is selected and calls back to the group when clicked.

Both controlled (`value` prop on RadioGroup) and uncontrolled (`defaultValue` on RadioGroup) modes are supported.

=== API / SYNTAX ===
```tsx
import { Radio, RadioGroup } from '@reactjit/core';
```

RadioGroup Props:
| Prop          | Type                        | Default   | Description                            |
|---------------|-----------------------------|-----------|----------------------------------------|
| value         | string                      | undefined | Controlled selected value              |
| defaultValue  | string                      | undefined | Initial value for uncontrolled mode    |
| onValueChange | (value: string) => void     | undefined | Fires when selection changes           |
| disabled      | boolean                     | false     | Disable all children                   |
| style         | Style                       | undefined | Container style (defaults gap: 8)      |
| children      | ReactNode                   | required  | Radio children                         |

Radio Props:
| Prop           | Type   | Default   | Description                           |
|----------------|--------|-----------|---------------------------------------|
| value          | string | required  | Option value (matched against group)  |
| label          | string | undefined | Text label next to the radio circle   |
| disabled       | boolean| false     | Disable this specific option          |
| size           | number | 20        | Radio circle diameter in pixels       |
| color          | Color  | '#3b82f6' | Selected state color                  |
| uncheckedColor | Color  | '#6b7280' | Unselected state border color         |
| style          | Style  | undefined | Container style overrides             |

=== EXAMPLES ===
Example 1: Basic radio group
---
```tsx
const colors = useThemeColors();
const [size, setSize] = useState('medium');

<RadioGroup value={size} onValueChange={setSize}>
  <Radio value="small" label="Small" color={colors.primary} />
  <Radio value="medium" label="Medium" color={colors.primary} />
  <Radio value="large" label="Large" color={colors.primary} />
</RadioGroup>
```
---
Platforms: love2d

Example 2: Uncontrolled with default
---
```tsx
const colors = useThemeColors();

<RadioGroup
  defaultValue="light"
  onValueChange={(theme) => applyTheme(theme)}
>
  <Radio value="light" label="Light theme" color={colors.primary} />
  <Radio value="dark" label="Dark theme" color={colors.primary} />
  <Radio value="system" label="System default" color={colors.primary} />
</RadioGroup>
```
---
Platforms: love2d

Example 3: Disabled options
---
```tsx
const colors = useThemeColors();
const [plan, setPlan] = useState('free');

<RadioGroup value={plan} onValueChange={setPlan}>
  <Radio value="free" label="Free" color={colors.primary} />
  <Radio value="pro" label="Pro" color={colors.primary} />
  <Radio value="enterprise" label="Enterprise" disabled color={colors.muted} />
</RadioGroup>
```
---
Platforms: love2d

=== PLATFORM NOTES ===
Love2D:
  Each Radio emits a `Radio` host element. Lua renders the outer circle, inner fill dot for the selected state, and optional label text. The RadioGroup itself renders as a Box with gap spacing and provides React context -- it does not emit a special host element.

  Disabling a RadioGroup disables all Radio children within it. A Radio can also be individually disabled even if the group is enabled.

=== CRITICAL RULES ===
- Radio must be a child of RadioGroup. Without a RadioGroup ancestor, the Radio has no selection context and clicks do nothing.
- Each Radio's `value` prop must be unique within its RadioGroup. Duplicate values cause undefined selection behavior.
- RadioGroup lays out children in a column with 8px gap by default. Override via the `style` prop.
