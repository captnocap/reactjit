=== OVERVIEW ===
ContextMenu wraps content in a right-click context menu region. When the user right-clicks inside the wrapped area, Lua renders a popup menu with the provided items. All menu rendering, hover tracking, and keyboard navigation happen in Lua. React only receives boundary events: open, close, and item selection.

This is a "Lua-owned interaction" primitive. The JS side declares items and receives selection callbacks. Lua handles everything visual.

ContextMenu provides built-in items for text selection (Copy, Select All) automatically when the right-clicked node contains text. Custom items are appended to these built-in options.

=== API / SYNTAX ===
```tsx
import { ContextMenu } from '@reactjit/core';
```

Props:
| Prop     | Type                                  | Default   | Description                           |
|----------|---------------------------------------|-----------|---------------------------------------|
| items    | ContextMenuItem[]                     | undefined | Custom menu items                     |
| onSelect | (event: ContextMenuEvent) => void     | undefined | Fires when any item is selected       |
| onOpen   | () => void                            | undefined | Fires when the menu opens             |
| onClose  | () => void                            | undefined | Fires when the menu closes            |
| children | ReactNode                             | undefined | Content that responds to right-click  |

ContextMenuItem:
| Field     | Type    | Default   | Description                              |
|-----------|---------|-----------|------------------------------------------|
| label     | string  | required  | Display label                            |
| action    | string  | required  | Action identifier dispatched on selection|
| disabled  | boolean | false     | Gray out and disable the item            |
| separator | boolean | false     | Render a divider line instead of an item |

ContextMenuEvent:
| Field        | Type    | Description                                |
|--------------|---------|--------------------------------------------|
| action       | string  | The action string of the selected item     |
| targetId     | number  | ID of the node that was right-clicked      |
| hasSelection | boolean | Whether text was selected when menu opened |
| selectedText | string  | The selected text content (if any)         |

=== EXAMPLES ===
Example 1: Basic context menu
---
```tsx
const colors = useThemeColors();

<ContextMenu
  items={[
    { label: 'Edit', action: 'edit' },
    { label: 'Delete', action: 'delete' },
  ]}
  onSelect={(e) => {
    if (e.action === 'edit') editItem();
    if (e.action === 'delete') deleteItem();
  }}
>
  <Box padding={16} bg={colors.surface} radius={8}>
    <Text style={{ color: colors.text }}>Right-click me</Text>
  </Box>
</ContextMenu>
```
---
Platforms: love2d

Example 2: Menu with separators and disabled items
---
```tsx
const colors = useThemeColors();

<ContextMenu
  items={[
    { label: 'Cut', action: 'cut' },
    { label: 'Copy', action: 'copy' },
    { label: 'Paste', action: 'paste', disabled: !hasClipboard },
    { label: '', action: '', separator: true },
    { label: 'Select All', action: 'select-all' },
  ]}
  onSelect={(e) => handleAction(e.action)}
>
  <Box padding={16} bg={colors.surface} radius={8}>
    <Text style={{ color: colors.text }}>Document content</Text>
  </Box>
</ContextMenu>
```
---
Platforms: love2d

Example 3: Responding to open/close events
---
```tsx
const colors = useThemeColors();
const [menuOpen, setMenuOpen] = useState(false);

<ContextMenu
  items={[{ label: 'Inspect', action: 'inspect' }]}
  onSelect={(e) => handleAction(e.action)}
  onOpen={() => setMenuOpen(true)}
  onClose={() => setMenuOpen(false)}
>
  <Box
    padding={16}
    bg={menuOpen ? colors.bgElevated : colors.surface}
    radius={8}
  >
    <Text style={{ color: colors.text }}>
      {menuOpen ? 'Menu is open' : 'Right-click for options'}
    </Text>
  </Box>
</ContextMenu>
```
---
Platforms: love2d

=== PLATFORM NOTES ===
Love2D:
  The popup menu is rendered by Lua as an overlay on top of the scene. It supports keyboard navigation (arrow keys, Enter to select, Escape to dismiss) and mouse hover highlighting. The menu auto-positions to stay within the window bounds.

  Built-in items (Copy, Select All) are added automatically by Lua when applicable. Your custom items array is appended after the built-in items.

=== CRITICAL RULES ===
- ContextMenu wraps children -- it does not render a visible element itself. The children define the right-click target area.
- The `action` field on ContextMenuItem must be unique within the items array. This is how you identify which item was selected in onSelect.
- Separator items still require label and action fields, but they are ignored. Use empty strings.
