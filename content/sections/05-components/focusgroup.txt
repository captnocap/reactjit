=== OVERVIEW ===
FocusGroup creates a scoped focus region for controller and keyboard navigation. It restricts Tab key and gamepad D-pad navigation to elements within the group, enabling isolated UI sections and multiplayer support where each controller navigates its own region. FocusGroups support automatic player-based ring colors (P1 blue, P2 red, P3 green, P4 yellow) and optional controller binding for split-screen or couch multiplayer scenarios.

When a FocusGroup is bound to a specific controller via the controller prop, only that controller can navigate elements within the group. If no controller is specified, the group accepts input from any controller. FocusGroups can be nested, and focus is automatically restored when switching between mouse and controller input modes. The framework tracks the last focused element per group and automatically scrolls focused elements into view within ScrollView parents.

=== API / SYNTAX ===
```tsx
import { FocusGroup } from '@reactjit/core';
```

```tsx
<FocusGroup
  controller={0}
  ringColor={[0.3, 0.6, 1.0, 0.9]}
  style={{ padding: 16, backgroundColor: '#1e1e2e' }}
>
  <Pressable onPress={() => {}}>
    <Text fontSize={14}>Option 1</Text>
  </Pressable>
  <Pressable onPress={() => {}}>
    <Text fontSize={14}>Option 2</Text>
  </Pressable>
</FocusGroup>
```

Props:
| Prop       | Type                           | Default   | Description                                       |
|------------|--------------------------------|-----------|---------------------------------------------------|
| controller | number                         | undefined | Joystick ID (0-based). Omit for any controller.  |
| ringColor  | [number, number, number, number?] | undefined | Focus ring color as [r, g, b, a] (0-1 range). Default: player-based (P1 blue, P2 red, P3 green, P4 yellow). |
| style      | Style                          | undefined | Container style (wraps Box internally).           |
| children   | ReactNode                      | REQUIRED  | Focusable elements (Pressable, TextInput, etc.).  |

RingColor format:
```ts
type RingColor = [number, number, number, number?];
// [r, g, b, a] where each is 0-1
// Example: [0.3, 0.6, 1.0, 0.9] = semi-transparent blue
```

=== PLATFORM NOTES ===
Love2D:
  FocusGroup is implemented in lua/focus.lua. It registers as a scoped navigation region on every layout pass. Spatial navigation (D-pad / left stick) uses center-point distance with directional filtering — the closest element in the pressed direction becomes focused. Focus state is preserved when switching between mouse and controller modes. Focused elements automatically scroll into view within ScrollView parents. The focus ring animates with exponential smoothing for visual polish.

  Controller binding: joystick IDs are 0-based. Controller 0 is the first connected gamepad, Controller 1 is the second, etc. If a group has controller={0}, only joystick 0 can navigate it. If controller is undefined, any joystick can navigate the group. Unbound groups (no controller prop) default to accepting input from any controller that is not bound to a specific group.

  Ring colors: If ringColor is not set and the group is bound to a controller, the framework auto-assigns a player color (P1 blue, P2 red, P3 green, P4 yellow) based on joystick ID modulo 4. If ringColor is set explicitly, that color is used regardless of controller. If ringColor is undefined and controller is undefined, the default blue ring is used.

Web:
  FocusGroup renders as a <div> with tabIndex management. Tab key navigation is scoped using keyboard event handlers that trap focus within the group. Gamepad navigation is not implemented on web (browser gamepad API support is limited). Ring colors are rendered as CSS outline or box-shadow. Focus state is managed via React state and DOM focus events.

Terminal:
  Not available on grid targets. Use terminal-specific focus management.

=== CRITICAL RULES ===
- FocusGroup only navigates elements with focusable={true} or interactive handlers (onClick, onPress, onKeyDown, etc.). Non-interactive elements are skipped.
- Tab key and D-pad navigation wrap within the group. Pressing right on the last element does NOT jump to the next group — it wraps to the first element.
- Controller-bound groups (controller={0}) ignore input from other controllers. If controller 1 presses D-pad and is not bound to any group, it navigates the default group (elements outside all FocusGroups) and any unclaimed FocusGroups.
- Nested FocusGroups create independent navigation scopes. A child group's elements are not navigable from the parent group. Each group maintains its own focus state.
- If a focused element is removed from the tree (unmounted), focus resets to the first focusable element in the group. If the entire group is unmounted, focus is lost.
- Focus restoration: when switching from mouse to controller mode, the framework restores the last focused element per group. If no element was previously focused, it focuses the first focusable element.
- Spatial navigation (D-pad) uses center-point distance with directional filtering. The algorithm prefers elements in the pressed direction and penalizes perpendicular distance. This prevents diagonal jumps when pressing left/right in a vertical list.
