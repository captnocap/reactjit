=== OVERVIEW ===
Portal renders children at the top level of the render tree, escaping the normal parent-child hierarchy. This is essential for UI elements that need to appear above all other content, such as modals, tooltips, dropdowns, and context menus. PortalHost is the container that receives portal content in native mode. In web mode, Portal falls back to ReactDOM.createPortal when no PortalHost is present.

=== API / SYNTAX ===
```tsx
import { Portal, PortalHost } from '@reactjit/core';
```

**Portal Props**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| children | ReactNode | required | Content to render at the top level |
| name | string | undefined | Optional identifier for the portal (auto-generated if omitted) |

**PortalHost Props**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| children | ReactNode | required | The main app content (renders below portals) |

=== EXAMPLES ===

Example 1: Basic Portal Usage
---
```tsx
import { Portal, Box, Text } from '@reactjit/core';

function App() {
  const [showOverlay, setShowOverlay] = useState(false);

  return (
    <Box style={{ width: '100%', height: '100%' }}>
      <Text fontSize={16}>Main content</Text>
      <Pressable onPress={() => setShowOverlay(true)}>
        <Text fontSize={14}>Show Overlay</Text>
      </Pressable>

      {showOverlay && (
        <Portal>
          <Box
            style={{
              position: 'absolute',
              top: 0,
              left: 0,
              width: '100%',
              height: '100%',
              backgroundColor: [0, 0, 0, 0.7],
              justifyContent: 'center',
              alignItems: 'center',
            }}
          >
            <Box
              style={{
                width: 400,
                height: 200,
                backgroundColor: [1, 1, 1, 1],
                borderRadius: 8,
                padding: 20,
              }}
            >
              <Text fontSize={18}>Portal Overlay</Text>
              <Pressable onPress={() => setShowOverlay(false)}>
                <Text fontSize={14}>Close</Text>
              </Pressable>
            </Box>
          </Box>
        </Portal>
      )}
    </Box>
  );
}
```

Example 2: Setting Up PortalHost in Native Mode
---
```tsx
import { PortalHost, Box, Text } from '@reactjit/core';
import { createRoot } from '@reactjit/native';
import { NativeBridge } from '@reactjit/native';
import { BridgeProvider, RendererProvider } from '@reactjit/core';

function App() {
  return (
    <Box style={{ width: '100%', height: '100%' }}>
      <Text fontSize={16}>App content here</Text>
      {/* Portals will render above this content */}
    </Box>
  );
}

// Entry point
const bridge = new NativeBridge();
const root = createRoot(bridge);

root.render(
  <BridgeProvider bridge={bridge}>
    <RendererProvider mode="native">
      <PortalHost>
        <App />
      </PortalHost>
    </RendererProvider>
  </BridgeProvider>
);
```

Example 4: Portal with Modal Component
---
```tsx
import { Modal, Box, Text, Pressable } from '@reactjit/core';

function ModalDemo() {
  const [visible, setVisible] = useState(false);

  return (
    <Box style={{ width: '100%', height: '100%' }}>
      <Pressable onPress={() => setVisible(true)}>
        <Text fontSize={16}>Open Modal</Text>
      </Pressable>

      {/* Modal uses Portal internally */}
      <Modal
        visible={visible}
        onRequestClose={() => setVisible(false)}
        animationType="fade"
        backdropDismiss={true}
      >
        <Box
          style={{
            width: 500,
            height: 300,
            backgroundColor: [1, 1, 1, 1],
            borderRadius: 8,
            padding: 20,
          }}
        >
          <Text fontSize={18}>Modal Content</Text>
        </Box>
      </Modal>
    </Box>
  );
}
```

=== PLATFORM NOTES ===

Love2D:
  - Portals render with zIndex: 1000 in absolute positioning
  - PortalHost must wrap your root component in native-main.tsx
  - Multiple portals stack in registration order (later portals appear on top)
  - Portal content is positioned absolutely relative to the viewport

Web:
  - Falls back to ReactDOM.createPortal when no PortalHost is present
  - Creates a fixed-position div appended to document.body
  - PortalHost is optional in web mode but recommended for consistency
  - Supports CSS transitions and transforms on portal content

Terminal / Grid targets:
  - Portals render in absolute coordinates over the character grid
  - Use explicit positioning (top, left) to place portal content
  - PortalHost required in all grid-based targets
  - Z-ordering follows registration order

=== CRITICAL RULES ===

- In native mode (Love2D, terminal, grid targets), you MUST wrap your root component in PortalHost for portals to work
- Portal children should use absolute positioning (position: 'absolute') with explicit coordinates
- Portal content renders at zIndex: 1000 by default — override via style if needed
- Multiple portals stack in registration order (first registered = bottom, last = top)
- The name prop is optional but recommended for debugging and stable portal identity
- Portal content is NOT clipped by parent containers — it always renders at the top level
- In web mode without PortalHost, Portal automatically falls back to ReactDOM.createPortal
- Do not nest PortalHost components — only one PortalHost per app root
- Portal children receive the same dimensions as the viewport (width: '100%', height: '100%')
- Use Portal for overlays, modals, dropdowns, tooltips, context menus, and notifications
- Portal does NOT handle backdrop clicks or Escape key dismissal — use Modal component for that
- Portals persist until unmounted — conditional rendering ({showPortal && <Portal>...</Portal>}) controls visibility
