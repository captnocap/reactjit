=== OVERVIEW ===
ReactJIT provides a composable search system split into visual components and data hooks. The components handle presentation; the hooks handle filtering and scoring.

SearchBar is a text input with a magnifier icon, clear button, and Lua-side debouncing. It fires onSearch after the user pauses typing, avoiding per-keystroke bridge traffic.

SearchResults renders a flat scrollable list of result items with active-index highlighting, icon slots, and meta labels. Controlled externally via activeIndex for keyboard navigation.

SearchCombo combines SearchBar and SearchResults into a single dropdown combo. Provide static items with a searchKey for client-side filtering, or an async onSearch function for server-side search.

CommandPalette is a full-screen modal command launcher. Commands are fuzzy-matched by label, group, and keywords. Supports keyboard navigation (arrows, Enter, Escape), grouped display, and shortcut badges.

AppSearch searches the live rendered node tree via Lua RPC. Hot results are nodes currently in the tree; cold results come from a compile-time manifest for pre-mount content. Selecting a result scrolls to and highlights the matching node.

Searchable is a zero-overhead wrapper that marks a subtree for compile-time text indexing by `rjit search-index`.

Data hooks: useSearch (synchronous in-memory), useFuzzySearch (scored fuzzy matching), useAsyncSearch (async with loading/error), useCommandSearch (fuzzy-ranked commands), useSearchHistory (SQLite-persisted), useSearchHighlight (split text into match/non-match parts), useSearchSchema (inspect what fields are being searched), useAppSearch (live tree RPC search).

=== API / SYNTAX ===
```tsx
import { SearchBar, SearchResults, SearchCombo, CommandPalette, AppSearch, Searchable } from '@reactjit/core';
import { useSearch, useFuzzySearch, useAsyncSearch, useCommandSearch, useSearchHistory, useSearchHighlight, useSearchSchema, useAppSearch } from '@reactjit/core';
```

useFuzzySearch(items, query, options):
Returns `{ results: FuzzySearchResult<T>[], items: T[] }` where each result has `item`, `score`, and `index`.

useAsyncSearch(fetcher, query, options):
Returns `{ results: T[], loading: boolean, error: Error | null }`.

useSearchHighlight(text, query):
Returns `HighlightPart[]` where each part has `text: string` and `match: boolean`.

=== EXAMPLES ===
Example 1: SearchBar with results
---
```tsx
const [query, setQuery] = useState('');
const results = useSearch(users, query, { key: 'name' });

<SearchBar onSearch={setQuery} placeholder="Find users..." />
<SearchResults
  items={results.map(u => ({ id: u.id, label: u.name, description: u.email }))}
  onSelect={(item) => navigate(item.id)}
/>
```

=== PLATFORM NOTES ===
Love2D:
  SearchBar uses Lua-side debouncing via TextInput's liveChangeDebounce prop, avoiding per-keystroke bridge traffic. CommandPalette renders inside a Modal. AppSearch queries the live node tree via bridge RPC (search:query, search:navigate, search:clear). Hot search results contain structural paths that Lua resolves directly to live nodes.

=== CRITICAL RULES ===
- useSearch auto-detects searchable fields when no `key` is provided. It inspects the first item and picks string fields under 200 characters, skipping ID-like keys (id, key, uuid, url, src, etc.).
- SearchCombo requires either `items` + `searchKey` for client-side search, OR `onSearch` for async search. Do not provide both.
- CommandPalette fuzzy-matches against label, group, and keywords. The `shortcut` prop is display-only — it does not register actual hotkeys.
- AppSearch hot results resolve directly from the live node tree. They become stale if the tree re-renders between query and navigation. Cold results use text matching as a fallback.
