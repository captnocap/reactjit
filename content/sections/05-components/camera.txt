=== OVERVIEW ===
Camera sets the perspective viewpoint used when rendering a Scene. Place exactly one Camera as a direct child of Scene. All props are reactive — changing position, lookAt, or fov causes the camera to update each frame without full reconciliation overhead. If no Camera is present in a Scene, a default camera at position [0, -3, 2] looking at the origin is used automatically.

=== API / SYNTAX ===
```tsx
import { Camera } from '@reactjit/3d';
```

Props:
| Prop | Type | Default | Description |
|------|------|---------|-------------|
| position | [number, number, number] | [0, -3, 2] | Camera position in world space as [x, y, z] |
| lookAt | [number, number, number] | [0, 0, 0] | The point in world space the camera looks at as [x, y, z] |
| fov | number | Math.PI / 3 | Field of view in radians (Math.PI / 3 = 60 degrees) |
| near | number | 0.01 | Near clipping plane distance — geometry closer than this is not rendered |
| far | number | 1000 | Far clipping plane distance — geometry beyond this is not rendered |

=== EXAMPLES ===
Example 1: Basic camera looking at the origin
---
```tsx
import { Scene, Camera, Mesh } from '@reactjit/3d';

function App() {
  return (
    <Scene style={{ width: 800, height: 600 }}>
      <Camera position={[0, -4, 2]} lookAt={[0, 0, 0]} />
      <Mesh geometry="box" color="#89b4fa" />
    </Scene>
  );
}
```
---
Platforms: love2d

Example 2: Animated camera orbiting the scene
---
```tsx
import { Scene, Camera, Mesh, AmbientLight, DirectionalLight } from '@reactjit/3d';
import { useAnimationFrame } from '@reactjit/core';

function OrbitingCamera() {
  const [angle, setAngle] = React.useState(0);

  useAnimationFrame((dt) => {
    setAngle((a) => a + dt * 0.5);
  });

  const radius = 5;
  const camX = Math.sin(angle) * radius;
  const camY = -Math.cos(angle) * radius;

  return (
    <Scene style={{ width: 800, height: 600 }}>
      <Camera position={[camX, camY, 2]} lookAt={[0, 0, 0]} fov={Math.PI / 4} />
      <AmbientLight color="#1a1a2e" intensity={0.2} />
      <DirectionalLight direction={[1, -1, 1]} color="#ffffff" intensity={1.0} />
      <Mesh geometry="sphere" color="#cba6f7" />
    </Scene>
  );
}
```

Platforms: love2d

Example 3: Narrow field of view for a telephoto effect
---
```tsx
import { Scene, Camera, Mesh } from '@reactjit/3d';

function TelephotoView() {
  return (
    <Scene style={{ width: 800, height: 600 }}>
      <Camera
        position={[0, -10, 4]}
        lookAt={[0, 0, 0]}
        fov={Math.PI / 8}
        near={0.1}
        far={100}
      />
      <Mesh geometry="box" color="#a6e3a1" scale={1.5} />
    </Scene>
  );
}
```

=== CRITICAL RULES ===
- Place exactly one Camera per Scene — multiple Camera children in the same Scene produce undefined behavior
- Camera renders nothing on web targets and returns null; it is Love2D only
- The position and lookAt props use a right-handed coordinate system where Z is up
- fov is in radians, not degrees — use Math.PI / 3 for 60 degrees, Math.PI / 4 for 45 degrees
- Omitting Camera entirely is valid; the default camera at [0, -3, 2] will be used

=== SEE ALSO ===
- Scene — the 3D viewport that Camera must be placed inside
- Mesh — renderable geometry that Camera looks at
- DirectionalLight — sun-like light that interacts with mesh shading
- AmbientLight — base illumination applied across all meshes
