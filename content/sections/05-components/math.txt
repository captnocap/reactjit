=== OVERVIEW ===
Math renders LaTeX math expressions as pixel-perfect typeset output. It emits a `Math` host element that Lua parses, lays out, and renders directly using Love2D's font system. No external typesetting engine is required -- the LaTeX parser and layout engine run entirely in Lua.

Supports standard LaTeX math notation: fractions, superscripts, subscripts, square roots, Greek letters, operators, and more. Both inline and display (block) modes are available.

The component is named `Math` in JSX but is internally aliased from `MathBlock` to avoid shadowing JavaScript's built-in `Math` object.

=== API / SYNTAX ===
```tsx
import { Math } from '@reactjit/core';
```

Props:
| Prop     | Type    | Default | Description                              |
|----------|---------|---------|------------------------------------------|
| tex      | string  | required| LaTeX math expression                    |
| inline   | boolean | false   | Render inline (compact) vs display mode  |
| fontSize | number  | 16      | Base font size in pixels                 |
| color    | Color   | undefined | Text color for the expression           |
| style    | Style   | undefined | Container style (padding defaults to 4) |

=== EXAMPLES ===
Example 1: Simple equation
---
```tsx
const colors = useThemeColors();

<Math
  tex="E = mc^2"
  color={colors.text}
  style={{ padding: 12 }}
/>
```
---
Platforms: love2d

Example 2: Quadratic formula
---
```tsx
const colors = useThemeColors();

<Math
  tex="\frac{-b \pm \sqrt{b^2-4ac}}{2a}"
  fontSize={20}
  color={colors.text}
  style={{ padding: 16, backgroundColor: colors.surface, borderRadius: 8 }}
/>
```
---
Platforms: love2d

Example 3: Inline math in a text block

```tsx
const colors = useThemeColors();
```

=== PLATFORM NOTES ===
Love2D:
  The LaTeX parser and glyph renderer run in Lua. Parsing happens once when the `tex` prop changes, and the layout result is cached. Complex expressions with deeply nested fractions or large matrices may take a frame or two to parse on first render.

  The renderer uses Love2D's built-in font system for glyphs. Symbol coverage depends on the loaded font. The default font (DejaVu Sans) covers Greek letters, mathematical operators, and common symbols.

=== CRITICAL RULES ===
- The `tex` prop is required. Omitting it renders nothing.
- Backslashes in LaTeX must be escaped in JSX strings: use `"\\frac{a}{b}"` or template literals with raw strings.
- For inline use alongside Text, set `inline={true}` and match the `fontSize` to the surrounding text.
