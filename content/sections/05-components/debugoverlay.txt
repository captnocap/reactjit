=== OVERVIEW ===
DebugOverlay is a React context provider that wraps any subtree to visualize layout boundaries without opening the full inspector. It adds colored borders to every Box within its subtree and optionally displays dimension labels on each element. The overlay can be toggled on/off at runtime and customized per-subtree. Use DebugBox as a drop-in replacement for Box to respect the debug context automatically.

DebugOverlay is particularly useful when tracking down layout bugs, verifying flex container alignment, or understanding how nested boxes are sizing themselves. Unlike the full inspector (F12 in Love2D), DebugOverlay can be embedded directly in your component tree and toggled programmatically.

=== API / SYNTAX ===
```tsx
import { DebugOverlay, DebugBox, useDebugOverlay } from '@reactjit/core';
```

```tsx
<DebugOverlay enabled={true} borderColor="rgba(255, 100, 100, 0.6)" showLabels={false}>
  <DebugBox style={{ width: '100%', height: '100%' }}>
    {children}
  </DebugBox>
</DebugOverlay>
```

DebugOverlay Props:
| Prop        | Type             | Default                      | Description                               |
|-------------|------------------|------------------------------|-------------------------------------------|
| children    | ReactNode        | (required)                   | Subtree to wrap with debug context        |
| enabled     | boolean          | true                         | Toggle overlay on/off                     |
| borderColor | string           | 'rgba(255, 100, 100, 0.6)'   | Border color for debug outlines           |
| showLabels  | boolean          | false                        | Show dimension labels on each Box         |

DebugBox Props:
| Prop     | Type      | Default   | Description                                |
|----------|-----------|-----------|-------------------------------------------|
| children | ReactNode | undefined | Child elements                            |
| style    | Style     | undefined | Box style (border added when debug enabled)|
| ...props | any       | undefined | All other Box props (onClick, etc.)       |

useDebugOverlay Hook:
Returns `{ enabled: boolean, borderColor: string, showLabels: boolean }` from the nearest DebugOverlay context.

=== EXAMPLES ===
Example 1: Basic overlay for a section
---
```tsx
import { DebugOverlay, DebugBox, Text } from '@reactjit/core';

function MyComponent() {
  return (
    <DebugOverlay>
      <DebugBox style={{ width: '100%', height: '100%', flexDirection: 'column', gap: 16 }}>
        <DebugBox style={{ flexGrow: 1, backgroundColor: '#1e1e2e' }}>
          <Text style={{ fontSize: 16, color: '#cdd6f4' }}>Section 1</Text>
        </DebugBox>
        <DebugBox style={{ flexGrow: 2, backgroundColor: '#313244' }}>
          <Text style={{ fontSize: 16, color: '#cdd6f4' }}>Section 2</Text>
        </DebugBox>
      </DebugBox>
    </DebugOverlay>
  );
}
```

Example 2: Toggle overlay with a button
---
```tsx
import { useState } from 'react';
import { DebugOverlay, DebugBox, Box, Text, Pressable } from '@reactjit/core';

function LayoutDebugger() {
  const [debugEnabled, setDebugEnabled] = useState(false);

  return (
    <Box style={{ width: '100%', height: '100%', flexDirection: 'column' }}>
      <Pressable onPress={() => setDebugEnabled(!debugEnabled)}>
        <Box style={{ padding: 12, backgroundColor: '#313244' }}>
          <Text style={{ fontSize: 14, color: '#cdd6f4' }}>
            {debugEnabled ? 'Hide' : 'Show'} Debug Overlay
          </Text>
        </Box>
      </Pressable>

      <DebugOverlay enabled={debugEnabled}>
        <DebugBox style={{ flexGrow: 1, padding: 16 }}>
          <DebugBox style={{ width: '100%', flexDirection: 'row', gap: 12 }}>
            <DebugBox style={{ flexGrow: 1, height: 100, backgroundColor: '#45475a' }} />
            <DebugBox style={{ flexGrow: 1, height: 100, backgroundColor: '#585b70' }} />
          </DebugBox>
        </DebugBox>
      </DebugOverlay>
    </Box>
  );
}
```

Example 4: Using the hook to read debug state
---
```tsx
import { DebugOverlay, useDebugOverlay, Box, Text } from '@reactjit/core';

function DebugAwareComponent() {
  const { enabled, borderColor } = useDebugOverlay();

  return (
    <Box style={{ padding: 16 }}>
      <Text style={{ fontSize: 14, color: enabled ? '#f38ba8' : '#cdd6f4' }}>
        Debug mode: {enabled ? 'ON' : 'OFF'}
      </Text>
      {enabled && (
        <Text style={{ fontSize: 12, color: '#a6adc8' }}>
          Border color: {borderColor}
        </Text>
      )}
    </Box>
  );
}

function App() {
  return (
    <DebugOverlay enabled={true}>
      <DebugAwareComponent />
    </DebugOverlay>
  );
}
```

=== PLATFORM NOTES ===
- **Love2D**: DebugOverlay works alongside the built-in inspector (F12). Use DebugOverlay for focused subtree inspection, use the inspector for full tree navigation and interaction state.
- **Web**: Border colors are applied via CSS. Dimension labels (when showLabels is true) render as absolutely positioned Text elements overlaying each DebugBox.
- **Grid targets**: Not currently supported. DebugOverlay and DebugBox are only available in `@reactjit/core` (Love2D and web targets).

=== CRITICAL RULES ===
- **DebugBox is required** — Regular `<Box>` elements inside a DebugOverlay will NOT show debug borders. You must use `<DebugBox>` for context awareness.
- **Context nesting** — If you nest multiple DebugOverlay providers, the innermost context wins. Each subtree can have its own border color and enabled state.
- **Performance** — DebugOverlay adds a style merge operation to every DebugBox render. For large subtrees (100+ boxes), consider wrapping only the problematic section instead of the entire tree.
- **Remove before production** — DebugOverlay is a development tool. Disable it or remove it entirely before building production distributions.
- **showLabels not implemented** — The `showLabels` prop is accepted but dimension label rendering is not yet implemented. Currently it only affects the context value returned by useDebugOverlay.
