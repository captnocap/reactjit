=== OVERVIEW ===
Pressable wraps content in an interactive container that tracks press, hover, and focus states. It fires callbacks for press, long press, hover in/out, and press in/out events.

Pressable pre-computes style variants and passes them to Box as `hoverStyle`/`activeStyle`, so Lua handles visual feedback at zero latency — no bridge round-trip needed for hover/active effects. React state is only used when `children` is a render-prop function that needs the interaction state.

The `style` prop can be either a static Style object or a function that receives `{ pressed, hovered, focused }` and returns a Style. When a function is provided, Pressable computes the diff between base, hovered, and pressed variants and sends them as separate overlay styles to Lua.

=== API / SYNTAX ===
```tsx
import { Pressable } from '@reactjit/core';
```

Props:
| Prop           | Type                                            | Default   | Description                                      |
|----------------|-------------------------------------------------|-----------|--------------------------------------------------|
| onPress        | (event: LoveEvent) => void                      | undefined | Fires on completed press (down + up)             |
| onLongPress    | (event: LoveEvent) => void                      | undefined | Fires after holding for delayLongPress ms        |
| onPressIn      | (event: LoveEvent) => void                      | undefined | Fires on press down                              |
| onPressOut     | (event: LoveEvent) => void                      | undefined | Fires on press release                           |
| onHoverIn      | (event: LoveEvent) => void                      | undefined | Fires when pointer enters                        |
| onHoverOut     | (event: LoveEvent) => void                      | undefined | Fires when pointer leaves                        |
| disabled       | boolean                                         | false     | Disables all interaction                         |
| delayLongPress | number                                          | 500       | Milliseconds before onLongPress fires            |
| hitSlop        | number \| HitSlop                               | undefined | Extend touch target beyond visible bounds        |
| style          | Style \| ((state: PressableState) => Style)     | undefined | Static or state-dependent style                  |
| children       | ReactNode \| ((state: PressableState) => ReactNode) | required | Static or state-dependent children          |

PressableState:
| Field   | Type    | Description                  |
|---------|---------|------------------------------|
| pressed | boolean | Currently being pressed      |
| hovered | boolean | Pointer is over the element  |
| focused | boolean | Element has keyboard focus   |

HitSlop:
| Field  | Type   | Description                    |
|--------|--------|--------------------------------|
| top    | number | Extra tap area above element   |
| bottom | number | Extra tap area below element   |
| left   | number | Extra tap area left of element |
| right  | number | Extra tap area right of element|

=== EXAMPLES ===
Example 1: Basic button
---
```tsx
const colors = useThemeColors();

<Pressable
  onPress={() => console.log('pressed')}
  style={{ padding: 12, backgroundColor: colors.primary, borderRadius: 8 }}
>
  <Text style={{ color: '#ffffff', textAlign: 'center' }}>Submit</Text>
</Pressable>
```
---
Platforms: love2d

Example 2: Style function for state-dependent appearance
---
```tsx
const colors = useThemeColors();

<Pressable
  onPress={() => doAction()}
  style={({ pressed, hovered }) => ({
    padding: 12,
    backgroundColor: pressed ? colors.primary : hovered ? colors.bgElevated : colors.surface,
    borderRadius: 8,
    opacity: pressed ? 0.8 : 1,
  })}
>
  <Text style={{ color: colors.text, textAlign: 'center' }}>Interactive</Text>
</Pressable>
```
---
Platforms: love2d

Example 3: Long press
---
```tsx
const colors = useThemeColors();

<Pressable
  onPress={() => selectItem()}
  onLongPress={() => showContextMenu()}
  delayLongPress={600}
  style={{ padding: 16, backgroundColor: colors.surface }}
>
  <Text style={{ color: colors.text }}>Tap or hold</Text>
</Pressable>
```
---
Platforms: love2d

Example 4: Render prop children
---
```tsx
const colors = useThemeColors();

<Pressable
  onPress={() => toggle()}
  style={{ padding: 12, backgroundColor: colors.surface, borderRadius: 8 }}
>
  {({ pressed, hovered }) => (
    <Text style={{ color: hovered ? colors.primary : colors.text }}>
      {pressed ? 'Pressing...' : 'Press me'}
    </Text>
  )}
</Pressable>
```
---
Platforms: love2d

Example 5: Disabled state
---
```tsx
const colors = useThemeColors();

<Pressable
  onPress={() => submit()}
  disabled={!isValid}
  style={{ padding: 12, backgroundColor: isValid ? colors.primary : colors.muted, borderRadius: 8 }}
>
  <Text style={{ color: '#ffffff', textAlign: 'center' }}>
    {isValid ? 'Submit' : 'Fill required fields'}
  </Text>
</Pressable>
```
---
Platforms: love2d

=== PLATFORM NOTES ===
Love2D:
  Hover and active styles are applied by Lua at paint time, not by React re-renders. This means hover effects are immediate (same frame) with no bridge latency. The render-prop form (`children` as function) does cause React re-renders when state changes, but the common case (static children) is purely Lua-driven.
