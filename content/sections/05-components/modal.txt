=== OVERVIEW ===
Modal renders a full-screen overlay above all other content. It uses `position: absolute` and a high `zIndex` (1000) to layer above everything in the tree — no PortalHost required.

The backdrop is a semi-transparent layer that covers the entire viewport. Clicking the backdrop dismisses the modal (configurable via `backdropDismiss`). Pressing Escape also triggers `onRequestClose`. Content is centered within the viewport by default.

When `visible` is false, Modal renders nothing.

=== API / SYNTAX ===
```tsx
import { Modal } from '@reactjit/core';
```

Props:
| Prop            | Type       | Default          | Description                                     |
|-----------------|------------|------------------|-------------------------------------------------|
| visible         | boolean    | required         | Whether the modal is shown                      |
| onRequestClose  | () => void | undefined        | Called on backdrop tap or Escape key             |
| onShow          | () => void | undefined        | Called when modal becomes visible                |
| transparent     | boolean    | false            | Transparent backdrop (no dimming)                |
| backdropColor   | Color      | [0, 0, 0, 0.5]  | Backdrop color with alpha                       |
| backdropDismiss | boolean    | true             | Whether tapping backdrop calls onRequestClose   |
| style           | Style      | undefined        | Applied to the content wrapper                  |
| children        | ReactNode  | required         | Modal content                                   |

=== EXAMPLES ===
Example 1: Basic modal
---
```tsx
const colors = useThemeColors();
const [visible, setVisible] = useState(false);

<Pressable onPress={() => setVisible(true)} style={{ padding: 12, backgroundColor: colors.primary, borderRadius: 8 }}>
  <Text style={{ color: '#ffffff' }}>Open Modal</Text>
</Pressable>

<Modal visible={visible} onRequestClose={() => setVisible(false)}>
  <Box padding={24} bg={colors.bgElevated} radius={12} style={{ width: 300 }}>
    <Text size={18} color={colors.text} bold>Dialog Title</Text>
    <Text size={14} color={colors.muted} style={{ marginTop: 8 }}>
      This is the modal content.
    </Text>
    <Pressable
      onPress={() => setVisible(false)}
      style={{ marginTop: 16, padding: 10, backgroundColor: colors.primary, borderRadius: 8 }}
    >
      <Text style={{ color: '#ffffff', textAlign: 'center' }}>Close</Text>
    </Pressable>
  </Box>
</Modal>
```
---
Platforms: love2d

Example 2: Non-dismissible modal
---
```tsx
const colors = useThemeColors();

<Modal visible={loading} backdropDismiss={false}>
  <Box padding={24} bg={colors.bgElevated} radius={12} align="center">
    <Text size={16} color={colors.text}>Loading...</Text>
  </Box>
</Modal>
```
---
Platforms: love2d

Example 3: Transparent backdrop
---
```tsx
const colors = useThemeColors();

<Modal visible={showTooltip} onRequestClose={() => setShowTooltip(false)} transparent>
  <Box padding={16} bg={colors.surface} radius={8} style={{ borderWidth: 1, borderColor: colors.border }}>
    <Text size={14} color={colors.text}>Floating content</Text>
  </Box>
</Modal>
```
---
Platforms: love2d

=== PLATFORM NOTES ===
Love2D:
  Modal uses position:absolute to layer above the rest of the tree. The layout engine handles absolute children out-of-flow and the painter sorts by zIndex. Escape key detection uses Love2D key names (lowercase 'escape').
