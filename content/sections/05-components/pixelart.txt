=== OVERVIEW ===
PixelArt renders small bitmap icons as colored Box grids. It solves the problem of Unicode symbols not rendering reliably across all Love2D font configurations — instead of relying on font glyphs, PixelArt draws each pixel as a tiny colored Box.

There are two ways to use it: the PixelArt component (declarative) and the usePixelArt hook (returns a memoized React element). Both accept either a built-in symbol name or an array of custom string art lines where any non-space character represents a filled pixel.

Built-in symbols: play, pause, stop, skip-next, skip-prev, check, close, arrow-up, arrow-down, arrow-left, arrow-right, circle, circle-outline, dot, plus, minus, heart, star, volume, mute, chevron-right, chevron-left, menu.

Use `getPixelArtSymbols()` to get the full list of built-in symbol names at runtime.

=== API / SYNTAX ===
```tsx
import { PixelArt, usePixelArt, getPixelArtSymbols } from '@reactjit/core';
```

PixelArt Props:
| Prop   | Type             | Default   | Description                               |
|--------|------------------|-----------|-------------------------------------------|
| art    | string \| string[] | required | Built-in symbol name or string art lines |
| size   | number           | 4         | Pixel size in logical units               |
| color  | string           | '#ffffff' | Fill color for all pixels                 |
| colors | string[]         | undefined | Per-row fill colors (overrides color)     |
| gap    | number           | 0         | Gap between pixels in logical units       |
| style  | Style            | undefined | Container style                           |

usePixelArt(art, options):
| Option | Type     | Default   | Description                           |
|--------|----------|-----------|---------------------------------------|
| size   | number   | 4         | Pixel size in logical units           |
| color  | string   | '#ffffff' | Fill color for all pixels             |
| colors | string[] | undefined | Per-row fill colors (overrides color) |
| gap    | number   | 0         | Gap between pixels                    |

Returns: React.ReactElement (memoized).

getPixelArtSymbols():
Returns: string[] — all available built-in symbol names.

Example 2: Using the hook
---
```tsx
const playIcon = usePixelArt('play', { size: 3, color: '#ffffff' });
const pauseIcon = usePixelArt('pause', { size: 3, color: '#ffffff' });

<Box style={{ flexDirection: 'row', gap: 12 }}>
  {isPlaying ? pauseIcon : playIcon}
</Box>
```
---
Platforms: love2d

const heartArt = [

=== PLATFORM NOTES ===
Love2D:
  Each pixel is a Box element. Large pixel art grids (many rows/columns) create many nodes. Keep art patterns small — the built-in symbols are designed at 5-7 rows. For larger artwork, consider Image instead.

=== CRITICAL RULES ===
- When using a string (not an array) as the `art` prop, it first checks built-in symbols by name. If no match is found, it splits the string by newlines and treats it as custom art.
- The `colors` array maps to rows (index 0 = first row). Rows beyond the array length fall back to the `color` prop.
- usePixelArt is memoized on art, size, color, colors, and gap. Avoid creating new arrays/objects inline each render — hoist them to constants or useMemo.
