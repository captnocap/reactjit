=== OVERVIEW ===
PixelArt converts Unicode symbols (▶ ⏸ ● ✓ arrows, etc.) and custom string art into rendered Box grids. Love2D's default font does not include Unicode symbols, so PixelArt renders them as colored Box elements instead. It provides a library of built-in symbols (play, pause, check, heart, etc.) and supports custom bitmap art defined as string arrays.

The component and hook use a "filled pixel" convention: any non-space character in the input string represents a filled pixel, which renders as a Box with the specified color. Spaces render as transparent. Per-row coloring is supported for multi-color designs like gradients or layered effects.

=== API / SYNTAX ===
```tsx
import { PixelArt, usePixelArt, getPixelArtSymbols } from '@reactjit/core';
```

```tsx
// Component usage
<PixelArt art="play" size={4} color="#ffffff" />

// Hook usage
const playIcon = usePixelArt('play', { size: 4, color: '#ffffff' });
return <Box>{playIcon}</Box>;

// Get list of available symbols
const symbols = getPixelArtSymbols();
```

PixelArt Component Props:
| Prop   | Type              | Default     | Description                                    |
|--------|-------------------|-------------|------------------------------------------------|
| art    | string \| string[] | (required)  | Built-in symbol name OR array of string art    |
| size   | number            | 4           | Pixel size in logical units                    |
| color  | string            | '#ffffff'   | Single fill color for all pixels               |
| colors | string[]          | undefined   | Per-row fill colors (overrides color)          |
| gap    | number            | 0           | Gap between pixels in logical units            |
| style  | Style             | undefined   | Container style overrides                      |

usePixelArt Hook:
```tsx
function usePixelArt(
  art: string | string[],
  options?: PixelArtOptions
): React.ReactElement
```

Options:
| Key    | Type      | Default    | Description                              |
|--------|-----------|------------|------------------------------------------|
| size   | number    | 4          | Pixel size in logical units              |
| color  | string    | '#ffffff'  | Single fill color for all pixels         |
| colors | string[]  | undefined  | Per-row fill colors (overrides color)    |
| gap    | number    | 0          | Gap between pixels in logical units      |

getPixelArtSymbols():
Returns an array of all built-in symbol names.

Built-in Symbols:
play, pause, stop, skip-next, skip-prev, check, close, arrow-up, arrow-down, arrow-left, arrow-right, circle, circle-outline, dot, plus, minus, heart, star, volume, mute, chevron-right, chevron-left, menu

Example 4: Using the hook for dynamic icons
---
```tsx
import { useState } from 'react';
import { usePixelArt, Box, Pressable } from '@reactjit/core';

function MediaControls() {
  const [isPlaying, setIsPlaying] = useState(false);

  const icon = usePixelArt(
    isPlaying ? 'pause' : 'play',
    { size: 5, color: isPlaying ? '#f38ba8' : '#a6e3a1' }
  );

  return (
    <Pressable onPress={() => setIsPlaying(!isPlaying)}>
      <Box style={{ padding: 12, backgroundColor: '#313244', borderRadius: 8 }}>
        {icon}
      </Box>
    </Pressable>
  );
}
```

Example 5: Icon grid with all built-in symbols

import { getPixelArtSymbols, PixelArt, Box, Text } from '@reactjit/core';

function IconGallery() {

  return (
    <Box style={{ width: '100%', padding: 16, flexDirection: 'row', flexWrap: 'wrap', gap: 16 }}>

          </Box>
          <Text style={{ fontSize: 10, color: '#a6adc8' }}>{symbol}</Text>
        </Box>
      ))}
    </Box>
  );
}

Example 6: Animated arrow with pixel gap
---
```tsx
import { useState, useEffect } from 'react';
import { PixelArt, Box } from '@reactjit/core';

function AnimatedArrow() {
  const [direction, setDirection] = useState<'arrow-up' | 'arrow-down' | 'arrow-left' | 'arrow-right'>('arrow-right');

  useEffect(() => {
    const directions = ['arrow-right', 'arrow-down', 'arrow-left', 'arrow-up'] as const;
    let index = 0;
    const interval = setInterval(() => {
      index = (index + 1) % 4;
      setDirection(directions[index]);
    }, 500);
    return () => clearInterval(interval);
  }, []);

  return (
    <Box style={{ padding: 24, backgroundColor: '#1e1e2e' }}>
      <PixelArt art={direction} size={8} color="#89dceb" gap={2} />
    </Box>
  );
}
```

=== PLATFORM NOTES ===
- **Love2D**: PixelArt is essential for Love2D because the default font lacks Unicode symbol support. All UI controls (play buttons, checkmarks, arrows) should use PixelArt or custom Box-based shapes.
- **Web**: PixelArt works but is unnecessary if you have a web font that includes Unicode symbols. Prefer native symbols for performance. PixelArt is useful for retro/pixel aesthetics or cross-target consistency.
- **Grid targets**: Not currently supported. PixelArt is only available in `@reactjit/core` (Love2D and web targets). Terminal/ComputerCraft targets can render Unicode symbols natively.

=== CRITICAL RULES ===
- **Never put Unicode symbols in `<Text>`** — Characters like ▶ ⏸ █ ● ✓ arrows, dingbats, and geometric shapes won't render in Love2D's default font. Always use PixelArt or Box-based geometry instead. The linter enforces this via `no-unicode-symbol-in-text`.
- **Filled pixel convention** — Any non-space character in the art string = filled pixel. Spaces = transparent. Use any character (█, X, #, etc.) for readability — they all render identically.
- **Size units are logical** — The `size` prop is in logical units, not pixels. On a 2x DPI screen, `size={4}` renders as 8 physical pixels.
- **Colors array falls back** — If `colors` array is shorter than the number of rows, remaining rows use the `color` prop. If neither is provided, defaults to white.
- **Memoization is automatic** — The `usePixelArt` hook memoizes the result. Changing `art`, `size`, `color`, `colors`, or `gap` triggers a re-render, but otherwise the element is cached.
- **Gap adds spacing** — The `gap` prop adds space between pixels, useful for retro/chunky aesthetics. Default is 0 (no gap).
- **Container style wraps** — When using the PixelArt component with a `style` prop, it wraps the rendered grid in an additional Box. When using the hook, no wrapper is added.
