=== OVERVIEW ===
MonacoMirror wraps an Input in a VS Code-styled editor chrome. It provides an activity bar, file explorer sidebar, tab bar, breadcrumb navigation, code minimap, and status bar -- all built from Box/Text/Pressable primitives. The actual text editing is handled by an embedded Input with `multiline`, `lineNumbers`, and `syntaxHighlight` enabled.

MonacoMirror is a compound component (not a Lua-owned primitive). It constructs its chrome in React and delegates editing to the Input component. The editor automatically adapts its layout: at small sizes (controlled by `layoutMode` and `compactMaxWidth`/`compactMaxHeight`), it switches to a compact mode that hides the activity bar, sidebar, minimap, and breadcrumbs.

The file explorer sidebar displays a tree built from either the `explorerFiles` prop or a fallback generated from `filePath`. Files and folders can be expanded/collapsed, and file selection fires `onFileSelect`.

=== API / SYNTAX ===
```tsx
import { MonacoMirror } from '@reactjit/core';
```

Props (inherits all Input props except multiline, lineNumbers, syntaxHighlight, and style):
| Prop              | Type                        | Default         | Description                              |
|-------------------|-----------------------------|-----------------|------------------------------------------|
| style             | Style                       | undefined       | Outer container style                    |
| inputStyle        | Style                       | undefined       | Style for the editor input area          |
| filePath          | string                      | 'src/App.tsx'   | Primary file path (tab label, breadcrumbs)|
| selectedFilePath  | string                      | undefined       | Controlled selected file in explorer     |
| explorerFiles     | string[]                    | undefined       | File paths for the explorer tree         |
| onFileSelect      | (path: string) => void      | undefined       | Fires when a file is selected in explorer|
| tabLabel          | string                      | undefined       | Override tab label text                  |
| workspaceLabel    | string                      | 'workspace'     | Label in explorer header                 |
| branch            | string                      | 'main'          | Branch name displayed in tab bar         |
| language          | string                      | auto-detected   | Override language label                  |
| sidebarWidth      | number                      | 190             | Sidebar width in pixels                  |
| minimapWidth      | number                      | 120             | Minimap panel width in pixels            |
| showActivityBar   | boolean                     | true            | Show left activity bar                   |
| showSidebar       | boolean                     | true            | Show file explorer sidebar               |
| showMinimap       | boolean                     | true            | Show code minimap panel                  |
| showBreadcrumbs   | boolean                     | true            | Show breadcrumb path bar                 |
| showStatusBar     | boolean                     | true            | Show bottom status bar                   |
| minimapMaxLines   | number                      | 120             | Max source lines to render in minimap    |
| layoutMode        | 'auto' \| 'full' \| 'compact' | 'auto'       | Layout mode selection                    |
| compactMaxWidth   | number                      | 560             | Width threshold for auto compact mode    |
| compactMaxHeight  | number                      | 260             | Height threshold for auto compact mode   |
| activityItems     | string[]                    | ['TAB','EX','CODE','MAP'] | Activity bar icon labels / shortcuts |

All Input props (value, defaultValue, onChangeText, onSubmit, onBlur, onFocus, onLiveChange, onChange, placeholder, editable, spellCheck, wordWrap, cursorColor, textStyle) are passed through to the embedded Input.

=== EXAMPLES ===
Example 1: Basic code editor
---
```tsx
const colors = useThemeColors();
const [code, setCode] = useState('function hello()\n  print("world")\nend');

<MonacoMirror
  value={code}
  onLiveChange={setCode}
  filePath="src/main.lua"
  language="lua"
  style={{ flexGrow: 1 }}
/>
```
---
Platforms: love2d

Example 2: Editor with file explorer
---
```tsx
const colors = useThemeColors();
const [code, setCode] = useState(initialCode);
const [selectedFile, setSelectedFile] = useState('src/App.tsx');

<MonacoMirror
  value={code}
  onLiveChange={setCode}
  filePath="src/App.tsx"
  selectedFilePath={selectedFile}
  explorerFiles={[
    'src/App.tsx',
    'src/components/Header.tsx',
    'src/components/Sidebar.tsx',
    'src/hooks/useData.ts',
    'package.json',
  ]}
  onFileSelect={(path) => {
    setSelectedFile(path);
    loadFile(path).then(setCode);
  }}
  style={{ flexGrow: 1 }}
/>
```
---
Platforms: love2d

Example 3: Compact mode for small spaces

const colors = useThemeColors();

=== PLATFORM NOTES ===
Love2D:
  MonacoMirror is a React compound component -- the chrome (tab bar, sidebar, minimap, status bar) is built from Box, Text, and Pressable elements. Only the editor area itself delegates to a Lua-owned Input. This means the chrome re-renders when React state changes (file selection, panel toggles), but the actual text editing remains zero-latency Lua.

  The minimap renders a simplified representation of the code as ink-density bars, not actual text. Its viewport highlight tracks the real Lua editor scroll position, and the status bar reflects the live cursor line/column from the Lua-owned editor.

  At small container sizes (below compactMaxWidth or compactMaxHeight), the layout automatically switches to compact mode, hiding the activity bar, sidebar, minimap, and breadcrumbs to maximize editor space.

=== CRITICAL RULES ===
- MonacoMirror needs explicit dimensions or `flexGrow: 1` to size itself. Without a known size, the layout collapses.
- The `style` prop controls the outer container. Use `inputStyle` for the editor input area and `textStyle` for text appearance.
- `explorerFiles` paths should use forward slashes. The tree builder normalizes backslashes automatically but forward slashes are canonical.
- When using controlled mode (`value` prop), always provide `onLiveChange` or `onChange` to update the value. Otherwise the editor appears frozen because keystrokes update Lua's buffer but React re-renders overwrite it with the stale value.
