=== OVERVIEW ===
Fleet renders a multi-agent panel where each agent is an independent Claude Code session. Agents appear as collapsible tiles with status indicators, permission bars, and embedded terminal canvases. The component manages session lifecycle, permission prompts, question prompts, and auto-accept toggling.

Each agent has a status (idle, running, thinking, waiting_permission, stopped), a model (sonnet, opus, haiku), and independent permission/question state. The permission bar shows inline approve/allow-all/deny buttons. Question prompts display as inline option bars.

Fleet uses the `useFleet` hook internally, which manages N concurrent sessions via session-scoped RPCs (every call includes `session: agentId` to target the right PTY). Auto-accept is global — one toggle for all agents.

Expanded agents show a ClaudeCanvas (visual PTY) and a text input prompt for sending tasks. Collapsed agents show only the header with status.

=== API / SYNTAX ===
```tsx
import { Fleet } from '@reactjit/core';
import { useFleet } from '@reactjit/core';
```

Fleet Props:
| Prop            | Type               | Default      | Description                         |
|-----------------|--------------------|--------------|-------------------------------------|
| workingDir      | string             | required     | Default working directory           |
| agents          | FleetAgentConfig[] | required     | Agent configurations                |
| defaultExpanded | string[]           | [first agent]| IDs of initially expanded agents    |
| style           | Style              | undefined    | Container style                     |

FleetAgentConfig:
| Field     | Type                          | Default  | Description                    |
|-----------|-------------------------------|----------|--------------------------------|
| id        | string                        | required | Unique agent ID                |
| model     | 'sonnet' \| 'opus' \| 'haiku' | 'sonnet' | Claude model                   |
| label     | string                        | id       | Display label                  |
| workingDir| string                        | parent   | Per-agent working directory    |

useFleet(options):
Returns FleetResult:
| Field           | Type                   | Description                          |
|-----------------|------------------------|--------------------------------------|
| agents          | FleetAgentState[]      | Per-agent state and actions          |
| autoAccept      | boolean                | Global auto-accept state             |
| toggleAutoAccept| () => void             | Toggle auto-accept for all agents    |
| focused         | string \| null         | Currently focused agent ID           |
| setFocused      | (id) => void           | Set focused agent                    |

FleetAgentState:
| Field            | Type                         | Description                        |
|------------------|------------------------------|------------------------------------|
| id               | string                       | Agent ID                           |
| model            | string                       | Active model name                  |
| label            | string                       | Display label                      |
| status           | string                       | Current status                     |
| perm             | FleetPermission \| null      | Pending permission request         |
| question         | FleetQuestion \| null        | Pending question prompt            |
| send             | (message: string) => void    | Send a message to this agent       |
| stop             | () => void                   | Stop current operation             |
| respond          | (choice: number) => void     | Respond to permission (1/2/3)      |
| respondQuestion  | (optionIndex: number) => void| Respond to question prompt         |

Example 3: Using useFleet directly
---
```tsx
const fleet = useFleet({
  workingDir: '/home/user/project',
  agents: [
    { id: 'main', model: 'sonnet', label: 'Main' },
    { id: 'helper', model: 'haiku', label: 'Helper' },
  ],
});

{fleet.agents.map(agent => (
  <Box key={agent.id}>
    <Text style={{ fontSize: 12 }}>{`${agent.label}: ${agent.status}`}</Text>
    {agent.perm && (
      <Pressable onPress={() => agent.respond(1)}>
        <Text style={{ fontSize: 12 }}>Approve</Text>
      </Pressable>
    )}
  </Box>
))}
```
---
Platforms: love2d

=== PLATFORM NOTES ===
Love2D:
  Fleet spawns ClaudeCode capability nodes on the Lua side. Each agent renders a ClaudeCanvas for visual PTY output. Permission responses use session-scoped RPCs (claude:respond with session ID). Auto-accept state is global in Lua — toggling it affects all sessions. Status colors are hardcoded (green=running, yellow=thinking, pink=waiting_permission).

=== CRITICAL RULES ===
- Every agent ID must be unique. Duplicate IDs cause session conflicts on the Lua side.
- Permission responses: 1=approve once, 2=allow all (auto-accept for this agent), 3=deny. These map to Claude Code's permission system.
- Auto-accept is global (Lua limitation). Toggling it for one agent toggles it for all.
- Fleet uses hardcoded dark-theme colors internally. For themed usage, use useFleet directly and build custom agent tiles.
