=== OVERVIEW ===
Text renders a string as a Lua `Text` node. The layout engine measures text from font metrics — width from glyph advances, height from line height — so Text auto-sizes to its content by default.

Text supports shorthand props for common styling (`size`, `color`, `bold`, `align`, `font`) alongside the full `style` prop. Shorthands are lower priority — `style={}` wins when both are set.

Text does not support mixed text and expression children. `<Text>Hello {name}!</Text>` creates three separate `__TEXT__` nodes that stack vertically instead of rendering inline. Always use template literals: `{`Hello ${name}!`}`.

=== API / SYNTAX ===
```tsx
import { Text } from '@reactjit/core';
```

Shorthand Props:
| Prop   | Type                      | Default   | Description               |
|--------|---------------------------|-----------|---------------------------|
| size   | number                    | undefined | Font size in pixels        |
| color  | Color                     | undefined | Text color                 |
| bold   | boolean                   | false     | Bold font weight           |
| italic | boolean                   | false     | Italic style               |
| align  | 'left' \| 'center' \| 'right' | undefined | Text alignment        |
| font   | string                    | undefined | Font family name           |
| lines  | number                    | undefined | Maximum number of lines    |

Style Props:
| Prop  | Type  | Default   | Description           |
|-------|-------|-----------|-----------------------|
| style | Style | undefined | Full style object     |

Relevant style properties for Text:
| Property           | Type                                     | Description                    |
|--------------------|------------------------------------------|--------------------------------|
| fontSize           | number                                   | Font size in pixels            |
| color              | Color                                    | Text color                     |
| fontFamily         | string                                   | Font family                    |
| fontWeight         | 'normal' \| 'bold' \| number             | Font weight                    |
| textAlign          | 'left' \| 'center' \| 'right'           | Horizontal text alignment      |
| lineHeight         | number                                   | Line height in pixels          |
| letterSpacing      | number                                   | Space between characters       |
| textDecorationLine | 'none' \| 'underline' \| 'line-through' | Text decoration                |
| textWrap           | 'wrap' \| 'nowrap'                       | Text wrapping behavior         |
| textOverflow       | 'clip' \| 'ellipsis'                     | Overflow behavior              |
| numberOfLines      | number                                   | Clamp to N lines with ellipsis |

Event Handlers:
| Prop        | Type                       | Description               |
|-------------|----------------------------|---------------------------|
| onKeyDown   | (event: LoveEvent) => void | Key pressed               |
| onKeyUp     | (event: LoveEvent) => void | Key released              |
| onTextInput | (event: LoveEvent) => void | Text character input      |

=== EXAMPLES ===
Example 1: Basic text
---
```tsx
const colors = useThemeColors();

<Text style={{ color: colors.text, fontSize: 16 }}>Hello, world</Text>
```
---
Platforms: love2d

Example 2: Using shorthand props
---
```tsx
const colors = useThemeColors();

<Text size={24} color={colors.text} bold align="center">
  Page Title
</Text>
```
---
Platforms: love2d

Example 3: Dynamic text with template literal
---
```tsx
const colors = useThemeColors();
const count = 42;

<Text style={{ color: colors.text, fontSize: 14 }}>
  {`You have ${count} items`}
</Text>
```
---
Platforms: love2d

Example 4: Truncated text with ellipsis
---
```tsx
const colors = useThemeColors();

<Text
  style={{
    color: colors.text,
    fontSize: 14,
    textOverflow: 'ellipsis',
  }}
  lines={2}
>
  This text will be clamped to two lines. Any overflow beyond the second line is replaced with an ellipsis character.
</Text>
```
---
Platforms: love2d

=== CRITICAL RULES ===
- Never mix text and expressions in Text children. `<Text>Hello {name}!</Text>` creates three stacked nodes. Use `{`Hello ${name}!`}` instead. The linter enforces this via `no-mixed-text-children`.
- Text is not a surface — it does not get proportional fallback sizing. It auto-sizes from font metrics.
- Always use `useThemeColors()` for text color. Never hardcode color values.
