=== OVERVIEW ===
Checkbox renders a toggleable check box with an optional text label. It emits a `Checkbox` host element -- all toggle state, check mark rendering, and click handling are in Lua.

Supports both controlled (`value` prop) and uncontrolled (`defaultValue` prop) modes. The checkbox size, checked color, and unchecked color are customizable.

=== API / SYNTAX ===
```tsx
import { Checkbox } from '@reactjit/core';
```

Props:
| Prop           | Type                        | Default   | Description                         |
|----------------|-----------------------------|-----------|-------------------------------------|
| value          | boolean                     | undefined | Controlled checked state            |
| defaultValue   | boolean                     | false     | Initial state for uncontrolled mode |
| onValueChange  | (value: boolean) => void    | undefined | Fires when toggled                  |
| disabled       | boolean                     | false     | Disable interaction                 |
| label          | string                      | undefined | Text label displayed next to checkbox|
| size           | number                      | 20        | Checkbox size in pixels             |
| color          | Color                       | '#3b82f6' | Checked state color                 |
| uncheckedColor | Color                       | '#6b7280' | Unchecked state border color        |
| style          | Style                       | undefined | Container style overrides           |

=== EXAMPLES ===
Example 1: Basic checkbox
---
```tsx
const colors = useThemeColors();
const [agreed, setAgreed] = useState(false);

<Checkbox
  value={agreed}
  onValueChange={setAgreed}
  label="I agree to the terms"
  color={colors.primary}
/>
```
---
Platforms: love2d

Example 2: Checkbox without label
---
```tsx
const colors = useThemeColors();
const [selected, setSelected] = useState(false);

<Box direction="row" align="center" gap={8}>
  <Checkbox value={selected} onValueChange={setSelected} color={colors.primary} />
  <Text style={{ color: colors.text, fontSize: 14 }}>Custom label layout</Text>
</Box>
```
---
Platforms: love2d

Example 3: Multiple checkboxes
---
```tsx
const colors = useThemeColors();
const [prefs, setPrefs] = useState({ email: true, sms: false, push: true });

<Box gap={8}>
  <Checkbox
    value={prefs.email}
    onValueChange={(v) => setPrefs({ ...prefs, email: v })}
    label="Email notifications"
    color={colors.primary}
  />
  <Checkbox
    value={prefs.sms}
    onValueChange={(v) => setPrefs({ ...prefs, sms: v })}
    label="SMS notifications"
    color={colors.primary}
  />
  <Checkbox
    value={prefs.push}
    onValueChange={(v) => setPrefs({ ...prefs, push: v })}
    label="Push notifications"
    color={colors.primary}
  />
</Box>
```
---
Platforms: love2d

=== PLATFORM NOTES ===
Love2D:
  The check mark glyph and toggle animation are rendered by Lua. The checkbox is laid out as a row with the check box on the left and the label text on the right. When a label is provided, the component auto-sizes to fit the label; without a label, it sizes to the `size` prop.

=== CRITICAL RULES ===
- When using a label, the component width auto-sizes to `size + 8 + 100`. For wider labels, override the width via the `style` prop.
- The `size` prop controls the checkbox square dimensions, not the overall component height.
