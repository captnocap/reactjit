=== OVERVIEW ===
Tabs renders a horizontal tab bar for switching between views. It supports two visual variants:

- `underline` (default) — tabs sit along a bottom border, with the active tab marked by a colored underline
- `pill` — tabs are rendered inside a rounded container, with the active tab getting an elevated pill background

Tabs is a controlled component. You manage the active tab ID in state and pass it as `activeId`. The `onSelect` callback fires with the selected tab's ID.

Colors are theme-aware via `useThemeColorsOptional()` with dark-theme fallbacks. Hover and press states are handled by Lua through Pressable's style function.

=== API / SYNTAX ===
```tsx
import { Tabs } from '@reactjit/core';
```

Props:
| Prop     | Type                   | Default     | Description                           |
|----------|------------------------|-------------|---------------------------------------|
| tabs     | Tab[]                  | required    | Array of tab definitions              |
| activeId | string                 | required    | ID of the currently active tab        |
| onSelect | (id: string) => void   | undefined   | Callback when a tab is pressed        |
| variant  | 'underline' \| 'pill'  | 'underline' | Visual style of the tab bar          |
| style    | Style                  | undefined   | Outer container style overrides       |

Tab:
| Field | Type   | Description           |
|-------|--------|-----------------------|
| id    | string | Unique tab identifier |
| label | string | Display text          |

=== EXAMPLES ===
Example 1: Underline tabs
---
```tsx
const [tab, setTab] = useState('overview');

<Tabs
  tabs={[
    { id: 'overview', label: 'Overview' },
    { id: 'details', label: 'Details' },
    { id: 'settings', label: 'Settings' },
  ]}
  activeId={tab}
  onSelect={setTab}
/>
```
---
Platforms: love2d

Example 2: Pill tabs
---
```tsx
const [tab, setTab] = useState('day');

<Tabs
  variant="pill"
  tabs={[
    { id: 'day', label: 'Day' },
    { id: 'week', label: 'Week' },
    { id: 'month', label: 'Month' },
  ]}
  activeId={tab}
  onSelect={setTab}
/>
```
---
Platforms: love2d

Example 3: Tabs with content switching
---
```tsx
const colors = useThemeColors();
const [tab, setTab] = useState('code');

<FlexColumn style={{ width: '100%' }}>
  <Tabs
    tabs={[
      { id: 'code', label: 'Code' },
      { id: 'preview', label: 'Preview' },
      { id: 'output', label: 'Output' },
    ]}
    activeId={tab}
    onSelect={setTab}
  />
  <Box style={{ flexGrow: 1, padding: 16 }}>
    {tab === 'code' && <Text style={{ color: colors.text }}>Code editor here</Text>}
    {tab === 'preview' && <Text style={{ color: colors.text }}>Preview pane here</Text>}
    {tab === 'output' && <Text style={{ color: colors.text }}>Build output here</Text>}
  </Box>
</FlexColumn>
```
---
Platforms: love2d

=== CRITICAL RULES ===
- Tabs is a controlled component. You must manage `activeId` in state and update it via `onSelect`.
- The underline variant sets `width: '100%'` on the container. The pill variant also sets `width: '100%'`. Both expect the parent to have a defined width.
- Tabs only renders the tab bar itself. You are responsible for rendering the content that corresponds to the active tab below (or beside) the Tabs component.
