=== OVERVIEW ===
SemanticTerminal is a visual capability node that renders a PTY terminal with Lua-side semantic classification. Each row in the terminal output is classified into semantic tokens (prompt, command, stdout, stderr, etc.) by a named classifier from `lua/classifiers/`.

Unlike the non-visual Terminal component, SemanticTerminal renders — it accepts a `style` prop for layout sizing. Use `flexGrow: 1` to fill available space.

SemanticTerminal supports two modes: `live` for interactive PTY sessions, and `playback` for replaying recorded `.rec.lua` session files with timeline scrubbing and speed control.

Use the `useSemanticTerminal` hook for managed state and playback controls.

=== API / SYNTAX ===
```tsx
import { SemanticTerminal } from '@reactjit/core';
```

Props:
| Prop              | Type                                                        | Default   | Description                                  |
|-------------------|-------------------------------------------------------------|-----------|----------------------------------------------|
| mode              | 'live' \| 'playback'                                       | 'live'    | Live PTY or recorded playback                |
| command           | string                                                      | 'bash'    | Command to run in live mode                  |
| args              | string                                                      | undefined | Command arguments (space-separated)          |
| cwd               | string                                                      | undefined | Working directory                            |
| rows              | number                                                      | 40        | Terminal rows                                |
| cols              | number                                                      | 120       | Terminal columns                             |
| classifier        | string                                                      | 'basic'   | Classifier name from lua/classifiers/        |
| recording         | boolean                                                     | false     | Record PTY output for later playback         |
| playbackSrc       | string                                                      | undefined | Path to .rec.lua file for playback mode      |
| playbackSpeed     | number                                                      | 1.0       | Playback speed multiplier                    |
| showTokens        | boolean                                                     | false     | Show token type badges on each row           |
| showGraph         | boolean                                                     | false     | Show semantic graph panel                    |
| showTimeline      | boolean                                                     | false     | Show timeline scrubber in playback mode      |
| style             | Style                                                       | undefined | Layout style                                 |
| onClassifiedRow   | (event: { row, token, text }) => void                       | undefined | Fired when a row is classified               |
| onGraphUpdate     | (event: { nodeCount, turnCount }) => void                   | undefined | Fired when the semantic graph updates        |
| onStateChange     | (event: { mode, streaming }) => void                        | undefined | Fired when session state changes             |
| onRecordingDone   | (event: { path, frames }) => void                           | undefined | Fired when recording is saved                |
| onPlaybackEnd     | (event: {}) => void                                         | undefined | Fired when playback reaches the end          |

=== EXAMPLES ===
Example 1: Live classified terminal
---
```tsx
<SemanticTerminal
  mode="live"
  command="bash"
  style={{ flexGrow: 1 }}
/>
```
---
Platforms: love2d

Example 2: Playback with timeline
---
```tsx
<SemanticTerminal
  mode="playback"
  playbackSrc="/tmp/session.rec.lua"
  showTimeline
  showTokens
  style={{ flexGrow: 1 }}
/>
```
---
Platforms: love2d

Example 3: With useSemanticTerminal hook
---
```tsx
const st = useSemanticTerminal({ mode: 'live', classifier: 'claude' });

<SemanticTerminal {...st.terminalProps} style={{ flexGrow: 1 }} />
```
---
Platforms: love2d

Example 4: Recording a session
---
```tsx
<SemanticTerminal
  mode="live"
  command="bash"
  recording
  onRecordingDone={(e) => console.log(`Saved ${e.frames} frames to ${e.path}`)}
  style={{ flexGrow: 1 }}
/>
```
---
Platforms: love2d

=== PLATFORM NOTES ===
Love2D:
  SemanticTerminal is rendered by Lua's vterm module. The classifier runs every frame on the Lua side, classifying each terminal row into semantic tokens. The 'basic' classifier handles generic shell output. The 'claude' classifier understands Claude Code session structure (prompts, tool calls, responses). Custom classifiers can be added to lua/classifiers/.

=== CRITICAL RULES ===
- SemanticTerminal is a visual component. Unlike Terminal (which renders nothing), it takes space in the layout. Always give it explicit dimensions or flexGrow.
- The `classifier` prop must match a classifier file in `lua/classifiers/`. The default 'basic' classifier is always available.
- Playback mode requires a valid `.rec.lua` file path in `playbackSrc`. Recording mode writes to disk when the session ends.
