=== OVERVIEW ===
ReactJIT is an npm workspaces monorepo. The core framework runs on Node.js for builds
and development tooling. The rendering target requires Love2D and a compiled QuickJS
shared library. This guide walks through every step from a fresh clone to a verified
installation.

=== API / SYNTAX ===
Prerequisites:
  - Node.js 18+ and npm
  - Love2D 11+ (https://love2d.org)
  - LuaJIT (bundled with Love2D)
  - make + gcc or clang (for QuickJS compilation)
  - git

=== EXAMPLES ===
Example 1: Full Installation
---
# 1. Clone the repository
git clone <repo-url> && cd reactjit

# 2. Install npm dependencies (workspaces monorepo)
npm install

# 3. Build QuickJS shared library
#    Clones quickjs-ng and compiles libquickjs.so
make setup

# 4. Propagate source-of-truth files to CLI runtime
#    Copies lua/*.lua, packages/core/, packages/renderer/, libquickjs.so
#    into cli/runtime/ so the CLI can scaffold projects
make cli-setup

# 5. Link the CLI globally
cd cli && npm link && cd ..

# 6. Verify the installation
reactjit help
---
Platforms: love2d

Example 2: Installing Love2D
---
# Linux (Debian/Ubuntu)
sudo apt install love

# Linux (AppImage)
# Download from https://love2d.org and chmod +x

# macOS (Homebrew)
brew install love

# macOS (direct download)
# Download from https://love2d.org

# Verify
love --version
---
Platforms: love2d

Example 3: Verify Everything Works
---
# Create a test project
reactjit init test-app
cd test-app

# Build and run
reactjit dev

# You should see a Love2D window with the default starter app.
# Edit src/App.tsx and save -- the window reloads automatically.
---
Platforms: love2d

=== PLATFORM NOTES ===
Love2D:
  Love2D 11+ is required. The framework uses LuaJIT (bundled with Love2D) and
  OpenGL 2.1 for rendering. The `make setup` step compiles libquickjs.so which
  requires a C compiler (gcc or clang). On Linux, both gcc and make are typically
  pre-installed. On macOS, install Xcode Command Line Tools (`xcode-select --install`).

=== CRITICAL RULES ===
- Always run `make cli-setup` after pulling framework changes. This propagates
  source-of-truth files into the CLI runtime directory.
- Never edit files in cli/runtime/ directly. They are overwritten by make cli-setup.
- The `npm link` step in cli/ must be done once to make the `reactjit` command
  available globally.
- Never edit lua/ or reactjit/ inside a consumer project. Those are disposable
  copies. Edit the monorepo source, then run `make cli-setup` and `reactjit update`.
