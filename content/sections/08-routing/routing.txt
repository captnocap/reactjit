=== OVERVIEW ===
The ReactJIT router provides declarative, URL-based navigation across all rendering targets. It uses a history adapter pattern to support both browser-based navigation (web) and in-memory navigation (Love2D, terminal, grid targets). Routes are matched via path patterns with parameter extraction, and the system provides both component-based navigation (Link) and imperative navigation (useNavigate).

=== API / SYNTAX ===

RouterProvider
--------------
Top-level provider that supplies routing state to the component tree.

```tsx
import { RouterProvider, createMemoryHistory } from '@reactjit/router';

<RouterProvider history={createMemoryHistory()}>
  <App />
</RouterProvider>
```

Type signature:
```tsx
interface RouterProviderProps {
  history: HistoryAdapter;
  children: ReactNode;
}
```

Routes
------
Container that evaluates all child Route elements and renders the best match based on pattern specificity.

```tsx
import { Routes, Route } from '@reactjit/router';

<Routes>
  <Route path="/" element={<Home />} />
  <Route path="/users/:id" element={<UserPage />} />
  <Route path="/users/new" element={<NewUser />} />
  <Route path="*" element={<NotFound />} />
</Routes>
```

Route matching precedence (highest to lowest):
- Static segments (/users/new) — score: 4 per segment
- Required parameters (/users/:id) — score: 3 per segment
- Optional parameters (/users/:id?) — score: 2 per segment
- Wildcards (/*) — score: 1

Route
-----
Marker component that defines a path pattern and corresponding element.

```tsx
<Route path="/users/:id" element={<UserPage />} />
```

Type signature:
```tsx
interface RouteProps {
  path: string;
  element: ReactNode;
}
```

Path pattern syntax:
- /users/new — static path
- /users/:id — named parameter (captured in useParams)
- /users/:id? — optional parameter
- /files/* — wildcard (captures rest as $rest param)
- / — root path

Link
----
Navigable element. Renders as an anchor tag and navigates on click.

```tsx
import { Link } from '@reactjit/router';

<Link to="/about">About</Link>
<Link to="/users/123" replace>View User</Link>
```

Type signature:
```tsx
interface LinkProps {
  to: string;
  replace?: boolean;
  children: ReactNode;
  style?: any;
}
```

Navigate
--------
Declarative redirect component. Triggers navigation when rendered.

```tsx
import { Navigate } from '@reactjit/router';

<Route path="/old-path" element={<Navigate to="/new-path" replace />} />
```

Useful for redirects and conditional navigation:
```tsx
function ProtectedRoute() {
  const isAuthenticated = useAuth();
  if (!isAuthenticated) return <Navigate to="/login" />;
  return <Dashboard />;
}
```

Outlet
------
Renders nested route content. Used in parent route elements to define where child routes appear.

```tsx
import { Outlet } from '@reactjit/router';

function Layout() {
  return (
    <Box flexDirection="column" width="100%" height="100%">
      <NavBar />
      <Outlet />
    </Box>
  );
}

<Routes>
  <Route path="/" element={<Layout />}>
    <Route path="home" element={<Home />} />
    <Route path="about" element={<About />} />
  </Route>
</Routes>
```

useRouter
---------
Returns full router state including location, params, and navigation methods.

```tsx
import { useRouter } from '@reactjit/router';

function Component() {
  const { location, navigate, back, forward } = useRouter();

  return (
    <Text fontSize={14}>
      Current path: {location.pathname}
    </Text>
  );
}
```

Type signature:
```tsx
interface RouterState {
  location: Location;
  params: Record<string, string>;
  navigate: (to: string, options?: NavigateOptions) => void;
  back: () => void;
  forward: () => void;
}

interface Location {
  pathname: string;
  search: string;
  hash: string;
}

interface NavigateOptions {
  replace?: boolean;
}
```

useNavigate
-----------
Returns navigation function for imperative navigation.

```tsx
import { useNavigate } from '@reactjit/router';

function Component() {
  const navigate = useNavigate();

  const handleSubmit = () => {
    // Save data...
    navigate('/success');
  };

  const handleCancel = () => {
    navigate('/home', { replace: true });
  };

  return <Form onSubmit={handleSubmit} />;
}
```

Type signature:
```tsx
function useNavigate(): (to: string, options?: NavigateOptions) => void
```

useLocation
-----------
Returns current location object.

```tsx
import { useLocation } from '@reactjit/router';

function Component() {
  const location = useLocation();

  return (
    <Box>
      <Text fontSize={14}>Path: {location.pathname}</Text>
      <Text fontSize={14}>Query: {location.search}</Text>
      <Text fontSize={14}>Hash: {location.hash}</Text>
    </Box>
  );
}
```

Type signature:
```tsx
function useLocation(): Location
```

useParams
---------
Returns route parameters extracted from the current path.

```tsx
import { useParams } from '@reactjit/router';

function UserPage() {
  const params = useParams<{ id: string }>();

  return <Text fontSize={16}>User ID: {params.id}</Text>;
}

// Route definition:
<Route path="/users/:id" element={<UserPage />} />

// When navigating to /users/123, params.id === "123"
```

Type signature:
```tsx
function useParams<T extends Record<string, string> = Record<string, string>>(): T
```

useRoute
--------
Matches current location against a pattern. Returns [matched, params] tuple.

```tsx
import { useRoute } from '@reactjit/router';

function Component() {
  const [isUserPage, params] = useRoute('/users/:id');

  if (isUserPage) {
    return <Text fontSize={14}>Viewing user {params.id}</Text>;
  }
  return <Text fontSize={14}>Not a user page</Text>;
}
```

Type signature:
```tsx
function useRoute(pattern: string): [boolean, Record<string, string>]
```

createMemoryHistory
-------------------
Creates an in-memory history adapter for non-browser environments.

```tsx
import { createMemoryHistory } from '@reactjit/router';

const history = createMemoryHistory({
  initialEntries: ['/home', '/about'],
  initialIndex: 1, // Start at /about
});
```

Type signature:
```tsx
interface MemoryHistoryOptions {
  initialEntries?: string[];
  initialIndex?: number;
}

function createMemoryHistory(options?: MemoryHistoryOptions): HistoryAdapter
```

createBrowserHistory
--------------------
Creates a browser history adapter using pushState/replaceState (web only).

```tsx
import { createBrowserHistory } from '@reactjit/router';

const history = createBrowserHistory();
```

Type signature:
```tsx
function createBrowserHistory(): HistoryAdapter
```

HistoryAdapter Interface
-------------------------
All history implementations conform to this interface.

```tsx
interface HistoryAdapter {
  readonly location: Location;
  push(path: string): void;
  replace(path: string): void;
  back(): void;
  forward(): void;
  subscribe(listener: (location: Location) => void): () => void;
}
```

=== EXAMPLES ===

Example 1: Basic routing setup
-------------------------------
```tsx
import React from 'react';
import { Box, Text } from '@reactjit/core';
import {
  RouterProvider,
  Routes,
  Route,
  Link,
  createMemoryHistory
} from '@reactjit/router';

function Home() {
  return (
    <Box padding={20}>
      <Text fontSize={24}>Home Page</Text>
      <Link to="/about">
        <Text fontSize={16} color="#4080FF">Go to About</Text>
      </Link>
    </Box>
  );
}

function About() {
  return (
    <Box padding={20}>
      <Text fontSize={24}>About Page</Text>
      <Link to="/">
        <Text fontSize={16} color="#4080FF">Back to Home</Text>
      </Link>
    </Box>
  );
}

function NotFound() {
  return (
    <Box padding={20}>
      <Text fontSize={24} color="#FF4040">404 Not Found</Text>
      <Link to="/">
        <Text fontSize={16} color="#4080FF">Go Home</Text>
      </Link>
    </Box>
  );
}

export function App() {
  const history = createMemoryHistory({ initialEntries: ['/'] });

  return (
    <RouterProvider history={history}>
      <Box width="100%" height="100%">
        <Routes>
          <Route path="/" element={<Home />} />
          <Route path="/about" element={<About />} />
          <Route path="*" element={<NotFound />} />
        </Routes>
      </Box>
    </RouterProvider>
  );
}
```

Example 2: Navigation with params
----------------------------------
```tsx
import React from 'react';
import { Box, Text, Pressable } from '@reactjit/core';
import {
  RouterProvider,
  Routes,
  Route,
  Link,
  useParams,
  useNavigate,
  createMemoryHistory
} from '@reactjit/router';

const users = [
  { id: '1', name: 'Alice', email: 'alice@example.com' },
  { id: '2', name: 'Bob', email: 'bob@example.com' },
  { id: '3', name: 'Charlie', email: 'charlie@example.com' },
];

function UserList() {
  return (
    <Box padding={20} flexDirection="column" gap={10}>
      <Text fontSize={24}>Users</Text>
      {users.map(user => (
        <Link key={user.id} to={`/users/${user.id}`}>
          <Box
            padding={10}
            backgroundColor="#2A2A2A"
            borderRadius={4}
          >
            <Text fontSize={16} color="#4080FF">{user.name}</Text>
          </Box>
        </Link>
      ))}
    </Box>
  );
}

function UserDetail() {
  const { id } = useParams<{ id: string }>();
  const navigate = useNavigate();
  const user = users.find(u => u.id === id);

  if (!user) {
    return (
      <Box padding={20}>
        <Text fontSize={18} color="#FF4040">User not found</Text>
        <Pressable onPress={() => navigate('/users')}>
          <Text fontSize={14} color="#4080FF">Back to list</Text>
        </Pressable>
      </Box>
    );
  }

  return (
    <Box padding={20} flexDirection="column" gap={15}>
      <Text fontSize={24}>{user.name}</Text>
      <Text fontSize={16} color="#CCCCCC">{user.email}</Text>
      <Text fontSize={14} color="#888888">ID: {user.id}</Text>

      <Pressable
        onPress={() => navigate('/users')}
        style={{ marginTop: 10 }}
      >
        <Box padding={10} backgroundColor="#3A3A3A" borderRadius={4}>
          <Text fontSize={14} color="#4080FF">Back to list</Text>
        </Box>
      </Pressable>
    </Box>
  );
}

export function App() {
  const history = createMemoryHistory({ initialEntries: ['/users'] });

  return (
    <RouterProvider history={history}>
      <Box width="100%" height="100%">
        <Routes>
          <Route path="/users" element={<UserList />} />
          <Route path="/users/:id" element={<UserDetail />} />
          <Route path="*" element={<Navigate to="/users" />} />
        </Routes>
      </Box>
    </RouterProvider>
  );
}
```

Example 3: Nested routes with Outlet
-------------------------------------
```tsx
import React from 'react';
import { Box, Text, Pressable } from '@reactjit/core';
import {
  RouterProvider,
  Routes,
  Route,
  Link,
  Outlet,
  useLocation,
  createMemoryHistory
} from '@reactjit/router';

function Layout() {
  const location = useLocation();

  const NavLink = ({ to, label }: { to: string; label: string }) => {
    const isActive = location.pathname === to;
    return (
      <Link to={to}>
        <Box
          padding={10}
          backgroundColor={isActive ? '#4080FF' : '#2A2A2A'}
          borderRadius={4}
        >
          <Text fontSize={14} color={isActive ? '#FFFFFF' : '#CCCCCC'}>
            {label}
          </Text>
        </Box>
      </Link>
    );
  };

  return (
    <Box width="100%" height="100%" flexDirection="column">
      {/* Navigation bar */}
      <Box
        flexDirection="row"
        gap={10}
        padding={15}
        backgroundColor="#1A1A1A"
      >
        <NavLink to="/dashboard" label="Dashboard" />
        <NavLink to="/settings" label="Settings" />
        <NavLink to="/profile" label="Profile" />
      </Box>

      {/* Content area */}
      <Box flexGrow={1} padding={20}>
        <Outlet />
      </Box>
    </Box>
  );
}

function Dashboard() {
  return <Text fontSize={24}>Dashboard Content</Text>;
}

function Settings() {
  return <Text fontSize={24}>Settings Content</Text>;
}

function Profile() {
  return <Text fontSize={24}>Profile Content</Text>;
}

export function App() {
  const history = createMemoryHistory({ initialEntries: ['/dashboard'] });

  return (
    <RouterProvider history={history}>
      <Routes>
        <Route path="/" element={<Layout />}>
          <Route path="dashboard" element={<Dashboard />} />
          <Route path="settings" element={<Settings />} />
          <Route path="profile" element={<Profile />} />
        </Route>
      </Routes>
    </RouterProvider>
  );
}
```

Example 4: Conditional navigation with useRoute
------------------------------------------------
```tsx
import React from 'react';
import { Box, Text } from '@reactjit/core';
import {
  RouterProvider,
  Routes,
  Route,
  useRoute,
  useLocation,
  createMemoryHistory
} from '@reactjit/router';

function Breadcrumbs() {
  const location = useLocation();
  const [isHome] = useRoute('/');
  const [isUser, userParams] = useRoute('/users/:id');
  const [isSettings] = useRoute('/settings/*');

  return (
    <Box flexDirection="row" gap={5} padding={10} backgroundColor="#1A1A1A">
      <Text fontSize={12} color="#888888">Current: </Text>
      {isHome && <Text fontSize={12} color="#4080FF">Home</Text>}
      {isUser && (
        <Text fontSize={12} color="#4080FF">
          User {userParams.id}
        </Text>
      )}
      {isSettings && (
        <Text fontSize={12} color="#4080FF">Settings</Text>
      )}
      {!isHome && !isUser && !isSettings && (
        <Text fontSize={12} color="#888888">{location.pathname}</Text>
      )}
    </Box>
  );
}

function Home() {
  return (
    <Box padding={20}>
      <Text fontSize={24}>Home</Text>
    </Box>
  );
}

export function App() {
  const history = createMemoryHistory();

  return (
    <RouterProvider history={history}>
      <Box width="100%" height="100%" flexDirection="column">
        <Breadcrumbs />
        <Routes>
          <Route path="/" element={<Home />} />
          {/* Other routes... */}
        </Routes>
      </Box>
    </RouterProvider>
  );
}
```

Example 5: Query string and hash handling
------------------------------------------
```tsx
import React from 'react';
import { Box, Text } from '@reactjit/core';
import {
  RouterProvider,
  useLocation,
  useNavigate,
  Routes,
  Route,
  createMemoryHistory
} from '@reactjit/router';

function SearchPage() {
  const location = useLocation();
  const navigate = useNavigate();

  // Parse query string
  const params = new URLSearchParams(location.search);
  const query = params.get('q') || '';
  const page = params.get('page') || '1';

  const handleSearch = (newQuery: string) => {
    navigate(`/search?q=${encodeURIComponent(newQuery)}&page=1`);
  };

  const handleNextPage = () => {
    const nextPage = parseInt(page) + 1;
    navigate(`/search?q=${encodeURIComponent(query)}&page=${nextPage}`);
  };

  return (
    <Box padding={20} flexDirection="column" gap={10}>
      <Text fontSize={24}>Search Results</Text>
      <Text fontSize={14} color="#CCCCCC">Query: {query}</Text>
      <Text fontSize={14} color="#CCCCCC">Page: {page}</Text>
      <Text fontSize={14} color="#CCCCCC">Hash: {location.hash}</Text>

      {/* Navigation examples */}
      <Box marginTop={10}>
        <Pressable onPress={handleNextPage}>
          <Text fontSize={14} color="#4080FF">Next Page</Text>
        </Pressable>
      </Box>
    </Box>
  );
}

export function App() {
  const history = createMemoryHistory({
    initialEntries: ['/search?q=react&page=1#results']
  });

  return (
    <RouterProvider history={history}>
      <Routes>
        <Route path="/search" element={<SearchPage />} />
      </Routes>
    </RouterProvider>
  );
}
```

=== PLATFORM NOTES ===

Love2D
------
- Uses createMemoryHistory (no URL bar)
- All routing state is in-memory
- History entries persist across hot reloads during development
- Use F12 inspector to debug current route state
- No browser navigation UI (back/forward buttons) — implement in-app navigation

Web
---
- Uses createBrowserHistory with pushState/replaceState API
- URL bar updates automatically on navigation
- Browser back/forward buttons work out of the box
- Link components render as <a> tags with proper href attributes
- Right-click on Link opens context menu with "Open in new tab"
- Search engines can crawl routes if using server-side rendering

Terminal / Grid Targets
------------------------
- Uses createMemoryHistory
- No URL concept — routes are purely application state
- Useful for multi-screen terminal UIs (menus, settings, views)
- Consider displaying current route in a status bar
- History navigation (back/forward) works normally

ComputerCraft
-------------
- Uses createMemoryHistory
- Routes can represent different screens on in-game computers
- Useful for multi-page UIs (control panels, menus, dashboards)
- No persistence — history resets on chunk unload

Neovim / Hammerspoon / AwesomeWM
---------------------------------
- Uses createMemoryHistory
- Routes can represent different tool modes or panels
- Useful for complex editor plugins or window manager widgets
- Can integrate with target-specific navigation (vim keybindings, WM shortcuts)

=== CRITICAL RULES ===

- Always wrap your app in RouterProvider with a history adapter
- Use createMemoryHistory for non-browser targets (Love2D, terminal, grid)
- Use createBrowserHistory for web targets
- Routes must be children of Routes component for best-match semantics
- Static routes (/users/new) are always matched before parameterized routes (/users/:id)
- Path patterns are normalized — trailing slashes are removed (except for "/")
- useParams only works inside a Route element's component tree
- useRouter throws an error if used outside RouterProvider
- Link components must have a "to" prop — empty strings navigate to "/"
- Navigate triggers on mount — wrap in conditional rendering for guarded navigation
- Optional parameters (:id?) match both /users and /users/123
- Wildcards (*) capture the rest of the path as $rest param
- Route matching is case-sensitive
- Query strings and hashes are preserved but not used for route matching
- Multiple Routes components can coexist (separate routing contexts)
- Outlet only renders content when used with nested route definitions
- back() and forward() do nothing at history boundaries (beginning/end of stack)
- replace: true navigation replaces current entry without growing history stack
- RouterProvider's history prop must be stable — don't recreate on every render
