=== OVERVIEW ===
@reactjit/router provides client-side routing with no external dependencies. There is no browser URL bar, no server, and no HTTP. Routes are purely in-memory path strings managed by a history adapter. The router matches the current path against route patterns and renders the best match.

The system has three layers:

1. History adapter -- manages a stack of location entries (pathname, search, hash). Memory history for Love2D and native targets; browser history for WASM web builds.

2. Route matching -- compiles path patterns into regexps with specificity scoring. Supports named parameters (`:id`), optional parameters (`:id?`), and wildcards (`*`). When multiple routes match, the most specific pattern wins.

3. React components and hooks -- `RouterProvider` wraps your app with routing context. `Routes` and `Route` declare which component renders at which path. Hooks (`useNavigate`, `useParams`, `useLocation`, `useRoute`) let any component read or change the current location.

All imports come from `@reactjit/router`:

```tsx
import { RouterProvider, Routes, Route, Link, Navigate, Outlet } from '@reactjit/router';
import { useRouter, useNavigate, useLocation, useParams, useRoute } from '@reactjit/router';
import { createMemoryHistory, createBrowserHistory } from '@reactjit/router';
```
