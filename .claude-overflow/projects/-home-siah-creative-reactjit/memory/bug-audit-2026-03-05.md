# Bug Audit Notes — 2026-03-05

## Key Patterns Found

### Color.set() API
- `lua/color.lua` `Color.set(c)` takes ONE argument (color value)
- Multiple chemistry capabilities call it with two args (color, opacity) — opacity silently dropped
- Correct pattern for opacity: `love.graphics.setColor(r, g, b, opacity)` directly

### Capability Event Dispatch
- Generic `capability` event subscription in eventDispatcher.ts handles ALL capability events via `event.handler` field
- No per-capability subscriptions needed — the architecture is sound
- BUT: some capabilities declare events in their schema but never actually fire them (SpectrumView's `onPeakSelect`, Imaging's `onPreview`)

### StoryScaffold Component Signatures
- `HeroBand({ accentColor, children })` — content goes in children, NOT as props
- `SectionLabel({ icon, children, accentColor? })` — text is children string, NOT `title` prop
- `CalloutBand({ borderColor, bgColor, children })` — no `title` or `accentColor` prop
- CaptureStory uses all of these wrong; FinanceStory and ChemistryStory use them correctly

### Dual Type Definitions (naming conflicts)
- `SpectrumViewProps` defined in BOTH `capabilities.tsx` and `spectra.ts` with different shapes
- `ReagentTestProps` defined in BOTH `capabilities.tsx` and `reagents.ts` with different shapes
- The `index.ts` re-export from `capabilities.tsx` shadows the other, but direct imports from the sub-files get the wrong type

### useRecorder Error Handling Gap
- All RPC errors silently swallowed with `.catch(() => {})`
- No error state exposed to the component
- Lua returns proper error messages but they never reach React

### Story Registration
- 18 story files exist in storybook/src/stories/ but aren't registered in index.ts
- Always check index.ts when adding a new story
