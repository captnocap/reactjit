**Step-by-Step Trace (layoutNode)**
1. Entry and early exits: `Layout.layoutNode(node, px, py, pw, ph)` begins with a nil guard and logging (lines 554-558). If `display == "none"`, it immediately sets `node.computed = {x=px, y=py, w=0, h=0}` with sources and returns, skipping children (lines 560-568). Non-visual capability nodes and capability nodes that render in their own surface also skip layout and set zero-sized computed rects (lines 570-599). Background effects and mask nodes are similarly skipped (lines 602-620).
2. Percent basis and min/max: `ru = Layout.resolveUnit` is cached (line 622). Percentage-based resolution uses `pctW` and `pctH`, which are taken from `node._parentInnerW/H` if present or fall back to `pw/ph` from the caller, then are cleared (lines 624-631). These bases are used to resolve `minWidth`, `maxWidth`, `minHeight`, `maxHeight` (lines 633-637) and explicit `width` and `height` (lines 639-641). `fit-content` flags are recorded (lines 642-643).
3. Width resolution: `w` is selected in this order: explicit width, `fit-content` via `estimateIntrinsicMain`, parent provided width `pw`, or intrinsic content via `estimateIntrinsicMain` (lines 648-662). The selection is recorded in `wSource` (lines 645-662). The key variables feeding width here are `explicitW`, `fitW`, `pw`, and the output of `estimateIntrinsicMain` (lines 639-662).
4. Height initial resolution: `h` starts as `explicitH` and `hSource` is set if explicit or fit-content; true auto height is deferred until after children are laid out (lines 664-669).
5. Aspect ratio derivation: if `aspectRatio` is set, a missing dimension is derived from the other dimension (explicit width to height, or height to width) and sources are updated to `aspect-ratio` (lines 671-680). Variables involved: `explicitW`, `h`, `ar`.
6. Parent flex overrides: if the parent set `_flexW`, that value overrides `w` and `wSource` becomes `flex` or `root` (lines 683-692). If the parent set `_stretchH`, and `h` is still nil, it becomes the height with source `stretch`, `flex`, or `root` based on `_flexGrowH` or `_rootAutoH` (lines 695-705). The flags `_stretchH`, `_rootAutoH`, `_flexGrowH` are cleared immediately (lines 707-709). These flags are fed into the final child sizes later (lines 1421-1461).
7. Padding resolution: padding is resolved early using `w` and `h` as the percentage bases (`padding`, `paddingLeft`, `paddingRight`, `paddingTop`, `paddingBottom`) so text measurement can use inner width (lines 711-717). The variables `padL`, `padR`, `padT`, `padB` feed text constraints and inner sizes.
8. Intrinsic measurement for this node: if the node is text and lacks explicit width or height, it computes a wrap constraint based on `explicitW` or `pw`, clamps to `maxW` if present, subtracts padding, and calls `measureTextNode` to get `mw/mh` (lines 726-750). It then sets `w = mw + padding` and/or `h = mh + padding` unless a parent assigned width (`parentAssignedW`) or explicit values exist (lines 742-750). CodeBlock and TextInput nodes have special measurement logic for height only (lines 754-776). Capabilities with a `measure` method can also supply an auto height (lines 778-795). These branches call `measureTextNode`, `CodeBlockModule.measure`, `Measure.getFont`, or `capDef.measure` (lines 739-795).
9. Clamping and re-measure: `w` is clamped to `minW/maxW`. If this changes a text node's width and there is no explicit height, the text is re-measured with the new inner constraint and `h` updated accordingly (lines 799-809). `h` is clamped to `minH/maxH` if present (lines 812-815).
10. Margins and inner sizes: margins are resolved against `pw/ph` (lines 819-824). The node origin is `x = px + marL`, `y = py + marT` (lines 826-827). Inner sizes are `innerW = w - padL - padR` and `innerH = (h or 9999) - padT - padB` (lines 828-829). The `9999` fallback means auto-height parents give children a very large available height during pre-measure.
11. Flex context: `isRow`, `gap`, `justify`, `align`, `wrap`, and `mainSize` are computed from style and inner sizes (lines 831-839). These variables control line splitting, flex distribution, and final placement.
12. Child filtering and measurement loop: children are filtered and pre-measured (lines 847-1053). Opaque leaf nodes (Scene3D or Emulator) have no children considered (lines 850-853).
13. Child branch: `display:none` children get `computed = 0` and are skipped (lines 861-863). `position:absolute` children are collected for a later absolute positioning pass (lines 864-866). All other children are visible and measured (lines 868-1053).
14. Child explicit size resolution: for each visible child, `cw` and `ch` are resolved from `cs.width/height` using `innerW/innerH` (lines 870-871). `grow` and `shrink` are read from style, and explicit widths/heights are cached to avoid losing intent if intrinsic estimation produces zero (lines 872-880).
15. Child min/max and padding: `cMinW/cMaxW/cMinH/cMaxH` are resolved from `innerW/innerH` (lines 882-886). Child padding `cpad*` is resolved from `innerW/innerH` for text measurement (lines 888-894).
16. Child text intrinsic measurement: if the child is text and missing `cw` or `ch`, it measures size. For `fit-content` width, it measures unconstrained and adds padding (lines 896-906). Otherwise it uses `outerConstraint = cw or innerW`, clamps to `cMaxW` if no explicit width, subtracts padding, and measures (lines 908-923). The measured `mw/mh` are used to fill `cw/ch` (lines 918-922). Variables feeding size here: `innerW`, `cw`, `cMaxW`, `cpadL/R/T/B`.
17. Child non-text intrinsic estimation: if the child is not text and missing `cw/ch`, it may call `estimateIntrinsicMain` to derive size from its content (lines 926-949). This is skipped for scroll containers and for flex-grow children on the main axis to avoid inflating the basis (lines 934-943). For `fit-content` width, it passes `nil` as the parent width to `estimateIntrinsicMain` (line 944).
18. Child aspect ratio: `cs.aspectRatio` can fill in missing `cw` or `ch` using the explicit dimensions first, or estimated dimensions if both explicit values were absent, with explicit handling to avoid Lua treating `0` as truthy (lines 952-971).
19. Child clamping and re-measure: `cw` is clamped to min/max; if text and width changes with no explicit height, the text is re-measured to update `ch` (lines 974-985). `ch` is clamped afterward (lines 989-992).
20. Child margins and flex basis: margins are resolved, then `mainMarginStart/End` are chosen based on row vs column (lines 994-1009). Flex basis is resolved with priority: `flexBasis` first, including a gap-aware correction for percentage basis in wrapping rows, otherwise it falls back to `cw` or `ch` (lines 1011-1031). If row and no `minWidth`, `computeMinContentW` computes a min-content floor (lines 1033-1039). All of these values are stored into `childInfos` (lines 1042-1051).
21. Line splitting: children are split into flex lines (lines 1056-1110). Without wrap, all visible children go into a single line (lines 1066-1072). With wrap, each item is measured as `max(floor, basis) + margins`, and lines break when `lineMain + gap + itemMain` would exceed `mainSize` (lines 1078-1103). `floor` is `minContent` or `minW` (line 1080).
22. Line totals and free space: for each line, `lineTotalBasis`, `lineTotalFlex`, `lineTotalMarginMain`, `lineGaps`, and `lineAvail` are computed (lines 1120-1145). `lineAvail = mainSize - lineTotalBasis - lineGaps - lineTotalMarginMain` is the free space used for grow or shrink (lines 1144-1145).
23. Flex grow and shrink: if there is positive free space and any grow, each child with `grow > 0` adds a proportional share to its basis (lines 1162-1168). If free space is negative, items shrink proportional to `flexShrink * basis`, with default `flexShrink = 1` when nil (lines 1171-1189). The updated `ci.basis` becomes the final main-axis size input for positioning.
24. Text re-measure after flex: for text nodes that grew and have no explicit height, the code re-measures with the new width. It chooses `finalW` as `ci.basis` in rows, or `ci.w or innerW` in columns, clamps to min/max, and if it changed, measures height with padding removed, clamps, and updates `ci.h` and `ci.w`. In column layouts, it also updates `ci.basis` to the new height (lines 1213-1246).
25. Row-only container re-estimation: in rows, non-text children without explicit height are re-estimated after flex distribution using the new width `ci.basis`, again clamped to min/max, to keep cross sizes accurate for text wrapping inside them (lines 1252-1273).
26. Line cross size: `lineCrossSize` is the maximum cross-axis extent of items in the line, including margins (lines 1276-1291). For nowrap, if the cross axis is definite, `lineCrossSize` is overridden to the full inner cross size (lines 1294-1309).
27. JustifyContent: line main-axis offset and extra gap are computed only if the main axis is definite (`isRow` or `explicitH`), otherwise no justification is applied to avoid huge offsets for auto-sized columns (lines 1312-1345). `lineMainOff` and `lineExtraGap` are then used for positioning.
28. Child positioning and final sizes: for each child, alignment is resolved (`alignSelf` or parent `alignItems`) (lines 1358-1359). In rows, `cx = x + padL + cursor`, `cw_final = ci.basis`, `ch_final = ci.h or lineCrossSize`, then both are clamped (lines 1366-1373). `crossAvail = lineCrossSize - vertical margins` and alignment on the cross axis sets `cy` or stretches `ch_final` when `align == stretch` and `ci.explicitH` is nil (lines 1375-1395). In columns, `cy = y + padT + cursor`, `ch_final = ci.basis`, `cw_final = ci.w or lineCrossSize`, with clamp, cross-availability, and alignment similarly applied (lines 1398-1418). The final child size depends on `ci.basis`, `ci.w`, `ci.h`, `lineCrossSize`, `min/max`, `align`, `crossCursor`, margins, padding, and `cursor`.
29. Parent-to-child size signals: before recursion, the parent sets flags on the child. In rows, if explicit width exists but differs from `cw_final`, `child._flexW` is set; or if no explicit width but `aspectRatio` would otherwise derive a different width, `_flexW` is also set (lines 1424-1435). In columns, if explicit height exists but differs, `_stretchH` is set (lines 1438-1441). For column cross-axis stretch without explicit width, `_flexW` is set (lines 1442-1445). If the child has no explicit height and is not fit-content, `_stretchH` is set for row cross-axis stretch or column flex-grow, and `_flexGrowH` marks column flex grow (lines 1449-1460). These feed the child width/height resolution at the top of `layoutNode` (lines 683-709).
30. Child computed assignment and recursion: this is the critical point requested. The child receives `child.computed = {x=cx, y=cy, w=cw_final, h=ch_final}` (line 1463). Then the parent passes `child._parentInnerW = innerW` and `child._parentInnerH = innerH` to control the child's percentage resolution (lines 1464-1466). The recursive call `Layout.layoutNode(child, cx, cy, cw_final, ch_final)` follows immediately (line 1467).
31. Post-recursion sizing and cursors: the parent uses `child.computed.w/h` (post-recursion) as `actualMainSize` to advance the main-axis cursor by `mainMarginStart + actualMainSize + mainMarginEnd + gap + lineExtraGap` (lines 1469-1479). **Potentially Surprising or Ambiguous Behaviors (documented, not labeled as bugs)**
- Percent-based padding and margins in `computeMinContentW` use a parent size of `0`, so percentage padding resolves to `0` when computing min-content width (lines 315-318). This can make min-content width ignore percent padding.
- When a node has no explicit height, `innerH` uses a `9999` fallback, which can affect percentage heights or child constraints during pre-measure (line 829). The code later guards `justifyContent` to avoid huge offsets (lines 1323-1329), but percent heights computed earlier still use that large value.
- For top-level width resolution, if no explicit width but `pw` is provided, the node width becomes `pw` rather than intrinsic content size (lines 655-658). This means auto width typically fills the parent unless `pw` is nil.
- For text nodes with no explicit width and no parent width (`pw` nil), the wrap constraint becomes `0` (line 729), so text measurement can collapse to padding-only width or a wrapped height depending on `measureText` behavior.
- For flex-grow children, intrinsic measurement of main-axis size is skipped (lines 936-943). This means their initial basis may be `0` if no explicit size or flexBasis is set, which can affect line breaking in wrap mode (lines 1078-1103).
- In nowrap rows, `lineCrossSize` is forced to the full cross size only when height is definite; if height is auto, lineCrossSize stays at the max child size (lines 1294-1309). This makes stretch behavior depend on whether the container height is resolved earlier.
- For absolute children, percentage sizes and offsets resolve against the parent outer `w/h`, not the inner content box, because `ru(cs.width, w)` and `ru(cs.left, w)` are used (lines 1581-1588). This is consistent within this engine but different from the percentage basis used for normal children (lines 624-631 and 1464-1466).
