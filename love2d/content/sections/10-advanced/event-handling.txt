=== OVERVIEW ===
ReactJIT events originate in Lua and flow to React. Love2D owns the event pump -- React
never polls the OS directly. When the user clicks, types, or moves a gamepad stick, Love2D
fires a callback, Lua does hit testing against the retained element tree, and the result is
queued as a bridge event. On the next frame tick, JS polls the queue and dispatches events to
React handlers via the handlerRegistry.

The event lifecycle for a mouse click:

1. Love2D fires love.mousepressed(x, y, button)
2. Lua runs Events.hitTest(root, x, y) -- walks children in reverse paint order (highest
   zIndex first) to find the topmost node with hasHandlers == true
3. Lua builds a bubble path from the hit node up to the root (Events.buildBubblePath)
4. The event is pushed to the bridge event queue with targetId and bubblePath
5. On the next JS poll, eventDispatcher.ts receives the event
6. dispatchWithBubbling walks the bubble path, calling onClick on each handler until
   stopPropagation() is called

Hit testing adjusts coordinates for scroll containers (adding scrollX/scrollY offsets) and
scaleToFit transforms (inverse-transforming mouse position). Nodes are hittable if:
- They have hasHandlers == true (React event handlers attached)
- They have hoverStyle or activeStyle props (Lua-owned visual feedback)
- They have a tooltip prop
- They are scroll containers (for wheel event routing)
- They are Lua-owned interactive types (TextInput, TextEditor, CodeBlock, Video, etc.)
- They are visual capabilities with hittable == true

=== API / SYNTAX ===
Event dispatch has three modes, determined by event type:

Bubbling events (walk bubble path from target to root):
  onClick, onRelease, onKeyDown, onKeyUp, onTextInput (when targetId present),
  onWheel, onTouchStart, onTouchEnd, onDragStart, onDrag, onDragEnd,
  onFileDrop, onDirectoryDrop, onFileDragEnter, onFileDragLeave

Target-only events (dispatch to target node only, no bubbling):
  onPointerEnter, onPointerLeave, onScroll, onLayout, onFocus, onBlur,
  onValueChange (slider/switch/checkbox/radio/select),
  onTextInputFocus, onTextInputBlur, onTextInputSubmit, onTextInputChange,
  onTextEditorFocus, onTextEditorBlur, onTextEditorSubmit, onTextEditorChange,
  onReady, onError, onTimeUpdate, onPlay, onPause, onEnded,
  onContextMenuSelect, onContextMenuOpen, onContextMenuClose

Broadcast events (sent to ALL nodes with the handler, no targeting):
  onKeyDown, onKeyUp, onTextInput (when no targetId -- unfocused keyboard),
  onTouchMove, onGamepadPress, onGamepadRelease, onGamepadAxis,
  onMidiNote, onMidiCC

Capability events:
  A single 'capability' subscription handles all registered capabilities.
  The Lua side specifies which handler to invoke via event.handler (e.g.
  "onProgress", "onTick"). No per-capability wiring is needed.

LoveEvent shape:
| Field           | Type     | Description                                    |
|-----------------|----------|------------------------------------------------|
| targetId        | number   | Node ID of the hit target                      |
| bubblePath      | number[] | Array of node IDs from target to root          |
| x               | number   | Mouse/touch X coordinate (screen space)        |
| y               | number   | Mouse/touch Y coordinate (screen space)        |
| button          | number   | Mouse button (1 = left, 2 = right, 3 = middle) |
| key             | string   | Keyboard key name                              |
| text            | string   | Text input character                           |
| stopPropagation | function | Call to stop bubbling                          |
| currentTarget   | number   | Node ID currently receiving the event          |

=== EXAMPLES ===
Example 1: Basic click handler with bubbling
---
```tsx
import { Box, Pressable, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function ClickExample() {
  const c = useThemeColors();

  return (
    <Box
      style={{ padding: 16 }}
      onClick={(e) => {
        // This fires if the Pressable doesn't stopPropagation
        console.log('Box received bubbled click');
      }}
    >
      <Pressable
        onPress={(e) => {
          e.stopPropagation();
          console.log('Pressable clicked');
        }}
        style={{ padding: 12, backgroundColor: c.primary, borderRadius: 8 }}
      >
        <Text style={{ color: c.bg, fontSize: 14 }}>Click me</Text>
      </Pressable>
    </Box>
  );
}
```

Platforms: love2d

Example 2: Keyboard events (broadcast vs focused)
---
```tsx
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';
import { useState } from 'react';

export function KeyboardExample() {
  const c = useThemeColors();
  const [lastKey, setLastKey] = useState('');

  return (
    <Box
      style={{ padding: 16 }}
      onKeyDown={(e) => {
        // Without a focused node, keyboard events broadcast to ALL handlers.
        // With a focused node, they bubble from the focus target.
        setLastKey(e.key);
      }}
    >
      <Text style={{ color: c.text, fontSize: 16 }}>
        {`Last key: ${lastKey || 'none'}`}
      </Text>
    </Box>
  );
}
```
---
Platforms: love2d

Example 3: Drag events
---
```tsx
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';
import { useState } from 'react';

export function DragExample() {
  const c = useThemeColors();
  const [offset, setOffset] = useState({ x: 100, y: 100 });

  return (
    <Box style={{ width: '100%', height: '100%' }}>
      <Box
        style={{
          position: 'absolute',
          left: offset.x,
          top: offset.y,
          width: 80,
          height: 80,
          backgroundColor: c.primary,
          borderRadius: 8,
        }}
        onDragStart={() => console.log('drag started')}
        onDrag={(e) => setOffset({ x: e.x - 40, y: e.y - 40 })}
        onDragEnd={() => console.log('drag ended')}
      >
        <Text style={{ color: c.bg, fontSize: 12, padding: 8 }}>Drag me</Text>
      </Box>
    </Box>
  );
}
```

=== CRITICAL RULES ===
- React does NOT poll events. Lua owns the event pump and pushes events to a queue. JS polls the queue once per frame via __hostGetEvents().
- Hit testing uses reverse paint order (highest zIndex checked first). If two overlapping nodes both have handlers, the visually topmost one wins.
- Scroll containers adjust hit test coordinates by their scroll offset. Mouse position (100, 300) inside a container scrolled to scrollY=200 tests children at (100, 500) in content space.
- stopPropagation() prevents the event from reaching ancestor handlers. It does not prevent other handlers on the same node from firing.
- To add a new event type: (1) add handler type to BoxProps in types.ts, (2) add to Box destructure in primitives.tsx, (3) add to createElement props in Box(), (4) subscribe in eventDispatcher.ts. Skipping steps 2-3 causes silent handler loss.
- Gamepad and MIDI events always broadcast to all nodes -- there is no focus-based routing for these input types.
