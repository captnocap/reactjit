=== OVERVIEW ===
The session recorder captures raw PTY output with high-resolution timestamps. Recordings
store the raw terminal byte stream, not classified tokens. This means you can replay the
same recording through different classifiers and see different interpretations -- useful
for testing and iterating on classification rules.

Recordings are serialized as Lua source files for zero-dependency loading. The format is
a table with metadata (CLI name, terminal dimensions, timestamp, duration, frame count)
and an array of frames, each containing a timestamp and raw byte data.

The recorder is designed for PTY-based terminal sessions (e.g. ClaudeCanvas). It captures
whatever the PTY emits and replays it at the original timing.

Recording format:
  {
    meta = {
      cli = "claude",           -- CLI tool name
      rows = 40,                -- terminal rows
      cols = 120,               -- terminal columns
      recorded = "2026-...",    -- ISO timestamp
      duration = 45.2,          -- total seconds
      frameCount = 1200,        -- number of frames
    },
    frames = {
      { t = 0.000000, data = "raw bytes..." },
      { t = 0.016667, data = "more bytes..." },
      ...
    },
  }

=== API / SYNTAX ===
Lua API:
```lua
local Recorder = require("lua.session_recorder")
```

Recorder.new(opts) -> recorder:
| Opt  | Type   | Default   | Description                |
|------|--------|-----------|----------------------------|
| cli  | string | "unknown" | CLI tool name for metadata |
| rows | number | 40        | Terminal rows              |
| cols | number | 120       | Terminal columns           |

Recorder instance methods:
| Method          | Description                                        |
|-----------------|----------------------------------------------------|
| recorder:start()     | Begin recording (auto-called on first capture) |
| recorder:stop()      | Stop recording, finalize metadata              |
| recorder:capture(data) | Record a chunk of PTY output with timestamp  |
| recorder:export()    | Return the recording table without saving      |
| recorder:save(path)  | Serialize and write to a .rec.lua file         |

Recorder.load(path) -> recording, err:
  Load a previously saved recording from a Lua source file.

=== EXAMPLES ===
Example 1: Recording a PTY session
---
```lua
local Recorder = require("lua.session_recorder")

-- Create a recorder for a Claude session
local rec = Recorder.new({ cli = "claude", rows = 40, cols = 120 })

-- In the PTY read callback:
function onPtyOutput(data)
  rec:capture(data)  -- auto-starts on first call
end

-- When the session ends:
rec:save("session.rec.lua")
```
---
Platforms: love2d

Example 2: Loading and inspecting a recording
---
```lua
local Recorder = require("lua.session_recorder")

local recording, err = Recorder.load("session.rec.lua")
if not recording then
  print("Error: " .. err)
  return
end

print("CLI: " .. recording.meta.cli)
print("Duration: " .. string.format("%.1fs", recording.meta.duration))
print("Frames: " .. recording.meta.frameCount)
print("Terminal: " .. recording.meta.cols .. "x" .. recording.meta.rows)

-- Iterate frames
for i, frame in ipairs(recording.frames) do
  print(string.format("  [%.3fs] %d bytes", frame.t, #frame.data))
end
```
---
Platforms: love2d

Example 3: Replaying a recording with timing
---
```lua
local Recorder = require("lua.session_recorder")

local recording = Recorder.load("session.rec.lua")
local frameIndex = 1
local elapsed = 0

function love.update(dt)
  elapsed = elapsed + dt
  while frameIndex <= #recording.frames do
    local frame = recording.frames[frameIndex]
    if frame.t <= elapsed then
      -- Feed the frame data to the terminal emulator
      vterm:write(frame.data)
      frameIndex = frameIndex + 1
    else
      break
    end
  end
end
```
---
Platforms: love2d

=== CRITICAL RULES ===
- Recordings store raw PTY bytes, not classified output. Classification happens at playback time.
- The .rec.lua format is valid Lua source. It uses string.format %q for safe byte escaping of binary PTY data.
- Recordings can be large (megabytes for long sessions). The Lua serializer handles this but load time scales linearly with frame count.
- The time source is love.timer.getTime() when available, falling back to os.clock(). All timestamps are relative to the recording start.
