=== OVERVIEW ===
ReactJIT has a Lua-owned focus system that supports both mouse and controller input modes.
Focus determines which node receives keyboard events and is visually indicated by an animated
focus ring.

The system supports two input modes:
- Mouse mode (default) -- focus follows clicks. No visible focus ring. Keyboard events
  without a targetId broadcast to all handlers.
- Controller mode -- activated automatically when a gamepad input is detected. An animated
  focus ring appears around the focused node. D-pad or left stick navigates spatially between
  focusable nodes. The system remembers the last focused node and restores it when switching
  back to controller mode.

A node is focusable if:
- It has focusable={true} in its props
- It has React event handlers (hasHandlers == true)
- It is a hittable capability (e.g. ClaudeCanvas)
- It does NOT have focusable={false}, display="none", or visibility="hidden"

FocusGroup scopes focus navigation to a region of the tree. Each group maintains its own
focused node, focusable node list, and optional controller binding. This enables split-screen
or multi-panel layouts where each controller navigates independently.

When focus changes to a node inside a ScrollView, the system automatically scrolls the
container to make the focused node visible.

=== API / SYNTAX ===
```tsx
import { FocusGroup } from '@reactjit/core';
```

FocusGroup Props:
| Prop       | Type                                   | Default       | Description                                          |
|------------|----------------------------------------|---------------|------------------------------------------------------|
| controller | number                                 | undefined     | Joystick ID that owns this group. Omit for any.      |
| ringColor  | [number, number, number, number?]      | player-based  | Focus ring RGBA (0-1). Default: P1 blue, P2 red.     |
| style      | Style                                  | undefined     | Style applied to the wrapping Box.                   |
| children   | ReactNode                              | required      | Content within the focus scope.                      |

Default player colors:
  P1: blue [0.3, 0.6, 1.0, 0.9]
  P2: red [1.0, 0.3, 0.3, 0.9]
  P3: green [0.3, 1.0, 0.5, 0.9]
  P4: yellow [1.0, 0.8, 0.2, 0.9]

Input keystrokeTarget/submitTarget (on Input component):
| Prop            | Type   | Description                                                    |
|-----------------|--------|----------------------------------------------------------------|
| keystrokeTarget | string | Node type to forward raw keystrokes to (Lua-to-Lua, no bridge)|
| submitTarget    | string | Node type to forward submitted text to on Enter (clears input)|
| escapeTarget    | string | Node type to forward Escape key to (Lua-to-Lua, no bridge)   |

Spatial navigation constants:
  Stick deadzone: 0.3
  Initial repeat delay: 0.4 seconds
  Repeat rate: 0.15 seconds

=== EXAMPLES ===
Example 1: Basic FocusGroup for a sidebar
---
```tsx
import { Box, FocusGroup, Pressable, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function SidebarNav() {
  const c = useThemeColors();
  const items = ['Home', 'Settings', 'About'];

  return (
    <FocusGroup style={{ width: 200, padding: 8 }}>
      {items.map((label) => (
        <Pressable
          key={label}
          onPress={() => console.log(label)}
          style={{ padding: 12, borderRadius: 6, backgroundColor: c.surface }}
        >
          <Text style={{ color: c.text, fontSize: 14 }}>{label}</Text>
        </Pressable>
      ))}
    </FocusGroup>
  );
}
```

Example 2: Split-screen with per-controller focus
---
```tsx
import { Box, FocusGroup, Pressable, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function SplitScreen() {
  const c = useThemeColors();

  return (
    <Box style={{ flexDirection: 'row', width: '100%', height: '100%' }}>
      <FocusGroup
        controller={1}
        ringColor={[0.3, 0.6, 1.0, 0.9]}
        style={{ flexGrow: 1, padding: 16 }}
      >
        <Text style={{ color: c.text, fontSize: 18 }}>Player 1</Text>
        <Pressable
          onPress={() => console.log('P1 action')}
          style={{ padding: 12, backgroundColor: c.surface, borderRadius: 8 }}
        >
          <Text style={{ color: c.text, fontSize: 14 }}>Action</Text>
        </Pressable>
      </FocusGroup>

      <FocusGroup
        controller={2}
        ringColor={[1.0, 0.3, 0.3, 0.9]}
        style={{ flexGrow: 1, padding: 16 }}
      >
        <Text style={{ color: c.text, fontSize: 18 }}>Player 2</Text>
        <Pressable
          onPress={() => console.log('P2 action')}
          style={{ padding: 12, backgroundColor: c.surface, borderRadius: 8 }}
        >
          <Text style={{ color: c.text, fontSize: 14 }}>Action</Text>
        </Pressable>
      </FocusGroup>
    </Box>
  );
}
```

Example 3: Input with keystrokeTarget forwarding
---
```tsx
import { Box, Input, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function ProxyInput() {
  const c = useThemeColors();

  // Keystrokes are forwarded to ClaudeCanvas at the Lua level.
  // The Input does not manage its own text buffer -- it displays
  // whatever value is passed via the value prop.
  return (
    <Box style={{ padding: 16 }}>
      <Input
        autoFocus
        keystrokeTarget="ClaudeCanvas"
        submitTarget="ClaudeCanvas"
        value=""
        placeholder="Type here..."
        style={{ backgroundColor: c.surface, color: c.text, fontSize: 14 }}
      />
    </Box>
  );
}
```

=== CRITICAL RULES ===
- FocusGroup creates a focus scope. Nodes inside a FocusGroup are navigated independently from nodes outside it.
- Controller mode activates automatically on gamepad input. Mouse movement switches back to mouse mode.
- Spatial navigation uses the node's computed bounding box to find the nearest focusable node in the pressed direction (up/down/left/right).
- keystrokeTarget routes keystrokes at the Lua level -- they never cross the bridge to JS. This is for proxy input patterns where the Input component is just a display.
- When a focusable node is inside a ScrollView and receives focus, the ScrollView scrolls to make it visible. This happens automatically -- no manual scrolling code is needed.
- The focusable prop explicitly opts a node in (true) or out (false) of the focusable set, overriding the default behavior based on hasHandlers.
