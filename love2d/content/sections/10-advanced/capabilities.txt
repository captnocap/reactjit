=== OVERVIEW ===
The capability system is how new features are added to ReactJIT. Instead of extending the
reconciler or adding new host element types, you register a capability in Lua and use it
as a component in React. The bridge handles the lifecycle automatically.

The pattern:

1. Lua registers a capability with Capabilities.register(typeName, definition)
2. React renders <Native type={typeName} {...props} />
3. Lua detects the new node in its per-frame tree sync, calls create()
4. When props change, Lua calls update()
5. Every frame, Lua calls tick() for animation, polling, or timed events
6. When React unmounts the component, Lua calls destroy()

Capabilities can be visual (they have a painted representation on screen) or non-visual
(they perform background work like audio playback or timers). Visual capabilities
participate in layout and painting. Non-visual capabilities skip both.

The schema property serves as documentation. It describes the props the capability
accepts, their types, defaults, and valid ranges. The useCapabilities() hook on the React
side queries all registered schemas via RPC -- an AI agent calls this once to know what
capabilities are available, then generates the JSX.

=== API / SYNTAX ===
Lua registration:
```lua
local Capabilities = require("lua.capabilities")

Capabilities.register("TypeName", {
  visual = false,        -- true if it paints to screen
  hittable = false,      -- true if it receives focus/keyboard events (visual only)
  rendersInOwnSurface = false,  -- true if it renders in a separate window

  schema = {
    propName = {
      type = "string",   -- "string", "number", "bool", "table"
      desc = "Description",
      default = "value",
      min = 0,           -- number constraints (optional)
      max = 100,
      values = { "a", "b" },  -- enum constraints (optional)
    },
  },

  events = { "onComplete", "onProgress" },

  create  = function(nodeId, props) return initialState end,
  update  = function(nodeId, props, prevProps, state) end,
  tick    = function(nodeId, state, dt, pushEvent, props) end,
  destroy = function(nodeId, state) end,
})
```

React consumption:
```tsx
import { Native } from '@reactjit/core';

function MyCapability(props) {
  return <Native type="TypeName" {...props} />;
}
```

React discovery:
```tsx
import { useCapabilities } from '@reactjit/core';
```

useCapabilities() return value:
| Field        | Type                                    | Description                        |
|--------------|-----------------------------------------|------------------------------------|
| capabilities | Record<string, CapabilitySchema> \| null | All registered capability schemas |
| loading      | boolean                                 | True while the RPC is in flight   |

CapabilitySchema:
| Field  | Type                          | Description                          |
|--------|-------------------------------|--------------------------------------|
| schema | Record<string, PropSchema>    | Prop definitions with types/defaults |
| events | string[]                      | Event handler names                  |
| visual | boolean                       | Whether it renders visually          |

Lifecycle methods:
| Method  | Called when                          | Arguments                          | Returns            |
|---------|--------------------------------------|------------------------------------|--------------------|
| create  | Node first appears in the tree       | nodeId, props                      | initial state table|
| update  | Props change (deep comparison)       | nodeId, newProps, prevProps, state  | void               |
| tick    | Every frame                          | nodeId, state, dt, pushEvent, props| void               |
| destroy | Node removed from the tree           | nodeId, state                      | void               |

=== EXAMPLES ===
Example 1: Registering a visual capability
---
```lua
-- lua/capabilities/health_bar.lua
local Capabilities = require("lua.capabilities")

Capabilities.register("HealthBar", {
  visual = true,

  schema = {
    value    = { type = "number", min = 0, max = 1, default = 1, desc = "Fill 0-1" },
    barColor = { type = "string", default = "#22c55e", desc = "Bar fill color" },
    bgColor  = { type = "string", default = "#1a1a2e", desc = "Bar background" },
  },

  events = {},

  create = function(nodeId, props)
    return { displayValue = props.value or 1 }
  end,

  update = function(nodeId, props, prev, state)
    -- Smoothly animate toward target value
    state.targetValue = props.value or 1
  end,

  tick = function(nodeId, state, dt, pushEvent, props)
    if state.targetValue then
      local speed = 3.0
      state.displayValue = state.displayValue +
        (state.targetValue - state.displayValue) * math.min(1, speed * dt)
    end
  end,

  paint = function(nodeId, state, node)
    local c = node.computed
    -- Background
    love.graphics.setColor(0.1, 0.1, 0.18, 1)
    love.graphics.rectangle("fill", c.x, c.y, c.w, c.h, 4, 4)
    -- Fill
    love.graphics.setColor(0.13, 0.77, 0.37, 1)
    love.graphics.rectangle("fill", c.x, c.y, c.w * state.displayValue, c.h, 4, 4)
  end,

  destroy = function(nodeId, state) end,
})
```

Example 2: Using useCapabilities for AI discovery
---
```tsx
import { useCapabilities } from '@reactjit/core';
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function CapabilityBrowser() {
  const c = useThemeColors();
  const { capabilities, loading } = useCapabilities();

  if (loading) {
    return <Text style={{ color: c.muted, fontSize: 14 }}>Loading...</Text>;
  }

  if (!capabilities) {
    return <Text style={{ color: c.muted, fontSize: 14 }}>No capabilities</Text>;
  }

  return (
    <Box style={{ padding: 16, gap: 8 }}>
      {Object.entries(capabilities).map(([name, cap]) => (
        <Box key={name} style={{ padding: 8, backgroundColor: c.surface, borderRadius: 6 }}>
          <Text style={{ color: c.text, fontSize: 14, fontWeight: 'bold' }}>{name}</Text>
          <Text style={{ color: c.muted, fontSize: 12 }}>
            {`Props: ${Object.keys(cap.schema).join(', ')}`}
          </Text>
          <Text style={{ color: c.muted, fontSize: 12 }}>
            {`Events: ${cap.events.join(', ') || 'none'}`}
          </Text>
        </Box>
      ))}
    </Box>
  );
}
```

Example 3: Pushing events from a capability
---
```lua
-- In the tick function of a capability:
tick = function(nodeId, state, dt, pushEvent, props)
  state.timer = (state.timer or 0) + dt
  if state.timer >= (props.interval or 1) then
    state.timer = state.timer - (props.interval or 1)

    -- Push an event to the React handler.
    -- type must be "capability", handler is the React prop name.
    pushEvent({
      type = "capability",
      payload = {
        type = "capability",
        targetId = nodeId,
        handler = "onTick",      -- matches the React prop name
        count = state.count,     -- custom data in the payload
      },
    })

    state.count = (state.count or 0) + 1
  end
end,
```

React receives this as:
```tsx
<Timer interval={1000} onTick={(e) => console.log(e.count)} />
```

=== CRITICAL RULES ===
- Capability files in lua/capabilities/ are loaded automatically. Place new files there -- no manual require is needed.
- The schema is documentation only -- it is not validated at runtime. Props arrive as-is from the React tree.
- The tick function runs every frame. Keep it fast. If you need expensive computation, spread it across frames using coroutines or frame budgets.
- pushEvent must use type: "capability" for the event to be routed correctly. The handler field determines which React prop receives it.
- Deep prop comparison (not reference equality) is used to detect changes. If you pass the same values, update will not be called.
- Visual capabilities participate in layout. Their node.computed gives the resolved {x, y, w, h} for painting.
- Non-visual capabilities are skipped by both layout and painter. They exist only for lifecycle management.
