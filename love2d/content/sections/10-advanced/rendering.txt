=== OVERVIEW ===
The Render component displays live external video sources inside the ReactJIT layout tree.
It supports screen capture, webcams, HDMI capture cards, window capture, virtual displays,
and full virtual machines -- all as a single declarative component.

The source prop determines what is captured:
- "screen:N" -- capture screen N via XShm (fast path, Linux only)
- "cam:N" -- webcam N via v4l2
- "hdmi:N" -- HDMI capture card N
- "window:Title" -- capture a specific window by its title
- "/dev/videoN" -- direct v4l2 device path
- "display" -- virtual display (Xephyr/Xvfb) where apps render into
- "file.iso" -- boot a virtual machine from an ISO file
- "vm:path.qcow2" -- boot a VM from a QEMU disk image

The Lua painter renders the live capture image into the node's computed bounds, applying
objectFit scaling (contain, cover, or fill). When interactive is true, mouse and keyboard
events within the node's bounds are forwarded to the capture source.

=== API / SYNTAX ===
```tsx
import { Render } from '@reactjit/core';
```

Props:
| Prop        | Type                           | Default   | Description                                          |
|-------------|--------------------------------|-----------|------------------------------------------------------|
| source      | string                         | required  | Source identifier (see overview for formats)          |
| fps         | number                         | 30        | Capture framerate                                    |
| resolution  | string                         | '1280x720'| Capture resolution (e.g. '1920x1080')                |
| interactive | boolean                        | varies    | Forward mouse/keyboard to source. Default false for capture, true for display/vm. |
| muted       | boolean                        | true      | Suppress audio from source                           |
| objectFit   | 'fill' \| 'contain' \| 'cover' | 'contain' | How the capture fits the node bounds                 |
| vmMemory    | number                         | 2048      | VM RAM in MB (ISO/QCOW2 sources only)                |
| vmCpus      | number                         | 2         | VM CPU count (ISO/QCOW2 sources only)                |
| w           | number/string                  | undefined | Shorthand for style.width                            |
| h           | number/string                  | undefined | Shorthand for style.height                           |
| radius      | number                         | undefined | Shorthand for style.borderRadius                     |
| style       | Style                          | undefined | Layout style                                         |
| onReady     | (event) => void                | undefined | Fires when the source is connected                   |
| onError     | (event) => void                | undefined | Fires on capture error                               |
| onFrame     | (event) => void                | undefined | Fires on each captured frame                         |
| onClick     | (event) => void                | undefined | Fires on click within the render bounds              |

=== EXAMPLES ===
Example 1: Webcam preview
---
```tsx
import { Box, Render } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function WebcamPreview() {
  const c = useThemeColors();

  return (
    <Box style={{ width: '100%', height: '100%', backgroundColor: c.bg }}>
      <Render
        source="cam:0"
        fps={30}
        resolution="640x480"
        style={{ flexGrow: 1, borderRadius: 8 }}
      />
    </Box>
  );
}
```
---
Platforms: love2d

Example 2: Screen capture with status callback
---
```tsx
import { Box, Render, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';
import { useState } from 'react';

export function ScreenCapture() {
  const c = useThemeColors();
  const [status, setStatus] = useState('connecting...');

  return (
    <Box style={{ width: '100%', height: '100%' }}>
      <Text style={{ color: c.muted, fontSize: 12, padding: 4 }}>{status}</Text>
      <Render
        source="screen:0"
        fps={60}
        onReady={() => setStatus('capturing')}
        onError={(e) => setStatus(`error: ${e.message}`)}
        style={{ flexGrow: 1 }}
      />
    </Box>
  );
}
```
---
Platforms: love2d

Example 3: Interactive virtual machine
---
```tsx
import { Box, Render, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function VMDisplay() {
  const c = useThemeColors();

  return (
    <Box style={{ width: '100%', height: '100%' }}>
      <Text style={{ color: c.text, fontSize: 14, padding: 8 }}>
        Ubuntu VM
      </Text>
      <Render
        source="vm:ubuntu.qcow2"
        interactive
        vmMemory={4096}
        vmCpus={4}
        objectFit="contain"
        style={{ flexGrow: 1 }}
      />
    </Box>
  );
}
```

Platforms: love2d

=== CRITICAL RULES ===
- Render is a surface node. Without explicit dimensions, it falls back to 1/4 of the parent's available space. Use flexGrow: 1 to fill available space.
- The objectFit prop controls scaling. "contain" preserves aspect ratio within bounds (letterboxing). "cover" fills bounds and clips. "fill" stretches to fit.
- VM sources (file.iso, vm:*.qcow2) require QEMU to be installed. The capability launches a QEMU process and connects its display to the node.
- interactive mode forwards all mouse and keyboard events to the source. The node consumes those events -- they do not bubble to parent React handlers.
