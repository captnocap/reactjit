=== OVERVIEW ===
ReactJIT provides several debugging tools that work entirely within the Love2D runtime --
there is no browser devtools to fall back on.

The F12 inspector is the primary debugging tool. Press F12 to open the devtools panel.
Hovering over the canvas highlights nodes with colored overlays showing margin (orange),
padding (green), and content (blue) areas. Clicking a node selects it and shows a detail
panel with computed styles, dimensions, and parent/child links.

The useDebug hook exposes component state to the inspector console. Any data registered
with useDebug is accessible via eval expressions and watch commands.

The DebugOverlay component wraps a subtree and adds colored borders to every Box, making
layout boundaries visible without the full inspector.

The error overlay appears automatically when React throws during render or when a Lua
error occurs. It shows the error message and stack trace as a full-screen overlay. Errors
are non-fatal -- the overlay can be dismissed and the app continues running.

=== API / SYNTAX ===
```tsx
import { useDebug, DebugOverlay, DebugBox } from '@reactjit/core';
```

useDebug(key, data):
| Param | Type   | Description                                              |
|-------|--------|----------------------------------------------------------|
| key   | string | Unique identifier (usually the component name)           |
| data  | any    | Any data to expose. Updated every render, cleaned on unmount. |

Registered data is accessible on globalThis.__debug[key].

DebugOverlay Props:
| Prop        | Type      | Default                    | Description                        |
|-------------|-----------|----------------------------|------------------------------------|
| enabled     | boolean   | true                       | Toggle the overlay on/off          |
| borderColor | string    | 'rgba(255, 100, 100, 0.6)' | Border color for debug outlines   |
| showLabels  | boolean   | false                      | Show dimension labels on each Box  |
| children    | ReactNode | required                   | Content to visualize               |

registerDebug(key, data) -> unregister:
  Imperative version for non-React code (game loops, services).
  Returns an unregister function.

getDebugData() -> Record<string, any>:
  Returns all currently registered debug data.

=== EXAMPLES ===
Example 1: useDebug for component state inspection
---
```tsx
import { Box, Text } from '@reactjit/core';
import { useDebug } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';
import { useState } from 'react';

function PlayerHUD({ health, mana }: { health: number; mana: number }) {
  const c = useThemeColors();

  // Expose state to the inspector console:
  //   > __debug['PlayerHUD']
  //   > __debug['PlayerHUD'].health
  //   :watch __debug['PlayerHUD'].health
  useDebug('PlayerHUD', { health, mana });

  return (
    <Box style={{ padding: 8 }}>
      <Text style={{ color: c.text, fontSize: 14 }}>{`HP: ${health}`}</Text>
      <Text style={{ color: c.text, fontSize: 14 }}>{`MP: ${mana}`}</Text>
    </Box>
  );
}
```

Example 2: DebugOverlay for layout visualization
---
```tsx
import { Box, Text, DebugOverlay } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

function LayoutDebug() {
  const c = useThemeColors();

  return (
    <DebugOverlay borderColor="rgba(100, 200, 255, 0.5)" showLabels>
      <Box style={{ flexDirection: 'row', width: '100%', gap: 8, padding: 16 }}>
        <Box style={{ flexGrow: 1, padding: 8, backgroundColor: c.surface }}>
          <Text style={{ color: c.text, fontSize: 14 }}>Left panel</Text>
        </Box>
        <Box style={{ flexGrow: 2, padding: 8, backgroundColor: c.surface }}>
          <Text style={{ color: c.text, fontSize: 14 }}>Right panel</Text>
        </Box>
      </Box>
    </DebugOverlay>
  );
}
```

Example 3: Imperative debug registration
---
```tsx
import { registerDebug } from '@reactjit/core';

// Register debug data from a non-React game loop
const unregister = registerDebug('GameEngine', {
  fps: 60,
  entityCount: 150,
  activeCollisions: 3,
});

// Update it periodically
globalThis.__debug['GameEngine'].fps = currentFps;

// Clean up when done
unregister();
```

=== PLATFORM NOTES ===
Love2D:
  The F12 inspector is a Lua-only module (lua/inspector.lua). It draws directly via
  Love2D graphics calls and does not touch the React tree. When disabled, it costs
  one boolean check per frame per hook.

  The inspector has three view modes (toggled in the tree panel):
  - Lua: shows the raw Lua node types (View, Text, Image)
  - Hybrid: shows both Lua types and React component names (debugName)
  - React: shows React component names only

  Console commands (press ` to open):
    Any JavaScript expression -- evaluated in the QuickJS context
    :watch <expr> -- add a live watch expression (updated every frame)
    :unwatch <index> -- remove a watch expression
    :clear -- clear the console output

=== CRITICAL RULES ===
- useDebug data updates every render. Do not pass expensive-to-compute objects unless the component already has that data. The hook does not memoize.
- DebugOverlay adds borderWidth: 1 to every Box in its subtree. This changes layout (by 2px per axis). Remove it when done debugging.
- The error overlay is dismissable but errors are also logged to stdout. Check the terminal for the full stack trace.
- registerDebug returns an unregister function. Call it to prevent memory leaks when the data source is removed.
