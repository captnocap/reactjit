=== OVERVIEW ===
ReactJIT renders at 60fps. React's job is to declare layout and diff the tree -- the less
React work per frame, the better. The most common performance problems come from React
re-rendering unnecessarily, which causes mutation commands to cross the bridge and triggers
Lua layout recalculation.

The golden rule: minimize the number of mutation commands per frame. Every prop change that
crosses the bridge causes the Lua tree to mark itself dirty and re-run layout. If nothing
changed, zero commands are sent and layout is skipped entirely.

=== EXAMPLES ===
Example 1: Hoist styles outside the component
---
```tsx
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

// BAD: creates a new object every render -> identity changes -> bridge update
function BadCard() {
  const c = useThemeColors();
  return (
    <Box style={{ padding: 16, backgroundColor: c.surface, borderRadius: 8 }}>
      <Text style={{ fontSize: 14, color: c.text }}>Content</Text>
    </Box>
  );
}

// GOOD: style objects are stable references (same object each render)
const styles = {
  card: { padding: 16, borderRadius: 8 } as const,
  text: { fontSize: 14 } as const,
};

function GoodCard() {
  const c = useThemeColors();
  // Only theme-dependent styles are inline
  return (
    <Box style={{ ...styles.card, backgroundColor: c.surface }}>
      <Text style={{ ...styles.text, color: c.text }}>Content</Text>
    </Box>
  );
}
```

Example 2: Static string constants for Text children
---
```tsx
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

// BAD: template literal in render body creates a new string every frame
function BadScore({ score }: { score: number }) {
  const c = useThemeColors();
  return <Text style={{ color: c.text, fontSize: 16 }}>{`Score: ${score}`}</Text>;
}

// GOOD for static text: hoist the string
const TITLE = 'Game Over';

function GoodTitle() {
  const c = useThemeColors();
  return <Text style={{ color: c.text, fontSize: 24 }}>{TITLE}</Text>;
}

// Template literals for dynamic text are fine -- they must be template literals
// (not mixed children). The key optimization is hoisting STATIC text.
function ScoreDisplay({ score }: { score: number }) {
  const c = useThemeColors();
  return <Text style={{ color: c.text, fontSize: 16 }}>{`Score: ${score}`}</Text>;
}
```

Platforms: love2d

Example 3: useMemo for derived data
---
```tsx
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';
import { useMemo } from 'react';

interface DataRow {
  name: string;
  value: number;
}

function DataList({ rows, filter }: { rows: DataRow[]; filter: string }) {
  const c = useThemeColors();

  // Memoize expensive filtering so it only re-runs when inputs change
  const filtered = useMemo(
    () => rows.filter((r) => r.name.includes(filter)),
    [rows, filter]
  );

  return (
    <Box style={{ padding: 8 }}>
      {filtered.map((row) => (
        <Text key={row.name} style={{ color: c.text, fontSize: 14 }}>
          {`${row.name}: ${row.value}`}
        </Text>
      ))}
    </Box>
  );
}
```

Example 4: The theme cache bug pattern
---
```tsx
import { useThemeColors } from '@reactjit/theme';

// BAD: module-level object captures theme values at eval time.
// When applyTheme() changes colors, these go STALE.
const COLORS = {
  accent: '#3b82f6',  // captured once at module load
  bg: '#0a0a0f',
};

// GOOD: use functions that read theme values at call time
function getColors() {
  const c = useThemeColors();
  return { accent: c.primary, bg: c.bg };
}

// Or better: just call useThemeColors() inside the component
function ThemedComponent() {
  const c = useThemeColors();
  return (
    <Box style={{ backgroundColor: c.bg }}>
      <Text style={{ color: c.primary, fontSize: 16 }}>Themed</Text>
    </Box>
  );
}
```
---
Platforms: love2d

=== CRITICAL RULES ===
- Never create objects or arrays inside a render function body if they can be hoisted. Each new identity triggers a bridge UPDATE command.
- Never wrap a Lua-owned primitive (CodeBlock, TextEditor) in a React component that re-renders every frame without memoizing its props. New string/object identities per frame cause downstream cache invalidation at 60fps.
- Module-level const objects that reference theme colors (C.accent, C.primary) will go stale after applyTheme(). Use functions or read colors inside components.
- Use React.memo() on components that receive the same props most frames. This prevents the reconciler from even diffing the subtree.
- The inspector (F12) shows layout and paint timing per frame. If layoutMs is consistently above 2ms, look for unnecessary tree mutations.
- Avoid useEffect with empty deps that sets up high-frequency intervals. Use useLuaInterval (with unique tick values) to keep timer logic in Lua.
