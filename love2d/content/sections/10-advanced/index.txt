=== OVERVIEW ===
This section covers everything beyond basic component usage. It is intended for developers
who are comfortable with ReactJIT's primitives and layout system and want to understand the
internals, extend the framework, or optimize their applications.

Topics covered:

- Event handling -- how pointer, keyboard, gamepad, and drag events flow from Love2D through
  Lua hit testing and the bridge into React handlers. Bubbling, target-only dispatch, and
  broadcast semantics.

- Focus management -- the FocusGroup component, controller-based spatial navigation,
  keystrokeTarget/submitTarget for Lua-routed input, and scroll-into-view behavior.

- Lua runtime -- LuaJIT, the QuickJS FFI bridge, how to add new Lua modules, and the
  source-of-truth sync pipeline.

- Performance -- avoiding unnecessary React re-renders, hoisting styles, using useMemo,
  static string constants for Text children, and the module-level cache bug pattern.

- Debugging -- the F12 inspector, useDebug hook, DebugOverlay component, error overlays,
  and the console with watch expressions.

- DevTools -- the unified Chrome-style bottom panel with Elements, Console, Network,
  Performance, Wireframe, Logs, and Source tabs. Pop-out mode and DevToolsEmbed.

- Custom rendering -- the Render component for screen capture, webcam, HDMI, and VM
  display. Raw Love2D drawing via paint callbacks.

- Capabilities -- the declarative capability system for adding new Lua-side features
  (audio, timers, GPIO, etc.) with automatic React component wrappers and AI discovery.

- Session recording -- capturing and replaying PTY terminal sessions for testing and
  debugging classifier behavior.
