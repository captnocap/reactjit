=== OVERVIEW ===
`@reactjit/audio` provides a modular audio synthesis and sequencing framework. The audio engine runs entirely in Lua (`lua/audio/`), processing audio at native sample rates. React hooks communicate with the engine via bridge RPCs for commands and bridge events for state updates.

The architecture follows a rack/module/port model:

- A **rack** holds modules and connections between them.
- A **module** is a self-contained audio unit (oscillator, filter, sampler, mixer, clock, sequencer).
- **Ports** are typed connection points (audio, control, midi) on each module.
- **Connections** route signals between module ports.

React's role is purely declarative: manage which modules exist, set parameters, and render UI for the state the Lua engine pushes back each frame.

=== API / SYNTAX ===
```tsx
import {
  useRack, useModule, useParam, useMIDI, useMIDINote, useMIDICC,
  useAudioInit, useClock, useClockEvent, useSampler, useRecorder, useSequencer,
} from '@reactjit/audio';
```

---- useAudioInit()

Initialize the audio engine. Call once at app startup. Returns `true` when the engine is ready.

```tsx
const audioReady = useAudioInit();
```

---- useRack(options?)

Access the full audio rack: list modules, manage connections.

Options:
| Option       | Type    | Default | Description                                    |
|--------------|---------|---------|------------------------------------------------|
| topologyOnly | boolean | false   | Only re-render on module/connection changes    |
| maxFps       | number  | unlimited | Cap state update rate                        |

Returns UseRackResult:
| Field        | Type                                              | Description            |
|--------------|---------------------------------------------------|------------------------|
| modules      | ModuleState[]                                     | All modules in rack    |
| connections  | Connection[]                                      | All connections        |
| addModule    | (type, id, params?) => Promise                    | Add a module           |
| removeModule | (id) => Promise                                   | Remove a module        |
| connect      | (fromId, fromPort, toId, toPort) => Promise       | Connect two ports      |
| disconnect   | (fromId, fromPort, toId, toPort) => Promise       | Disconnect ports       |

---- useModule(moduleId)

Access a specific module's parameters and state.

Returns UseModuleResult:
| Field       | Type                                    | Description              |
|-------------|-----------------------------------------|--------------------------|
| id          | string                                  | Module ID                |
| type        | string                                  | Module type              |
| params      | Record<string, any>                     | Current parameters       |
| ports       | Record<string, { type, direction }>     | Port definitions         |
| activeNotes | Record<string, { note, envelope }>      | Active note states       |
| setParam    | (name, value) => Promise                | Set a parameter          |

---- useParam(moduleId, paramName)

Read and write a single parameter on a module. Returns `[value, setValue]` tuple.

```tsx
const [cutoff, setCutoff] = useParam('filter1', 'cutoff');
setCutoff(800);
```

---- useMIDI()

Access MIDI devices, CC mappings, and learn mode.

Returns UseMIDIResult:
| Field    | Type                                 | Description                         |
|----------|--------------------------------------|-------------------------------------|
| available| boolean                              | MIDI system detected                |
| devices  | MIDIDevice[]                         | Connected devices                   |
| mappings | MIDIMapping[]                        | Active CC-to-param mappings         |
| learning | { moduleId, param } \| null          | Currently learning target           |
| learn    | (moduleId, param) => Promise         | Start MIDI learn for a param        |
| map      | (moduleId, param, ch, cc) => Promise | Map CC to param explicitly          |
| unmap    | (moduleId, param) => Promise         | Remove a mapping                    |

---- useMIDINote(handler)

Subscribe to MIDI note on/off events.

```tsx
useMIDINote((event) => {
  // event: { note: number, velocity: number, on: boolean, channel: number, device: string }
});
```

---- useMIDICC(handler)

Subscribe to MIDI CC events.

```tsx
useMIDICC((event) => {
  // event: { cc: number, value: number, channel: number, device: string }
});
```

---- useClock(moduleId)

Access a clock module's position and transport controls.

Returns UseClockResult:
| Field       | Type                    | Description                    |
|-------------|-------------------------|--------------------------------|
| beat        | number                  | Current beat                   |
| bar         | number                  | Current bar                    |
| step        | number                  | Current step                   |
| phase       | number                  | Continuous phase (0-1)         |
| running     | boolean                 | Transport running              |
| bpm         | number                  | Current BPM                    |
| start       | () => Promise           | Start transport                |
| stop        | () => Promise           | Stop transport                 |
| setBpm      | (bpm) => Promise        | Set BPM                        |
| setDivision | (division) => Promise   | Set step division              |
| setSwing    | (swing) => Promise      | Set swing amount               |

---- useClockEvent(handler)

Subscribe to clock tick events.

```tsx
useClockEvent((tick) => {
  // tick: { beat, bar, step, bpm }
});
```

---- useSampler(moduleId)

Manage a sampler module's sample slots: load, clear, trigger.

Returns UseSamplerResult:
| Field       | Type                                       | Description                 |
|-------------|--------------------------------------------|-----------------------------|
| slots       | Record<number, SampleSlot \| null>         | Loaded samples              |
| voices      | SamplerVoice[]                             | Active voices               |
| loadSample  | (slot, path, mode?) => Promise             | Load a WAV file into a slot |
| clearSample | (slot) => Promise                          | Remove a sample from a slot |
| trigger     | (slot, velocity?) => Promise               | Trigger playback            |

SampleSlot: `{ name, duration, sampleRate, mode: 'oneshot' | 'loop' }`
Mode defaults to 'oneshot'. Slot 1 maps to MIDI note 36 (GM kick drum).

---- useRecorder()

Record audio from input devices into sampler slots.

Returns UseRecorderResult:
| Field          | Type                                          | Description              |
|----------------|-----------------------------------------------|--------------------------|
| devices        | AudioRecordingDevice[]                        | Available input devices  |
| recording      | RecordingState                                | Current recording state  |
| listDevices    | () => Promise                                 | Refresh device list      |
| startRecording | (moduleId, slot, deviceIndex?) => Promise     | Start recording          |
| stopRecording  | () => Promise                                 | Stop and save to slot    |

---- useSequencer(moduleId)

Control a step sequencer module's pattern and track assignments.

Returns UseSequencerResult:
| Field          | Type                                               | Description               |
|----------------|----------------------------------------------------|-----------------------------|
| pattern        | Record<string, Record<string, StepData>>           | Pattern grid               |
| currentStep    | number                                             | Active playback step       |
| trackTargets   | Record<string, string>                             | Track-to-module mappings   |
| setStep        | (track, step, active, note?, velocity?) => Promise | Set a step                 |
| setTrackTarget | (track, targetModuleId) => Promise                 | Assign track output        |
| clearPattern   | () => Promise                                      | Clear all steps            |

StepData: `{ active: boolean, note: number, velocity: number }`

=== EXAMPLES ===
Example 1: Basic synth rack
---
```tsx
import { useAudioInit, useRack, useModule, useParam } from '@reactjit/audio';
import { Knob } from '@reactjit/controls';
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';
import { useEffect } from 'react';

function SynthRack() {
  const c = useThemeColors();
  const ready = useAudioInit();
  const rack = useRack();
  const [cutoff, setCutoff] = useParam('filter1', 'cutoff');

  useEffect(() => {
    if (!ready) return;
    rack.addModule('oscillator', 'osc1', { waveform: 'saw' });
    rack.addModule('filter', 'filter1', { cutoff: 1000, resonance: 0.5 });
    rack.addModule('mixer', 'out', {});
    rack.connect('osc1', 'audio_out', 'filter1', 'audio_in');
    rack.connect('filter1', 'audio_out', 'out', 'input_1');
  }, [ready]);

  return (
    <Box style={{ padding: 16, gap: 8 }}>
      <Text style={{ color: c.text, fontSize: 14 }}>{`Modules: ${rack.modules.length}`}</Text>
      <Knob value={cutoff ?? 1000} min={20} max={20000} onChange={setCutoff} label="Cutoff" />
    </Box>
  );
}
```

Example 2: Sampler with clock
---
```tsx
import { useAudioInit, useRack, useSampler, useClock } from '@reactjit/audio';
import { Box, Text, Pressable } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';
import { useEffect } from 'react';

function DrumMachine() {
  const c = useThemeColors();
  const ready = useAudioInit();
  const rack = useRack();
  const sampler = useSampler('drums');
  const clock = useClock('clock1');

  useEffect(() => {
    if (!ready) return;
    rack.addModule('sampler', 'drums', {});
    rack.addModule('clock', 'clock1', { bpm: 120 });
    sampler.loadSample(1, 'samples/kick.wav');
    sampler.loadSample(2, 'samples/snare.wav');
  }, [ready]);

  return (
    <Box style={{ padding: 16, gap: 8 }}>
      <Text style={{ color: c.text, fontSize: 14 }}>
        {`Beat: ${clock.beat} | Bar: ${clock.bar} | BPM: ${clock.bpm}`}
      </Text>
      <Box style={{ flexDirection: 'row', gap: 8 }}>
        <Pressable onPress={() => clock.start()}>
          <Text style={{ color: c.success, fontSize: 14 }}>{'Play'}</Text>
        </Pressable>
        <Pressable onPress={() => clock.stop()}>
          <Text style={{ color: c.error, fontSize: 14 }}>{'Stop'}</Text>
        </Pressable>
      </Box>
      <Box style={{ flexDirection: 'row', gap: 8 }}>
        <Pressable onPress={() => sampler.trigger(1)}>
          <Text style={{ color: c.primary, fontSize: 14 }}>{'Kick'}</Text>
        </Pressable>
        <Pressable onPress={() => sampler.trigger(2)}>
          <Text style={{ color: c.primary, fontSize: 14 }}>{'Snare'}</Text>
        </Pressable>
      </Box>
    </Box>
  );
}
```

Example 3: MIDI learn
---
```tsx
import { useMIDI, useModule } from '@reactjit/audio';
import { Box, Text, Pressable } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

function MIDIPanel() {
  const c = useThemeColors();
  const midi = useMIDI();
  const filter = useModule('filter1');

  return (
    <Box style={{ padding: 16, gap: 8 }}>
      <Text style={{ color: c.text, fontSize: 14 }}>
        {`MIDI: ${midi.available ? 'Available' : 'Not detected'}`}
      </Text>
      {midi.devices.map(d => (
        <Text key={d.id} style={{ color: c.textSecondary, fontSize: 12 }}>
          {`${d.name} (${d.connected ? 'connected' : 'disconnected'})`}
        </Text>
      ))}
      <Pressable onPress={() => midi.learn('filter1', 'cutoff')}>
        <Text style={{ color: c.accent, fontSize: 14 }}>
          {midi.learning ? 'Move a knob...' : 'MIDI Learn: Cutoff'}
        </Text>
      </Pressable>
    </Box>
  );
}
```

=== PLATFORM NOTES ===
Love2D:
  The audio engine runs in Lua using Love2D's audio sources and thread system. Audio processing happens in a separate Love2D thread at native sample rates -- React hooks only see state snapshots pushed at ~30fps via bridge events.

  useRack receives `audio:state` events from Lua. By default it filters updates to topology changes only (module add/remove, connection changes). For per-frame parameter updates (envelope values, VU levels), use useModule or useParam on specific modules.

  MIDI support requires Love2D's MIDI capabilities (luamidi). The midi.available flag indicates whether the system has MIDI support compiled in.

=== CRITICAL RULES ===
- Call useAudioInit() once before using any other audio hook
- useRack re-renders only on topology changes by default -- use useModule for live parameter tracking
- useSampler slot numbering starts at 1, mapping to GM drum MIDI notes (slot 1 = note 36)
- Audio processing is Lua-owned -- React cannot process audio samples directly
- useClock only re-renders for discrete changes (beat, bar, step, running) -- continuous phase is skipped to avoid 30fps re-renders
