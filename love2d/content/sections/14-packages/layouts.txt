=== OVERVIEW ===
@reactjit/layouts provides structural layout components organized into three tiers:

Page Layouts -- full-viewport skeletons that divide the screen into named regions (header, sidebar, footer, content). These replace the need to manually wire up root-level flex containers.

Container Layouts -- inner-region components for arranging content within a section (stacked lists, wrapping clusters, hero+supporting grids, horizontal scrolling reels).

Nav Layouts -- page-level layouts with a dedicated navigation slot (top bar, side rail, bottom tabs, drawer overlay, command bar).

All layouts are style-free -- they allocate space but apply no colors, borders, or typography. Bring your own theme.

=== API / SYNTAX ===
```tsx
import { AppShell, HolyGrail, Centered, Stage, Mosaic, Pinboard, Curtain } from '@reactjit/layouts';
import { Stack, Cluster, Sidebar, Shelf, Keystone, Frame, Ladder, Reel } from '@reactjit/layouts';
import { TopNav, SideNav, BottomNav, CommandShell, Drawer, Bookshelf, Crumb } from '@reactjit/layouts';
```

---- Page Layouts ----

| Component | Props                                | Description                                        |
|-----------|--------------------------------------|----------------------------------------------------|
| AppShell  | header?, footer?, style?             | Header / body / footer column                      |
| HolyGrail | header?, footer?, left?, leftWidth?, right?, rightWidth?, style? | Header / [left \| main \| right] / footer |
| Centered  | maxWidth?, style?                    | Content centered in viewport                       |
| Stage     | tray?, trayHeight?, style?           | Full content + docked bottom tray                  |
| Mosaic    | columns?, gap?, style?               | Equal-width columns, each child gets one cell       |
| Pinboard  | header?, sidebar?, sidebarWidth?, sidebarSide?, style? | Fixed header + sidebar, no footer          |
| Curtain   | left?, right?, split?, gap?, style?  | Two full-height panels with configurable split      |

---- Container Layouts ----

| Component | Props                                | Description                                        |
|-----------|--------------------------------------|----------------------------------------------------|
| Stack     | gap?, style?                         | Vertical list with uniform gap                     |
| Cluster   | gap?, justify?, align?, style?       | Horizontal wrapping group (tags, badges)            |
| Sidebar   | side?, sideWidth?, gap?, children[]  | Fixed sidebar + flexible content                   |
| Shelf     | gap?, align?, justify?, style?       | Single non-wrapping horizontal row                 |
| Keystone  | heroRatio?, gap?, style?             | First child is hero, rest share bottom area         |
| Frame     | padding?, style?                     | Uniform inset on all sides                         |
| Ladder    | gap?, style?                         | Children alternate left/right alignment             |
| Reel      | itemWidth?, gap?, style?             | Horizontal scrolling strip with fixed item widths   |

---- Nav Layouts ----

| Component    | Props                                        | Description                              |
|--------------|----------------------------------------------|------------------------------------------|
| TopNav       | nav, style?                                  | Horizontal nav bar above content         |
| SideNav      | nav, navWidth?, side?, style?                | Vertical nav rail beside content         |
| BottomNav    | nav, style?                                  | Tab bar below content                    |
| CommandShell | command, style?                              | Persistent command/search bar at top     |
| Drawer       | drawer, drawerWidth?, side?, open?, style?   | Overlay panel (absolute positioned)      |
| Bookshelf    | tabs, tabsWidth?, side?, style?              | Vertical tabs + content beside them      |
| Crumb        | trail, style?                                | Breadcrumb trail above content           |

=== EXAMPLES ===
Example 1: AppShell with header and footer
---
```tsx
import { Text, Box } from '@reactjit/core';
import { AppShell } from '@reactjit/layouts';
import { useThemeColors } from '@reactjit/theme';

export function App() {
  const colors = useThemeColors();

  return (
    <AppShell
      header={
        <Box style={{ paddingLeft: 16, paddingTop: 12, paddingBottom: 12, backgroundColor: colors.bgElevated }}>
          <Text style={{ color: colors.text, fontSize: 18, fontWeight: 'bold' }}>My App</Text>
        </Box>
      }
      footer={
        <Box style={{ paddingLeft: 16, paddingTop: 8, paddingBottom: 8, backgroundColor: colors.bgElevated }}>
          <Text style={{ color: colors.muted, fontSize: 12 }}>v1.0.0</Text>
        </Box>
      }
    >
      <Box style={{ flexGrow: 1, justifyContent: 'center', alignItems: 'center' }}>
        <Text style={{ color: colors.text }}>Main Content</Text>
      </Box>
    </AppShell>
  );
}
```

Example 2: HolyGrail layout
---
```tsx
import { Text, Box } from '@reactjit/core';
import { HolyGrail } from '@reactjit/layouts';
import { useThemeColors } from '@reactjit/theme';

export function Dashboard() {
  const colors = useThemeColors();

  return (
    <HolyGrail
      leftWidth={200}
      rightWidth={240}
      header={<Box style={{ height: 48, backgroundColor: colors.bgElevated }} />}

      right={<Box style={{ backgroundColor: colors.surface, height: '100%' }}><Text style={{ color: colors.text }}>Details</Text></Box>}

      <Box style={{ flexGrow: 1, paddingLeft: 16, paddingTop: 16 }}>
        <Text style={{ color: colors.text }}>Main Panel</Text>
      </Box>
    </HolyGrail>
  );
}
```

Example 3: Drawer navigation
---
```tsx
import { useState } from 'react';
import { Text, Box, Pressable } from '@reactjit/core';
import { Drawer } from '@reactjit/layouts';
import { useThemeColors } from '@reactjit/theme';

export function DrawerDemo() {
  const colors = useThemeColors();
  const [open, setOpen] = useState(false);

  return (
    <Drawer
      open={open}
      drawerWidth={260}
      drawer={
        <Box style={{ backgroundColor: colors.surface, height: '100%', paddingLeft: 16, paddingTop: 16 }}>
          <Pressable onPress={() => setOpen(false)}>
            <Text style={{ color: colors.primary }}>Close Drawer</Text>
          </Pressable>
        </Box>
      }
    >
      <Box style={{ flexGrow: 1, justifyContent: 'center', alignItems: 'center' }}>
        <Pressable onPress={() => setOpen(true)}>
          <Text style={{ color: colors.primary }}>Open Drawer</Text>
        </Pressable>
      </Box>
    </Drawer>
  );
}
```

Example 4: Keystone hero layout
---
```tsx
import { Box, Text } from '@reactjit/core';
import { Keystone } from '@reactjit/layouts';
import { useThemeColors } from '@reactjit/theme';

export function HeroLayout() {
  const colors = useThemeColors();

  return (
    <Keystone heroRatio={0.6} gap={4} style={{ height: 400 }}>
      <Box style={{ backgroundColor: colors.surface, borderRadius: 8, justifyContent: 'center', alignItems: 'center', height: '100%' }}>
        <Text style={{ color: colors.text, fontSize: 24 }}>Hero Content</Text>
      </Box>
      <Box style={{ backgroundColor: colors.bgElevated, borderRadius: 8, justifyContent: 'center', alignItems: 'center', height: '100%' }}>
        <Text style={{ color: colors.text }}>Panel A</Text>
      </Box>
      <Box style={{ backgroundColor: colors.bgElevated, borderRadius: 8, justifyContent: 'center', alignItems: 'center', height: '100%' }}>
        <Text style={{ color: colors.text }}>Panel B</Text>
      </Box>
    </Keystone>
  );
}
```

=== CRITICAL RULES ===
- All layouts fill 100% width and height by default (page layouts). Override with the style prop.
- Layouts allocate space only. They apply no colors, borders, or padding. Add your own.
- AppShell body and all "content" regions use flexGrow: 1 -- they absorb remaining space after fixed-size slots.
- Curtain's split prop is 0-1 (fraction of width for the left panel). Default 0.5 = equal split.
- Drawer uses position: absolute for the drawer panel. Content behind stays full-size and interactive.
- Sidebar expects exactly two children: [sidebar content, main content].
- Reel wraps children in a horizontal ScrollView. Each item gets the specified itemWidth.
