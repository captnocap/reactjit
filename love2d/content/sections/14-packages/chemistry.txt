=== OVERVIEW ===
@reactjit/chemistry provides a complete chemistry toolkit. Pure data (118 elements, compounds, enthalpies) lives in TypeScript. Molecular computation (balancing, geometry, bond analysis) runs via Lua RPCs. Interactive visualizations (reagent tests, spectra, phase diagrams, Bohr models) are Lua capabilities painted at 60fps. PubChem REST API integration enables live compound lookups.

The package has four layers:

1. **Static data** -- ELEMENTS array (118 elements with mass, category, electronegativity, shells, cpkColor), COMPOUNDS library, ENTHALPIES reference, and physical CONSTANTS (Avogadro, Boltzmann, Planck, etc.)
2. **Pure TS utilities** -- formula parsing, molar mass, mass composition, electron configuration, bond character, stoichiometry (mass/moles/particles), ideal gas law, molarity/dilution
3. **React hooks and widgets** -- useElement, useMolecule, useReaction, useEquilibrium, usePeriodicTableFilter, PeriodicTable widget, ElementTile/Card/Detail, MoleculeCard, ElectronShell, ReactionView
4. **Lua capabilities** -- ReagentTest, SpectrumView, PhaseDiagram, BohrModel, StructureView (all rendered natively via `<Native>`)

Chemical notation uses the LaTeX typesetter (latex_parser.lua + latex_layout.lua) via ChemFormula, ChemEquation, IsoNotation, and ChemFig components.

Importing `@reactjit/chemistry` also registers chemistry-specific unit conversions into `@reactjit/convert` as a side effect.

=== EXAMPLES ===
Example 1: Element lookup
---
```tsx
import { useElement, CONSTANTS } from '@reactjit/chemistry';
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function ElementInfo() {
  const c = useThemeColors();
  const carbon = useElement('C');

  if (!carbon) return null;

  return (
    <Box style={{ padding: 16 }}>
      <Text style={{ fontSize: 18, color: c.text }}>
        {`${carbon.name} (${carbon.symbol}) — Z=${carbon.number}`}
      </Text>
      <Text style={{ fontSize: 14, color: c.muted }}>
        {`Mass: ${carbon.mass} g/mol | Config: ${carbon.electronConfig}`}
      </Text>
    </Box>
  );
}
```

Example 2: Periodic table with selection
---
```tsx
import { useState } from 'react';
import { PeriodicTable, useElement } from '@reactjit/chemistry';
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function PeriodicTableDemo() {
  const c = useThemeColors();
  const [selected, setSelected] = useState<number | null>(null);
  const el = useElement(selected ?? 1);

  return (
    <Box style={{ width: '100%', height: '100%' }}>
      <PeriodicTable onSelect={(e) => setSelected(e.number)} selected={selected} />
      {el && (
        <Text style={{ fontSize: 14, color: c.text, padding: 8 }}>
          {`${el.name} — ${el.category} — ${el.phase} at STP`}
        </Text>
      )}
    </Box>
  );
}
```

Example 5: Stoichiometry utilities
---
```tsx
import { molarMass, massToMoles, molesToParticles, idealGasPressure, CONSTANTS } from '@reactjit/chemistry';

// Molar mass of water
const mmH2O = molarMass('H2O'); // 18.015

// 36g of water = how many moles?
const moles = massToMoles(36, 'H2O'); // ~2.0

// How many molecules?
const particles = molesToParticles(moles); // ~1.204e24

// Ideal gas: 1 mol at 273K in 22.4L
const P = idealGasPressure(1, 273, 0.0224); // ~101325 Pa

// Physical constants
CONSTANTS.AVOGADRO;       // 6.022e23
CONSTANTS.BOLTZMANN;      // 1.381e-23
CONSTANTS.SPEED_OF_LIGHT; // 2.998e8
```

=== PLATFORM NOTES ===
Love2D:
  Lua capabilities (ReagentTest, SpectrumView, PhaseDiagram, BohrModel, StructureView) are rendered by dedicated Lua painters at 60fps. They route through the `<Native>` component and are registered via the capability system. Molecular computation (useMolecule, useReaction) runs via bridge RPCs to Lua. Static data (elements, utilities) is pure TS with no bridge dependency.

=== CRITICAL RULES ===
- useMolecule and useReaction are async -- they return null until the Lua RPC resolves. Always handle the null case.
- Lua capabilities (ReagentTest, SpectrumView, etc.) are surface elements. If no explicit dimensions are set, they use proportional surface fallback (1/4 of parent). Set width/height or use flexGrow for precise sizing.
- ChemFormula, ChemEquation, IsoNotation, and ChemFig render via the LaTeX typesetter. They produce inline elements, not block elements.
- Importing @reactjit/chemistry registers chemistry conversions into @reactjit/convert as a side effect. No manual registration needed.
