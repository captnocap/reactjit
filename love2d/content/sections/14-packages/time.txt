=== OVERVIEW ===
@reactjit/time provides time-related hooks, widgets, and utilities. Time-sensitive operations (stopwatch, countdown, precision scheduling) run on the Lua side inside Love2D's update loop, making them frame-accurate and immune to JavaScript event-loop jitter.

The package has three layers:

1. **Hooks** -- useTime (JS wall clock), useLuaTime (Lua monotonic + wall clock), useStopwatch (Lua-driven elapsed timer), useCountdown (Lua-driven countdown), useOnTime (frame-precise delayed callback), useInterval (Lua-driven repeating callback), useFrameInterval (fire every N rendered frames)
2. **Widgets** -- Clock, Stopwatch, Countdown (self-contained with optional controls and progress bars), Ticker (invisible repeating callback), FrameTicker (invisible frame-synced callback)
3. **Pure utilities** -- Unix timestamp helpers, date arithmetic (addDays, diffHours, etc.), day boundary functions, predicates (isToday, isPast), formatting (formatDuration, formatDate, relativeTime), timezone conversion, parsing

Hooks that run on the Lua side (useStopwatch, useCountdown, useOnTime, useInterval, useFrameInterval) create Lua timer objects that accumulate time via love.update(dt). This means a callback scheduled with useOnTime fires in the exact Love2D frame that crosses the delay threshold -- no JS GC pauses, no event-loop delays.

=== API / SYNTAX ===
```tsx
import {
  // Hooks
  useTime, useLuaTime,
  useStopwatch, useCountdown,
  useOnTime, useInterval, useFrameInterval,
  // Widgets
  Clock, Stopwatch, Countdown, Ticker, FrameTicker,
  // Utilities
  nowMs, nowSec, fromUnixSec, toUnixSec,
  addDays, addHours, addMinutes, addSeconds,
  diffDays, diffHours, diffMinutes, diffSeconds,
  startOfDay, endOfDay, startOfHour,
  isToday, isYesterday, isTomorrow, isSameDay, isPast, isFuture,
  formatDuration, formatDurationLong, formatDate, formatTimeOfDay, formatDateOnly,
  relativeTime,
  tzOffsetMinutes, tzConvert, listTimezones,
  parseDate, parseDuration,
} from '@reactjit/time';
```

useStopwatch options and return:
| Option    | Type    | Default | Description                             |
|-----------|---------|---------|-----------------------------------------|
| tickRate  | number  | 100     | Update frequency (ms) from Lua to React |
| autoStart | boolean | false   | Start running on mount                  |

| Return  | Type       | Description                          |
|---------|------------|--------------------------------------|
| elapsed | number     | Elapsed time in milliseconds         |
| running | boolean    | Whether stopwatch is running         |
| start   | () => void | Start                                |
| stop    | () => void | Stop                                 |
| reset   | () => void | Reset to 0                           |
| restart | () => void | Reset + start                        |

useCountdown return:
| Return    | Type       | Description                          |
|-----------|------------|--------------------------------------|
| remaining | number     | Remaining time in milliseconds       |
| running   | boolean    | Whether countdown is running         |
| complete  | boolean    | Whether countdown reached zero       |
| progress  | number     | 0-1 progress fraction                |
| start     | () => void | Start                                |
| stop      | () => void | Pause                                |
| reset     | () => void | Reset to initial duration            |
| restart   | () => void | Reset + start                        |

Clock props:
| Prop     | Type                           | Default | Description                 |
|----------|--------------------------------|---------|-----------------------------|
| format   | 'time' | 'date' | 'datetime'  | 'time'  | Display format              |
| timezone | string                         | local   | IANA timezone name          |
| rateMs   | number                         | 1000    | Update rate                 |
| style    | Style                          | undefined | Container style           |
| textStyle| Style                          | undefined | Text element style        |

Countdown widget props:
| Prop       | Type       | Default | Description                        |
|------------|------------|---------|------------------------------------|
| duration   | number     | required| Duration in milliseconds           |
| autoStart  | boolean    | false   | Start on mount                     |
| controls   | boolean    | true    | Show start/stop/reset buttons      |
| showBar    | boolean    | true    | Show progress bar                  |
| showMs     | boolean    | false   | Show milliseconds                  |
| onComplete | () => void | undefined| Called when countdown reaches zero |

Example 1: Drop-in clock
---
```tsx
import { Clock } from '@reactjit/time';

// One-liner wall clock
<Clock />

// With timezone and format
<Clock format="datetime" timezone="America/New_York" />

<Clock format="date" />
```
---
Platforms: love2d

Example 2: Stopwatch with controls
---
```tsx
import { Stopwatch } from '@reactjit/time';

// Self-contained with start/stop/reset buttons
<Stopwatch autoStart />

// Display only (no controls)
<Stopwatch controls={false} />

// With milliseconds and tick callback
<Stopwatch showMs onTick={(ms) => console.log(ms)} />
```
---
Platforms: love2d

Example 3: Countdown timer
---
```tsx
import { useState } from 'react';
import { Countdown } from '@reactjit/time';
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function CountdownDemo() {
  const c = useThemeColors();
  const [done, setDone] = useState(false);

  return (
    <Box style={{ padding: 16, gap: 8 }}>
      <Countdown
        duration={30_000}
        autoStart
        onComplete={() => setDone(true)}
      />
      {done && (
        <Text style={{ color: c.primary, fontSize: 16 }}>Time is up!</Text>
      )}
    </Box>
  );
}
```
---
Platforms: love2d

Example 4: Precision scheduling with useOnTime
---
```tsx
import { useOnTime, useInterval } from '@reactjit/time';

// Fire exactly 500ms after mount (frame-accurate via Lua)
useOnTime(() => playNote(60, 127), 500);

// Repeating callback every 2 seconds via Lua timer
useInterval(() => fetchData(), 2000);
```

Platforms: love2d

Example 6: Date utilities
---
```tsx
import {
  addDays, diffHours, isToday, formatDuration, relativeTime,
  startOfDay, tzConvert,
} from '@reactjit/time';

const tomorrow = addDays(Date.now(), 1);
const hoursDiff = diffHours(Date.now(), tomorrow);  // 24
const today = isToday(Date.now());                   // true

// Duration formatting
formatDuration(90_000);                              // "1:30"
formatDuration(90_000, { ms: true });                // "1:30.000"

// Relative time
relativeTime(Date.now() - 3_600_000);                // "1 hour ago"

// Timezone conversion
const tokyoMs = tzConvert(Date.now(), 'Asia/Tokyo');
```

Platforms: love2d

Example 7: Using hooks for custom timer logic
---
```tsx
import { useStopwatch, useCountdown } from '@reactjit/time';
import { formatDuration } from '@reactjit/time';
import { Box, Text, Pressable } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function CustomTimer() {
  const c = useThemeColors();
  const sw = useStopwatch();

  return (
    <Box style={{ padding: 16, gap: 8 }}>
      <Text style={{ color: c.text, fontSize: 24 }}>
        {formatDuration(sw.elapsed)}
      </Text>
      <Box style={{ flexDirection: 'row', gap: 8 }}>
        <Pressable onPress={sw.running ? sw.stop : sw.start}>
          <Text style={{ color: c.primary, fontSize: 14 }}>
            {sw.running ? 'Stop' : 'Start'}
          </Text>
        </Pressable>
        <Pressable onPress={sw.reset}>
          <Text style={{ color: c.muted, fontSize: 14 }}>Reset</Text>
        </Pressable>
      </Box>
    </Box>
  );
}
```

=== PLATFORM NOTES ===
Love2D:
  Lua-driven timers (useStopwatch, useCountdown, useOnTime, useInterval) accumulate time via love.update(dt) in the Lua main loop. This provides sub-millisecond precision tied to the frame clock. useTime and the pure utility functions run entirely in JavaScript. useFrameInterval counts actual rendered frames rather than wall-clock time -- at lower framerates it fires less frequently, keeping logic synchronized with the render cadence.

=== CRITICAL RULES ===
- useTime is a JS setInterval -- it is subject to event-loop jitter. Use useOnTime or useInterval for frame-accurate timing.
- Widgets (Clock, Stopwatch, Countdown) use useThemeColors() internally. They adapt to the active theme.
- Ticker and FrameTicker render nothing. They exist only for side effects. Use them when you need a repeating callback without a visible widget.
- useCountdown fires onComplete once when remaining hits zero. It does not fire again unless restart() is called.
- formatDuration formats milliseconds as "M:SS" by default. Pass { ms: true } for "M:SS.mmm" and { forceHours: true } for "H:MM:SS".
