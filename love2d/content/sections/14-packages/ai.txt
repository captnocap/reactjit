=== OVERVIEW ===
`@reactjit/ai` provides provider-agnostic LLM integration for ReactJIT applications. It supports OpenAI-compatible and Anthropic APIs with streaming, multi-round tool execution (agentic loops), MCP server connections, API key management, and pre-built chat UI components.

The package is organized in three tiers:

1. **Hooks** (Tier 1) -- `useChat`, `useCompletion`, `useModels` for building custom UIs.
2. **Components** (Tier 2) -- `AIMessageList`, `AIChatInput`, `AIModelSelector`, `AISettingsPanel`, `AIConversationSidebar`, `AIMessageWithActions` for composing chat interfaces from parts.
3. **Templates** (Tier 3) -- `MinimalChat`, `SimpleChatUI`, `PowerChatUI` for drop-in fully-functional chat UIs.

All streaming goes through Love2D's HTTP/proxy support via SSE (Server-Sent Events). No browser APIs required.

=== API / SYNTAX ===
```tsx
import {
  AIProvider, useChat, useCompletion, useModels, useAPIKeys, useMCPServer,
  AIMessageList, AIChatInput, AIModelSelector,
  MinimalChat, SimpleChatUI, PowerChatUI,
  openai, anthropic, getProvider,
} from '@reactjit/ai';
```

---- AIProvider

Context provider for default AI configuration. Wrap your app to provide API keys, model selection, and provider settings to all hooks.

| Prop   | Type     | Description                          |
|--------|----------|--------------------------------------|
| config | AIConfig | Default provider/model/key settings  |

AIConfig:
| Field        | Type                               | Default   | Description                |
|--------------|------------------------------------|-----------|----------------------------|
| provider     | 'openai' \| 'anthropic' \| 'custom' | required | LLM provider             |
| model        | string                             | required  | Model ID                   |
| apiKey       | string                             | undefined | API key                    |
| baseURL      | string                             | undefined | Custom endpoint URL        |
| temperature  | number                             | undefined | Sampling temperature       |
| maxTokens    | number                             | undefined | Max response tokens        |
| systemPrompt | string                             | undefined | System message prepended   |
| proxy        | string                             | undefined | SOCKS5/HTTP proxy URL      |

---- useChat(options?)

Conversational AI with streaming and automatic tool execution loops.

Options (extends AIConfig):
| Field           | Type              | Default   | Description                          |
|-----------------|-------------------|-----------|--------------------------------------|
| initialMessages | Message[]         | []        | Pre-seed conversation                |
| tools           | ToolDefinition[]  | undefined | Tool definitions for function calling|
| maxToolRounds   | number            | 10        | Max tool call rounds per send        |
| onChunk         | (chunk) => void   | undefined | Called per streaming content token    |
| onToolCall      | (call) => void    | undefined | Called when model invokes a tool      |
| onError         | (error) => void   | undefined | Error callback                       |

Returns ChatResult:
| Field        | Type                                 | Description                   |
|--------------|--------------------------------------|-------------------------------|
| messages     | Message[]                            | Full conversation history     |
| send         | (content: string) => Promise<void>   | Send a user message           |
| isLoading    | boolean                              | Request in progress           |
| isStreaming   | boolean                              | Currently receiving tokens    |
| stop         | () => void                           | Abort the current request     |
| error        | Error \| null                        | Last error                    |
| setMessages  | setter                               | Manually update messages      |

---- useCompletion(options?)

Single-shot text completion with streaming. Options extend AIConfig with onChunk and onError.

Returns CompletionResult:
| Field       | Type                                | Description              |
|-------------|-------------------------------------|--------------------------|
| completion  | string                              | Current completion text  |
| complete    | (prompt: string) => Promise<string> | Run a completion         |
| isLoading   | boolean                             | Request in progress      |
| isStreaming  | boolean                             | Currently receiving      |
| stop        | () => void                          | Abort                    |
| error       | Error \| null                       | Last error               |

---- useModels(options?)

Fetch available models from the provider. Auto-fetches on mount and config changes.

Returns ModelsResult:
| Field   | Type        | Description           |
|---------|-------------|-----------------------|
| models  | ModelInfo[] | Available models      |
| loading | boolean     | Fetching in progress  |
| error   | Error \| null | Last error          |
| refetch | () => void  | Manually re-fetch     |

---- useAPIKeys()

Manage API keys persisted to local storage.

Returns APIKeysResult:
| Field     | Type                                  | Description                      |
|-----------|---------------------------------------|----------------------------------|
| keys      | APIKeyRecord[]                        | All stored keys                  |
| setKey    | (record) => Promise<string>           | Add or update a key              |
| deleteKey | (id: string) => Promise<void>         | Remove a key                     |
| getKey    | (provider) => APIKeyRecord \| undefined | Get key for a provider          |
| loading   | boolean                               | Loading state                    |

---- useMCPServer(config)

Connect to an MCP (Model Context Protocol) server and expose its tools.

MCPServerConfig:
| Field       | Type                    | Description                    |
|-------------|-------------------------|--------------------------------|
| command     | string                  | Server command to run          |
| args        | string[]                | Command arguments              |
| transport   | 'stdio' \| 'sse'       | Transport type                 |
| permissions | MCPPermissionsConfig    | Tool permission rules          |

---- ToolDefinition

Tool definition for function-calling / agentic loops.

| Field       | Type                             | Description               |
|-------------|----------------------------------|---------------------------|
| name        | string                           | Tool name                 |
| description | string                           | What the tool does        |
| parameters  | Record<string, any>              | JSON Schema for arguments |
| execute     | (args: any) => Promise<any>      | Implementation function   |

---- Components (Tier 2)

| Component                | Description                              |
|--------------------------|------------------------------------------|
| AIMessageList            | Renders a Message[] array as a chat view |
| AIChatInput              | Text input with send button              |
| AIModelSelector          | Dropdown to pick a model                 |
| AISettingsPanel          | Full settings form (key, model, temp)    |
| AIConversationSidebar    | Conversation list with create/delete     |
| AIMessageWithActions     | Single message with copy/retry actions   |

---- Templates (Tier 3)

| Template      | Description                                              |
|---------------|----------------------------------------------------------|
| MinimalChat   | Bare-minimum chat: message list + input. One prop: config|
| SimpleChatUI  | Chat with model selector and settings                    |
| PowerChatUI   | Full-featured: sidebar, tools, MCP, settings panel       |

=== EXAMPLES ===
Example 1: Minimal chat
---
```tsx
import { useChat, AIProvider } from '@reactjit/ai';
import { Box, Text, Pressable, ScrollView, Input } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';
import { useState } from 'react';

function Chat() {
  const c = useThemeColors();
  const { messages, send, isStreaming } = useChat({ model: 'gpt-4' });
  const [input, setInput] = useState('');

  const handleSend = async () => {
    if (!input.trim()) return;
    const msg = input;
    setInput('');
    await send(msg);
  };

  return (
    <Box style={{ flexGrow: 1, padding: 16, gap: 8 }}>
      <ScrollView style={{ flexGrow: 1 }}>
        {messages.map((m, i) => (
          <Box key={i} style={{ padding: 8 }}>
            <Text style={{ color: m.role === 'user' ? c.primary : c.text, fontSize: 14 }}>
              {`${m.role}: ${m.content}`}
            </Text>
          </Box>
        ))}
      </ScrollView>
      <Box style={{ flexDirection: 'row', gap: 8 }}>
        <Input
          value={input}
          onChangeText={setInput}
          onSubmit={handleSend}
          placeholder="Type a message..."
          style={{ flexGrow: 1 }}
        />
        <Pressable onPress={handleSend}>
          <Text style={{ color: c.primary, fontSize: 14 }}>{isStreaming ? 'Streaming...' : 'Send'}</Text>
        </Pressable>
      </Box>
    </Box>
  );
}

export function App() {
  return (
    <AIProvider config={{ provider: 'openai', model: 'gpt-4', apiKey: 'sk-...' }}>
      <Chat />
    </AIProvider>
  );
}
```

Example 2: Agent with tool calling
---
```tsx
import { useChat } from '@reactjit/ai';
import type { ToolDefinition } from '@reactjit/ai';

const tools: ToolDefinition[] = [
  {
    name: 'calculate',
    description: 'Evaluate a math expression',
    parameters: {
      type: 'object',
      properties: {
        expression: { type: 'string', description: 'Math expression to evaluate' },
      },
      required: ['expression'],
    },
    execute: async (args) => {
      const result = Function(`"use strict"; return (${args.expression})`)();
      return { result };
    },
  },
];

function AgentChat() {
  const { messages, send, isLoading } = useChat({
    model: 'gpt-4',
    tools,
    maxToolRounds: 5,
    onToolCall: (call) => console.log('Tool called:', call.name),
  });

  // The model can now call calculate() and continue the conversation
  // with the result. Tool execution loops run automatically up to maxToolRounds.
}
```

Example 4: useCompletion for single-shot tasks
---
```tsx
import { useCompletion } from '@reactjit/ai';
import { Box, Text, Pressable } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

function Summarizer() {
  const c = useThemeColors();
  const { completion, complete, isStreaming } = useCompletion({
    model: 'gpt-4',
    systemPrompt: 'You summarize text concisely in 2-3 sentences.',
  });

  return (
    <Box style={{ padding: 16, gap: 8 }}>
      <Pressable onPress={() => complete('Summarize the history of computing...')}>
        <Text style={{ color: c.primary, fontSize: 14 }}>{'Summarize'}</Text>
      </Pressable>
      {completion ? (
        <Text style={{ color: c.text, fontSize: 14 }}>{completion}</Text>
      ) : null}
    </Box>
  );
}
```

=== PLATFORM NOTES ===
Love2D:
  All HTTP streaming goes through Love2D's networking layer (luasec + luasocket). The startStream function in the package handles SSE parsing over these connections. Proxy support (SOCKS5/HTTP) is passed through to Love2D's existing proxy infrastructure.

  Anthropic's API does not expose a models listing endpoint -- useModels returns a hardcoded list of available Claude models for the 'anthropic' provider.

  The 'custom' provider type uses OpenAI-compatible request/response format, so it works with any OpenAI-compatible API (Ollama, LM Studio, vLLM, etc.) by setting baseURL.

=== CRITICAL RULES ===
- AIProvider is optional -- hooks accept config directly via options
- useChat runs tool execution loops automatically up to maxToolRounds (default: 10). Each loop is one LLM call + tool execution. Set a reasonable limit to prevent runaway costs
- Tool execute functions must return serializable values (passed back to the model as JSON)
- Message.content can be a string or ContentBlock[] array (for multimodal messages with images)
- The stop() function aborts the current streaming request but preserves accumulated messages
