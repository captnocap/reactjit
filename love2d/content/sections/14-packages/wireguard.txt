=== OVERVIEW ===
@reactjit/wireguard provides two tiers of encrypted peer-to-peer networking, both using the same X25519 key format:

Tier 1: useWireGuard -- real kernel WireGuard via the wg CLI. Requires wireguard-tools installed and root/sudo. Keys are held by the kernel, never in process memory. Full VPN-grade isolation.

Tier 2: usePeerTunnel -- userspace encrypted UDP using X25519 + XChaCha20-Poly1305 via libsodium FFI. No root needed, no system dependencies beyond libsodium. Works everywhere Love2D runs. Keys live in process memory.

Both tiers share useWireGuardIdentity for key generation. Choose your trust boundary: kernel-level for infrastructure, userspace for applications.

Lua-side: lua/wireguard.lua (wg subprocess), lua/peer_tunnel.lua (encrypted UDP), lua/stun.lua (RFC 5389 NAT traversal).

=== API / SYNTAX ===
```tsx
import { useWireGuardIdentity, useWireGuard, usePeerTunnel } from '@reactjit/wireguard';
```

---- useWireGuardIdentity ----

Returns:

| Field      | Type                     | Description                    |
|------------|--------------------------|--------------------------------|
| publicKey  | string \| null           | Generated public key           |
| privateKey | string \| null           | Generated private key          |
| generate   | () => Promise<KeyPair>   | Generate a new key pair        |
| ready      | boolean                  | True after keys are generated  |

---- useWireGuard(interfaceName?) ----

| Field        | Type                                        | Description                    |
|--------------|---------------------------------------------|--------------------------------|
| available    | boolean                                     | wireguard-tools is installed   |
| hasPrivilege | boolean                                     | Has root/sudo access           |
| status       | WireGuardStatus \| null                     | Current tunnel status          |
| up           | (config: WireGuardConfig) => Promise<boolean>| Bring up the tunnel           |
| down         | () => Promise<boolean>                      | Tear down the tunnel          |
| addPeer      | (peer: WireGuardPeerConfig) => Promise<boolean>| Add a peer                 |
| removePeer   | (publicKey: string) => Promise<boolean>     | Remove a peer                 |
| generateKeys | () => Promise<KeyPair>                     | Generate a key pair           |
| error        | string \| null                              | Last error message            |

WireGuardConfig:

| Field      | Type                  | Default     | Description              |
|------------|-----------------------|-------------|--------------------------|
| privateKey | string                | required    | Interface private key    |
| listenPort | number                | undefined   | UDP listen port          |
| address    | string                | undefined   | Interface address (CIDR) |
| dns        | string                | undefined   | DNS server               |
| peers      | WireGuardPeerConfig[] | undefined   | Initial peers            |

---- usePeerTunnel(config) ----

| Field       | Type                                      | Description                    |
|-------------|-------------------------------------------|--------------------------------|
| ready       | boolean                                   | Tunnel is operational          |
| tunnelInfo  | TunnelInfo \| null                        | Tunnel state (port, public IP) |
| peers       | PeerInfo[]                                | Connected peers                |
| send        | (publicKey, data) => void                 | Send to a specific peer        |
| broadcast   | (data) => void                            | Send to all peers              |
| addPeer     | (publicKey, endpoint?) => void            | Add a peer by public key       |
| removePeer  | (publicKey) => void                       | Remove a peer                  |
| lastMessage | PeerMessage \| null                       | Last received message          |
| error       | string \| null                            | Last error                     |

PeerTunnelConfig:

| Field             | Type   | Default              | Description                 |
|-------------------|--------|----------------------|-----------------------------|
| privateKey        | string | required             | Your X25519 private key     |
| listenPort        | number | undefined            | UDP port to bind            |
| stunServer        | string | undefined            | STUN server hostname        |
| stunPort          | number | 3478                 | STUN server port            |
| keepaliveInterval | number | undefined            | Keepalive interval (ms)     |

PeerInfo states: 'waiting' (no endpoint), 'handshaking' (key exchange), 'established' (ready), 'failed'.

=== EXAMPLES ===
Example 1: Userspace P2P chat
---
```tsx
import { useState, useEffect } from 'react';
import { Box, Text, Pressable } from '@reactjit/core';
import { useWireGuardIdentity, usePeerTunnel } from '@reactjit/wireguard';
import { useThemeColors } from '@reactjit/theme';

export function P2PChat() {
  const colors = useThemeColors();
  const identity = useWireGuardIdentity();
  const [messages, setMessages] = useState<string[]>([]);

  const tunnel = usePeerTunnel(identity.ready ? {
    privateKey: identity.privateKey!,
    stunServer: 'stun.l.google.com',
    stunPort: 19302,
  } : null);

  useEffect(() => {
    if (tunnel.lastMessage) {
      setMessages(prev => [...prev, `${tunnel.lastMessage!.publicKey.slice(0, 8)}: ${tunnel.lastMessage!.data}`]);
    }
  }, [tunnel.lastMessage]);

  return (
    <Box style={{ gap: 8 }}>
      {!identity.ready && (
        <Pressable onPress={() => identity.generate()}>
          <Text style={{ color: colors.primary }}>Generate Identity</Text>
        </Pressable>
      )}
      {identity.ready && (
        <Text style={{ color: colors.muted, fontSize: 10 }}>
          {`Your key: ${identity.publicKey!.slice(0, 16)}...`}
        </Text>
      )}
      {tunnel.tunnelInfo?.publicEndpoint && (
        <Text style={{ color: colors.text, fontSize: 10 }}>
          {`Endpoint: ${tunnel.tunnelInfo.publicEndpoint}`}
        </Text>
      )}
      {tunnel.peers.map(p => (
        <Text key={p.publicKey} style={{ color: p.state === 'established' ? colors.primary : colors.muted, fontSize: 10 }}>
          {`Peer ${p.publicKey.slice(0, 8)}... [${p.state}]`}
        </Text>
      ))}
      {messages.map((msg, i) => (
        <Text key={i} style={{ color: colors.text, fontSize: 12 }}>{msg}</Text>
      ))}
    </Box>
  );
}
```

Example 2: Kernel WireGuard tunnel
---
```tsx
import { useWireGuard } from '@reactjit/wireguard';
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function WireGuardPanel() {
  const colors = useThemeColors();
  const wg = useWireGuard('wg-rjit0');

  if (!wg.available) return <Text style={{ color: colors.primary }}>Install wireguard-tools</Text>;
  if (!wg.hasPrivilege) return <Text style={{ color: colors.primary }}>Needs sudo</Text>;

  const bringUp = async () => {
    const keys = await wg.generateKeys();
    await wg.up({
      privateKey: keys.privateKey,
      listenPort: 51820,
      address: '10.0.0.1/24',
    });
  };

  return (
    <Box style={{ gap: 4 }}>
      <Text style={{ color: colors.text }}>
        {wg.status ? `Interface: ${wg.status.interface}, Port: ${wg.status.listenPort}` : 'Tunnel down'}
      </Text>
      {wg.status?.peers.map(p => (
        <Text key={p.publicKey} style={{ color: colors.muted, fontSize: 10 }}>
          {`Peer ${p.publicKey.slice(0, 8)} rx:${p.transferRx} tx:${p.transferTx}`}
        </Text>
      ))}
    </Box>
  );
}
```

=== PLATFORM NOTES ===
Love2D:
  Tier 1 (useWireGuard) requires: wireguard-tools installed, root/sudo access, and the wg/wg-quick binaries in PATH.
  Tier 2 (usePeerTunnel) requires: libsodium available via FFI (bundled with LuaJIT). No root needed.
  STUN NAT traversal uses RFC 5389. The public endpoint is resolved after tunnel creation and available in tunnelInfo.publicEndpoint.
  Status polling: useWireGuard polls every 5s, usePeerTunnel every 2s.

=== CRITICAL RULES ===
- Pass null to usePeerTunnel to disable the tunnel. The tunnel is created when config is non-null and destroyed on unmount.
- Key format is X25519 for both tiers. Keys generated by useWireGuardIdentity work with either hook.
- usePeerTunnel creates the tunnel on mount and destroys it on unmount. The tunnelId is managed internally.
- Peer state transitions: waiting -> handshaking -> established. The 'failed' state means the handshake timed out.
- lastMessage updates on every incoming message. If you need message history, accumulate in your own state.
- Tier 1 (kernel WireGuard) is a real VPN interface. It affects system routing. Use with care.
