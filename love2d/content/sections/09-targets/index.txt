=== OVERVIEW ===
ReactJIT has one renderer: Love2D. Every target -- native desktop and WASM web builds -- runs the same rendering pipeline: React reconciler, QuickJS bridge, Lua layout engine, Love2D painter, OpenGL 2.1.

The target interface is defined in `lua/target_love2d.lua`. A target is a Lua table with five fields:

- `name` -- target identifier (currently `"love2d"`)
- `measure` -- text measurement and font cache (`lua/measure.lua`)
- `images` -- image loading and lifecycle (`lua/images.lua`)
- `videos` -- video playback via libmpv (`lua/videos.lua`, optional)
- `painter` -- rendering backend (`lua/painter.lua`)

What differs between native and WASM is the transport layer:

- Native (Love2D desktop): JavaScript runs inside an embedded QuickJS runtime. The bridge uses C FFI trampolines (`bridge_quickjs.lua`) to pass mutation commands and events between JS and Lua in-process. This is the default mode.

- WASM (love.js web): Lua runs inside Emscripten-compiled Love2D in the browser. JavaScript runs natively in the browser context. The bridge uses `Module.FS` file polling (`bridge_fs.lua`) to exchange data -- React writes commands to a virtual file, Lua reads them each frame.

In both cases, the layout engine, painter, and all rendering logic are identical. The only difference is how bytes cross the JS-Lua boundary.
