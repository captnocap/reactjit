=== OVERVIEW ===
`rjit test` runs test specs inside the Love2D process itself. Because ReactJIT owns the
full stack -- instance tree, layout results, event system -- tests run with direct access
to everything. No browser automation, no ports, no sockets.

The test runner bundles your spec file with esbuild, launches Love2D in test mode
(RJIT_TEST=1), and streams results back to the terminal. Test globals (test, page, expect)
are injected automatically -- no imports needed.

Tests interact with the rendered tree using page.find() selectors and locator actions
(click, type, key). Assertions use expect() matchers that work with the real layout engine.

The app must be built first (rjit build). The test runner loads the existing bundle.js and
then evaluates the spec on top of it.

=== API / SYNTAX ===
Usage:
  rjit test <spec-file.ts> [flags]

Flags:
  --timeout=<seconds>    Test timeout (default: 30)
  --verbose, -v          Pass Love2D stderr through to the terminal

Requirements:
  - The app bundle must exist (run rjit build first)
  - Love2D must be installed and in PATH
  - xvfb-run is used automatically on Linux for headless operation

Globals (injected, no import needed):
  test(name, async fn)   Define a test case
  page                   Page object for finding elements and taking screenshots
  expect(locator)        Assertion wrapper for locators

Selectors -- page.find(componentName, props?):
  page.find('Pressable', { testId: 'submit' })    By component type + prop
  page.find('Text')                                 Any Text node
  page.find('TextInput', { placeholder: 'Search...' })  By any prop value
  page.find('Box', { testId: 'sidebar' })           By testId

  Component name matches debugName (Pressable, Box, ScrollView) or raw type (View, Text).
  Add testId props to components you want to target -- they flow through to the node tree.

Locator actions:
  await locator.click()           Press + release at element center, wait 1 frame
  await locator.type('hello')     Click to focus + inject characters, wait 1 frame
  await locator.key('return')     Inject keypressed/keyreleased (Enter, Escape, arrows)
  await locator.text()            Returns text content of the element
  await locator.rect()            Returns { x, y, w, h }
  await locator.all()             Returns array of all matching nodes

Page actions:
  await page.wait()               Wait 1 frame explicitly
  await page.wait(3)              Wait N frames
  await page.screenshot('/tmp/s.png')  Capture current frame

Matchers:
  await expect(locator).toBeVisible()            Element exists with non-zero size
  await expect(locator).toBeFound()              Element exists in the tree
  await expect(locator).toHaveText('exact')      Exact text match
  await expect(locator).toContainText('substr')  Substring match
  await expect(locator).toHaveRect({ x, y, w, h })  Pixel rect within +/-1px tolerance

Timing model:
  Every await in a test naturally waits one frame (bridge round-trip). click() and type()
  add an extra wait so React has time to re-render before the next assertion. For animations
  or async work, use await page.wait(N) to give it more time.

=== EXAMPLES ===
Example 1: Basic visibility test
---
// tests/app.test.ts
// No imports needed -- test, page, expect are globals

test('header is visible', async () => {
  const header = page.find('Text', { testId: 'header' });
  await expect(header).toBeVisible();
  await expect(header).toHaveText('My App');
});
---
Platforms: love2d

Example 2: Button interaction
---
test('clicking toggle changes status', async () => {
  await page.find('Pressable', { testId: 'toggle' }).then(l => l.click());
  await expect(page.find('Text', { testId: 'status' })).toHaveText('on');
});
---
Platforms: love2d

Example 3: Text input
---
test('search box filters results', async () => {
  const input = page.find('TextInput', { testId: 'search' });
  await input.type('hello');
  await expect(page.find('Text', { testId: 'result-count' })).toContainText('hello');
});
---
Platforms: love2d

Example 4: Layout geometry assertion
---
test('sidebar has correct width', async () => {
  const sidebar = page.find('Box', { testId: 'sidebar' });
  await expect(sidebar).toHaveRect({ x: 0, y: 0, w: 250, h: 720 });
});
---
Platforms: love2d

Example 5: Running the test
---
cd my-app
rjit build              # App must be built first

=== PLATFORM NOTES ===
Love2D:
Tests run inside the Love2D process via QuickJS. The test shim (cli/lib/test-shim.js)
injects globals before the spec evaluates. Results stream back via stdout markers
(TEST_PASS, TEST_FAIL, TEST_DONE). On Linux, xvfb-run is used for headless operation
when available.

=== CRITICAL RULES ===
- The app must be built first (rjit build) -- the test runner loads the existing bundle.js
- Test globals (test, page, expect) are injected automatically -- do not import them
- Each await naturally waits one frame -- click() and type() add an extra frame wait
- Add testId props to components you want to test: <Box testId="sidebar"> flows through to the node tree
- Use rjit test the same way you use rjit lint -- run it before calling a feature done
