=== OVERVIEW ===
`rjit dev` starts esbuild in watch mode, monitoring your source files for changes. On every
save, esbuild rebundles the project and Love2D's HMR system picks up the new bundle.js
in-place -- no restart needed.

On the first successful build, dev mode automatically launches Love2D (via `love .` or
`love love/` depending on project structure). Subsequent rebuilds are picked up by Love2D's
hot reload system without restarting the process.

The command uses esbuild's --watch flag with the --jsx-dev flag enabled, which populates
_debugSource on React fibers for better error messages and the visual inspector (F12).

TSL files (.tsl) in src/ are transpiled to Lua before the initial build and watched for
changes alongside TypeScript sources. TSL transpilation errors block the build.

=== API / SYNTAX ===
Usage:
  rjit dev [target]

Arguments:
  target                 Build target (default: 'love'). Available: 'love', 'web'.

Behavior:
  1. Transpiles any .tsl files in src/ to lua/ (Love2D target only)
  2. Starts esbuild in watch mode with the correct flags for the target
  3. On first successful build, launches Love2D automatically
  4. Subsequent rebuilds trigger HMR -- Love2D detects the changed bundle.js

esbuild flags (set automatically via targets.mjs):
  --bundle --format=iife --global-name=ReactJIT --target=es2020 --jsx=automatic --jsx-dev
  --external:react-dom --external:child_process --watch

Output file:
  love/bundle.js (Love2D target)
  web/bundle.js (Web target)

Entry point resolution (checked in order):
  Love2D: src/main-love.tsx, src/main.tsx
  Web: src/main-wasm.tsx, src/main-web.tsx

=== EXAMPLES ===
Example 1: Start development (most common)
---
cd my-app
rjit dev
# esbuild watches src/, Love2D launches on first build
# Edit src/App.tsx -- save -- see changes instantly
---
Platforms: love2d

Example 2: Dev mode for web target
---
rjit dev web
# Bundles to web/bundle.js, does not launch a runtime
# Serve web/ with any HTTP server to view
---
Platforms: love2d

=== PLATFORM NOTES ===
Love2D:
Dev mode detects whether the project uses a love/ subdirectory (has love/main.lua) or the
project root as the Love2D directory. Love2D is launched with `love <dir>` and stdio is
inherited so Love2D output appears in your terminal. If Love2D exits with a non-zero code,
dev mode reports the error but keeps esbuild watching.

=== CRITICAL RULES ===
- Dev mode does NOT run the linter -- use rjit lint separately or rely on rjit build for lint-gated builds
- The --jsx-dev flag is only added in dev mode, not in production builds
- If the entry point is not found, dev mode exits with an error -- make sure src/main.tsx (or src/main-love.tsx) exists
- Ctrl+C cleanly shuts down both esbuild and the Love2D process
