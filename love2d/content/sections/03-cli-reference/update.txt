=== OVERVIEW ===
`rjit update` copies the latest framework runtime files from the CLI's bundled runtime
into your project. It updates lua/, lib/, reactjit/ (framework TypeScript source), fonts/,
data/, and bin/ directories. It never touches your application code in src/.

The command merges files: it copies every file from the CLI runtime into the project,
creating directories as needed, but preserves any user-created files that do not exist in
the source. This means custom Lua modules you add to lua/ are kept.

Symlinks are protected. If a destination directory is a symlink (e.g., the storybook's
lua/ symlink to the monorepo root), the update skips that directory and prints a message.
Running update from the storybook directory is explicitly blocked -- it reads from source
via symlinks and should never have stale copies.

The build command (rjit build) runs update automatically before bundling. Use --no-update
to skip this if you manage runtime files manually. In monorepo development, update also
overlays critical source-of-truth files (init.lua, bsod.lua, masks.lua, scene3d.lua,
capabilities.lua) directly from the monorepo root, since the CLI runtime snapshot can lag
behind active development.

=== API / SYNTAX ===
Usage:
  rjit update

No flags or arguments. Must be run from inside a ReactJIT project directory (one that has
main.lua or src/).

Directories synced:
  lua/                   Lua runtime (layout engine, painter, bridge, events, etc.)
  lib/                   Native libraries (libquickjs.so)
  reactjit/              Framework TypeScript source (core, renderer, optional packages)
  fonts/                 Font packs (also copied to love/fonts/ if love/ subdirectory exists)
  data/                  Data files (dictionary, etc.; also copied to love/data/ if applicable)
  bin/                   Bundled binaries (tor)

Source of runtime files:
  cli/runtime/           Populated by `make cli-setup` from monorepo source-of-truth files

=== EXAMPLES ===
Example 1: Update after framework changes
---
# After editing lua/ or packages/ in the monorepo:
make cli-setup

# Then in your consumer project:
cd examples/my-app
rjit update
rjit build
---
Platforms: love2d

Example 2: Automatic update during build
---
# rjit build runs update automatically:
cd examples/my-app
rjit build
# Output includes:
#   ok  lua/ (42 files merged, user files preserved)
#   ok  lib/
#   ok  reactjit/
---
Platforms: love2d

=== CRITICAL RULES ===
- Never edit files in a project's lua/ or reactjit/ directly -- those are disposable copies. Edit the source of truth (monorepo root lua/, packages/) and then run make cli-setup + rjit update
- Never run rjit update from the storybook directory -- it reads from source via symlinks
- Symlink destinations are skipped, not overwritten -- this protects the storybook's source-of-truth links
- Update merges files (overwrites framework files, preserves user files) -- it does not delete files that exist in the project but not in the CLI runtime
