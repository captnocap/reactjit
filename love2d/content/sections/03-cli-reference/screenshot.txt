=== OVERVIEW ===
`rjit screenshot` captures a single frame of your application as a PNG image. It runs the
full build pipeline (lint, bundle, launch Love2D), waits for the first frame to render,
saves the screenshot, and exits. This is useful for visual verification in CI pipelines,
documentation, and quick layout checks without manually launching Love2D.

The command runs three steps:
  1. Lint check (errors block the screenshot)
  2. Bundle with esbuild (same flags as rjit build)
  3. Launch Love2D in screenshot mode with REACTJIT_SCREENSHOT=1

On Linux, if xvfb-run is available, the screenshot runs headlessly (no window appears).
Without xvfb-run, a Love2D window briefly appears to render the frame.

The process times out after 15 seconds. If Love2D fails to initialize or render within
that window, the command exits with an error.

=== API / SYNTAX ===
Usage:
  rjit screenshot [flags]

Flags:
  --output <path>        Output file path (default: screenshot.png in project root)
  -o <path>              Short form of --output

Environment variables set for Love2D:
  REACTJIT_SCREENSHOT=1
  REACTJIT_SCREENSHOT_OUTPUT=<path>

The Love2D runtime detects REACTJIT_SCREENSHOT and saves a frame to the specified path,
then prints SCREENSHOT_SAVED: to stdout and exits.

=== EXAMPLES ===
Example 1: Default screenshot
---
cd my-app
rjit screenshot
# Output: my-app/screenshot.png
---
Platforms: love2d

Example 2: Custom output path
---
rjit screenshot --output /tmp/preview.png
---
Platforms: love2d

Example 3: Screenshot in CI (headless)
---
# Install xvfb for headless operation
sudo apt-get install xvfb

# Screenshot runs without a display
rjit screenshot --output artifacts/screenshot.png
---
Platforms: love2d

=== PLATFORM NOTES ===
Love2D:
Screenshot mode uses xvfb-run on Linux when available, allowing headless capture on CI
servers without a display. On macOS and Windows, a Love2D window briefly appears to render
the frame. The 15-second timeout accounts for shader compilation and asset loading on first
launch.

=== CRITICAL RULES ===
- Lint errors block screenshots -- fix lint issues first
- The screenshot times out after 15 seconds -- if your app has slow initialization, the capture may fail
- The output file is overwritten without warning
- Love2D must be installed and in PATH (install from https://love2d.org)
