=== OVERVIEW ===
Emulator renders a Libretro-based game emulation surface. It displays the video output of a Libretro core (NES, SNES, Game Boy, etc.) as a node in the React tree. The Lua side manages core loading, ROM loading, and frame stepping.

The component is a surface element — without explicit dimensions, it falls back to 1/4 of parent size (proportional surface fallback). ROMs can be loaded via the `src` prop or by dropping a file onto the emulator window.

Emulator state (running cores, save states, RAM) is managed entirely on the Lua side. Use `useLoveRPC('libretro:list')` to query active instances and `useLoveRPC('libretro:memory')` to read emulator RAM for live game data overlays.

=== API / SYNTAX ===
```tsx
import { Emulator } from '@reactjit/core';
```

Props:
| Prop        | Type                                              | Default   | Description                         |
|-------------|----------------------------------------------------|-----------|-------------------------------------|
| src         | string                                             | undefined | ROM file path (e.g. "game.nes")     |
| playing     | boolean                                            | true      | Whether emulation is running        |
| w           | number \| string                                   | undefined | Width shorthand                     |
| h           | number \| string                                   | undefined | Height shorthand                    |
| style       | Style                                              | undefined | Full style object                   |
| onROMLoaded | (event: { filename, fileSize, filePath }) => void  | undefined | Fired when a ROM loads              |

=== PLATFORM NOTES ===
Love2D:
  Requires a Libretro core installed on the system (e.g. nestopia_libretro.so for NES). The Lua capabilities system manages core lifecycle. ROM paths are relative to the Love2D filesystem. Input forwarding (gamepad/keyboard) is handled by the Lua side.

=== CRITICAL RULES ===
- A Libretro core must be installed and accessible on the system for the target console. Without a core, the emulator surface renders but does nothing.
- The `src` prop is optional. If omitted, the emulator waits for a ROM to be dropped onto the window.
- Emulator is a surface element. Give it explicit dimensions or flexGrow to size it properly.
