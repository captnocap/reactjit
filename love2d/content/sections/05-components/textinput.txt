=== OVERVIEW ===
Input is the unified text input component. It renders as either a single-line `TextInput` or a multi-line `TextEditor` host element depending on props. Lua owns all interaction: cursor movement, text selection, keyboard handling, and rendering. Only boundary events cross the bridge: focus, blur, submit, and change.

By default, Input is single-line. Set `multiline={true}` for multi-line editing. Editor features (`lineNumbers`, `syntaxHighlight`, `tooltipLevel`, `sessionId`) imply multiline automatically and escalate to the TextEditor host element.

Input supports both controlled (`value` prop) and uncontrolled (`defaultValue` prop) modes. In uncontrolled mode, the text lives entirely in Lua and only crosses the bridge on submit or blur.

The `keystrokeTarget` and `submitTarget` props enable the proxy input pattern: raw keystrokes are forwarded to another Lua node (e.g. a PTY canvas) without crossing the bridge. The Input displays a value read from an external source while forwarding all keyboard input to the target. This is how terminal input bars work.

`TextInput` is a deprecated alias that forwards to `Input`.

=== API / SYNTAX ===
```tsx
import { Input } from '@reactjit/core';
```

Props:
| Prop               | Type                         | Default     | Description                                        |
|--------------------|------------------------------|-------------|----------------------------------------------------|
| value              | string                       | undefined   | Controlled text value                              |
| defaultValue       | string                       | undefined   | Initial text for uncontrolled mode                 |
| onChangeText       | (text: string) => void       | undefined   | Fires on blur and submit with final text           |
| onSubmit           | (text: string) => void       | undefined   | Fires when Enter is pressed (single-line) or submitted |
| onFocus            | () => void                   | undefined   | Fires when input gains focus                       |
| onBlur             | (text: string) => void       | undefined   | Fires when input loses focus, receives current text|
| onLiveChange       | (text: string) => void       | undefined   | Debounced per-keystroke callback (Lua-side debounce) |
| liveChangeDebounce | number                       | 300         | Debounce delay in ms for onLiveChange              |
| onChange           | (text: string) => void       | undefined   | Fires after idle timeout (editor mode)             |
| changeDelay        | number                       | 3           | Seconds of idle before onChange fires              |
| placeholder        | string                       | undefined   | Placeholder text                                   |
| placeholderColor   | Color                        | undefined   | Placeholder text color                             |
| maxLength          | number                       | undefined   | Maximum character count                            |
| multiline          | boolean                      | false       | Enable multi-line editing                          |
| editable           | boolean                      | true        | Whether text can be edited                         |
| secureTextEntry    | boolean                      | false       | Mask text (password field)                         |
| live               | boolean                      | false       | Forward keystrokes via onLiveChange/onChangeText   |
| keyboardType       | string                       | 'default'   | On-screen keyboard hint                            |
| autoFocus          | boolean                      | false       | Focus on mount                                     |
| cursorColor        | Color                        | undefined   | Cursor color                                       |
| style              | Style                        | undefined   | Container style                                    |
| textStyle          | Style                        | undefined   | Text style (fontSize, fontFamily, color, fontWeight)|

Proxy Props:
| Prop              | Type   | Default   | Description                                           |
|-------------------|--------|-----------|-------------------------------------------------------|
| keystrokeTarget   | string | undefined | Node type to forward raw keystrokes to (Lua-to-Lua)   |
| submitTarget      | string | undefined | Node type to dump submitted text to on Enter           |
| escapeTarget      | string | undefined | Node type to forward Escape key to                     |

Editor Props (imply multiline):
| Prop             | Type    | Default | Description                                   |
|------------------|---------|---------|-----------------------------------------------|
| lineNumbers      | boolean | false   | Show line numbers in gutter                   |
| syntaxHighlight  | boolean | false   | Enable JSX syntax highlighting                |
| tooltipLevel     | string  | undefined | Hover tooltip verbosity ('beginner' \| 'guided' \| 'clean') |
| sessionId        | string  | undefined | PTY session ID for keystroke passthrough       |
| submitOnEnter    | boolean | false   | Enter submits, Shift+Enter inserts newline     |
| spellCheck       | boolean | false   | Enable inline spell checking                   |
| wordWrap         | boolean | false   | Wrap long lines at viewport edge               |

=== EXAMPLES ===
Example 1: Basic single-line input
---
```tsx
const colors = useThemeColors();
const [name, setName] = useState('');

<Input
  placeholder="Enter your name"
  onSubmit={(text) => setName(text)}
  style={{ padding: 8, backgroundColor: colors.surface, borderRadius: 6 }}
  textStyle={{ color: colors.text, fontSize: 14 }}
/>
```
---
Platforms: love2d

Example 2: Controlled input with live updates
---
```tsx
const colors = useThemeColors();
const [query, setQuery] = useState('');

<Input
  value={query}
  onLiveChange={(text) => setQuery(text)}
  liveChangeDebounce={200}
  placeholder="Search..."
  autoFocus
  style={{ padding: 8, backgroundColor: colors.surface, borderRadius: 6 }}
  textStyle={{ color: colors.text, fontSize: 14 }}
/>
```
---
Platforms: love2d

Example 3: Multi-line editor with line numbers
---
```tsx
const colors = useThemeColors();

<Input
  defaultValue="function hello()\n  print('world')\nend"
  multiline
  lineNumbers
  syntaxHighlight
  style={{ flexGrow: 1, backgroundColor: colors.bg }}
  textStyle={{ fontFamily: 'monospace', fontSize: 12, color: colors.text }}
/>
```
---
Platforms: love2d

Example 4: Proxy input pattern (keystroke forwarding)
---
```tsx
const colors = useThemeColors();
const promptText = usePromptText();

<Input
  autoFocus
  value={promptText}
  keystrokeTarget="ClaudeCanvas"
  submitTarget="ClaudeCanvas"
  style={{ padding: 8, backgroundColor: colors.surface }}
  textStyle={{ color: colors.text, fontSize: 14 }}
/>
```
---
Platforms: love2d

Example 5: Password field
---
```tsx
const colors = useThemeColors();

<Input
  placeholder="Password"
  secureTextEntry
  onSubmit={(text) => authenticate(text)}
  style={{ padding: 8, backgroundColor: colors.surface, borderRadius: 6 }}
  textStyle={{ color: colors.text, fontSize: 14 }}
/>
```

=== PLATFORM NOTES ===
Love2D:
  Lua owns the cursor, selection, and text rendering. React never touches the text buffer directly. In single-line mode, the TextInput host element handles horizontal scrolling for overflow text. In editor mode, the TextEditor host element adds gutter line numbers, syntax highlighting, and code intelligence.

  The proxy input pattern (keystrokeTarget/submitTarget) is entirely Lua-to-Lua. Keystrokes never cross the JS bridge, so there is zero latency between key press and target response.

  onLiveChange debouncing runs in Lua. No per-keystroke bridge traffic occurs during rapid typing -- only the debounced result crosses.

=== CRITICAL RULES ===
- An Input with keystrokeTarget/submitTarget is NOT a text editor. It is a display rectangle that forwards keystrokes. It must never maintain its own text buffer. Set `value` from an external source and let the target own the state.
- Do not mix `value` (controlled) with `defaultValue` (uncontrolled). Pick one mode.
- Editor features (lineNumbers, syntaxHighlight, tooltipLevel, sessionId) automatically imply multiline. You do not need to set `multiline={true}` alongside them.
- `onChangeText` fires on both blur and submit. If you only want the submit value, use `onSubmit` instead.
- `textStyle` only extracts fontSize, fontFamily, fontWeight, and color. Other style properties go in `style`.
