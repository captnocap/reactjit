=== OVERVIEW ===
DebugOverlay is a context-based wrapper that enables visual layout debugging for a subtree. Wrap any section of your component tree to add colored borders to every DebugBox inside it, making flex layout boundaries visible at a glance.

DebugOverlay itself is just a context provider — it renders no extra nodes. DebugBox is a drop-in replacement for Box that reads the debug context and conditionally adds a colored border.

The useDebugOverlay hook returns the current debug context (enabled, borderColor, showLabels) for components that need to react to debug state.

This is lighter than the full F12 inspector. Use it when you want quick visual feedback on a specific section's layout without opening devtools.

=== API / SYNTAX ===
```tsx
import { DebugOverlay, DebugBox, useDebugOverlay } from '@reactjit/core';
```

DebugOverlay Props:
| Prop        | Type    | Default                    | Description                        |
|-------------|---------|----------------------------|------------------------------------|
| children    | ReactNode | required                 | Subtree to debug                   |
| enabled     | boolean | true                       | Toggle debug borders on/off        |
| borderColor | string  | 'rgba(255, 100, 100, 0.6)'| Border color for debug outlines    |
| showLabels  | boolean | false                      | Show dimension labels on each Box  |

DebugBox Props:
Same as Box. When inside an enabled DebugOverlay, automatically adds a 1px border in the debug color.

useDebugOverlay():
Returns `{ enabled: boolean, borderColor: string, showLabels: boolean }`.

=== EXAMPLES ===
Example 1: Visualize a row layout
---
```tsx
<DebugOverlay>
  <Box style={{ flexDirection: 'row', width: '100%' }}>
    <DebugBox style={{ flexGrow: 1 }}>
      <Text style={{ fontSize: 14 }}>Left panel</Text>
    </DebugBox>
    <DebugBox style={{ flexGrow: 2 }}>
      <Text style={{ fontSize: 14 }}>Right panel</Text>
    </DebugBox>
  </Box>
</DebugOverlay>
```
---
Platforms: love2d

Example 2: Toggle debug mode
---
```tsx
const [debug, setDebug] = useState(false);

<Pressable onPress={() => setDebug(d => !d)}>
  <Text style={{ fontSize: 12 }}>{`Debug: ${debug ? 'ON' : 'OFF'}`}</Text>
</Pressable>

<DebugOverlay enabled={debug} borderColor="rgba(0, 200, 255, 0.5)">
  <MyComplexLayout />
</DebugOverlay>
```
---
Platforms: love2d

Example 3: Custom component using the hook
---
```tsx
function MyPanel({ children }) {
  const { enabled, borderColor } = useDebugOverlay();

  return (
    <Box style={{
      padding: 8,
      borderWidth: enabled ? 2 : 0,
      borderColor: enabled ? borderColor : undefined,
    }}>
      {children}
    </Box>
  );
}
```
---
Platforms: love2d

=== CRITICAL RULES ===
- DebugOverlay only affects DebugBox components, not regular Box. Replace Box with DebugBox in the sections you want to visualize.
- DebugOverlay is a context provider with zero rendering cost when disabled. Safe to leave in production code with `enabled={false}`.
