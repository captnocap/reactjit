=== OVERVIEW ===
Portal and PortalHost work together to render children outside their position in the component tree. A PortalHost wraps a section of your app and collects any Portal content from its descendants. Portal content is rendered as absolute-positioned overlays (zIndex: 1000) at the PortalHost level.

This is useful for dropdown menus, tooltips, and other UI that needs to visually escape its parent's clipping or stacking context — without restructuring your component hierarchy.

Portal registers its children with the nearest PortalHost via React context. When the Portal unmounts, its content is automatically removed from the host.

=== API / SYNTAX ===
```tsx
import { Portal, PortalHost } from '@reactjit/core';
```

PortalHost Props:
| Prop     | Type      | Default  | Description                        |
|----------|-----------|----------|------------------------------------|
| children | ReactNode | required | App content that may contain Portals|

Portal Props:
| Prop     | Type      | Default   | Description                    |
|----------|-----------|-----------|--------------------------------|
| children | ReactNode | required  | Content to render at the host  |
| name     | string    | undefined | Optional name for the portal   |

=== EXAMPLES ===
Example 1: Basic portal setup
---
```tsx
const colors = useThemeColors();

function App() {
  return (
    <PortalHost>
      <Box fill>
        <Toolbar />
        <Content />
      </Box>
    </PortalHost>
  );
}

function Content() {
  const [showMenu, setShowMenu] = useState(false);

  return (
    <Box padding={16}>
      <Pressable onPress={() => setShowMenu(true)} style={{ padding: 8, backgroundColor: colors.surface }}>
        <Text style={{ color: colors.text }}>Show Menu</Text>
      </Pressable>

      {showMenu && (
        <Portal>
          <Pressable onPress={() => setShowMenu(false)} style={{ width: '100%', height: '100%' }}>
            <Box style={{ position: 'absolute', top: 100, left: 50, padding: 12, backgroundColor: colors.bgElevated, borderRadius: 8 }}>
              <Text style={{ color: colors.text }}>Menu Item 1</Text>
              <Text style={{ color: colors.text }}>Menu Item 2</Text>
            </Box>
          </Pressable>
        </Portal>
      )}
    </Box>
  );
}
```

Example 2: Named portal
---
```tsx
<Portal name="dropdown-menu">
  <Box style={{ position: 'absolute', top: anchorY, left: anchorX, backgroundColor: colors.bgElevated, borderRadius: 8, padding: 8 }}>
    {menuItems.map((item, i) => (
      <Pressable key={i} onPress={() => onSelect(item)} style={{ padding: 8 }}>
        <Text style={{ color: colors.text }}>{item.label}</Text>
      </Pressable>
    ))}
  </Box>
</Portal>
```

=== PLATFORM NOTES ===
Love2D:
  Portal content is rendered as absolute-positioned children of the PortalHost with zIndex 1000. The Lua painter sorts by zIndex, so portal content draws above regular content. Each portal gets its own full-viewport Box wrapper.

=== CRITICAL RULES ===
- A PortalHost must be an ancestor of any Portal that uses it. Without a PortalHost in the tree, Portal content will not render.
- Portal content is positioned absolute at zIndex 1000. You are responsible for positioning the content within the viewport (using `position: 'absolute'` with `top`/`left`).
- For simple overlay needs, Modal may be more appropriate — it includes backdrop handling and dismiss logic built in.
