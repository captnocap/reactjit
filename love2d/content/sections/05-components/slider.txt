=== OVERVIEW ===
Slider renders a draggable range control. All interaction (drag, click-to-position) and painting (track, active track, thumb) are handled entirely in Lua for zero-latency response. Value changes are pushed back to React via events.

Slider supports both controlled (`value` prop) and uncontrolled (`defaultValue` prop) modes. The track, active track, and thumb colors are customizable. By default, Slider fills the available width and sizes its height from the thumb size.

=== API / SYNTAX ===
```tsx
import { Slider } from '@reactjit/core';
```

Props:
| Prop             | Type                       | Default   | Description                             |
|------------------|----------------------------|-----------|-----------------------------------------|
| value            | number                     | undefined | Controlled current value                |
| defaultValue     | number                     | 0         | Initial value for uncontrolled mode     |
| onValueChange    | (value: number) => void    | undefined | Fires on every value change during drag |
| onSlidingStart   | () => void                 | undefined | Fires when drag begins                  |
| onSlidingEnd     | (value: number) => void    | undefined | Fires when drag ends with final value   |
| minimumValue     | number                     | 0         | Minimum selectable value                |
| maximumValue     | number                     | 1         | Maximum selectable value                |
| step             | number                     | undefined | Snap to step increments                 |
| disabled         | boolean                    | false     | Disable interaction                     |
| style            | Style                      | undefined | Container style                         |
| trackColor       | Color                      | '#333333' | Inactive track color                    |
| activeTrackColor | Color                      | '#4A90D9' | Active (filled) track color             |
| thumbColor       | Color                      | '#ffffff' | Thumb color                             |
| thumbSize        | number                     | 20        | Thumb diameter in pixels                |
| trackHeight      | number                     | 4         | Track height in pixels                  |
| vertical         | boolean                    | false     | Render as vertical slider               |

Example 1: Basic slider
---
```tsx
const colors = useThemeColors();
const [volume, setVolume] = useState(0.5);

<Slider
  value={volume}
  onValueChange={setVolume}
  activeTrackColor={colors.primary}
/>
```

=== PLATFORM NOTES ===
Love2D:
  Drag and click interaction happen in Lua at paint framerate. The thumb position updates on the same frame as the mouse event -- there is no bridge latency during dragging. The onValueChange callback fires per-frame during active drag, so avoid expensive work in the handler.

  The slider auto-sizes its width to 100% of the parent unless an explicit width is set via style. Height is derived from thumbSize plus internal padding.

=== CRITICAL RULES ===
- Do not hardcode a pixel width on Slider. It defaults to `width: '100%'` which adapts to the parent container. If you must constrain it, set width on a wrapping Box instead.
- `onValueChange` fires on every frame during drag. Keep the handler lightweight. Use `onSlidingEnd` for expensive operations like saving to storage.
- When using `step`, the value snaps to the nearest step increment. The step value must evenly divide the range (maximumValue - minimumValue) for predictable behavior.
