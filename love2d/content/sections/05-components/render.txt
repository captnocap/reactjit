=== OVERVIEW ===
Render is a surface element that captures and displays live video from external sources: screen regions, webcams, HDMI capture cards, application windows, virtual displays, and even VMs booted from ISO/disk images.

The `source` prop is a string identifier that determines what to capture:
- `"screen:N"` — screen capture via XShm (fast path)
- `"cam:N"` — webcam via v4l2
- `"hdmi:N"` — HDMI capture card
- `"window:Title"` — capture a window by its title
- `"/dev/videoN"` — direct v4l2 device
- `"display"` — virtual display (Xephyr/Xvfb, apps render to it)
- `"file.iso"` or `"vm:path.qcow2"` — boot a VM from ISO or disk image

For VM sources, additional configuration props control RAM and CPU allocation. The `interactive` prop enables mouse/keyboard input forwarding to the capture source (default for display and VM sources).

Render is a surface element — without explicit dimensions, it falls back to 1/4 of parent size.

=== API / SYNTAX ===
```tsx
import { Render } from '@reactjit/core';
```

Props:
| Prop        | Type                                               | Default      | Description                                   |
|-------------|-----------------------------------------------------|--------------|-----------------------------------------------|
| source      | string                                              | required     | Capture source identifier (see overview)      |
| fps         | number                                              | 30           | Capture framerate                             |
| resolution  | string                                              | '1280x720'   | Capture resolution (e.g. '1920x1080')         |
| interactive | boolean                                             | false/true   | Enable input forwarding to source             |
| muted       | boolean                                             | true         | Suppress audio from source                    |
| objectFit   | 'fill' \| 'contain' \| 'cover'                      | 'contain'    | Scaling mode                                  |
| vmMemory    | number                                              | 2048         | VM RAM in MB (VM sources only)                |
| vmCpus      | number                                              | 2            | VM CPU count (VM sources only)                |
| w           | number \| string                                    | undefined    | Width shorthand                               |
| h           | number \| string                                    | undefined    | Height shorthand                              |
| radius      | number                                              | undefined    | Border radius shorthand                       |
| style       | Style                                               | undefined    | Full style object                             |
| onReady     | (event?: { displayNumber?, vmInfo? }) => void       | undefined    | Source is ready to display                    |
| onError     | (event: { message: string }) => void                | undefined    | Capture error                                 |
| onFrame     | (event: { frameNumber: number }) => void            | undefined    | Frame received                                |
| onClick     | (event: LoveEvent) => void                          | undefined    | Click on the render surface                   |

=== PLATFORM NOTES ===
Love2D:
  Render routes through the Native host element and is rendered by Lua's capture pipeline. Screen capture uses XShm for zero-copy performance on Linux. Webcam and HDMI use v4l2. VM sources use QEMU with VNC bridging. The `interactive` prop defaults to false for capture sources (screen, cam, hdmi, window) and true for controlled sources (display, vm). All sources run on the Lua side — React only declares the surface.

=== CRITICAL RULES ===
- The `source` prop format determines the capture backend. Invalid source strings will trigger onError.
- Render is a surface element. Give it explicit dimensions or flexGrow.
- VM sources (ISO, qcow2) require QEMU installed on the system. Without it, onError fires.
- The `interactive` prop defaults differ by source type. Explicitly set it to avoid surprises.
- Multiple Render elements capturing the same source share the underlying capture stream.
