=== OVERVIEW ===
ScrollView wraps content in a scrollable container. Lua handles the scroll translation, scissor clipping, scrollbar indicators, and wheel/touch events. React just declares the container and its children.

ScrollView supports vertical (default) and horizontal scrolling. It exposes an imperative `scrollTo()` method via `React.forwardRef` for programmatic scroll control.

ScrollView is not a surface — it does not get proportional fallback sizing. You must give it an explicit height (or `flexGrow`) to define its scroll viewport. Without a known viewport size, there is nothing to scroll within.

=== API / SYNTAX ===
```tsx
import { ScrollView } from '@reactjit/core';
```

Props:
| Prop                  | Type                           | Default   | Description                              |
|-----------------------|--------------------------------|-----------|------------------------------------------|
| horizontal            | boolean                        | false     | Scroll horizontally instead of vertically|
| showScrollIndicator   | boolean                        | true      | Show scrollbar indicator                 |
| style                 | Style                          | undefined | Style for the scroll container           |
| contentContainerStyle | Style                          | undefined | Style for the inner content wrapper      |
| onScroll              | (event: ScrollEvent) => void   | undefined | Fires during scroll                      |
| onScrollBegin         | () => void                     | undefined | Fires when scrolling starts              |
| onScrollEnd           | () => void                     | undefined | Fires when scrolling stops               |
| children              | ReactNode                      | undefined | Scrollable content                       |

ScrollEvent:
| Field          | Type   | Description                    |
|----------------|--------|--------------------------------|
| scrollX        | number | Horizontal scroll offset       |
| scrollY        | number | Vertical scroll offset         |
| contentWidth   | number | Total content width            |
| contentHeight  | number | Total content height           |
| viewportWidth  | number | Visible viewport width         |
| viewportHeight | number | Visible viewport height        |

Imperative API (via ref):
| Method   | Signature                                              | Description              |
|----------|--------------------------------------------------------|--------------------------|
| scrollTo | (options: { x?: number; y?: number; animated?: boolean }) => void | Scroll to position |

=== EXAMPLES ===
Example 1: Basic vertical scroll
---
```tsx
const colors = useThemeColors();

<ScrollView style={{ flexGrow: 1 }}>
  {items.map((item, i) => (
    <Box key={i} padding={16} style={{ borderBottomWidth: 1, borderColor: colors.border }}>
      <Text style={{ color: colors.text }}>{item.title}</Text>
    </Box>
  ))}
</ScrollView>
```
---
Platforms: love2d

Example 2: Horizontal scroll
---
```tsx
const colors = useThemeColors();

<ScrollView horizontal style={{ height: 120 }}>
  {images.map((src, i) => (
    <Image key={i} src={src} w={100} h={100} radius={8} style={{ marginRight: 12 }} />
  ))}
</ScrollView>
```
---
Platforms: love2d

Example 3: Imperative scroll control
---
```tsx
const scrollRef = useRef<ScrollViewRef>(null);

<Box>
  <Pressable onPress={() => scrollRef.current?.scrollTo({ y: 0 })}>
    <Text style={{ color: colors.primary }}>Scroll to top</Text>
  </Pressable>
  <ScrollView ref={scrollRef} style={{ flexGrow: 1 }}>
    {content}
  </ScrollView>
</Box>
```
---
Platforms: love2d

Example 4: Tracking scroll position
---
```tsx
const [scrollY, setScrollY] = useState(0);
const colors = useThemeColors();

<ScrollView
  style={{ flexGrow: 1 }}
  onScroll={(e) => setScrollY(e.scrollY)}
>
  {children}
</ScrollView>
```
---
Platforms: love2d

=== CRITICAL RULES ===
- ScrollView needs explicit height or `flexGrow` to define its viewport. It is excluded from proportional surface fallback. Without a known size, nothing scrolls.
- The Lua painter handles scissor clipping. Content outside the viewport is clipped automatically.
- Do not nest ScrollViews in the same direction. Vertical-inside-vertical or horizontal-inside-horizontal creates ambiguous scroll targets.
