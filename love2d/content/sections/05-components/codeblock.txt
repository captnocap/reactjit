=== OVERVIEW ===
CodeBlock is a read-only, syntax-highlighted code display. It renders as a `CodeBlock` host element that Lua paints directly -- tokenization, coloring, line rendering, and scrolling all happen in Lua with no bridge involvement after mount.

Pass code via the `code` string prop. The `language` prop controls syntax highlighting; it defaults to `'auto'` which attempts to detect the language from content. Explicit language values produce more reliable highlighting.

CodeBlock is a Lua-owned primitive. React declares what to display; Lua does all the rendering work. This means CodeBlock performs well even with large files because no per-line React elements are created.

=== API / SYNTAX ===
```tsx
import { CodeBlock } from '@reactjit/core';
```

Props:
| Prop     | Type      | Default | Description                                      |
|----------|-----------|---------|--------------------------------------------------|
| code     | string    | undefined | Source code to display                          |
| language | string    | 'auto'  | Language for syntax highlighting                 |
| fontSize | number    | 10      | Font size in pixels                              |
| style    | Style     | undefined | Container style (padding defaults to 10)        |
| children | ReactNode | undefined | Alternative to code prop (Text nodes)           |

Supported languages include: typescript, javascript, lua, json, css, html, markdown, python, rust, go, and others. When set to `'auto'`, Lua inspects the content for heuristic detection.

=== EXAMPLES ===
Example 1: Basic code display
---
```tsx
const colors = useThemeColors();

<CodeBlock
  code={`function greet(name)\n  print("Hello, " .. name)\nend`}
  language="lua"
  style={{ backgroundColor: colors.bg, borderRadius: 8 }}
/>
```
---
Platforms: love2d

Example 2: TypeScript with custom font size
---
```tsx
const colors = useThemeColors();

const snippet = `import { Box, Text } from '@reactjit/core';

export function App() {
  return (
    <Box fill>
      <Text style={{ fontSize: 24 }}>Hello</Text>
    </Box>
  );
}`;

<CodeBlock
  code={snippet}
  language="typescript"
  fontSize={12}
  style={{ flexGrow: 1, backgroundColor: colors.bg, borderRadius: 8 }}
/>
```
---
Platforms: love2d

Example 3: Auto-detected language
---
```tsx
const colors = useThemeColors();

<CodeBlock
  code={jsonString}
  style={{ padding: 16, backgroundColor: colors.surface, borderRadius: 8 }}
/>
```

=== PLATFORM NOTES ===
Love2D:
  CodeBlock renders entirely in Lua. The tokenizer runs once when the code prop changes and caches the result. Scrolling within the code block (when content overflows) is handled by Lua's scroll system.

  Always pass code as a static string constant or a memoized value. If the code string has a new identity every render (e.g. created inline in the render body), the tokenizer re-runs every frame, which wastes CPU. Hoist string constants or use useMemo.

=== CRITICAL RULES ===
- Pass code as a string prop, not as children. The `code` prop is the preferred API. Children (Text nodes) also work but create unnecessary React elements.
- Hoist code strings to constants or memoize them. A new string identity per render forces Lua to re-tokenize every frame.
- CodeBlock is read-only. For editable code, use Input with `syntaxHighlight` or MonacoMirror.
- Do not set explicit `language` prop when the code is empty. Auto-detection handles this gracefully.
