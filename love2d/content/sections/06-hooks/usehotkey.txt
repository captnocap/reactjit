=== OVERVIEW ===
useHotkey registers a global keyboard shortcut that fires a callback when the key combination is pressed. It works even when a TextEditor is focused (for unhandled combos like Ctrl+Z). The combo string supports modifier keys: ctrl, shift, alt, meta (or cmd/gui).

The hook subscribes to bridge `keydown` events and matches against the parsed combo. The handler ref is kept fresh automatically -- no stale closures.

=== API / SYNTAX ===
```tsx
import { useHotkey } from '@reactjit/core';
```

Signature:
```tsx
function useHotkey(
  combo: string,
  handler: (event: LoveEvent) => void,
  options?: { enabled?: boolean }
): void
```

Parameters:
| Parameter | Type                        | Default   | Description                      |
|-----------|-----------------------------|-----------|----------------------------------|
| combo     | string                      | required  | Key combination (e.g. 'ctrl+s') |
| handler   | (event: LoveEvent) => void  | required  | Callback on match                |
| options   | { enabled?: boolean }       | undefined | Set enabled=false to disable     |

Combo format:
- Single key: `'escape'`, `'space'`, `'f1'`
- With modifiers: `'ctrl+s'`, `'ctrl+shift+z'`, `'alt+tab'`
- Modifier names: `ctrl`/`control`, `shift`, `alt`, `meta`/`cmd`/`gui`

=== EXAMPLES ===
Example 1: Undo/Redo
---
```tsx
import { useHotkey } from '@reactjit/core';

function Editor() {
  useHotkey('ctrl+z', () => undo());
  useHotkey('ctrl+shift+z', () => redo());
  useHotkey('ctrl+s', () => save());

  return <Text>Editor with keyboard shortcuts</Text>;
}
```
---
Platforms: love2d

Example 2: Conditional hotkey
---
```tsx
import { useHotkey } from '@reactjit/core';

function Modal({ isOpen, onClose }: { isOpen: boolean; onClose: () => void }) {
  useHotkey('escape', () => onClose(), { enabled: isOpen });

  if (!isOpen) return null;

  return (
    <Box style={{ backgroundColor: 'rgba(0,0,0,0.5)', flexGrow: 1 }}>
      <Text>Press Escape to close</Text>
    </Box>
  );
}
```
---
Platforms: love2d

Example 3: Tool switching
---
```tsx
import { useHotkey } from '@reactjit/core';

function ToolPalette() {
  const [tool, setTool] = useState('brush');

  useHotkey('1', () => setTool('brush'));
  useHotkey('2', () => setTool('eraser'));
  useHotkey('3', () => setTool('fill'));

  return <Text>{`Active: ${tool}`}</Text>;
}
```
---
Platforms: love2d

=== CRITICAL RULES ===
- The combo string is case-insensitive. `'Ctrl+S'` and `'ctrl+s'` are equivalent.
- Modifier keys must come before the final key in the combo string. `'s+ctrl'` will not work.
- When `enabled` is false, the subscription is removed entirely. This is more efficient than checking a condition inside the handler.
- Multiple useHotkey calls with the same combo all fire. There is no conflict resolution -- be careful not to register the same combo in multiple mounted components unless intended.
