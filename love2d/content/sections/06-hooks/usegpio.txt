=== OVERVIEW ===
GPIO hooks wrap the declarative GPIO capability components (Pin, PWM, SerialPort, I2CDevice) as imperative hooks. Each hook renders an invisible capability component internally and returns state plus controls. Use these when you want to interact with hardware without writing JSX for the hardware parts.

The family includes:
- usePin -- read/write digital GPIO pins with edge detection
- usePWM -- pulse-width modulation output control
- useSerial -- serial port communication (UART)
- useI2C -- I2C bus device reading

Each hook returns an `element` property that MUST be rendered somewhere in your component tree. This element is the invisible capability node that owns the hardware lifecycle.

=== API / SYNTAX ===
```tsx
import { usePin, usePWM, useSerial, useI2C } from '@reactjit/core';
```

usePin:
```tsx
function usePin(pin: number, mode?: 'input' | 'output', opts?: UsePinOptions): UsePinResult

interface UsePinOptions {
  chip?: string;
  pull?: 'none' | 'up' | 'down';
  edge?: 'none' | 'rising' | 'falling' | 'both';
  activeLow?: boolean;
}

interface UsePinResult {
  value: boolean;
  setValue: (v: boolean) => void;
  lastEdge: string | null;
  element: React.ReactElement;   // MUST render this
}
```

usePWM:
```tsx
function usePWM(pin: number, opts?: UsePWMOptions): UsePWMResult

interface UsePWMOptions {
  chip?: string;
  frequency?: number;
  enabled?: boolean;
}

interface UsePWMResult {
  duty: number;
  setDuty: (d: number) => void;
  frequency: number;
  setFrequency: (f: number) => void;
  element: React.ReactElement;   // MUST render this
}
```

useSerial:
```tsx
function useSerial(port: string, baud?: number, opts?: UseSerialOptions): UseSerialResult

interface UseSerialOptions {
  dataBits?: number;
  stopBits?: number;
  parity?: 'none' | 'even' | 'odd';
  flowControl?: 'none' | 'hardware';
}

interface UseSerialResult {
  lastLine: string | null;
  lines: string[];
  lastData: string | null;
  send: (data: string) => void;
  element: React.ReactElement;   // MUST render this
}
```

useI2C:
```tsx
function useI2C(bus: number, address: number, opts?: UseI2COptions): UseI2CResult

interface UseI2COptions {
  register?: number;
  readLength?: number;
  pollInterval?: number;
  enabled?: boolean;
}

interface UseI2CResult {
  value: number;
  bytes: number[];
  element: React.ReactElement;   // MUST render this
}
```

=== EXAMPLES ===
Example 1: Read a button and control an LED
---
```tsx
import { Box, Text } from '@reactjit/core';
import { usePin } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function ButtonAndLED() {
  const colors = useThemeColors();
  const button = usePin(17, 'input', { pull: 'up', edge: 'both' });
  const led = usePin(27, 'output');

  return (
    <Box style={{ gap: 8, padding: 16 }}>
      {button.element}
      {led.element}
      <Text style={{ color: colors.text }}>
        {`Button: ${button.value ? 'Pressed' : 'Released'}`}
      </Text>
    </Box>
  );
}
```
---
Platforms: love2d

Example 2: PWM brightness control
---
```tsx
import { Box, Text } from '@reactjit/core';
import { usePWM } from '@reactjit/core';
import { Slider } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function LEDBrightness() {
  const colors = useThemeColors();
  const pwm = usePWM(18, { frequency: 1000 });

  return (
    <Box style={{ gap: 8, padding: 16 }}>
      {pwm.element}
      <Text style={{ color: colors.text }}>{`Duty: ${(pwm.duty * 100).toFixed(0)}%`}</Text>
      <Slider value={pwm.duty} onValueChange={pwm.setDuty} min={0} max={1} step={0.01} />
    </Box>
  );
}
```
---
Platforms: love2d

Example 3: Serial port monitor
---
```tsx
import { Box, Text, ScrollView } from '@reactjit/core';
import { useSerial } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function SerialMonitor() {
  const colors = useThemeColors();
  const serial = useSerial('/dev/ttyUSB0', 115200);

  return (
    <Box style={{ flexGrow: 1, padding: 16 }}>
      {serial.element}
      <ScrollView style={{ flexGrow: 1 }}>
        {serial.lines.map((line, i) => (
          <Text key={i} style={{ color: colors.text, fontSize: 12 }}>{line}</Text>
        ))}
      </ScrollView>
    </Box>
  );
}
```
---
Platforms: love2d

=== CRITICAL RULES ===
- You MUST render the `element` property in your component tree. Without it, the capability node is never created and no hardware interaction occurs.
- GPIO requires appropriate system permissions. Linux GPIO needs /sys/class/gpio access or gpiod. Serial requires access to the device file.
- useSerial.send() is currently a placeholder -- serial writes need to go through the capability's Lua side via bridge.rpc() for now.
- useI2C address is a number (hex notation: 0x48 for a temperature sensor, for example).
