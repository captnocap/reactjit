=== METADATA ===
title: useLoveEvent
description: Subscribe to Lua events with a callback handler
category: Hooks
platforms: love2d

related: useLove, useLoveState, useLoveSend
difficulty: intermediate

=== OVERVIEW ===
useLoveEvent subscribes to a named event from Lua and calls your handler whenever it fires. The handler ref is kept fresh automatically -- no stale closure bugs. The subscription is cleaned up on unmount.

This is the fire-and-forget listener. It does not store state or cause re-renders by itself. If you need the latest event payload as React state, use useLove or useLoveState instead.

=== API / SYNTAX ===
```tsx
import { useLoveEvent } from '@reactjit/core';
```

Signature:
```tsx
function useLoveEvent(eventType: string, handler: (payload: any) => void): void
```

Parameters:
| Parameter | Type                    | Description                         |
|-----------|-------------------------|-------------------------------------|
| eventType | string                  | Bridge event name to subscribe to   |
| handler   | (payload: any) => void  | Callback invoked on each event      |

=== EXAMPLES ===
Example 1: Log entity spawns
---
```tsx
import { useLoveEvent } from '@reactjit/core';

function SpawnLogger() {
  useLoveEvent('entity:spawned', (data) => {
    console.log('Spawned:', data.type, 'at', data.x, data.y);
  });

  return null;
}
```
---
Platforms: love2d

Example 2: Accumulate events with a ref
---
```tsx
import { useLoveEvent } from '@reactjit/core';

function ErrorCollector() {
  const errors = useRef<string[]>([]);
  const [count, setCount] = useState(0);

  useLoveEvent('error', (data) => {
    errors.current.push(data.message);
    setCount(c => c + 1);
  });

  return <Text>{`Errors: ${count}`}</Text>;
}
```
---
Platforms: love2d

Example 3: React to Lua lifecycle events
---
```tsx
import { useLoveEvent } from '@reactjit/core';

function LoadingScreen() {
  const [loaded, setLoaded] = useState(false);

  useLoveEvent('assets:loaded', () => setLoaded(true));

  if (!loaded) {
    return <Text>Loading...</Text>;
  }
  return <App />;
}
```
---
Platforms: love2d

=== CRITICAL RULES ===
- The handler function is stored in a ref and updated every render. You can safely reference current state or props inside it without adding them to a dependency array.
- The subscription re-registers only when eventType changes. Changing the handler function does not cause a re-subscription.
- If you need the event payload as state (for rendering), use useLove or useLoveState. useLoveEvent is for side effects only.
