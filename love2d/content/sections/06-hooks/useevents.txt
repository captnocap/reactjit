=== OVERVIEW ===
A lightweight event bus that lives entirely in JavaScript. No bridge, no Lua, no serialization -- pure in-memory pub/sub. Use it for React-to-React communication between sibling components, feature modules, or state coordinators.

For events that need to cross into Lua, use useLoveEvent/useLoveSend instead.

The family includes:
- useEventBus -- create a bus instance (stable across re-renders)
- useEvent -- subscribe to one or more channels
- useEventState -- subscribe and keep the latest payload as state
- useEmit -- create a bound emitter function for a channel
- useEventChannel -- convenience: bus + emit in one call

=== API / SYNTAX ===
```tsx
import { useEventBus, useEvent, useEventState, useEmit, useEventChannel } from '@reactjit/core';
```

useEventBus:
```tsx
function useEventBus(): EventBus
```

useEvent:
```tsx
function useEvent<T>(bus: EventBus, channel: string | string[], handler: (payload: T, channel: string) => void): void
```

useEventState:
```tsx
function useEventState<T>(bus: EventBus, channel: string, initialValue?: T): T | undefined
```

useEmit:
```tsx
function useEmit(bus: EventBus, channel: string): (payload?: any) => void
```

useEventChannel:
```tsx
function useEventChannel(): [EventBus, (channel: string, payload?: any) => void]
```

EventBus interface:
| Method        | Signature                                    | Description                    |
|---------------|----------------------------------------------|--------------------------------|
| emit          | (channel, payload?) => void                  | Fire event on a channel        |
| on            | (channel, handler) => unsubscribe            | Subscribe to a channel         |
| once          | (channel, handler) => unsubscribe            | Subscribe for one event only   |
| clear         | (channel?) => void                           | Remove listeners               |
| listenerCount | readonly number                              | Active listener count          |

=== EXAMPLES ===
Example 1: Decouple sibling communication
---
```tsx
import { useState } from 'react';
import { Box, Text, Pressable } from '@reactjit/core';
import { useEventBus, useEvent, useEmit } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function App() {
  const bus = useEventBus();
  return (
    <Box style={{ gap: 8, padding: 16 }}>
      <AddButton bus={bus} />
      <CartCount bus={bus} />
    </Box>
  );
}

function AddButton({ bus }: { bus: any }) {
  const colors = useThemeColors();
  const addToCart = useEmit(bus, 'cart:add');

  return (
    <Pressable onPress={() => addToCart({ id: 42, qty: 1 })}>
      <Text style={{ color: colors.primary }}>Add to Cart</Text>
    </Pressable>
  );
}

function CartCount({ bus }: { bus: any }) {
  const colors = useThemeColors();
  const [count, setCount] = useState(0);
  useEvent(bus, 'cart:add', () => setCount(c => c + 1));

  return <Text style={{ color: colors.text }}>{`Cart: ${count}`}</Text>;
}
```

Example 2: Wildcard listener for logging
---
```tsx
import { useEventBus, useEvent } from '@reactjit/core';

export function useEventLogger(bus: ReturnType<typeof useEventBus>) {
  useEvent(bus, '*', (payload, channel) => {
    console.log(`[Event] ${channel}:`, payload);
  });
}
```
---
Platforms: love2d

Example 3: Latest value with useEventState
---
```tsx
import { Text } from '@reactjit/core';
import { useEventState } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function LatestPrice({ bus }: { bus: any }) {
  const colors = useThemeColors();
  const price = useEventState<number>(bus, 'price:update', 0);

  return <Text style={{ color: colors.text }}>{`Price: $${price}`}</Text>;
}
```

=== CRITICAL RULES ===
- The bus is pure JavaScript. Events never cross the bridge. For Lua-side events, use useLoveEvent.
- useEventBus returns the same instance for the component's lifetime. Share it via props, context, or module-level variable.
- useEvent handler stays fresh (captured by ref) -- no stale closures. Automatically unsubscribes on unmount.
- Wildcard '*' receives all events. Multi-channel subscription accepts a string array.
- Events fire synchronously. All listeners for a channel execute before emit() returns.
