=== OVERVIEW ===
usePTY manages a Terminal capability node and provides accumulated output, send helpers, and resize control. It wraps the Terminal component's event-driven API into a stateful hook with rolling output buffer, cursor tracking, and convenience methods.

The hook returns a `terminalProps` object that you spread onto a `<Terminal>` element in your JSX tree. That element is the invisible capability node that owns the PTY lifecycle.

Supports three PTY types:
- `user` (default) -- standard user shell
- `root` -- root shell (needs sudo or shows password prompt)
- `template` -- one fresh PTY per command via `runCommand()`

=== API / SYNTAX ===
```tsx
import { usePTY } from '@reactjit/core';
```

Signature:
```tsx
function usePTY(opts?: UsePTYOptions): UsePTYResult
```

Options:
| Option      | Type                          | Default     | Description                            |
|-------------|-------------------------------|-------------|----------------------------------------|
| type        | 'user' \| 'root' \| 'template'| 'user'     | PTY type                               |
| shell       | string                        | 'bash'      | Shell executable                       |
| cwd         | string                        | undefined   | Working directory                      |
| env         | Record<string, string \| false>| undefined  | Env var overrides (false = unset)      |
| rows        | number                        | 24          | Terminal rows                          |
| cols        | number                        | 80          | Terminal columns                       |
| session     | string                        | auto        | Named session ID for RPC scoping       |
| transport   | 'bridge' \| 'ws' \| 'http' \| 'tor' | 'bridge' | Transport backend              |
| autoConnect | boolean                       | true        | Auto-spawn shell on mount              |
| maxOutput   | number                        | 100000      | Max chars in rolling output buffer     |

Returns:
| Field         | Type                       | Description                            |
|---------------|----------------------------|----------------------------------------|
| output        | string                     | Accumulated output (rolling buffer)    |
| dirtyRows     | DirtyRow[]                 | Latest settled dirty rows from vterm   |
| cursor        | CursorState                | Cursor position and visibility         |
| connected     | boolean                    | Whether the shell is running           |
| send          | (data: string) => void     | Write raw bytes to the PTY             |
| sendLine      | (command: string) => void  | Send command + newline                 |
| interrupt     | () => void                 | Send Ctrl+C                            |
| sendEOF       | () => void                 | Send Ctrl+D                            |
| runCommand    | (command: string) => void  | Template mode: run in fresh PTY        |
| resize        | (rows, cols) => void       | Notify PTY of window size change       |
| clearOutput   | () => void                 | Clear the output buffer                |
| terminalProps | TerminalCapabilityProps     | Spread onto <Terminal>                 |

=== EXAMPLES ===
Example 1: Basic terminal with output
---
```tsx
import { Box, Text, Pressable } from '@reactjit/core';
import { usePTY, Terminal } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function ShellPanel() {
  const colors = useThemeColors();
  const { output, sendLine, connected, terminalProps } = usePTY({
    shell: 'bash',
    session: 'main',
  });

  return (
    <Box style={{ flexGrow: 1, gap: 8 }}>
      <Terminal {...terminalProps} />
      <Text style={{ color: colors.muted }}>
        {connected ? 'Connected' : 'Disconnected'}
      </Text>
      <Pressable onPress={() => sendLine('ls -la')}>
        <Text style={{ color: colors.primary }}>Run ls</Text>
      </Pressable>
    </Box>
  );
}
```
---
Platforms: love2d

Example 2: Template mode -- one command per PTY
---
```tsx
import { Box, Text, Pressable } from '@reactjit/core';
import { usePTY, Terminal } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function CommandRunner() {
  const colors = useThemeColors();
  const { output, runCommand, terminalProps } = usePTY({
    type: 'template',
    env: { MY_KEY: 'abc' },
  });

  return (
    <Box style={{ flexGrow: 1, gap: 8 }}>
      <Terminal {...terminalProps} />
      <Pressable onPress={() => runCommand('echo $MY_KEY')}>
        <Text style={{ color: colors.primary }}>Run Command</Text>
      </Pressable>
    </Box>
  );
}
```
---
Platforms: love2d

=== CRITICAL RULES ===
- You MUST render `<Terminal {...terminalProps} />` in your component tree. Without it, no PTY process is spawned.
- The output buffer is a rolling window capped at maxOutput characters. Old output is trimmed from the front.
- Session IDs must be unique across concurrent terminals. Auto-generated if omitted.
- interrupt() sends Ctrl+C (0x03), sendEOF() sends Ctrl+D (0x04). These are raw byte writes.
