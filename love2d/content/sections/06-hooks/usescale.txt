=== OVERVIEW ===
The scaling system provides viewport-proportional sizing for your UI. Wrap a subtree in ScaleProvider with a reference resolution, and all numeric style values inside automatically scale based on the current window dimensions vs. the reference.

At the reference size, scale=1 and everything renders as authored. At 2x the reference, scale=2 and all pixel values double. The scale never goes below 1 -- at the reference size or smaller, content renders at its authored size.

The system includes:
- ScaleProvider -- context provider that computes scale from viewport vs. reference
- useScale -- read the current scale factor (returns 1 with no provider)
- useScaledStyle -- multiply a style object's numeric values by the scale
- NoScale -- opt a subtree out of scaling

=== API / SYNTAX ===
```tsx
import { ScaleProvider, useScale, useScaledStyle, NoScale } from '@reactjit/core';
```

ScaleProvider:
```tsx
function ScaleProvider(props: {
  reference: { width: number; height: number };
  children: React.ReactNode;
}): React.ReactElement
```

Hooks:
```tsx
function useScale(): number
function useScaledStyle(style: Style | undefined): Style | undefined
```

NoScale:
```tsx
function NoScale(props: { children: React.ReactNode }): React.ReactElement
```

=== EXAMPLES ===
Example 1: Scale a UI designed for 800x600
---
```tsx
import { Box, Text } from '@reactjit/core';
import { ScaleProvider } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function App() {
  const colors = useThemeColors();

  return (
    <ScaleProvider reference={{ width: 800, height: 600 }}>
      <Box style={{ padding: 16, gap: 8 }}>
        <Text style={{ fontSize: 24, color: colors.text }}>Title</Text>
        <Text style={{ fontSize: 14, color: colors.muted }}>
          This text scales proportionally with the window
        </Text>
      </Box>
    </ScaleProvider>
  );
}
```
---
Platforms: love2d

Example 2: Read the scale factor
---
```tsx
import { Text } from '@reactjit/core';
import { useScale } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function ScaleIndicator() {
  const colors = useThemeColors();
  const scale = useScale();

  return (
    <Text style={{ color: colors.muted, fontSize: 12 }}>
      {`Scale: ${scale.toFixed(2)}x`}
    </Text>
  );
}
```
---
Platforms: love2d

Example 3: Opt out of scaling for fixed-size UI
---
```tsx
import { Box, Text } from '@reactjit/core';
import { NoScale } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function Overlay() {
  const colors = useThemeColors();

  return (
    <NoScale>
      <Box style={{ padding: 8, backgroundColor: colors.bgElevated }}>
        <Text style={{ fontSize: 12, color: colors.text }}>
          Always 12px regardless of scale
        </Text>
      </Box>
    </NoScale>
  );
}
```
---
Platforms: love2d

=== CRITICAL RULES ===
- Scale is computed as `min(viewportWidth / refWidth, viewportHeight / refHeight)`, clamped to a minimum of 1. Content never shrinks below its authored size.
- useScaledStyle returns the original object when scale=1, avoiding allocation. Do not rely on object identity changing when scale is 1.
- ScaleProvider reads viewport dimensions from useWindowDimensions. It reacts to window resizes automatically.
