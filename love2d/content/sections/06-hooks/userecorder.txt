=== OVERVIEW ===
Two hooks for recording the Love2D window:

useGifRecorder captures frames at a configurable FPS using Love2D's screenshot API, then assembles them into a GIF via ffmpeg.

useRecorder pipes raw RGBA frames to ffmpeg in a subprocess for MP4 or WebM output. No temp files, no PNG encoding overhead.

Both require ffmpeg to be installed and on PATH.

=== API / SYNTAX ===
```tsx
import { useGifRecorder } from '@reactjit/core';
import { useRecorder } from '@reactjit/core';
```

useGifRecorder:
```tsx
function useGifRecorder(): GifRecorderResult

interface GifRecorderResult {
  recording: boolean;
  frames: number;
  gifPath: string | null;
  start: (opts?: { fps?: number; output?: string }) => void;
  stop: () => Promise<string | null>;
}
```

useRecorder:
```tsx
function useRecorder(): RecorderResult

interface RecorderResult {
  recording: boolean;
  frames: number;
  duration: number;
  filePath: string | null;
  format: string | null;
  start: (opts?: { fps?: number; format?: 'mp4' | 'webm'; output?: string }) => void;
  stop: () => Promise<string | null>;
}
```

GIF options:
| Option | Type   | Default        | Description              |
|--------|--------|----------------|--------------------------|
| fps    | number | 15             | Capture framerate        |
| output | string | recording.gif  | Output file path         |

Video options:
| Option | Type           | Default        | Description              |
|--------|----------------|----------------|--------------------------|
| fps    | number         | 30             | Capture framerate        |
| format | 'mp4' \| 'webm'| 'mp4'         | Output format            |
| output | string         | recording.mp4  | Output file path         |

=== EXAMPLES ===
Example 1: GIF recording toggle
---
```tsx
import { Box, Text, Pressable } from '@reactjit/core';
import { useGifRecorder } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function GifButton() {
  const colors = useThemeColors();
  const { recording, start, stop, frames } = useGifRecorder();

  return (
    <Pressable onPress={() => recording ? stop() : start({ fps: 15 })}>
      <Box style={{ padding: 8, backgroundColor: colors.surface, borderRadius: 4 }}>
        <Text style={{ color: colors.text }}>
          {recording ? `Recording (${frames} frames)` : 'Record GIF'}
        </Text>
      </Box>
    </Pressable>
  );
}
```
---
Platforms: love2d

Example 2: MP4 video recording with duration
---
```tsx
import { Box, Text, Pressable } from '@reactjit/core';
import { useRecorder } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function VideoButton() {
  const colors = useThemeColors();
  const { recording, start, stop, duration } = useRecorder();

  return (
    <Pressable onPress={() => recording ? stop() : start({ fps: 30, format: 'mp4' })}>
      <Box style={{ padding: 8, backgroundColor: colors.surface, borderRadius: 4 }}>
        <Text style={{ color: colors.text }}>
          {recording ? `Recording ${duration.toFixed(1)}s` : 'Record Video'}
        </Text>
      </Box>
    </Pressable>
  );
}
```
---
Platforms: love2d

=== CRITICAL RULES ===
- ffmpeg MUST be installed and on PATH. Without it, start() silently fails.
- Recording polls status every 500ms via setInterval. Frames and duration update at that cadence, not per-frame.
- stop() returns a Promise that resolves to the output file path, or null on error.
- GIF recording assembles frames after stop -- there is a delay before the file is ready for large captures.
