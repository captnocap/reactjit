=== OVERVIEW ===
useIFTTT wires any trigger to any action in a single line. Both sides accept either a string DSL or a function. Mix and match freely.

String triggers subscribe to bridge events. String actions fire bridge commands. Function triggers edge-detect false-to-true transitions. Function actions receive the trigger payload.

=== API / SYNTAX ===
```tsx
import { useIFTTT } from '@reactjit/core';
```

Signature:
```tsx
function useIFTTT(trigger: Trigger, action: Action): IFTTTResult

type Trigger = string | (() => boolean);
type Action = string | ((event?: any) => void);

interface IFTTTResult {
  fired: number;       // Times the rule has fired
  lastEvent: any;      // Payload from most recent trigger
  fire: (event?) => void; // Manually fire the action
}
```

String triggers:
| Trigger                | Description                              |
|------------------------|------------------------------------------|
| 'key:<key>'            | Keydown (e.g. 'key:space', 'key:escape') |
| 'key:up:<key>'         | Keyup                                    |
| 'key:ctrl+<k>'         | Key combo (e.g. 'key:ctrl+s')            |
| 'click'                | Any mouse click                          |
| 'timer:every:<ms>'     | Repeating Lua timer                      |
| 'timer:once:<ms>'      | Single-shot delay                        |
| 'gamepad:<button>'     | Gamepad button press                     |
| 'midi:note:<n>'        | MIDI note on                             |
| 'midi:cc:<n>'          | MIDI CC message                          |
| 'filedrop'             | File dropped on window                   |
| 'mount'                | Fires once on component mount            |
| 'state:<key>:<value>'  | Fires when Lua state matches value       |
| '<event>'              | Any raw bridge event name                |

String actions:
| Action                   | Description                          |
|--------------------------|--------------------------------------|
| 'state:set:<key>:<val>'  | Set Lua state                       |
| 'state:toggle:<key>'     | Toggle boolean Lua state            |
| 'notification:<msg>'     | Push a notification                 |
| 'clipboard:<text>'       | Copy text to clipboard              |
| 'send:<type>'            | Fire a bridge event                 |
| 'rpc:<method>'           | Call a Lua RPC                      |
| 'log:<message>'          | console.log for debugging           |

=== EXAMPLES ===
Example 1: Keyboard shortcuts
---
```tsx
import { useIFTTT } from '@reactjit/core';

export function useToolShortcuts() {
  useIFTTT('key:1', 'state:set:tool:brush');
  useIFTTT('key:2', 'state:set:tool:eraser');
  useIFTTT('key:3', 'state:set:tool:fill');
  useIFTTT('key:ctrl+s', () => save());
  useIFTTT('key:space', 'state:toggle:paused');
}
```
---
Platforms: love2d

Example 2: Reactive condition trigger
---
```tsx
import { useIFTTT } from '@reactjit/core';

export function useWinCondition(score: number) {
  useIFTTT(
    () => score > 100,
    'notification:You win!'
  );
}
```
---
Platforms: love2d

Example 3: Timer-based actions
---
```tsx
import { useIFTTT } from '@reactjit/core';

export function useAutoSave() {
  useIFTTT('timer:every:30000', () => saveDocument());
  useIFTTT('timer:once:5000', 'log:Ready to go');
}
```
---
Platforms: love2d

Example 4: Gamepad and MIDI
---
```tsx
import { useIFTTT } from '@reactjit/core';

export function useGameControls() {
  useIFTTT('gamepad:a', () => jump());
  useIFTTT('gamepad:start', 'state:toggle:paused');
  useIFTTT('midi:note:60', 'state:set:lastNote:C4');
}
```
---
Platforms: love2d

Example 5: File drop handler
---
```tsx
import { useIFTTT } from '@reactjit/core';

export function useFileDrop(onFile: (path: string) => void) {
  useIFTTT('filedrop', (e) => onFile(e.path));
}
```

=== CRITICAL RULES ===
- Function triggers edge-detect false-to-true transitions. They run after every render. Keep them pure and cheap. Do NOT read `fired` or `lastEvent` inside the trigger function -- this creates feedback loops.
- String action values are coerced: 'true'->true, 'false'->false, numeric strings->numbers.
- Timer triggers use Lua-side timers via bridge RPC. timer:once fires exactly once, then cancels the timer.
- State match triggers compare with coercion: state:paused:true matches both string "true" and boolean true.
- mount trigger fires once when the component mounts. It does not re-fire on re-render.
