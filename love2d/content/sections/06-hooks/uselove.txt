=== OVERVIEW ===
useLove combines event subscription and command sending into a single hook. It subscribes to a named event type from Lua, maintains the latest payload as state, and returns a send function for fire-and-forget commands back to Lua.

This is the Swiss Army knife hook -- use it when you need both directions of communication. For listen-only, use useLoveEvent. For send-only, use useLoveSend. For RPC with responses, use useLoveRPC.

=== API / SYNTAX ===
```tsx
import { useLove } from '@reactjit/core';
```

Signature:
```tsx
function useLove<T>(eventType: string, initialState: T): [T, (cmd: string, payload?: any) => void]
```

Parameters:
| Parameter    | Type   | Description                              |
|--------------|--------|------------------------------------------|
| eventType    | string | Bridge event name to subscribe to        |
| initialState | T      | Initial state before any events arrive   |

Returns a tuple:
| Index | Type                                 | Description                      |
|-------|--------------------------------------|----------------------------------|
| [0]   | T                                    | Latest event payload (state)     |
| [1]   | (cmd: string, payload?: any) => void | Send function for Lua commands   |

=== EXAMPLES ===
Example 1: Game state subscription
---
```tsx
import { useLove } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

function GameHUD() {
  const colors = useThemeColors();
  const [game, send] = useLove('game:state', { score: 0, lives: 3 });

  return (
    <Box style={{ flexDirection: 'row', gap: 16, padding: 8 }}>
      <Text style={{ color: colors.text, fontSize: 18 }}>{`Score: ${game.score}`}</Text>
      <Text style={{ color: colors.text, fontSize: 18 }}>{`Lives: ${game.lives}`}</Text>
      <Pressable onPress={() => send('game:reset')}>
        <Text style={{ color: colors.primary, fontSize: 14 }}>Reset</Text>
      </Pressable>
    </Box>
  );
}
```
---
Platforms: love2d

Example 2: Player movement commands
---
```tsx
import { useLove } from '@reactjit/core';

function Controls() {
  const [player, send] = useLove('player:position', { x: 0, y: 0 });

  return (
    <Box>
      <Text>{`Position: ${player.x}, ${player.y}`}</Text>
      <Pressable onPress={() => send('player:move', { dx: 0, dy: -1 })}>
        <Text>Up</Text>
      </Pressable>
    </Box>
  );
}
```
---
Platforms: love2d

=== CRITICAL RULES ===
- The send function is fire-and-forget. It does not return a result. If you need a response from Lua, use useLoveRPC instead.
- The state updates every time the subscribed event fires. If the event fires at 60fps, React will re-render at 60fps. For high-frequency events, consider useLoveEvent with a ref instead.
- The eventType string must match exactly what Lua emits. It is case-sensitive.
