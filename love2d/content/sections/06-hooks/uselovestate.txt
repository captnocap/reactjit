title: useLoveState
description: Bidirectional shared state between React and Lua
category: Hooks
platforms: love2d

related: useLove, useHotState, useLocalStore
difficulty: intermediate

=== OVERVIEW ===
useLoveState creates a shared state value that both React and Lua can read and write. When React calls the setter, it updates both local React state and the Lua-side state. When Lua changes the value, the update flows back to React via a bridge event.

The API matches useState: it returns a [value, setValue] tuple. The key string namespaces the state so multiple useLoveState calls with different keys are independent.

=== API / SYNTAX ===
```tsx
import { useLoveState } from '@reactjit/core';
```

Signature:
```tsx
function useLoveState<T>(key: string, initialValue: T): [T, (value: T) => void]
```

Parameters:
| Parameter    | Type   | Description                               |
|--------------|--------|-------------------------------------------|
| key          | string | Unique state key (namespaces the value)   |
| initialValue | T      | Initial value before Lua sends an update  |

Returns:
| Index | Type             | Description                                |
|-------|------------------|--------------------------------------------|
| [0]   | T                | Current value                              |
| [1]   | (value: T) => void | Setter -- updates React and Lua atomically |

=== EXAMPLES ===
Example 1: Player health
---
```tsx
import { useLoveState } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

function HealthBar() {
  const colors = useThemeColors();
  const [health, setHealth] = useLoveState('player.health', 100);

  return (
    <Box style={{ flexDirection: 'row', gap: 8 }}>
      <Text style={{ color: colors.text }}>{`HP: ${health}`}</Text>
      <Pressable onPress={() => setHealth(100)}>
        <Text style={{ color: colors.primary }}>Heal</Text>
      </Pressable>
    </Box>
  );
}
```
---
Platforms: love2d

Example 2: Pause toggle
---
```tsx
import { useLoveState } from '@reactjit/core';

function PauseButton() {
  const [paused, setPaused] = useLoveState('game.paused', false);

  return (
    <Pressable onPress={() => setPaused(!paused)}>
      <Text>{paused ? 'Resume' : 'Pause'}</Text>
    </Pressable>
  );
}
```
---
Platforms: love2d

=== CRITICAL RULES ===
- The key string is used as the state namespace. Two components using the same key share the same state -- changes from one are reflected in the other.
- The setter updates React state immediately (optimistic) and sends the update to Lua. There is no round-trip delay for the local update.
- Lua-side changes are pushed as `state:<key>` bridge events. The hook subscribes to this automatically.
- For state that needs to survive app restarts, use useLocalStore instead. For state that survives HMR but not restarts, use useHotState.
