=== OVERVIEW ===
useScrape fetches a URL, parses the HTML (no DOM required -- works in QuickJS), and extracts data using CSS selectors. It supports two modes:

Expert mode: provide a selector map up front and get extracted data immediately.
Guided mode: omit selectors, browse a tagged catalog of page elements, then call pick() to extract what you want by element ID.

The hook includes a minimal HTML parser that runs entirely in JavaScript -- no browser APIs needed. It supports tag, class, and ID selectors, descendant combinators, attribute extraction via `@attr` suffix, and `:first` for single results.

=== API / SYNTAX ===
```tsx
import { useScrape } from '@reactjit/core';
```

Expert mode:
```tsx
const { data, loading, error } = useScrape('https://example.com', {
  title: 'h1',
  price: '.product-price',
  link: 'a@href',
});
```

Guided mode:
```tsx
const scrape = useScrape('https://example.com');
// scrape.elements -> tagged catalog
// scrape.pick({ title: 1, price: 5, link: { id: 3, attr: 'href' } })
```

Selector syntax:
| Selector            | Matches                              | Extracts        |
|---------------------|--------------------------------------|-----------------|
| 'h1'                | First h1 tag                         | Text content    |
| '.price'            | Elements with class "price"          | Text content    |
| '#main'             | Element with id "main"               | Text content    |
| 'div.card'          | div elements with class "card"       | Text content    |
| 'div .title'        | .title descendants of div            | Text content    |
| 'a@href'            | All a tags                           | href attribute  |
| 'img@src'           | All img tags                         | src attribute   |
| 'selector@html'     | Matched elements                     | Inner HTML      |
| 'h1:first'          | First h1 only (force single result)  | Text content    |

Options:
| Option    | Type                       | Default   | Description                           |
|-----------|----------------------------|-----------|---------------------------------------|
| interval  | number                     | undefined | Auto-refetch interval (ms)            |
| headers   | Record<string, string>     | undefined | Request headers                       |
| transform | (html: string) => string   | undefined | Pre-parse HTML transform              |
| arrays    | boolean                    | false     | Always return arrays for matches      |

Returns:
| Field    | Type                   | Description                              |
|----------|------------------------|------------------------------------------|
| data     | Record \| null         | Extracted data keyed by selector names   |
| loading  | boolean                | True while fetching                      |
| error    | string \| null         | Error message if fetch/parse failed      |
| html     | string \| null         | Raw HTML from last successful fetch      |
| elements | ScrapeElement[]        | Tagged element catalog (guided mode)     |
| refetch  | () => void             | Re-fetch and re-parse                    |
| setUrl   | (url) => void          | Change the URL dynamically               |
| set      | (overrides) => void    | Manually override data values            |
| pick     | (map) => void          | Guided mode: extract by element IDs      |

=== EXAMPLES ===
Example 1: Expert mode with CSS selectors
---
```tsx
import { Box, Text } from '@reactjit/core';
import { useScrape } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function PriceScraper() {
  const colors = useThemeColors();
  const { data, loading, error } = useScrape('https://example.com/product', {
    title: 'h1',
    price: '.product-price',
    image: 'img.hero@src',
  });

  if (loading) return <Text style={{ color: colors.muted }}>Scraping...</Text>;
  if (error) return <Text style={{ color: colors.primary }}>{error}</Text>;

  return (
    <Box style={{ gap: 4, padding: 16 }}>
      <Text style={{ color: colors.text }}>{`${data?.title}`}</Text>
      <Text style={{ color: colors.text }}>{`Price: ${data?.price}`}</Text>
    </Box>
  );
}
```

Example 2: Guided mode with element picking
---
```tsx
import { useEffect } from 'react';
import { Box, Text } from '@reactjit/core';
import { useScrape } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function GuidedScraper() {
  const colors = useThemeColors();
  const scrape = useScrape('https://example.com');

  // After elements load, pick by catalog ID
  useEffect(() => {
    if (scrape.elements.length > 0) {
      scrape.pick({
        title: 1,
        link: { id: 3, attr: 'href' },
      });
    }
  }, [scrape.elements.length]);

  return (
    <Box style={{ gap: 4, padding: 16 }}>
      <Text style={{ color: colors.muted }}>{`${scrape.elements.length} elements found`}</Text>
      {scrape.data && (
        <Text style={{ color: colors.text }}>{`Title: ${scrape.data.title}`}</Text>
      )}
    </Box>
  );
}
```

Platforms: love2d

Example 3: Auto-refetching price monitor
---
```tsx
import { useScrape } from '@reactjit/core';

export function usePriceMonitor(url: string) {
  return useScrape(url, { price: '.price' }, { interval: 60000 });
}
```

=== CRITICAL RULES ===
- The HTML parser is minimal -- it handles most well-formed HTML but may struggle with severely malformed markup. Use the `transform` option to fix known issues before parsing.
- Selector syntax is simplified CSS. It supports tag, class, ID, descendant combinators, and @attr extraction. It does not support pseudo-classes (except :first), sibling combinators, or attribute selectors like [data-id="x"].
- The element catalog (guided mode) skips structural/invisible tags (html, head, body, meta, link, noscript, template).
- Auto-refetch uses Lua timers via the bridge. The interval must be unique across all useLuaInterval calls.
