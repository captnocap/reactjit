=== OVERVIEW ===
useSystemInfo returns real system information gathered by Lua from /proc, io.popen, and os.getenv. It provides OS name, kernel version, hostname, user, shell, CPU model, architecture, memory usage, and uptime.

Every sub-object (memory, uptime, and the top-level result) carries a `.toSysLog(path)` method for structured JSON logging to a file.

By default, the hook fetches once on mount. Pass a refresh interval in milliseconds to re-poll dynamic fields (memory, uptime).

=== API / SYNTAX ===
```tsx
import { useSystemInfo, formatUptime, formatMemory } from '@reactjit/core';
```

Signature:
```tsx
function useSystemInfo(refreshInterval?: number): SystemInfo
```

Types:
```tsx
interface SystemInfo {
  os: string;            // e.g. "Ubuntu 24.04 LTS"
  kernel: string;        // e.g. "6.14.0-37-generic"
  hostname: string;
  user: string;
  shell: string;         // e.g. "bash"
  cpu: string;           // e.g. "AMD Ryzen 9 7950X (32 cores)"
  arch: string;          // e.g. "x86_64"
  memory: MemoryInfo;
  uptime: UptimeInfo;
  loading: boolean;
  toSysLog: (path: string) => void;
}

interface MemoryInfo {
  used: number;
  total: number;
  unit: 'GiB' | 'MiB';
  toSysLog: (path: string) => void;
}

interface UptimeInfo {
  days: number;
  hours: number;
  minutes: number;
  toSysLog: (path: string) => void;
}
```

Parameters:
| Param           | Type   | Default | Description                                |
|-----------------|--------|---------|--------------------------------------------|
| refreshInterval | number | 0       | Ms between re-polls. 0 = fetch once.      |

Helpers:
| Function      | Signature                                 | Description                         |
|---------------|-------------------------------------------|-------------------------------------|
| formatUptime  | (u: UptimeInfo) => string                 | "2 days, 3 hours, 15 mins"         |
| formatMemory  | (m: MemoryInfo) => string                 | "12.4 GiB / 31.2 GiB"             |
| formatBytes   | (bytes: number) => string                 | Human-readable byte string          |

=== EXAMPLES ===
Example 1: Display system information
---
```tsx
import { Box, Text } from '@reactjit/core';
import { useSystemInfo, formatUptime, formatMemory } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function SystemInfoPanel() {
  const colors = useThemeColors();
  const info = useSystemInfo(5000);

  if (info.loading) return <Text style={{ color: colors.muted }}>Loading...</Text>;

  return (
    <Box style={{ gap: 4, padding: 16 }}>
      <Text style={{ color: colors.text }}>{`OS: ${info.os}`}</Text>
      <Text style={{ color: colors.text }}>{`CPU: ${info.cpu}`}</Text>
      <Text style={{ color: colors.text }}>{`Memory: ${formatMemory(info.memory)}`}</Text>
      <Text style={{ color: colors.text }}>{`Uptime: ${formatUptime(info.uptime)}`}</Text>
      <Text style={{ color: colors.muted }}>{`${info.user}@${info.hostname}`}</Text>
    </Box>
  );
}
```

Platforms: love2d

Example 2: Log system info to file
---
```tsx
import { useSystemInfo } from '@reactjit/core';
import { useEffect } from 'react';

export function SysLogger() {
  const info = useSystemInfo();

  useEffect(() => {
    if (!info.loading) {
      info.toSysLog('/tmp/sysinfo.log');
    }
  }, [info.loading]);

  return null;
}
```

Platforms: love2d

=== CRITICAL RULES ===
- When using a refresh interval, pick a unique value to avoid timer collisions with other useLuaInterval calls. See useLuaInterval docs for staggering rules.
- The hook uses useBridgeOptional internally. If no bridge is present, it falls back to Node.js os module (for grid targets).
- toSysLog writes JSON lines with an _ts timestamp to the given path via bridge RPC.
