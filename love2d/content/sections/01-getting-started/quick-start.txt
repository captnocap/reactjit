=== OVERVIEW ===
This guide gets you from zero to a running ReactJIT app in five minutes. You will
scaffold a new project, edit the entry component, run it in watch mode, and optionally
produce a standalone binary. All you need is a completed installation (see
installation.txt).

=== EXAMPLES ===
Example 1: Scaffold and Run
---
# Scaffold a new project (interactive package selection)
reactjit init my-app
cd my-app

# Start watch mode -- rebuilds on file changes, launches Love2D automatically
reactjit dev
---
Platforms: love2d

Example 2: Edit src/App.tsx
---
```tsx
import { useState } from 'react';
import { Box, Text, Pressable } from '@reactjit/core';

export function App() {
  const [message, setMessage] = useState('Hello ReactJIT!');

  return (
    <Box style={{
      width: '100%',
      height: '100%',
      backgroundColor: '#0f172a',
      justifyContent: 'center',
      alignItems: 'center',
      gap: 20,
    }}>
      <Text style={{ fontSize: 32, color: '#f8fafc' }}>
        {message}
      </Text>
      <Pressable onPress={() => setMessage('You clicked!')}>
        {({ pressed }) => (
          <Box style={{
            backgroundColor: pressed ? '#2563eb' : '#3b82f6',
            paddingLeft: 24,
            paddingRight: 24,
            paddingTop: 12,
            paddingBottom: 12,
            borderRadius: 8,
          }}>
            <Text style={{ fontSize: 16, color: '#ffffff' }}>Click Me</Text>
          </Box>
        )}
      </Pressable>
    </Box>
  );
}
```
---
Platforms: love2d

Example 3: The Entry Point (src/main.tsx)
---
```tsx
import React from 'react';
import { createLove2DApp } from '@reactjit/renderer';
import { App } from './App';

const app = createLove2DApp();
app.render(<App />);
```

This is generated by `reactjit init`. You rarely need to edit it.
The renderer creates the reconciler, connects the QuickJS bridge,
and starts pumping mutation commands to Lua.
---
Platforms: love2d

Example 4: Lint and Build
---
# Run the static linter to catch layout errors
reactjit lint

# Take a screenshot to verify layout
reactjit screenshot --output /tmp/preview.png

# Build a self-extracting Linux binary
reactjit build linux

# Other build targets:
#   reactjit build macos          macOS bundle (Intel x64)
#   reactjit build macmseries     macOS bundle (Apple Silicon arm64)
#   reactjit build windows        Windows archive (x64)
#   reactjit build dist:love      Self-extracting Linux binary (Love2D + glibc)
---
Platforms: love2d

Example 5: Non-Interactive Init
---
# Minimal (core only, no optional packages)
reactjit init my-app --minimal

# All optional packages
reactjit init my-app --all

# Specific packages
reactjit init my-app --router --storage --audio

# Convert from another framework
reactjit init my-app --convert path/to/existing-app.py
---
Platforms: love2d

=== CRITICAL RULES ===
- Always use `reactjit init` to scaffold projects. Do not mkdir and copy manually.
- Always use `reactjit dev` for watch mode. Do not run esbuild --watch manually.
- Always use `reactjit build` for production builds. Do not run esbuild manually.
- The CLI encodes correct esbuild flags, enforces lint gates, and handles runtime
  file placement. Raw esbuild commands use wrong flags and skip linting.
- After modifying any component, run `reactjit lint` to catch layout errors.
