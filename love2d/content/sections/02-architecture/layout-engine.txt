=== OVERVIEW ===
The layout engine (lua/layout.lua) is a complete flexbox implementation written in Lua. It
takes the retained instance tree produced by tree.lua and computes absolute pixel positions
and sizes ({x, y, w, h}) for every node. These computed values are then used by the painter
to issue draw calls.

The engine is pixel-perfect. Flex distribution math, cursor positions, and post-layout sizes
are exact to the pixel. It has been exhaustively verified. If you encounter a layout overflow
or sizing problem, look at the component first -- does it hardcode a width that overflows its
parent? The layout engine itself is not the source of bugs.

Every Box is a flex container by default. flexDirection defaults to 'column', alignItems
defaults to 'stretch', flexShrink defaults to 1. There are no block, inline, or grid display
modes -- everything is flex.

Three sizing tiers. Elements resolve their dimensions through three tiers, checked in order.
The first one that applies wins:

Tier 1: Explicit dimensions. You set width, height, flexGrow, or flexBasis. This always
takes priority.

Tier 2: Content auto-sizing. Containers with children auto-size from their content. Column
containers sum children heights plus gaps and padding; row containers sum children widths.
Text nodes measure from Love2D's font rasterizer using the resolved font size and wrap
constraint.

Tier 3: Proportional surface fallback. Empty surface nodes (Box, Image, Video, VideoPlayer,
Scene3D, Emulator, Render, Chart2D) with no explicit dimensions and no children fall back to
1/4 of their parent's available space. This cascades recursively: in an 800px window, an
unsized Box is 200px; a nested unsized Box inside it is 50px. Scroll containers and
interactive elements (Text, TextInput, Pressable, CodeBlock) are NOT surfaces -- they size
from content or require explicit dimensions.

Text measurement. The layout engine delegates text measurement to Love2D's font rasterizer via
the Measure module. It resolves font size (default 14px), font family, line height, letter
spacing, and number-of-lines constraints. Text wrapping uses the parent's inner width (after
padding) as the wrap constraint. When maxWidth is set on a text node, it clamps the wrap
constraint for correct wrapping.

Unit resolution. Style values can be plain numbers (pixels), percentages ('50%'), viewport
units ('50vw', '50vh'), calc expressions ('calc(100% - 40px)'), or fit-content. Percentages
resolve against the parent container's computed size. Viewport units resolve against the
window dimensions. The resolveUnit function handles all conversions.

=== API / SYNTAX ===
Supported layout properties:

  Flex container:
    flexDirection: 'column' | 'row'          -- default: 'column'
    flexWrap: 'nowrap' | 'wrap'              -- default: 'nowrap' (but rows auto-wrap unless a child has flexGrow)
    justifyContent: 'flex-start' | 'center' | 'flex-end' | 'space-between' | 'space-around' | 'space-evenly'
    alignItems: 'stretch' | 'flex-start' | 'center' | 'flex-end'  -- default: 'stretch'
    gap: number                              -- space between children and between wrap lines

  Flex child:
    flexGrow: number                         -- default: 0
    flexShrink: number                       -- default: 1
    flexBasis: number | string               -- initial main-axis size before grow/shrink
    alignSelf: 'auto' | 'flex-start' | 'center' | 'flex-end' | 'stretch'

  Sizing:
    width: number | string                   -- pixels, '%', 'vw', 'calc()', 'fit-content'
    height: number | string                  -- same units
    minWidth: number | string
    minHeight: number | string
    maxWidth: number | string
    maxHeight: number | string
    aspectRatio: number                      -- width/height ratio (requires one explicit dimension)

  Spacing:
    padding: number                          -- all sides
    paddingLeft: number
    paddingRight: number
    paddingTop: number
    paddingBottom: number
    margin: number                           -- all sides
    marginLeft: number
    marginRight: number
    marginTop: number
    marginBottom: number

  Positioning:
    position: 'relative' | 'absolute'        -- default: 'relative'
    top: number | string
    left: number | string
    right: number | string
    bottom: number | string

  Display:
    display: 'flex' | 'none'                 -- default: 'flex'. 'none' skips the node entirely.

Surface types (receive proportional fallback):
  View (Box), Image, Video, VideoPlayer, Scene3D, Emulator, Render, Chart2D

Non-surface types (size from content or need explicit dimensions):
  Text, TextInput, Pressable, CodeBlock, ScrollView, Math

=== EXAMPLES ===
Example 1: Three sizing tiers in action
---
import { Box, Text, Image } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

function SizingDemo() {
  const c = useThemeColors();
  return (
    <Box style={{ width: '100%', height: '100%', backgroundColor: c.bg, padding: 16, gap: 12 }}>
      {/* Tier 1: Explicit -- 200px tall */}
      <Box style={{ height: 200, backgroundColor: c.surface, borderRadius: 8 }} />

      {/* Tier 2: Content auto-sized -- height from text */}
      <Box style={{ backgroundColor: c.surface, padding: 12, borderRadius: 8 }}>
        <Text style={{ fontSize: 16, color: c.text }}>{`This box auto-sizes to fit its content`}</Text>
      </Box>

      {/* Tier 3: Proportional fallback -- empty surface gets parent/4 */}
      <Box style={{ backgroundColor: c.primary, borderRadius: 8 }} />
    </Box>
  );
}

Example 2: Header + content + footer with flexGrow
---
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

function Layout() {
  const c = useThemeColors();
  return (
    <Box style={{ width: '100%', height: '100%', backgroundColor: c.bg }}>
      <Box style={{ padding: 16, backgroundColor: c.bgElevated }}>
        <Text style={{ fontSize: 20, color: c.text }}>{`Header`}</Text>
      </Box>
      <Box style={{ flexGrow: 1, padding: 16 }}>
        <Text style={{ fontSize: 14, color: c.text }}>{`Content fills remaining space`}</Text>
      </Box>
      <Box style={{ padding: 12, backgroundColor: c.bgElevated }}>
        <Text style={{ fontSize: 12, color: c.muted }}>{`Footer`}</Text>
      </Box>
    </Box>
  );
}

Example 3: Row layout with percentage widths
---
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

function SidebarLayout() {
  const c = useThemeColors();
  return (
    <Box style={{ width: '100%', height: '100%', flexDirection: 'row', backgroundColor: c.bg }}>
      <Box style={{ width: '25%', backgroundColor: c.surface, padding: 12 }}>
        <Text style={{ fontSize: 14, color: c.text }}>{`Sidebar`}</Text>
      </Box>
      <Box style={{ flexGrow: 1, padding: 16 }}>
        <Text style={{ fontSize: 14, color: c.text }}>{`Main content`}</Text>
      </Box>
    </Box>
  );
}

=== CRITICAL RULES ===
- The layout engine is pixel-perfect. If something overflows, look at the component first. Do not blame the layout engine until you have verified the component does not hardcode dimensions that exceed its parent.
- Root containers MUST use width: '100%', height: '100%'. The proportional fallback does not apply at the root because the root IS the viewport.
- Use flexGrow: 1 for space-filling elements. Never hardcode pixel heights to fill a known window size.
- ScrollView needs explicit height or flexGrow. Scroll containers are excluded from the proportional surface fallback.
- Percentage-based children in auto-sized parents resolve to 0. The parent needs a known size for percentages to work.
- aspectRatio requires at least one explicit dimension. Without width or height, there is nothing to derive from.
- Do not use paddingHorizontal or paddingVertical. The linter rejects them. Use paddingLeft/paddingRight and paddingTop/paddingBottom.
- flexDirection defaults to 'column', not 'row'. Every Box is a vertical stack unless you set flexDirection: 'row'.
