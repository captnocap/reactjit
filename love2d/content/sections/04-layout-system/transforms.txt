=== OVERVIEW ===
ReactJIT supports a transform object on any element's style for applying geometric
transformations. Transforms are applied after layout -- they affect visual rendering
without changing the element's position in the layout tree. A transformed element still
occupies the same layout space it would without the transform. Siblings are not affected.

The transform system supports translation, rotation, scaling, and skewing. The transform
origin defaults to the center of the element (0.5, 0.5) and can be adjusted with originX
and originY values ranging from 0 (top-left) to 1 (bottom-right). Transforms are commonly
combined with the animation system (useAnimation, useSpring) or CSS transitions for
motion effects.

=== API / SYNTAX ===
Transform style property:

  transform: {
    translateX?: number     // Horizontal offset in pixels
    translateY?: number     // Vertical offset in pixels
    rotate?: number         // Rotation in degrees (clockwise)
    scaleX?: number         // Horizontal scale factor (1 = no change)
    scaleY?: number         // Vertical scale factor (1 = no change)
    skewX?: number          // Horizontal skew in degrees
    skewY?: number          // Vertical skew in degrees
    originX?: number        // Transform origin X, 0-1 (default: 0.5 = center)
    originY?: number        // Transform origin Y, 0-1 (default: 0.5 = center)
  }

All transform sub-properties are optional. Only the properties you specify are applied.
Unspecified properties default to their identity values (0 for translate/rotate/skew,
1 for scale, 0.5 for origin).

Transform origin:
  originX and originY are normalized values from 0 to 1, where:
    0, 0 = top-left corner
    0.5, 0.5 = center (default)
    1, 1 = bottom-right corner

Transforms do NOT affect:
  - Layout position -- siblings do not move
  - Hit testing bounds -- interaction uses the untransformed rectangle
  - Overflow clipping -- scissor uses layout coordinates, not transformed ones

=== EXAMPLES ===
Example 1: Simple translation
---
import { Box } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

function Translated() {
  const c = useThemeColors();
  return (
    <Box style={{
      width: 100,
      height: 100,
      backgroundColor: c.primary,
      borderRadius: 8,
      transform: { translateX: 50, translateY: 20 },
    }} />
  );
}

Example 7: Transform with CSS transition
---
import { Box, Pressable, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';
import { useState } from 'react';

function AnimatedCard() {
  const c = useThemeColors();
  const [pressed, setPressed] = useState(false);
  return (
    <Pressable
      onPress={() => setPressed(!pressed)}
      style={{
        width: 120,
        height: 80,
        backgroundColor: c.surface,
        borderRadius: 8,
        transform: pressed
          ? { scaleX: 0.95, scaleY: 0.95 }
          : { scaleX: 1, scaleY: 1 },
        transition: {
          transform: { duration: 150, easing: 'easeOut' },
        },
      }}
    >
      <Text style={{ fontSize: 14, color: c.text, textAlign: 'center' }}>
        {`Tap me`}
      </Text>
    </Pressable>
  );
}
---
Platforms: love2d

=== PLATFORM NOTES ===
Love2D:
Transforms are applied via love.graphics.translate(), love.graphics.rotate(),
love.graphics.scale(), and love.graphics.shear() in the painter. The transform stack
is pushed before rendering the node and popped after, so child nodes inherit parent
transforms (a rotated Box rotates its children too). The origin is translated to the
node's center (or custom origin point) before rotation/scale/skew, then translated back.

=== CRITICAL RULES ===
- Transforms are post-layout -- they do not change the element's position in flex
- Hit testing uses the untransformed layout rectangle -- a rotated button's clickable area is still its axis-aligned bounding box
- Child nodes inherit parent transforms -- rotating a Box rotates all its children
- Combine transforms with transition or useAnimation for smooth motion effects
