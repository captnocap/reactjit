=== OVERVIEW ===
ReactJIT uses Flexbox for all layout, mirroring React Native's approach. Every Box is
implicitly a flex container -- there is no need to set display: 'flex' (it is the default).
The critical difference from CSS is that flexDirection defaults to 'column', not 'row'.
This means children stack vertically unless you explicitly set flexDirection: 'row'.

The Lua-side layout engine resolves flex properties into absolute pixel positions and sizes.
It supports main axis distribution (justifyContent), cross axis alignment (alignItems,
alignSelf), flex sizing (flexGrow, flexShrink, flexBasis), wrapping (flexWrap), and gap
spacing. Alignment values use 'start' and 'end' instead of CSS's 'flex-start' and
'flex-end'. The CSS-prefixed forms ('flex-start', 'flex-end') are normalized automatically,
so both work.

Rows auto-wrap by default unless any child has flexGrow > 0 (which signals a structural
layout row like sidebar + content). Columns never wrap unless explicitly set. This
behavior can be overridden with flexWrap: 'nowrap' or flexWrap: 'wrap'.

=== API / SYNTAX ===
Flexbox style properties (all set via the `style` prop on Box):

  flexDirection: 'row' | 'column'
    Default: 'column'
    Sets the main axis. 'column' stacks children vertically, 'row' stacks horizontally.

  justifyContent: 'start' | 'center' | 'end' | 'space-between' | 'space-around' | 'space-evenly'
    Default: 'start'
    Distributes children along the main axis.

  alignItems: 'start' | 'center' | 'end' | 'stretch'
    Default: 'stretch'
    Aligns children along the cross axis. 'stretch' makes children fill the full cross-axis
    extent of the container (or the tallest sibling in a row).

  alignSelf: 'auto' | 'start' | 'center' | 'end' | 'stretch'
    Default: 'auto' (inherits parent's alignItems)
    Overrides the parent's alignItems for a specific child.

  flexGrow: number
    Default: 0
    How much of the remaining free space this child should absorb. When multiple children
    have flexGrow, the free space is distributed proportionally to their grow values.

  flexShrink: number
    Default: 1 (CSS spec default)
    How much this child should shrink when the container is too small. Shrinking is
    proportional to flexShrink * basis. Set to 0 to prevent shrinking.

  flexBasis: number | string | 'auto'
    Default: 'auto'
    The initial size of the child along the main axis before flexGrow/flexShrink are applied.
    When 'auto', falls back to the child's width (row) or height (column). Percentage values
    in wrapping rows with gap are automatically corrected to account for gap consumption.

  flexWrap: 'nowrap' | 'wrap'
    Default: rows auto-wrap, columns do not wrap.
    Whether children wrap to the next line when they exceed the container's main axis size.
    In a row, if any child has flexGrow > 0, auto-wrap is suppressed (treated as a
    structural layout row).

  gap: number | string
    Spacing between children on the main axis. Also applies between wrap lines.

  display: 'flex' | 'none'
    Default: 'flex'
    Set to 'none' to hide an element entirely -- it occupies no space and its children
    are not laid out.

Shorthand props on Box (alternative to style):
  direction: 'row' | 'col' -- maps to flexDirection
  gap: number | string -- maps to style.gap
  align: -- maps to alignItems
  justify: -- maps to justifyContent
  wrap: boolean -- maps to flexWrap: 'wrap'
  grow: boolean -- maps to flexGrow: 1
  fill: boolean -- maps to width: '100%', height: '100%'

=== EXAMPLES ===
Example 1: Vertical stack (default column direction)
---
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

function Stack() {
  const c = useThemeColors();
  return (
    <Box style={{ width: '100%', height: '100%', backgroundColor: c.bg }}>
      <Text style={{ fontSize: 20, color: c.text }}>First</Text>
      <Text style={{ fontSize: 20, color: c.text }}>Second</Text>
      <Text style={{ fontSize: 20, color: c.text }}>Third</Text>
    </Box>
  );
}
---

Example 4: flexGrow to fill remaining space
---
const c = useThemeColors();
<Box style={{ flexDirection: 'row', width: '100%', height: 50 }}>
  <Box style={{ width: 100, backgroundColor: c.primary }} />
  <Box style={{ flexGrow: 1, backgroundColor: c.surface }} />
  <Box style={{ width: 100, backgroundColor: c.primary }} />
</Box>

Example 5: Header + content + footer layout
---
const c = useThemeColors();
<Box style={{ width: '100%', height: '100%', backgroundColor: c.bg }}>
  <Box style={{ padding: 12, backgroundColor: c.bgElevated }}>
    <Text style={{ fontSize: 18, color: c.text }}>Header</Text>
  </Box>

    <Text style={{ fontSize: 14, color: c.text }}>
      {`Content fills remaining space via flexGrow: 1`}
    </Text>
  </Box>
  <Box style={{ padding: 12, backgroundColor: c.bgElevated }}>

  </Box>

Example 6: Wrapping row with gap
---
const c = useThemeColors();
<Box style={{
  flexDirection: 'row',
  flexWrap: 'wrap',
  width: '100%',
  gap: 8,
  padding: 16,
}}>
  {Array.from({ length: 12 }, (_, i) => (
    <Box key={i} style={{
      width: 80,
      height: 80,
      backgroundColor: c.surface,
      borderRadius: 4,
    }} />
  ))}
</Box>

=== PLATFORM NOTES ===
Love2D:
The Lua layout engine processes the flex algorithm in a single pass per frame. Flex
distribution math (grow, shrink, basis) is exact to the pixel. Text nodes are re-measured
after flex distribution to account for width changes that affect wrapping height.

=== CRITICAL RULES ===
- flexDirection defaults to 'column' -- children stack vertically unless you set 'row'
- alignItems defaults to 'stretch' -- children expand to fill the cross axis
- flexShrink defaults to 1 -- children will shrink proportionally when the container overflows
- Rows auto-wrap by default unless a child has flexGrow > 0, which signals a structural layout
- Use flexGrow: 1 on the element that should absorb remaining space, not hardcoded heights
- Gap applies between children on the main axis AND between wrap lines
