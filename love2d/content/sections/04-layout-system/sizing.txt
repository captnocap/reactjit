=== OVERVIEW ===
Every element in ReactJIT resolves its dimensions through three tiers, checked in order.
The first tier that applies wins:

1. Explicit dimensions -- you set width, height, flexGrow, or flexBasis. This always
   takes priority.

2. Content auto-sizing -- containers with children auto-size from their content. Text
   nodes measure from font metrics. This is the default for any element with children.

3. Proportional surface fallback -- empty surface nodes (Box, Image, Video, VideoPlayer,
   Scene3D, Emulator, Render, Chart2D) with no explicit dimensions and no children fall
   back to 1/4 of their parent's available space. This cascades recursively: in an 800px
   window, an unsized Box is 200px; a nested unsized Box inside it is 50px.

Interactive elements (Text, TextInput, Pressable, CodeBlock, ScrollView) are NOT surfaces.
They size from content or require explicit dimensions. ScrollView is specifically excluded
from auto-sizing -- it needs explicit height or flexGrow to define its scroll viewport.

Root containers must always use width: '100%', height: '100%' to fill the viewport. The
proportional fallback does not apply at the root because the root IS the viewport.

=== API / SYNTAX ===
Sizing style properties:

  width: number | string
    Width in pixels (number), percentage of parent ('50%'), viewport units ('50vw'),
    or calc expression ('calc(100% - 40px)'). Special value 'fit-content' (or 'fit')
    shrink-wraps to the element's content width.

  height: number | string
    Height in pixels (number) or percentage of parent. Same unit support as width.
    'fit-content' shrink-wraps to content height.

  minWidth: number | string
    Minimum width constraint. The element will not shrink below this value even under
    flex shrink pressure. Applied after flex distribution.

  minHeight: number | string
    Minimum height constraint.

  maxWidth: number | string
    Maximum width constraint. The element will not grow beyond this value even with
    flexGrow. When maxWidth is set on a text node without explicit width, the text
    measurement constraint is clamped to maxWidth for correct wrapping.

  maxHeight: number | string
    Maximum height constraint.

  aspectRatio: number
    Maintains a width-to-height ratio. For example, aspectRatio: 16/9 with width: 320
    produces height: ~178. Requires at least one explicit dimension -- if neither width
    nor height is set, aspectRatio has nothing to derive from.

The three sizing tiers in detail:

  Tier 1 -- Explicit dimensions:
    Set width, height, flexGrow, or flexBasis directly. This always wins.

  Tier 2 -- Content auto-sizing (containers with children):
    Column: height = sum of children heights + gaps + padding.
             width = max child width + padding.
    Row:    width = sum of children widths + gaps + padding.
            height = max child height + padding.
    Text:   width and height from font measurement (wrap constraint = parent inner width).

  Tier 3 -- Proportional surface fallback (empty surfaces only):
    Applies when: node is a surface type AND has no children AND has no explicit
    height AND height resolved to < 1px AND flexGrow is 0 or unset.
    Fallback: height = parent_height / 4 (using the definite, resolved parent height).
    Cascading: 800px -> 200px -> 50px -> 12px ...

  Surface types (get fallback):
    View (Box), Image, Video, VideoPlayer, Scene3D, Emulator, Render, Chart2D.
    Excludes scroll containers (overflow: 'scroll' or 'auto').

  Non-surface types (size from content or need explicit dims):
    Text, TextInput, Pressable, CodeBlock, ScrollView, Math.

=== EXAMPLES ===
Example 1: Content auto-sizing -- no explicit dimensions needed
---
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

function AutoSized() {
  const c = useThemeColors();
  return (
    <Box style={{ width: '100%', height: '100%', backgroundColor: c.bg, padding: 16 }}>
      <Box style={{ backgroundColor: c.surface, padding: 12, borderRadius: 8 }}>
        <Text style={{ fontSize: 16, color: c.text }}>Title</Text>
        <Text style={{ fontSize: 14, color: c.muted }}>Subtitle</Text>
      </Box>
    </Box>
  );
}
---
Platforms: love2d

Example 2: Proportional surface fallback -- empty boxes get 1/4 parent
---
const c = useThemeColors();
// In an 800px-tall container:
// - First Box: 200px (800/4)
// - Nested Box inside it: 50px (200/4)
<Box style={{ width: '100%', height: '100%', backgroundColor: c.bg }}>
  <Box style={{ backgroundColor: c.surface }}>
    <Box style={{ backgroundColor: c.primary }} />
  </Box>
</Box>
---
Platforms: love2d

Example 3: Fixed-size box with min/max constraints
---
const c = useThemeColors();
<Box style={{
  width: '80%',
  minWidth: 200,
  maxWidth: 600,
  height: 100,
  backgroundColor: c.surface,
  borderRadius: 8,
}} />
---
Platforms: love2d

Example 4: Aspect ratio
---
const c = useThemeColors();
<Box style={{
  width: 320,
  aspectRatio: 16 / 9,
  backgroundColor: c.primary,
  borderRadius: 8,
}} />
---

Example 5: Viewport units
---
const c = useThemeColors();
// 50% of viewport width, 30% of viewport height
<Box style={{
  width: '50vw',
  height: '30vh',
  backgroundColor: c.surface,
  borderRadius: 8,
}} />
---
Platforms: love2d

Example 6: Calc expression
---
const c = useThemeColors();
// Full width minus 40px for margins
<Box style={{
  width: 'calc(100% - 40px)',
  height: 60,
  backgroundColor: c.surface,
  borderRadius: 8,
  marginLeft: 20,
}} />
---
Platforms: love2d

Example 7: Fit-content width
---
const c = useThemeColors();
// Box shrink-wraps to text width instead of filling parent
<Box style={{
  width: 'fit-content',
  backgroundColor: c.surface,
  padding: 12,
  borderRadius: 8,
}}>
  <Text style={{ fontSize: 14, color: c.text }}>
    {`This box is only as wide as this text`}
  </Text>
</Box>
---
Platforms: love2d

Example 8: flexGrow for space-filling (not hardcoded heights)
---
const c = useThemeColors();
<Box style={{ width: '100%', height: '100%', backgroundColor: c.bg }}>
  <Box style={{ padding: 16, backgroundColor: c.bgElevated }}>
    <Text style={{ fontSize: 18, color: c.text }}>Header</Text>
  </Box>
  <Box style={{ flexGrow: 1 }}>
    <Text style={{ fontSize: 14, color: c.text, padding: 16 }}>
      {`Content fills remaining space`}
    </Text>
  </Box>
  <Box style={{ padding: 12, backgroundColor: c.bgElevated }}>
    <Text style={{ fontSize: 12, color: c.muted }}>Footer</Text>
  </Box>
</Box>
---
Platforms: love2d

=== PLATFORM NOTES ===
Love2D:
Pixel units. The canvas has a fixed resolution set in conf.lua (typically 1280x720 or
1920x1080). Percentage values resolve against the parent container's computed size.
Viewport units (vw, vh) resolve against the window dimensions. calc() supports
'calc(X% + Ypx)' and 'calc(X% - Ypx)' expressions.

=== CRITICAL RULES ===
- Root containers MUST use width: '100%', height: '100%' -- the proportional fallback does not apply at the root
- Containers with children auto-size from content -- explicit dimensions are usually unnecessary
- Empty surface nodes (Box, Image, Video, Scene3D, Emulator, Render, Chart2D) without children or explicit sizing get the proportional fallback (parent/4)
- ScrollView is NOT a surface -- it needs explicit dimensions or flexGrow to define its viewport
- Percentage-based children in auto-sized parents resolve to 0 -- the parent needs a known size for percentages to work
- aspectRatio requires at least one explicit dimension to derive the other
