=== OVERVIEW ===
ReactJIT supports three spacing mechanisms: padding (inner spacing), margin (outer spacing),
and gap (spacing between flex children). These work the same way as in CSS Flexbox: padding
creates space inside the element's border, margin creates space outside, and gap creates
uniform spacing between sibling children in a flex container.

All spacing values accept numbers (pixels) or percentage strings. The shorthand `padding`
and `margin` apply to all four sides. Per-side properties (paddingLeft, marginTop, etc.)
override the shorthand for the specified side.

There are NO `paddingHorizontal` or `paddingVertical` shorthand properties. The linter
rejects these. Use paddingLeft + paddingRight or paddingTop + paddingBottom instead.

Padding affects text measurement: the layout engine uses inner width (width minus horizontal
padding) as the wrap constraint for text nodes, so text wraps correctly inside padded
containers.

=== API / SYNTAX ===
Padding properties:

  padding: number | string
    Applies uniform padding to all four sides.
    Per-side overrides take priority when both are set.

  paddingLeft: number | string
  paddingRight: number | string
  paddingTop: number | string
  paddingBottom: number | string
    Per-side padding. Overrides the shorthand `padding` for the specified side.

  NOT SUPPORTED: paddingHorizontal, paddingVertical
    These do not exist in ReactJIT. The linter rejects them. Use the per-side
    properties instead: paddingLeft + paddingRight, paddingTop + paddingBottom.

Margin properties:

  margin: number | string
    Applies uniform margin to all four sides.

  marginLeft: number | string
  marginRight: number | string
  marginTop: number | string
  marginBottom: number | string
    Per-side margin. Overrides the shorthand `margin` for the specified side.

Gap property:

  gap: number | string
    Spacing between children along the main axis of a flex container. Also applies
    between lines when flexWrap: 'wrap' is active. Gap does not add space before the
    first child or after the last child -- only between adjacent pairs.

Spacing resolution:
  Percentage values resolve against the parent's width (for horizontal spacing) or
  height (for vertical spacing). The shorthand `padding` resolves against the node's
  own width. Per-side properties resolve against the appropriate parent dimension.

Box shorthand props:
  padding: number | string -- maps to style.padding
  px: number | string -- maps to paddingLeft + paddingRight
  py: number | string -- maps to paddingTop + paddingBottom
  margin: number | string -- maps to style.margin
  gap: number | string -- maps to style.gap

=== EXAMPLES ===
Example 1: Uniform padding
---
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

function Padded() {
  const c = useThemeColors();
  return (
    <Box style={{ width: '100%', height: '100%', padding: 24, backgroundColor: c.bg }}>
      <Text style={{ fontSize: 16, color: c.text }}>
        {`24px of space on all sides`}
      </Text>
    </Box>
  );
}
---
Platforms: love2d

Example 2: Per-side padding
---
const c = useThemeColors();
<Box style={{
  width: 300,
  paddingLeft: 20,
  paddingRight: 20,
  paddingTop: 8,
  paddingBottom: 8,
  backgroundColor: c.surface,
  borderRadius: 8,
}}>
  <Text style={{ fontSize: 14, color: c.text }}>Asymmetric padding</Text>
</Box>
---
Platforms: love2d

Example 3: Using px/py shorthand props
---
const c = useThemeColors();
<Box px={20} py={8} bg={c.surface} radius={8} w={300}>
  <Text style={{ fontSize: 14, color: c.text }}>Same result as per-side</Text>
</Box>
---
Platforms: love2d

Example 4: Margin for spacing between siblings
---
const c = useThemeColors();
<Box style={{ width: '100%', height: '100%', padding: 16, backgroundColor: c.bg }}>
  <Box style={{ height: 50, marginBottom: 12, backgroundColor: c.primary, borderRadius: 4 }} />
  <Box style={{ height: 50, marginBottom: 12, backgroundColor: c.surface, borderRadius: 4 }} />
  <Box style={{ height: 50, backgroundColor: c.primary, borderRadius: 4 }} />
</Box>
---
Platforms: love2d

Example 5: Gap for uniform child spacing (preferred over margin)
---
const c = useThemeColors();
<Box style={{ width: '100%', height: '100%', padding: 16, gap: 12, backgroundColor: c.bg }}>
  <Box style={{ height: 50, backgroundColor: c.primary, borderRadius: 4 }} />
  <Box style={{ height: 50, backgroundColor: c.surface, borderRadius: 4 }} />
  <Box style={{ height: 50, backgroundColor: c.primary, borderRadius: 4 }} />
</Box>
---
Platforms: love2d

Example 6: Row with gap
---
const c = useThemeColors();
<Box style={{
  flexDirection: 'row',
  width: '100%',
  gap: 8,
  padding: 16,
}}>
  <Box style={{ width: 80, height: 80, backgroundColor: c.primary, borderRadius: 4 }} />
  <Box style={{ width: 80, height: 80, backgroundColor: c.surface, borderRadius: 4 }} />
  <Box style={{ width: 80, height: 80, backgroundColor: c.primary, borderRadius: 4 }} />
</Box>

=== PLATFORM NOTES ===
Love2D:
Spacing values are in pixels. The Lua layout engine accounts for padding when computing
child positions (children are offset by padLeft and padTop), for margin when spacing
siblings, and for gap when distributing children along the main axis. Padding reduces the
inner width available for child layout and text wrapping.

=== CRITICAL RULES ===
- Do NOT use paddingHorizontal or paddingVertical -- they do not exist, the linter rejects them
- Use paddingLeft + paddingRight instead of paddingHorizontal
- Use paddingTop + paddingBottom instead of paddingVertical
- Prefer gap over margin for uniform child spacing -- gap is cleaner and applies between wrap lines
- Padding is inside the border, margin is outside -- padding affects content area, margin affects layout position
