=== OVERVIEW ===
These are the critical layout rules in ReactJIT. The layout engine itself is pixel-perfect --
flex distribution math, cursor positions, and post-layout sizes are exact. If you encounter
a layout bug, check the component first: does it hardcode a width that overflows? Does it
ignore container bounds? The engine is not the source of the problem.

The layout engine has been refined to handle many cases that older versions required strict
rules for. These have been removed because they are no longer needed:
- Text has a sensible default fontSize (14px) -- no longer required to set explicitly
- Unicode symbols render correctly via DejaVu Sans -- no special handling needed
- Rows resolve width from parents correctly -- no explicit width required for justifyContent
- Deep flex nesting works without explicit dimensions (8+ levels)
- Large component trees handle correctly (12+ children without explicit container size)
- FlexRow/FlexColumn components work fine
- flexGrow works without explicit sibling sizes

Four rules remain because they reflect fundamental constraints of how the layout system
operates. The linter (rjit lint) runs automatically during builds and catches violations
at compile time. Rules can be bypassed per-line with `// rjit-ignore-next-line`.

=== API / SYNTAX ===
Linter escape hatch:

  // rjit-ignore-next-line
  <Text>Intentionally bare text</Text>

This suppresses all linter rules for the next line. Use sparingly -- the rules exist
because violating them produces layout bugs that are difficult to diagnose.

Rule 1: Root containers must have width: '100%', height: '100%'

  The root element IS the viewport. Without explicit dimensions, it has zero size and
  nothing will be visible. flexGrow on the root does not work because there is no parent
  container to grow into. The proportional surface fallback also does not apply at the
  root because the root IS the reference point.

Rule 2: Use flexGrow: 1 for space-filling elements

  When you want an element to absorb remaining space after its siblings measure, use
  flexGrow: 1. Never hardcode pixel heights to fill a known window size -- that creates
  dead space at different resolutions. In a column of header + content + footer, the
  content should have flexGrow: 1. The header and footer auto-size from their content.

Rule 3: ScrollView needs explicit height

  Scroll containers (overflow: 'scroll') are excluded from auto-sizing and from the
  proportional surface fallback. They need height: <number>, height: '50%', or
  flexGrow: 1 to define their scroll viewport. Without explicit height, a ScrollView
  has zero size and no content is visible. overflow: 'auto' containers auto-size to
  content but get scroll behavior when externally constrained.

Rule 4: No mixed text and expressions in Text -- use template literals

  <Text>Score: {score}</Text> creates two separate __TEXT__ nodes ("Score: " and the
  value of score). The layout engine lays these out as separate flex items in a column,
  causing them to stack vertically instead of rendering inline. Using {`Score: ${score}`}
  produces a single __TEXT__ node with the correct concatenated text.
  Linter rule: no-mixed-text-children (error, blocks build).

The Scissor Rule (Lua renderers):

  Never use raw love.graphics.setScissor(x, y, w, h) with content-space coordinates
  inside a Lua renderer. setScissor operates in screen/window coordinates and ignores
  the current transform stack. When a node is inside a ScrollView, the painter applies
  love.graphics.translate(-scrollX, -scrollY) -- drawing coordinates shift correctly,
  but a raw scissor stays in the wrong position.

  The correct pattern is to convert content coordinates to screen coordinates using
  love.graphics.transformPoint(), then use love.graphics.intersectScissor() instead
  of setScissor() to respect parent clip regions.

=== EXAMPLES ===
Example 1: WRONG -- root uses flexGrow instead of explicit dimensions
---
// flexGrow on the root has no parent to grow into. Nothing will be visible.
function App() {
  return (
    <Box style={{ flexGrow: 1, backgroundColor: '#1e1e2e' }}>
      <Text style={{ fontSize: 20, color: '#cdd6f4' }}>Invisible</Text>
    </Box>
  );
}
---
Platforms: love2d

Example 2: RIGHT -- root uses explicit dimensions
---
import { Box, Text } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

function App() {
  const c = useThemeColors();
  return (
    <Box style={{ width: '100%', height: '100%', backgroundColor: c.bg }}>
      <Text style={{ fontSize: 20, color: c.text }}>Visible</Text>
    </Box>
  );
}
---
Platforms: love2d

Example 3: WRONG -- mixed text children create overlapping nodes
---
// Creates two __TEXT__ nodes that stack vertically
<Text style={{ fontSize: 14, color: '#cdd6f4' }}>
  Score: {score}
</Text>
---
Platforms: love2d

Example 4: RIGHT -- template literal creates a single text node
---
const c = useThemeColors();
<Text style={{ fontSize: 14, color: c.text }}>
  {`Score: ${score}`}
</Text>
---
Platforms: love2d

Example 5: RIGHT -- flexGrow: 1 for space-filling content
---
const c = useThemeColors();
<Box style={{ width: '100%', height: '100%', backgroundColor: c.bg }}>
  <Box style={{ padding: 12, backgroundColor: c.bgElevated }}>
    <Text style={{ fontSize: 18, color: c.text }}>Header</Text>
  </Box>
  <Box style={{ flexGrow: 1, padding: 16 }}>
    <Text style={{ fontSize: 14, color: c.text }}>
      {`Content fills remaining space`}
    </Text>
  </Box>
  <Box style={{ padding: 12, backgroundColor: c.bgElevated }}>
    <Text style={{ fontSize: 14, color: c.muted }}>Footer</Text>
  </Box>
</Box>
---
Platforms: love2d

Example 6: WRONG -- hardcoded pixel heights to fill a known window size
---
// 48 + 260 + 80 = 388. At any other resolution this leaves dead space or clips.
<Box style={{ width: '100%', height: '100%' }}>
  <Box style={{ height: 48 }}><Text>Header</Text></Box>
  <Box style={{ height: 260 }}><Text>Content</Text></Box>
  <Box style={{ height: 80 }}><Text>Footer</Text></Box>
</Box>
---
Platforms: love2d

Example 7: RIGHT -- ScrollView with explicit height
---
const c = useThemeColors();
<Box style={{ width: '100%', height: '100%', backgroundColor: c.bg }}>
  <ScrollView style={{ flexGrow: 1, width: '100%' }}>
    {items.map((item, i) => (
      <Text key={i} style={{ fontSize: 14, color: c.text, padding: 8 }}>
        {`${item.name}`}
      </Text>
    ))}
  </ScrollView>
</Box>
---
Platforms: love2d

Example 8: RIGHT -- Lua scissor pattern for custom renderers
---
-- Save previous scissor and convert content coords to screen coords
local psx, psy, psw, psh = love.graphics.getScissor()
local sx, sy = love.graphics.transformPoint(x, y)
local sx2, sy2 = love.graphics.transformPoint(x + w, y + h)
love.graphics.intersectScissor(sx, sy, math.max(0, sx2 - sx), math.max(0, sy2 - sy))

-- ... draw content ...

-- Restore previous scissor
if psx then
  love.graphics.setScissor(psx, psy, psw, psh)
else
  love.graphics.setScissor()
end
---
Platforms: love2d

=== COMMON PATTERNS ===
Layout anti-patterns to avoid:

Hardcoding pixel heights to fit a known window size:
  <Scene style={{ height: 260 }}/> in a 600px-tall parent leaves 340px of dead air.
  Use flexGrow: 1 instead.

Budgeting pixels manually:
  Adding 48 + 260 + 80 + gaps to hit a target height. Let flex do this -- one element
  grows, the rest auto-size from content.

Using fixed dimensions where auto-sizing works:
  If a panel contains text and buttons, it knows its own size. Do not constrain it with
  a hardcoded height. Let it shrink-wrap, and give a sibling flexGrow: 1 to fill the gap.

Using paddingHorizontal or paddingVertical:
  These do not exist in ReactJIT. Use paddingLeft + paddingRight or paddingTop +
  paddingBottom. The linter rejects the shorthands.

=== PLATFORM NOTES ===
Love2D:
The layout engine runs in Lua (layout.lua) with a per-pass budget of 100,000 node visits
to catch infinite loops. Legitimate trees never approach this limit. The engine resolves
flex distribution, min-content floors, text re-measurement, and cross-axis alignment in
a single pass. The proportional surface fallback uses the fully-resolved parent height,
not the viewport height, so nested surfaces cascade correctly (800 -> 200 -> 50 -> 12).

=== CRITICAL RULES ===
1. Root containers MUST have width: '100%', height: '100%' -- the root has no parent to grow into or fallback from

2. Use flexGrow: 1 for space-filling elements -- never hardcode pixel heights. One element grows, the rest auto-size.

3. ScrollView needs explicit height or flexGrow -- scroll containers are excluded from both auto-sizing and the proportional surface fallback.

4. No mixed text and expressions in Text -- use template literals ({`text ${var}`}). Mixed children create separate __TEXT__ nodes that stack vertically.

5. The Scissor Rule -- in Lua renderers, always use love.graphics.transformPoint() + intersectScissor(), never raw setScissor() with content-space coordinates. setScissor ignores the transform stack and breaks inside ScrollViews.
