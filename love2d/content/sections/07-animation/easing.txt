=== OVERVIEW ===
The Easing object provides easing functions that control how an animation's progress maps to its output value. An easing function takes a normalized time value (0 to 1) and returns a normalized progress value.

Easing functions are used in two contexts:

JS-driven animations pass Easing functions directly to timing configs:
  anim.timing({ toValue: 1, easing: Easing.bounce })

Lua-driven animations use string names in style.transition and style.animation:
  transition: { all: { easing: 'bounce' } }
  animation: { keyframes: {...}, easing: 'elastic' }

Available Lua easing strings: 'linear', 'easeIn', 'easeOut', 'easeInOut', 'bounce', 'elastic'.

=== API / SYNTAX ===
```tsx
import { Easing } from '@reactjit/core';
```

Static functions (no arguments):
| Function       | Curve shape                                              |
|----------------|----------------------------------------------------------|
| Easing.linear  | Constant speed, no acceleration                          |
| Easing.easeIn  | Starts slow, accelerates (quadratic)                     |
| Easing.easeOut | Starts fast, decelerates (quadratic)                     |
| Easing.easeInOut | Slow start, fast middle, slow end (quadratic)          |
| Easing.bounce  | Bounces at the end like a ball hitting the floor         |

Factory functions (return an EasingFunction):
| Function       | Parameters                     | Description                                    |
|----------------|--------------------------------|------------------------------------------------|
| Easing.elastic | bounciness?: number (default 1) | Elastic snap with configurable overshoot      |
| Easing.bezier  | x1, y1, x2, y2: number        | Custom cubic bezier curve (CSS-style control points) |

Type:
```tsx
type EasingFunction = (t: number) => number;
```

=== EXAMPLES ===
Example 1: Built-in easings with timing
---
```tsx
import { useEffect } from 'react';
import { useAnimation, Easing } from '@reactjit/core';

export function EasingDemo() {
  const [anim, value] = useAnimation(0);

  useEffect(() => {
    const a = anim.timing({
      toValue: 200,
      duration: 600,
      easing: Easing.bounce,
    });
    a.start();
    return () => a.stop();
  }, [anim]);

  return value; // use in style
}
```
---
Platforms: love2d

Example 2: Elastic with configurable bounciness
---
```tsx
import { useEffect } from 'react';
import { useAnimation, Easing } from '@reactjit/core';

// Higher bounciness = more oscillations before settling
const gentleElastic = Easing.elastic(0.5);
const wildElastic = Easing.elastic(3);

export function ElasticDemo() {
  const [anim, value] = useAnimation(0);

  useEffect(() => {
    const a = anim.timing({
      toValue: 1,
      duration: 800,
      easing: wildElastic,
    });
    a.start();
    return () => a.stop();
  }, [anim]);

  return value;
}
```
---
Platforms: love2d

Example 3: Custom cubic bezier
---
```tsx
import { Easing } from '@reactjit/core';

// Overshoot-and-settle (like CSS cubic-bezier(0.68, -0.6, 0.32, 1.6))
const overshoot = Easing.bezier(0.68, -0.6, 0.32, 1.6);

// Snappy ease-out
const snappy = Easing.bezier(0.25, 1, 0.5, 1);

// Use in timing config
anim.timing({ toValue: 100, duration: 400, easing: overshoot });
```
---
Platforms: love2d

Example 4: Easing in Lua-driven transitions
---
```tsx
import { Box } from '@reactjit/core';
import { useThemeColors } from '@reactjit/theme';

export function LuaEasingDemo() {
  const c = useThemeColors();

  return (
    <Box style={{
      backgroundColor: c.surface,
      transition: {
        backgroundColor: { duration: 300, easing: 'easeInOut' },
        opacity: { duration: 200, easing: 'linear' },
      },
    }} />
  );
}
```

=== PLATFORM NOTES ===
Love2D:
  Easing.bezier uses Newton-Raphson iteration (8 iterations) to solve the cubic bezier. This is computed per frame per animated value, so avoid using many simultaneous bezier-eased animations on low-end hardware.

=== CRITICAL RULES ===
- Easing.elastic() and Easing.bezier() are factory functions that return an EasingFunction. Call them before passing to a config, not inline in a render body (to avoid creating a new function identity each frame).
- Lua-driven transitions and keyframe animations use string easing names, not Easing.* functions. The available strings are: 'linear', 'easeIn', 'easeOut', 'easeInOut', 'bounce', 'elastic'.
- Easing.bezier control points follow the CSS cubic-bezier() convention: (x1, y1, x2, y2) where x values should be between 0 and 1, but y values can go negative or above 1 for overshoot effects.
