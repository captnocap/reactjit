# Bug Audit — 2026-03-05

Comprehensive codebase audit of uncommitted changes and new files.

## CRITICAL BUGS

### 1. CaptureStory uses wrong scaffold component props
**Files:** `storybook/src/stories/CaptureStory.tsx`
**Lines:** 246-250, 256, 282, 298, 314, 339-346

The CaptureStory passes non-existent props to shared scaffold components:

- **HeroBand** (line 246-250): Passes `pill`, `title`, `subtitle` — none of these exist. HeroBand only accepts `accentColor` + `children`.
- **SectionLabel** (lines 256, 282, 298, 314): Uses `n`, `title`, `color` — should be `icon`, `children`, `accentColor`.
- **CalloutBand** (lines 339-346): Passes `title`, `accentColor` — should only have `borderColor`, `bgColor`, `children`.

**Impact:** The story renders but all HeroBand/SectionLabel/CalloutBand content is missing (props silently ignored). The story looks broken in the storybook.

**Compare:** FinanceStory and ChemistryStory use these components correctly.

### 2. ColorUtils.set() called with wrong signature in chemistry capabilities
**Files:** `lua/capabilities/spectrum_view.lua` (lines 209, 256)
**Root:** `lua/color.lua` — `Color.set(c)` accepts ONE argument

```lua
-- WRONG (two args — opacity silently dropped):
ColorUtils.set(lineColorHex, opacity)

-- CORRECT (one arg):
ColorUtils.set(lineColorHex)
-- or direct:
love.graphics.setColor(r, g, b, opacity)
```

**Impact:** Spectrum lines render without custom opacity. The extra argument is silently ignored by Lua. All non-highlighted peaks render at full opacity instead of fading.

### 3. Imaging `onComplete` event payload doesn't match TypeScript type
**Files:**
- `lua/capabilities/imaging.lua` (line 570-578) — pushes `{ width, height }`
- `packages/imaging/src/types.ts` (line 60) — expects `{ outputPath?: string }`

```lua
-- Lua sends:
{ handler = "onComplete", width = result:getWidth(), height = result:getHeight() }

-- TypeScript expects:
onComplete?: (result: { outputPath?: string }) => void;
```

**Impact:** React handlers receive `width`/`height` but TypeScript says they get `outputPath`. The type is a lie — users expecting `outputPath` get `undefined`.

## HIGH SEVERITY

### 7. Dual `SpectrumViewProps` type definitions (naming conflict)
**Files:**
- `packages/chemistry/src/capabilities.tsx` (lines 23-32) — `{ spectrumType, compound, ... }`
- `packages/chemistry/src/spectra.ts` (lines 24-29) — `{ spectrum: Spectrum, ... }`

Two completely different interfaces with the same name. The `index.ts` export on line 48 explicitly re-exports the `capabilities.tsx` version, shadowing the `spectra.ts` one. But:
- A developer importing directly from `spectra.ts` gets the wrong type
- The `spectra.ts` version expects a full `Spectrum` object; the `capabilities.tsx` version expects `spectrumType` + `compound` strings
- The Lua capability expects the `capabilities.tsx` version

---

### 8. useRecorder error states silently swallowed
**File:** `packages/core/src/useRecorder.ts`

- Line 73: `.catch(() => {})` — polling errors silently eaten
- Line 90: `.catch(() => {})` — start errors silently eaten
- Line 86-87: If `result.error` exists, the hook just returns without exposing it

**Impact:** If ffmpeg isn't installed or the pipe breaks, the user gets no feedback. The hook reports `recording: true` while nothing is actually recording. Lua properly returns error messages (`"ffmpeg pipe failed — is ffmpeg installed?"`) but they never reach React.

## LOW SEVERITY

### 13. `shader_grade.lua` not in Masks.loadAll() list
**File:** `lua/masks.lua` (lines 406-435)

The `shader_grade` module is used by CRT, VHS, and Watercolor via `require("lua.masks.shader_grade")` but isn't in the explicit mask list in `Masks.loadAll()`. Works because `require()` loads it on demand, but it's inconsistent with how other mask modules are registered.

**File:** `claudeshome/lua/capabilities.lua` has deep equality checking that the main `lua/capabilities.lua` doesn't have yet. The consumer project is ahead of the source of truth.
